    摘    要：

    采用当前方法采集农业大棚的环境信息时, 采集信息所用的时间较长, 采集得到的信息完整性较低, 存在采集效率低和丢包率高的问题。提出云计算下农业大棚环境信息自动采集方法, 通过控制层、支撑层、操作层、传输层和服务层建立农业大棚环境信息采集平台, 根据平台中存在的骨干节点建立环结构, 在普通节点与骨干节点之间依赖关系的基础上搭建骨干路径, 将存在农业大棚环境信息的数据包通过骨干路径传送到sink节点中, 利用独立成分分析法初步提取数据包中存在的农业大棚环境信息, 再采用模糊线性鉴别方法进行一步提取农业大棚环境信息, 在云计算的基础下完成农业大棚环境信息的自动采集。仿真结果表明, 所提方法的采集效率高、丢包率低。

    关键词：

    云计算; 农业大棚; 信息采集;

    Automatic Collection and Simulation of Environmental Information in Agricultural Greenhouses under Cloud Computing

    BU Xiao-bo LIU De-lei WANG Hao-ru NIU Ya-man

    College of Software, Shanxi Agricultural University

    Abstract：

    At present, collection method of environment information of agricultural greenhouse is time-consuming. Meanwhile, integrity of the collected information is low. Collection efficiency is low and packet loss rate is high. Therefore, an automatic collection method of the environment information under cloud computing is proposed. Collection platform of the environment information was built via control layer, support layer, operation layer, transport layer, and service layer. According to backbone node existing in the platform, ring structure was built based on dependence between normal node and backbone node. Data package having environmental message of agricultural greenhouse was transmitted into sink node via backbone path. Independent component analysis was used to extract the environmental message existing in the data package preliminarily. Then, the environmental message was further extracted via fuzzy linear identification method. Based on cloud computing, the automatic collection of environment information was completed. Simulation results show that the method has high collection efficiency and low packet loss rate.

    Keyword：

    Cloud computing; Agricultural greenhouse; Information collection;

    1 引言

    农业大棚被广泛的应用在工厂化育苗、蔬菜生产、观赏农业、花卉盆景、畜禽饲养、果木育苗和水产养殖等领域中[1]。利用计算机技术、网络技术和传感器技术可以通过执行节点和传感器节点测得农业大棚中的二氧化碳浓度、基质温湿度、光照强度、PH值、气压等环境因子, 通过数据分析和处理获得农业大棚的环境信息, 根据采集得到的环境信息自动调控农业大棚的施肥、灌溉和室温环境等作业, 使农业大棚内的植物和农作物在最佳的条件下生长[2]。当前农业大棚环境信息采集方法存在采集效率低和丢包率高的问题, 需要深入的分析并研究农业大棚环境信息采集方法[3]。

    邓雪峰、孙瑞志、聂娟等人提出基于时间自动机的农业大棚环境信息采集方法, 该方法通过分析农业大棚环境监控物联网系统, 划分农业大棚环境监测物联网系统, 在时间自动机理论的基础上建立组件模型, 根据组件模型构建时间自动机模型, 利用时间自动机模型采集农业大棚的环境信息。但该方法采集农业大棚环境信息所用的时间较长, 存在采集效率低的问题[4]。杨宏业、张维庭、冯文涛等人提出基于ZigBee技术的农业大棚环境信息采集方法, 该方法通过ZigBee技术建立传感网络, 在农业大棚中安置采集终端, 采集农业大棚环境信息, 并将采集得到的信息通过无线发送功能传送到中间层, 通过中间层分析并处理采集得到的农业大棚环境信息, 将最终信息传送到顶层监控应用层中, 完成农业大棚环境信息的采集。但该方法采集得到的信息完整性较低, 存在丢包率高的问题[5]。俞昆、谭继文、李善等人提出基于KPCA-LVQ的农业大棚环境信息采集方法, 该方法根据农业大棚环境信号的非平稳性和非线性特征, 提取信号的能量特征值, 结合特征值、方差、频率和峰度等特征量构成农业大棚环境信息的初始特征集, 通过KPCA法去除特征集中存在的冗余信息, 构建农业大棚环境信息采集模型, 完成农业大棚环境信息的采集。但该方法的过程较为繁琐, 计算所用的时间较长, 存在采集效率低的问题[6]。

    为了解决上述方法中存在的问题, 提出云计算下农业大棚环境信息自动采集方法。

    2 基于云计算的农业大棚环境信息采集平台

    云计算下农业大棚环境信息自动采集方法通过五个层次构建农业大棚环境信息采集平台, 如图1所示。

    1) 支撑层:

    支撑层具有云计算分析平台和云计算存储平台。其中硬件具有以太局域网、PC云集群和NameNode。软件包括用户管理、信息采集、农业大棚环境云分析和数据云存储系统等。支撑层具有分析并存储大数据、处理能力强和响应快的特点[7]。

    2) 服务层:

    服务层的硬件具有邮件服务器、门户服务器、语音服务器、信息收发器和数据库服务器等, 云计算下农业大棚环境信息自动采集方法将固态硬盘应用到数据库服务器中, 提高农业大棚环境信息的采集速度。门户服务层的软件包括语音服务系统、数据采集系统、邮件服务系统、数据库系统、农业专家系统和生产管理系统等。服务层的主要功能是处理响应快、交互性强的事务, 不处理和存储规模较大的历史数据。

    3) 传输层:

    主要包括农业大棚环境连接互联网和传感器网组的传输模块。农业大棚环境传感器网中的主要组网设备是物联网网关, 可以连接上位机和农业大棚现场的传感网。具有安全认证、网络连接、编解码、协议转换、信令交换和转发信息等功能, 可以完成农业大棚环境监控平台内节点的管理, 如采集节点的能量、标识、属性和状态等, 以及远程维护、唤醒、升级、诊断和控制等。

    4) 控制层:

    可以通过控制层控制农业大棚机械、采集传感器数据。传感器通信和采集模块通常分为有线方式和无线方式, 分别采用TCP/IP协议、RS485协议的编码技术和RF射频技术。控制程序的预警信息、存储命令、报警信息和发送命令通过控制模块完成, 用于控制开膜、喷淋、卷帘和滴灌等设备和机械。

    5) 操作层:

    云计算下农业大棚环境信息采集平台的最底层是操作层, 通过操作层感应农业大棚中环境要素的改变, 并采用数字量描述农业大棚环境要素的变化[8,9]。主要包括农业大棚内的土壤水分、空气温度、土壤温度、空气湿度、二氧化碳浓度、光照和农业大棚外的视频图像、风速、空气湿度、空气温度和风向等传感器。操作模块主要是农业大棚生产设备和机械等执行机构, 如开膜机、灌溉设备、卷帘机、补光灯和二氧化碳气罐等, 可以在控制层的基础上根据农户需求和控制程序执行机械或设备的关闭指令和开启指令, 实现农业大棚生产的高效化、智能化和现代化。

    3 农业大棚环境信息自动采集方法

    云计算下农业大棚环境信息自动采集方法在农业大棚环境信息采集平台中选择骨干节点建立环结构, 根据普通节点和骨干节点之间存在的依赖关系, 建立骨干路径, 将含有农业大棚环境信息的数据包传送到sink节点, 采用独立成分分析法提取数据包中的关键信息, 通过模糊k近邻方法获得分布信息, 再利用模糊线性鉴别分析方法在分布信息的基础上进一步提取农业大棚环境信息, 完成农业大棚环境信息的自动采集, 具体步骤如下:

    当目标进入农业大棚环境信息采集平台的监测区域内时, 在目标附近的传感器节点可以感应到农业大棚中的目标。传感器节点开始测量目标当前位置和节点之间的距离, 构成一个三元组的感知数据包DPacket, 数据包DPacket的表达式如下:

    DPacket=<cdi, sdi, t> (1)

    式中, cdi代表的是传感器节点感应到的目标当前的节点坐标 (xi, yi) ;sdi代表的是感应到目标的节点i与当前目标位置之间的距离;t代表的是发现农业大棚中目标的时间。

    感知数据包DPacket通过感应到目标的传感器节点i传送到节点i的依赖节点。农业大棚环境信息采集平台中的骨干节点将收到的感知数据包DPacket传送到本节点的依赖节点, 通过依赖节点保持转发操作, 将感知数据包DPacket转发到小环号的骨干节点, 重复上述步骤, 直到感知数据包DPacket传送到中心节点终止迭代。

    当农业大棚环境信息采集平台采集到的全部感知数据包DPacket传送到中心节点后, 选取感知数据包DPacket中的距离数值sdi较小的m个节点, 即与目标之间距离最近的m个节点, 将其作为候选节点, 参与农业大棚采集目标定位, 采用加权质心算法通过下式得到目标当前的位置 (Tx, Ty) :

    Τx=∑i=1mxi/sdi∑i=1m1/sdi (2)

    Τy=∑i=1myi/sdi∑i=1m1/sdi (3)

    根据式 (2) 和式 (3) 计算的目标位置 (Tx, Ty) 得到定位节点中存在的数据包LPacket。

    根据骨干路径利用中心节点将数据包LPacket传送到sink。骨干路径选择中心节点作为初始节点, 将数据包LPacket传送到可以与中心节点直接通信的第二个骨干节点, 第二个骨干节点再将数据包LPacket传送到下一个骨干节点, 重复上述步骤, 直到含有农业大棚环境信息的数据包LPacket到达sink[10]。

    对数据包LPacket作ICA变化, 获得独立成分IC, IC的表达式如下

    IC= (IC1, IC2, …ICm) (4)

    式中, ICi代表的是数据包LPacket经过ICA变化的独立子成分。对数据包LPacket进行投影变化, 得到农业大棚环境信息的特征向量Y, 特征向量Y的计算公式如下

    Y=ICT·LPacket (5)

    式中, T代表的是转置矩阵。设uij代表的是第j个数据包LPacket相对于第i类的隶属度, 其计算公式如下

    设wi代表的是调整后的类中心点, 其计算公式如下

    ωi=∑j=1uijnij/∑j=1uij (7)

    5 结束语

    农业大棚的数量不断增加, 仅仅通过农户无法采集到农业大棚中完整的环境信息, 农业大棚环境信息的采集成为目前亟待解决的问题。当前农业大棚环境信息采集方法存在采集效率低和丢包率高的问题。提出云计算下农业大棚环境信息自动采集方法, 可在较短的时间内采集到完整的农业大棚环境信息, 解决了当前方法中存在的问题, 为我国农业的发展提供了条件。
