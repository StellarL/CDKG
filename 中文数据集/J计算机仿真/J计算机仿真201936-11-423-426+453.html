    摘    要：

    为了增强网络攻击防护,保护网络信息安全,需要进行网络数据流异常检测,以便于发现网络异常状态,及时进行修复,为此,提出基于数学建模的网络数据流异常检测方法。通过对网络数据流的分析,将其划分为周期性与随机性网络数据流两类,结合周期性数据流的数据包生成速度、数据包大小及传输延时,构建周期性网络数据流数学模型,利用泊松分布对随机性网络数据流进行建模;通过深度学习训练机对网络数据流正常与异常状态进行样本分析,引入逻辑回归模型,结合网络数据流模型对应的数据流特征实现异常检测。实验结果表明,该网络数据流异常检测方法的检测结果精准度及灵敏度高于实验对比方法,适用性较强,具有较好的鲁棒性。

    关键词：

    数学模型; 网络数据流; 异常检测; 泊松分布;

    Network Data Flow Anomaly Detection Simulation Based on Mathematical Modeling

    ZHANG Cheng SHANG Hai-tao

    Institute of Technology, East China Jiaotong University

    Abstract：

    In order to enhance the network protection and protect network information security, it is necessary to detect abnormal network data flow to facilitate the discovery of network abnormal state. Therefore, this paper presented a method to detect the abnormal network data flow based on mathematical modeling. By analyzing the network data flow, we divided it into two types including periodic network data flow and random network data flow. Combined with the data packet generation speed, packet size and transmission delay of the periodic data flow, we built the mathematical model of periodic network data flow. Then, we used Poisson distribution to construct the model of random network data flow. Based on the deep learning machine, we analyzed the normal and abnormal state of network data flow. Finally, we introduced the logistic regression model and combined with the data flow characteristics corresponding to the network data flow model to achieve the abnormal detection. Simulation results show that the accuracy and sensitivity of proposed detection method is higher than the comparison method. Meanwhile, the applicability is strong and this method has good robustness.

    Keyword：

    Mathematical model; Network data flow; Abnormal detection; Poisson distribution;

    1 引言

    互联网技术迅猛发展,并广泛应用于各生产、生活领域,成为了日常工作与学习,及各行业发展过程中不可分割的一部分[1]。随着网络攻击数量的逐渐增长,网络安全与入侵防护问题引起了相关研究领域与各界人士的广泛关注。网络信息安全对个人生活、企业生产及社会安定的重要性逐渐增加。网络异常检测是网络及信息安全领域的研究内容之一,也是当前研究的重点与难点[2,3]。

    网络用户数量的增加,致使网络数据流量猛增,且网络攻击时间的类型与目的也呈现出一定的增长趋势。在网络安全防护技术中,对异常网络数据流的挖掘与分析至关重要,根据分析所得的网络数据流特点来进行异常检测[4]。国内外很多学者进行了网络数据流异常检测的相关研究,但受到网络数据流规模过大、存在时间连续性,且到达速度较快,随时间变化等特性的影响,当前的网络数据流异常检测方法难以全面推广[5]。

    通过对当前方法的分析与研究,当前网络数据流异常检测方法中存在的主要问题如下:

    1)难以对网络数据流进行及时有效的存储与处理,致使新的网络数据会覆盖部分旧数据,该部分网络数据流的遗漏,会导致检测结果的精准度降低。

    2)网络数据流规模较大,且类型复杂,增加了网络数据流区分与异常检测的难度,导致网络数据流异常检测过程中的计算量过大,过于复杂,降低了网络数据流异常检测的效率,难以满足其时效性需求。

    针对上述问题,结合数学模型进行网络数据流分析,提出基于数学建模的网络数据流异常检测方法,并通过实验检验了该方法的可行性。

    2 基于数学建模的网络数据流异常检测方法

    2.1 建立网络数据流数学模型

    网络数据流数量庞大,为便于进行数据流分析与网络运行状态管理,可依据各类数据的属性进行数据流分类。根据网络数据流的产生方式,结合随机过程理论,对网络运行的连续、随机事件动态关系进行定量描述,并将网络数据流划分为周期性与随机性网络数据流两类(网络数据流产生图示如图1所示),在此基础上,针对不同类型的网络数据流分别建立对应的数学模型。

    图1 网络数据流产生图示   下载原图

    周期性网络数据流为一定时间间隔内不间断产生的网络数据类,在网络数据流中,该类数据流所占的比例较大。周期性网络数据流具备产生稳定、规模较大等特点,且通常消耗的网络带宽资源较多,可能致使通信网络数据传输出现时延[6]。

    周期性网络数据流一般为时驱型数据,是指每一类网络报文均在预设时间触发,且每一类网络报文的分组长度与文件大小通常为定值,可以进行预设,网络报文的到达间隔呈现周期性变化趋势。

    假设周期性网络数据流的报文分组长度为Lc,报文内含有网络帧头、数据域等网络相关信息;网络运行单位时间内生成的数据包数量为Nc,Nc的值与网络传感器的采用率f0紧密关联;网络数据包传输时延为Dc,主要包括网络时延Ec,数据包发送端及接收端的信息处理时延Sc和Rc,综合分析得到如式(1)所示的周期性网络数据流数学模型表达式

    Mc=f(Lc,Nc,Dc) (1)

    其中,数据包数量Nc及网络数据包传输时延Dc的描述式分别为

    Nc=f0 (2)

    Dc=Sc+Ec+Rc (3)

    根据各类网络数据的应用范围与所要达成的目标差异,设网络正常运行情况下能够支持的最大传输时延为Ds,则各类周期性数据流的传输时延均应满足以下约束条件

    Dc≤Ds (4)

    周期性网络数据流的应用类型与对应的时延及数据包大小可分为以下几种形式:

    1)应用于网络保护的周期性数据流传输时延为1～10ms之间,数据包的大小约为几十字节;

    2)应用于网络控制的周期性数据流的传输时延约为80ms,数据包大小也在几十字节左右;

    3)应用于网络监测的周期性数据流传输时延大约为1s,根据各类网络差异,数据包大小在几十字节至几百字节不等;

    4)应用于量测与计费的周期性数据流传输时延通常以小时计,数据包大小约为几百字节;

    5)应用于日志记录与版本更新的周期性数据流传输时延较长,可按天或周计算,数据包大小也较其它应用更大,以千字节或兆字节进行计量[7]。

    随机性网络数据流是网络运行过程中的随机事件产生的,如网络运行过程中出现攻击信息入侵等情况时,网络保护程序随即产相应指令,迅速隔离并清除攻击信息,以此保障网络的正常运行[8]。该类网络数据流具备的特征如下:

    在某一时间范围内,网络数据产生报文分组的概率为P,报文长度可设置为定值,也可随时间变动而随机变化,数据端到端传输到达顺序互无关联,也就是在2个不同的时间段内传输成功的报文数量之间不存在任何关联。根据先验知识与相关研究,可利用泊松分布来近似上述过程。

    设在时段区间[τ,τ+t]内,网络报文的平均成功传输数量为λ,成功传输报文的总数量为N(t),则时间间隔t内有k个网络报文成功进行端到端传输的概率服从参数为λ的泊松分布,对应的描述式如下

    k=N(τ+t)-N(τ) (5)

    Ρ(k)=(λt)keλtk! (6)

    根据上式可知,服从泊松分布的连续网络数据包之间的端到端到达时差服从参数为1/λ的负指数分布,将其作为数据包到达间隔分析的可靠性模型,利用式(7)给出函数表达式

    g(t)=λeλt,t≥0 (7)

    利用OPNET网络仿真技术软件包进行数据流传输特性研究,分析周期性与随机性网络数据流的特点,得到以下结果:

    1)周期性网络数据流可利用数据包生成速率、大小及时延几项内容来描述,网络传输时延基本为定值,网络带宽资源比较充裕的情况下,该类数据流的传输实验与数据包大小为正比例关系,与数据包生成速率不相关。

    2)随机性网络数据流可利用泊松分布来描述,该类数据流的传输特性与数据包大小及到达率相关,数据传输时延不稳定。

    3 实验结果与分析

    在网络数据流异常检测实验中,经典的异常检测数据集包括:KDD99数据集与清洗、去噪后所得的NSL-KDD数据集,为消除数据的滞后性影响,本次实验中,实验数据集为根据Morre等人在2009年所提出的网络数据流特征提取标准所获取的数据集合。

    为了检验方法中时间滑动窗口的宽度变化对网络数据流异常检测结果的精准度影响,采用本文方法在实验数据集上反复进行实验,对比时间滑动窗口宽度变化情况下,网络数据流异常检测结果的精准度变化趋势,实验中设置的时间滑动窗口宽度分别为1min、2min和3min,得到实验结果如图2所示。

    根据图2可知,在实验数据集上,时间滑动窗口宽度为3min时的异常检测结果准确度最高,原因是时间滑动窗口的宽度越大,该窗口内所包含的网络数据流越多,异常数据流量也随之增加,更容易被检测出来,但检测的灵敏度较低,需要在异常发生3min后才能检测到网络数据流的异常状态;时间窗口宽度为2min和1min时,检测结果的精准度相对较低,但时间灵敏度明显增加,能够在更短的时间内发现网络数据流的异常状态。综合上述分析,网络数据流异常检测时间滑动窗口的宽度可根据实际应用进行具体分析,选择适当的宽度值,避免因窗口宽度较大导致灵敏度过低,或因窗口宽度过小而引起检测精准度不够。
