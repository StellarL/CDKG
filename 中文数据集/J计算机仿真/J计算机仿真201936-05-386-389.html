    摘    要：

    针对当前方法在实现过程中由于云平台下终端设备较多导致网络资源负载不均衡, 吞吐量较小、稳定性较差等问题, 提出一种基于模糊循环迭代控制的多终端自适应负载均衡算法, 通过构建多终端资源负载时间模型, 对终端资源节点所完成终端任务的总时间进行计算, 得到云平台下多终端资源负载均衡的目标函数;建立资源权重分配机制, 控制多终端输出和输入之间的空间模糊距离, 当正态分布概率迭代计算结果最小时, 实现了多终端自适应负载均衡。仿真验证, 采用方法能够提高平均吞吐量、降低平均延时、提高资源任务分配率和资源利用率, 保证多终端自适应负载均衡的稳定性和灵活性。

    关键词：

    多终端; 负载均衡; 平均吞吐量; 平均延时; 任务分配率;

    Multi-Terminal Adaptive Load Balancing Algorithm Simulation under Cloud Platform

    GENG Qiang HUANG Xue-qin

    Networking Academy, Haikou College of Economics

    Abstract：

    Aiming at the problem that the current method is unbalanced due to the large number of terminal devices under the cloud platform, the throughput is small and the stability is poor, a multi-terminal adaptive load balancing algorithm based on fuzzy loop iterative control is proposed. By constructing a multi-terminal resource load time model, the total time of the terminal tasks completed by the terminal resource node is calculated, and the objective function of multi-terminal resource load balancing under the cloud platform is obtained; the resource weight distribution mechanism is established, and the spatial fuzzy distance between the multi-terminal output and the inputs is controlled. When the iterative calculation result of the normal distribution probability is the smallest, the multi-terminal adaptive load balancing is realized. Simulations show that the proposed method can improve the average throughput, reduce the average delay, improve the resource task allocation rate and resource utilization, and ensure the stability and flexibility of multi-terminal adaptive load balancing.

    Keyword：

    Multi-terminal; Load balancing; Average throughput; Average delay; Task allocation rate;

    1 引言

    随着计算机网络的不断发展, 云平台下云服务的服务范围和领域也在不断扩大, 广泛应用到移动网络中, 终端设备也从承担最基本的信息输入输出工作到具备一定的数据运算和处理的能力[1,2]。在一个大型的计算机中, 云平台下连接多个终端设备将客户的数据进行输入或输出, 当信息数据不断增加时, 在网络结构中构建一个负载平衡的法则, 已经应用到Web服务器、FTP服务器、企业关键应用服务器和其它关键任务服务器等网络结构中[3]。随着计算机网络技术的不断提升, 终端设备不断增多, 由于云平台下终端设备较多导致网络资源负载不均衡, 吞吐量较小、稳定性较差等问题, 需要探究一种多终端自适应负载均衡算法, 该方法可以提高资源吞吐量、提高资源利用率, 保证多终端负载均衡性和稳定性。具备一定的使用效果[4]。

    文献[5]提出一种基于自适应权值最小负载的多终端自适应负载均衡算法, 该方法通过运用不同种类的负载因素体现出各终端节点的综合负载, 依据各终端数据负载情况对终端节点的权值进行动态调节, 同时计入多终端节点前一时间断内的实际负载情况, 确保集群中负载最小的终端节点提供的服务, 实现了多终端自适应负载均衡该方法存在多终端自适应负载均衡平均延时的缺点。文献[6]提出一种基于分布式服务器集群的多终端自适应负载均衡算法, 该方法通过利用信息数据、信息控制和应用分力, 并对多终端设备中一个终端设备的实际负载进行计算;经过查找资源采样记录来确定连接资源最少的终端设备, 降低客户访问请求的响应时间, 实现了多终端自适应负载均衡。该算法存在终端自适应负载均衡平均吞吐量较差的缺点。文献[7]提出一种基于动态自适应权重轮询的多终端自适应负载均衡算法, 该算法通过对云平台下多终端服务设备节点性能的各种因素进行计算, 依据多终端节点在运行时的实际负载动态情况计算出多终端中节点的实际负载值, 同时对实际负载值进行加权, 实现了多终端自适应负载均衡。该算法存在终端自适应负载均衡性较差的缺点。

    综上所述, 提出一种基于模糊循环迭代控制的多终端自适应负载均衡算法。建立资源权重分配机制, 控制多终端输出和输入之间的空间模糊距离, 当正态分布概率迭代计算结果最小时, 实现了多终端自适应负载均衡, 克服传统方法负载均衡能力较差的缺点。仿真试验表明, 该方法能够提高平均吞吐量、减小平均延时、提高资源利用率和自适应负载均衡性。

    2 云平台下多终端自适应负载均衡算法原理

    云平台下多终端自适应负载均衡算法原理是通过利用排队理论计算多终端节点处理能力, 同时计算容纳信息处理能力, 依据排队定律得到期望信息平均抵达率, 当某个终端节点的负载较高时, 所需要的信息数量将会减少时, 实现了多终端自适应负载均衡[7]。

    在云平台下, 利用排队理论依据多终端服务器的负载情况进行均衡, 整个终端系统是属于一个排队系统, 在多终端系统中是要有信息队列, 一个终端节点是属于一个单独的排队系统, 终端系统要平等对待全部的信息请求。

    3.1 云平台下多终端资源负载均衡目标函数计算

    云平台下多终端资源负载均衡目标函数计算是通过构建多终端资源负载时间模型, 对终端资源节点所完成终端任务的总时间进行计算, 得到云平台下多终端资源负载均衡的目标函数[8]。

    在云平台下, 多终端资源分配是非常重要的, 而且资源负载均衡能够有效提高客户任务, 云服务终端资源类型较多, 同种类型并具有异构性。在大型的云平台下终端设备的计算任务通过分成多个终端的子任务, 依据多个终端子任务的要求来分配对应的资源节点, 在不同时间段, 多终端任务和多终端资源间是一种动态转变过程[9]。假设E={e1, e2, …, en}表示多终端资源集合;F={f1, f2, …, fn}表示多终端节点集合;G={g1, g2, …gn}表示多终端任务集合;Lfe表示多终端资源和终端设备之间的映射关系;Lfg表示多终端资源与物理之间的映射关系, 构建多终端资源负载均衡模型计算公式如下
