    摘    要：

    当前协同数据库分布交互相关方法存在用户访问满意度低和数据同步性能差的问题, 提出基于WEB技术的协同数据库分布交互方法。以WEB技术在数据交互中应用分析为依据, 构建WEB技术下协同数据库分布交互框架。数据库交互框架中的表示层主要负责将交互响应结果提交给用户。业务逻辑层主要责任为处理客户端提交的请求及联系服务层, 完成业务逻辑层运算;服务层主要负责业务逻辑层的数据供给。分别分析交互过程中数据库信息分类、WEB适配器和数据同步更新等关键技术, 为数据库分布交互提供技术支撑。依据交互框架和关键技术, 给出数据库交互流程:初始化交互界面, 数据库发布对外提供网络服务信息。用户提交交互申请并认证用户身份;用户检索所需资源, 应用服务器绑定用户请求并获取接口信息, 将用户所需信息数据传输至服务接口。用户接收交互结果, 完成一次数据库分布交互。实验结果表明, 上述方法实现后用户满意度高、数据同步更新性能强, 具有实用性。

    关键词：

    万维网技术; 协同数据库; 分布交互;

    Research on Distributed Interactive Simulation of Collaborative Database Based on WEB Technology

    GAO Hui GUI Cai-hong

    Zhixing College, Northwest Normal University Bowen College, 2.Lanzhou Jiaotong University

    Abstract：

    At present, the relevant methods of collaborative database distribution interaction have the problems of low user access satisfaction degree and poor data synchronization performance. In this article, a method of collaborative database distributed interaction based on WEB technology was presented. Based on the application analysis of WEB technology in data interaction, the distributed interaction framework of collaborative database based on WEB technology was constructed. The presentation layer in database interaction framework was mainly responsible for submitting the interactive response result to the user. The business logic layer was mainly responsible for processing the request submitted by the client and contacting the service layer, and thus to complete the operation of business logic layer. The service layer was mainly responsible for the data supply of business logic layer. Respectively, the key technologies such as database information classification, WEB adapter and data synchronization update during the interaction were analyzed, which could provide technical support for database distribution interaction. According to the interaction framework and key technology, the database interaction process was given: the interaction interface was initialized, and the database release information provided external network service. Then, the user submitted the interaction demand and authenticated the user identity. Meanwhile, the user retrieved the required resource, and the application server binded the user request and obtained the interface information. The information required by the user was transferred to the service interface. Finally, the user received the interaction result and completed a databasedistribution interaction. Simulation results show that the proposed method has high user satisfaction degree, strong data synchronous update performance and practicality.

    WEB technology; Collaborative database; Distributed interaction;

    1 引言

    随着互联网逐渐发展与应用, 其为各种数据的集成与交互提供了很多机会[1]。目前多数信息资源交互服务应用的平台及协议等均存在一定差异, 且对外交互的数据形式也存在很大不同, 导致难以落实不同实体数据库之间的资源交互共享。综上, 如何解决语言差异和平台差异及数据差异实现数据交互共享, 成为了互联网领域亟待解决的问题。尽管当前已经存在一些数据交互共享技术, 并于一定程度上解决了信息孤岛问题, 但数据交互共享的内容相对单一, 且交互的程度比较有限[2,3]。鉴于数据交互共享在大众工作与生活中影响较大, 引起了相关研究人员的重视, 部分研究成果如下。

    刘婧[4]等人提出基于元数据的数据交互共享方法。该方法根据分布式分级数据信息共享机制组织资源交互共享, 结合多种元数据标准存在的共性提取交互共享元数据具体规范;通过模块化目录树框架设计并构建反映任务结构特性的元数据集合, 同时描述元数据的生成和访问之间的交互过程。实验结果表明, 该方法规范性强, 但交互后用户访问满意度较低。潘伟丰[5]等人提出基于组合历史的交互式服务方法。该方法根据隶属网抽象复合服务和原子服务两者间存在的隶属关系, 利用单模投影得到服务之间组合关系, 使用骨干网筛选掉无效服务组合之间的联系;根据服务使用度间分布探究服务使用模式;基于服务应用的不同场景, 给出相应服务推荐模式, 实现数据交互推荐服务。实验结果表明, 该方法安全性较高, 但数据同步性能较差, 即数据更新延迟高。刘明超[6]等人提出基于多视图协同的数据可视交互方法。该方法将可视化分析功能分成数据变换、分析和可视映射以及可视绘制四个部分;将协同主题定义为可视分析中参数, 通过观察者角度实现关联视图在数据交互时的协同一致;根据实现结果构建基于图数协同与图图协同的数据交互式协同可视化架构。实验结果表明, 该方法运行过程复杂度较低, 但存在数据同步性差的问题。

    数据交互服务是计算机领域的重点研究课题, 针对当前相关研究方法存在的问题, 提出基于万维网 (WEB) 技术的协同数据库分布交互方法。

    2 基于WEB技术的协同数据库分布交互

    2.1 WEB技术在数据交互中应用分析

    WEB技术为三层C/S, WEB将C/S模型服务器终端分解为一个应用服务器与一个或者多个数据库服务终端, 详细如图1所示。

    图1 WEB模型   下载原图

    根据图1可知, WEB简约了C/S模型的客户端, 仅需安装操作系统和网络协议以及浏览器, WEB服务器端集所有开发与维护工作于一身[7]。WEB应用服务器端将数据信息构建成分布式超文本形式, 利用超文本标识语言, 即HTML以及文本传输协议, 即HTTP表征与组织信息, 以此完成以整个网络为背景的数据存取。

    WEB交互式应用表现方面为:与动态型数据链接的使用者查询;处理动态数据。利用扩充HTML文档, WEB能够采用各种方式完成数据交互。比较常见的方式为以下三种:第一种是提供WEB服务器的扩展性能, 其中包含网关接口CGI与应用编程接口API, 此种方式将客户端交互式访问利用URL传输至WEB服务器, 然后启动与之对应的程序, 将结果转换成动态HTML文档返回给客户端。第二种是将应用程序嵌入至HTML文档, 实现相应解释设备的浏览器下载和执行。第三种方式为根据部件化软件将WEB客户端浏览器作为下载内容, 同时执行WEB端部件化软件中的包容器, 一旦需要数据交互, 浏览器端将下载HTML文档中相应部件并执行。

    2.2 WEB技术下协同数据库分布交互框架

    综合上述WEB数据交互模式, 构建的WEB技术下协同数据库分布交互框架如图2所示。

    图2 WEB技术下协同数据库分布交互框架   下载原图

    框架分析:基于WEB技术的协同数据库分布交互框架主要由表示层和业务逻辑层及服务层等部分组成。各组成部分具体描述为:

    表示层:该层即为客户端与用户交互点, 可为浏览器或者某个应用程序。其通过传输协议HTTP, 将用户请求和有关参数传输至WEB服务器, 利于和业务逻辑层的交互, 同时将响应结果提交给用户, 完成基本操作。

    业务逻辑层:该层为WEB技术下数据交互共享模式核心层。其主要责任为处理客户端提交的请求及联系服务层, 完成业务逻辑层运算, 同时将处理结果传输至客户端。另外, 其还负责处理网络信息资源以及线程等。

    业务逻辑层利用融合和使用不同服务器上的网络服务, 完成数据库分布式WEB交互逻辑。该层的WEB服务器为客户端与中间层的桥梁[8]。客户端提交HTTP请求过程中, WEB服务器收到用户提交的请求, 负责和应用服务器连接。此时, 应用服务器解析用户提交的请求, 同时执行相应的应用逻辑, 调度相应的本地线程和远程WEB协同处理业务, 将处理结果利用WEB服务器传输给客户端。

    服务层:该层主要负责业务逻辑层的数据提供, 例如存储逻辑层返回的处理结果等, 同时屏蔽数据源变化, 进而当数据源有变化时, 仅需修改和连接数据源语句 (更新数据库中数据) , 即能够达到协同数据库分布交互共享的需求。

    2.3 协同数据库分布交互关键技术分析

    在协同数据库分布交互中, 数据分类是其中必不可少的一个环节, 其能够有效提升数据分布交互共享效率和精度, 进而提升用户对交互结果的满意程度[9]。针对协同数据库复杂性, 利用SMOTEBoost算法实现协同数据库信息资源分类。SMOTEBoost算法实际上是一种信息资源合成和集成技术相结合的分类算法, 其利用SMOTE技术实现少数信息数据类样本的人工合成, 以提升少数类样本比例, 同时降低数据倾斜度。SMOTE技术和AdaBoost相结合能够高效防止因赋予少数类样本较大权重导致的过度拟合, 从而提升数据分类器泛化性能。AdaBoost算法依据多轮训练获取若干数据分类器, 同时于每轮训练之后调节各样本权重, 增加错分样本权重, 最终利用加权弱分类器获得最强数据分类器。综上, 可得到基于SMOTEBoost算法的协同数据库信息分类详细过程。

    WEB适配器关键技术:WEB适配器是根据同步WEB服务以及数据同步WEB服务构成, 其中, 结构同步WEB服务主要性能为:使本地数据库中数据信息结构和远端的异构信息库同步更新。同步WEB服务主要功能为:协同数据库各组成部分在实行增、删、改操作之后, 还能够与远端中间数据库保持一致。

    在协同数据库中, 针对各组成部分, 对于数据库的一些请求能够通过本地信息系统中数据接口与协同数据库连接读取数据, 构建相应WEB服务以供数据交互共享调用。因协同数据库WEB服务多少存在差异, 由此以XML统一格式当作数据交互共享时的公开格式, 以此消除异构信息存在的异构性。

    数据同步关键技术:当前数据同步法均存在各自的优势, 但也存在着局限性[10]。在协同数据库分布交互时, 一旦数据产生变化, 则数据库必须同步更新, 该部分功能由适配器中同步WEB服务完成。该部分通过构建源数据库表与本地临时表及副本实现。临时表依据基础表中源库表数据以及源表元数据生成。将相应数据引入中间数据库, 使用者访问时, 主要对中间数据库进行访问, 数据利用完毕后, 将中间数据库中信息与副本对比, 假设产生了变化, 则利用管理器与WEB服务适配器更新数据元数据, 详细如图3所示。

    2.4 WEB技术下协同数据库分布交互运行分析

    基于WEB技术的协同数据库分布交互运行主要由以下两方面组成:一为WEB服务平台利用对网络上下协同数据库信息的发布对外提供数据资源网络服务, 并将服务发布至公共注册处理中心, 以供用户检索。WEB服务还具备用户认证服务, 为各信息资源服务端提供各自调用数据, 并将交互结果返回接口。二为数据库分布交互的客户端利用注册中心检索WEB服务平台供给的各种数据资源网络服务, 再利用应用服务器完成绑定请求, 得到接口信息, 同时将用户所需信息数据传输至服务接口, 用户接收请求结果。详细过程如图4所示。

    3 实验结果与分析

    为验证基于WEB技术的协同数据库分布交互方法可靠性, 将所提方法应用在某电子商务平台, 观察所提方法性能。实验指标为:

    1) 用户满意度

    2) 数据同步性能 (数据库中数据更新实时性)

    实验结果如图5所示。

    图5 不同方法用户满意度对比   下载原图

    根据图5可知, 基于WEB技术的协同数据库分布交互方法用户满意度曲线相较当前方法更为平稳, 且交互结果用户满意度较高。所提方法关键技术中, 利用SMOTEBoost算法实现协同数据库信息资源分类, 有效提升了数据分布交互共享效率和精度, 从而提升用户数据检索交互时的满意度。

    图6 不同方法数据同步性能对比   下载原图

    由图6可知, 当前相关方法数据更新延迟高, 可靠性较差, 而基于WEB技术的协同数据库分布交互方法数据更新延迟平均在0.2μs以下。所提方法通过构建源数据库表与本地临时表及副本实现数据库更新。其中, 临时表依据基础表中源库表数据以及源表元数据生成, 且在数据利用完毕后, 将中间数据库中信息与副本对比, 一旦数据产生了变化, 则利用管理器与WEB服务适配器更新数据元数据, 以此增强数据更新实时性, 即同步性能。

    4 结束语

    针对海量数据分布交互重要意义, 提出基于WEB技术的协同数据库分布交互方法。利用构建交互框架、分析关键技术和交互流程, 实现协同数据库交互研究。实验结果表明, 所提方法整体性能优越, 具有合理性。下一步将以网络数据智能化融合以及管理为研究方向, 优化网络资源, 进一步提升网络服务质量。
