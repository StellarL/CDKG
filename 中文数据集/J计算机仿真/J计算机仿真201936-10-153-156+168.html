    摘    要：

    由于当前方法无法较好监控云存储系统用户访问行为变化,导致云存储系统访问过程中数据丢包率较高、动态授权验证响应较较慢,提出了基于属性的云存储系统动态授权访问控制方法。采用模糊层次分析法度量云存储系统用户访问行为的信誉度,以确保后续进行系统访问用户动态授权验证结果的实时性与可靠性。根据云存储系统用户访问信誉度评价结果,利用AES加密方法分别加密数据和对称密钥,将加密后的数据和对称密钥共同存储到云服务中心,为了减小由于密钥数量过多给云存储系统带来的巨大开销,同时降低计算复杂度,引入密钥分配中心来提供动态授权访问验证密钥,将云存储系统访问密钥与属性集分配给访问用户,并传输这些属性信息,如果用户可以从密钥分配中心成功获取相匹配属性,说明通过动态授权验证,可以访问系统中的数据。仿真结果证明,提出方法动态授权验证响应较快,基本不受系统待访问文件大小和用户数量影响,访问过程能够较好地保证系统数据的完整性。

    Dynamic Authorization for Cloud Storage Verifiable Access Control Simulation

    LI Ling-rui LIU Sheng

    Department of Mathematics and Computer, Wuyi University

    Abstract：

    The current method often leads to high data packet loss rate and slow dynamic authentication response during cloud storage system access. This article puts forward a method to control the dynamic authorized access for cloud storage system based on attribute. Firstly, this research used fuzzy analytical hierarchy process to measure the credibility of user access behavior of cloud storage system, so as to ensure the real-time performance and reliability of the user dynamic authorization verification results of subsequent system access. According to the credibility evaluation result of cloud storage system user access, our research used AES encryption to respectively encrypt the data and the symmetric key. And then, we stored the encrypted data and the symmetric key in the cloud service center. In order to reduce the huge overhead caused by the excessive number of keys and lower the computational complexity, we introduced the key distribution center to provide the dynamic authorization access verification key. After that, we assigned the access key and attribute set of cloud storage system to the access user, and we transmitted the attribute information. If the user could successfully obtain the matching attributes from the key distribution center, the data in the system could be accessed through dynamic authorization verification. Simulation results show that the dynamic authorization verification response of the proposed method is fast. This method is basically not affected by the size of file to be accessed and the number of users. The access process can guarantee the integrity of data well.

    Keyword：

    Cloud storage; Dynamic authorization; Verification; Access control;

    1 引言

    由于云计算服务具有开放、共享的特点,人们可以通过随时访问云存储系统获取需要的信息[1],但是这种没有权限,无需动态授权的访问必然会给云存储系统带来危险,许多有不良企图的访问者冒充正常用户通过访问云存储系统窃取企业或个人的私密信息,轻则造成云存储系统的数据丢失、被删减、篡改,重则可能会导致整个云存储系统的瘫痪[2]。近年来,云存储系统的访问安全受到各行各业的关注,涌现了许多较好的访问控制方法。

    文献[3]提出了基于属性和RBAC相结合的云存储动态授权可验证访问控制方法,该方法描述了RBAC模型中的元素、角色、规则、时间约束,并通过引入权限过滤机制验证云存储系统会话角色的身份,对于通过该检验机制的访问者允许访问,相反,则拒绝访问,实现了云存储动态授权可验证访问。文献[4]提出基于ABE加密机制的云存储动态授权可验证访问控制方法,该方法针对云存储系统中数据结构复杂,且数据量较大的特点,利用ABE加密机制实现对云存储数据密文和访问角色的动态授权验证,实现云存储系统访问权限控制管理。文献[5]研究并提出了基于哈希函数的云存储动态授权可验证访问控制方法,该访问控制过程主要分为云存储系统终端身份动态验证和系统终端数据动态访问控制两部分,主要利用哈希函数、私有密钥和随机密钥验证访问者身份信息,保证云存储系统信息的安全性和完整性。

    上述几种方法在云存储系统访问过程中存在数据丢包率较高、完整性较差、受系统待访问文件大小和用户数量影响较大,动态授权验证响应较慢的问题,提出了基于属性的云存储系统动态授权访问控制方法。

    2 云存储的动态授权可验证访问控制研究

    2.1 云存储系统用户访问信誉模型建立

    建立云存储系统用户访问信誉模型的最终目的是为用户访问云存储系统机密数据时提供安全保障。研究采用模糊层次分析法度量云存储系统用户访问行为的信誉度,以确保后续进行系统访问用户动态授权验证结果的实时性与可靠性[6]。假设系统访问用户访问行为属性有n个证据类型队列,可以通过监测程序监测获得,记作A=〈a1,a2,…,an〉,为了方便计算,将系统访问用户访问行为证据作归一化处理,可得

    A′=〈e1,e2,⋯en〉   (1)

    则可得现系统访问用户访问行为证据数据的对应权值为

    W=〈ω1,ω2,⋯,ωn〉   (2)∑i=1nωi=1   (3)

    根据上述计算可得云存储系统用户访问信誉度评价值为

    Rcu=A′×W=∑i=1nei⋅ωi   (4)

    其中,Rcu∈[0,1],当Rcu=0时,表示系统访问用户的访问行为完全不可信;当Rcu=1时,表示系统访问用户的访问行为完全可信。

    云存储系统用户访问信誉评价主要有四部分构成,分别为用户访问行为直接信誉度、用户访问行为推荐信誉度、用户访问行为综合信誉度以及用户访问行为综合信誉度,其中用户访问行为直接信誉度评价是基于系统访问用户和系统直接交互的历史经验,以及云认证中心根据访问用户当前访问行为变化情况给出的系统信誉评价结果;用户访问行为推荐信誉度其它云存储系统对同一个访问用户的历史访问行为评价;用户访问行为综合信誉度主要是用户访问行为直接信誉度和用户访问行为推荐信誉度的综合评价结果;用户访问行为综合信誉度则是对系统访问历史综合访问信誉度的加权求和结果[7,8]。

    为了便于计算和说明,将云存储系统用户访问信誉模型中的实体按角色划分为四类,假设CU表示目标实体,即想要获取用户访问行为信誉度的系统用户;CSo表示源服务,即想要获取其他用户访问信誉度的服务;CSr表示推荐服务,即向源服务反馈用户访问信誉信息的服务;BM表示实时监控系统访问用户访问行为变化的实体,即云认证中心。

    1)用户访问行为直接信誉度评价值

    已知影响用户访问行为直接信誉度的主要因素有用户申请访问次数、用户申请访问时间间隔、用户申请访问的历史信誉度等,假设RCSo:CUt分别表示时刻t,CSo对CU的信誉度评价值;RBΜ:CUt表示BM的信誉度评价值,为了尽可能体现云存储系统最近的信誉度,防止久远信誉度的干扰影响,引入信誉度时间衰减因子λ,即认为距离当前时刻越近的云存储系统信誉度所占权值越大,可信度越高;T为离散值,表示CSo和CU在信誉度时间衰减窗口win的交互时刻;T′为连续值,表示在信誉度时间衰减窗口win,BM对CU的监控时刻;Tcurrent表示表示当前时刻,则可得用户访问行为直接信誉度评价值为

    2.2 基于属性的云存储动态授权访问控制方法

    根据云存储系统用户访问信誉评价结果,利用AES加密方法分别加密数据和对称密钥,将加密后的数据和对称密钥共同存储到云服务中心,为了减小由于密钥数量过多给云存储系统带来的巨大开销,同时降低计算复杂度,引入密钥分配中心来提供云存储系统数据动态授权访问验证密钥[9,10]。

    1)云存储系统用户的添加与身份动态认证

    已知密钥分配中心KDC是由政府或其它机构等第三方承担的,通常情况下KDC不只有一个,系统中所有的KDC都有不相交的属性集,当一个新用户想要注册并访问云存储系统时,密钥分配中心KDC将云存储系统访问密钥与属性集分配给访问用户,并利用ssh协议传输这些属性信息,假设系统密钥属性信息在传输过程中是绝对安全的,用户可以从KDC成功获取系统访问密钥与属性集,如果通过动态授权验证,即可获得系统中需要的数据。具体实现过程描述如下。

    假设一个数据拥有者Uu希望向云服务中心存储数据M;Uu采用ABE加密算法的生成对称密钥Kd用于加密数据M,并设置云存储系统访问结构S,用于判断那些用户拥有动态授权可以访问系统数据,同时创建了一个大小为m×R的矩阵,其中,m表示S中包含的属性数目;R表示系统访问矩阵,按照以下式(13)加密对称密钥Kd可得

    C=〈R,π,C0,{C1,x,C2,x,C3,x,∀x}〉   (12)

    其中,π表示访问矩阵R到属性i的映射,x∈X,X为访问矩阵R的行元素集合;Cx为常量。

    根据上述计算,将对称密钥Kd的加密密文与数据密文CM共同存储到云服务中心。当用户Uu申请访问云存储系统中的密文时,系统会利用ssh传输协议将对称密钥密文C和数据密文CM发送给用户,如果Uu拥有有效的属性集,即可利用以下式(14)解密得到对称密钥Kd和数据明文M。

    Μ=C0/∏x∈X,Rcu   (13)

    2)系统用户的撤销

    被撤销的用户无权访问云存储系统中的数据,即便这些用户拥有相匹配的属性也无法通过动态授权。Uu必须及时更改希望存储的数据,发送数据更新消息给其他系统用户,将Iu做标记,收到消息的其他系统用户更改包含Iu的数据,防止被撤销用户在没有通过动态授权验证的情况下解密新值C0成功访问云存储系统。
