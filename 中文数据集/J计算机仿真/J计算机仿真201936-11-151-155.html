    摘    要：

    移动网关作为不同终端的翻译器,需要时刻确保软件数据的安全与保密,但传统数据传输方法稳定性能较差,容易发生丢失。为解决上述问题,提出一种基于移动网关的软件机密数据防丢失传输方法。根据移动网关特点,分析软件机密数据传输过程,提出防丢失传输基本要求,其次使用混沌加密算法处理软件机密数据,并得出其密钥与哈希验证码,进一步加强用户隐私信息安全性,利用同态融合算法得到二者密钥,传输到接收节点中,完成软件机密数据的防丢失传输。仿真结果表明,上述方法自适应性强,具有较高的稳定性和可靠性,传输数据完整准确,能有效的防止机密数据丢失,可广泛的应用于现实移动网关中。

    Simulation of Software Secret DataLoss Prevention Transmission Based on Mobile Gateway

    GENG Jiao

    College of Humanities & Information Changchun University of Technology

    Abstract：

    As the translator of different terminals, the mobile gateway needs to ensure the security and confidentiality of software data at all times. The stability of traditional data transmission method is poor and it is easy to be lost. Therefore, a method to prevent the loss of software confidential data based on mobile gateway was put forward. According to the characteristics of mobile gateway, the process of software confidential data transmission was analyzed and the basic requirements for preventing transmission loss were put forward. Moreover, the chaotic encryption algorithm was used to deal with the software confidential data, so that the key and hash verification code were obtained. Furthermore, the security of user privacy information was strengthened. In addition, the homomorphism fusion algorithm was used to get two keys, which needed to be transmitted to the receiving node. Finally, the loss prevention of software confidential data transmission was completed. Simulation results verify that the proposed method has stronger self-adaptability, higher stability and higher reliability. The transmission data is complete and accurate. Meanwhile, this method can effectively prevent the loss of confidential data. Thus, it can be widely applied in real mobile gateways.

    1 引言

    近些年,随着科技地高速发展,社会对网络越来越依赖,各种类型应用软件[1]充斥在人类的日常生活中。同时移动网关[2]作为不同网络、系统或者设备间的沟通桥梁,它可以将接收到的信息重新打包,调整为对方能够接受的方式,再进行数据传输,为工作学习提供了极大的便利。与此同时,这个过程需要时刻保持数据的安全与保密,防止目标设备接受到的信息出现丢失,对用户隐私信息安全造成极大的威胁。那么,如何防止软件的机密数据在传输过程中发生丢失问题,成为现今计算机领域的热点研究课题。

    相关研究学者提出如下解决方案,徐金甫等人[3]建立一种GCM认证加密下数据防丢失方法,将软件机密数据划分在同一安全区域内,并使用Diffie-Hellman协议做加密处理,保证处理后的数据在GCM算法中可以正常运行,进而在传输中能够有效保持机密数据的完整性和安全性。石方夏[4]提出了数据混合加密制度,通过搭建KEM/DEM框架,建立混合密码钥匙算法,使得数据包在加密传输的过程中,可以根据不同时间段反馈的验证密钥来检验信息的完整程度,有效防止数据出现丢失。

    但上述两种方法都不够稳定,较容易受到外来因素的影响,并且二者传输效率低,数据保密性和完整性都不能满足现阶段现实生活的需求。为此,提出一种基于移动网关的软件机密数据防丢失传输方法,使用混沌加密算法进行研究,以有效加强其稳定性,方便运算,减少传输延时时间,同时该方法能够在最大程度上防止软件机密数据在传输过程中的丢失,保证了用户隐私信息的安全,极大地提高了传输效率和精准度。

    2 移动网关下软件机密数据防丢失传输

    2.1 软件机密数据传输中防丢失基本功能设计

    为了防止软件机密数据在传输过程中的丢失,提出其基本功能流程设计图,如图1。

    图1 机密数据防丢失基本流程图   下载原图

    如图1描述,其基本功能内容可分为两部分:

    第一部分:发送终端和接收终端可能相同,也可能不同。二者不同时,发送终端就需要将软件机密数据通过移动网关进行交换处理,将数据转化为对方能够接受的信号再进行混沌加密,并向接受终端传达加密处理后的结果,防止机密数据的泄露。

    第二部分:发送终端将加密好的数据信息进行同态数据融合,之后再传输,保证信息在传输中的不丢失。接收终端得到信息后,首先解密数据信号,再将翻译后的数据转化为接收终端能够解读的信号,得到最初的原始数据模拟信息。

    2.2 软件机密数据防丢失传输实现

    通过软件机密数据传输防丢失基本功能设计,可知,使用混沌加密和同态融合算法来处理软件机密数据传输丢失问题,该过程主要包括数据加密保护阶段、处理数据融合阶段以及机密数据加密防丢失验证阶段三部分,各阶段详解说明如下。

    2.2.1 软件机密数据加密保护阶段

    加密阶段也是软件机密数据防丢失的预处理阶段,这个过程可以加强数据的机密性,若加密数据遭受恶意攻击,对方也无法得出原始信息,有效保证用户隐私信息安全。

    使用mi表示明文,ci表示密文,同时用p表示素数,n为软件机密数据传输中移动网络节点数量,并且p>mi,p>∑i=1nmi,使用离散时间非线性方程[5]构建混沌加密算法,其定义如下

    xn+1=T(xn) (1)

    式中,xn∈V,n=0,1,2,3…,称为数据序列状态,同时T:V→V为映射关系[6],可以将当前状态xn映射为下一序列状态xn+1。假设式(1)的原始值为x0,那么反复利用T,就能获得一个序列{xn,n=0,1,2,3…},这也可以定义为该离散时间动力方程中的任意一条轨迹。利用Logistic映射[7]表达如下

    xn+1=μ×xn×(1-xn) (2)

    式中,μ为映射参量,μ∈(0,4),xn∈(0,1)。并且当μ∈(3.5699456…,4)时,Logistic的映射出现混沌态,换句话说,原始条件x0在Logistic的映射处理下,所呈现的序列{xn,n=0,1,2,3…}是无周期的,同时对原始数据极为敏感,并且无法收敛。

    通过式(2)的Logistic映射混沌函数处理后,得到序列的概率分布矩阵ρ(x),可表示为

    利用ρ(x),比较容易得到混沌序列中一些具有实际意义的函数特性,例如x所用时间的平均值x¯,同时也是混沌序列的轨迹均值,即

    x¯=limn→∞∑i=1nx=∫01xρ(x)dx=0.5 (4)

    这时对于相互关联的函数来说,可以独立使用两个原始值x0与y0,那么混沌序列的相互关联函数c(l)可表示为:

    c(l)=limn→∞1n∑i=1n(x0-x¯)(y0-y¯)  =∫01∫01ρ(x,y)(x-x¯)(Τ´(y)-y¯)dxdy  =0         (5)

    其中,x¯0为原始值x0的平均值,y¯0为原始值y0的平均值,T′表示两个原始值的关联规则系数。

    利用式(5)证明了混沌序列的ACF函数等同于delta函数。

    通过上述过程了解,混沌加密函数不仅拥有较高的确定性,还具有便捷简单的运算形式,对机密数据的原始条件极为敏感,并且还可以对白噪声进行统计,减少外界干扰向量。因此,混沌加密算法可以较好的应用在软件机密数据防丢失传输上。

    在实际运算过程中,加密算法处理的是数字量,所以可以将软件机密数据转化为实数构建的序列{xk},并进行映射处理,转化为整数构建的随机序列来作为混沌加密密钥。常使用的映射手段为,提取xk小数点后的有效数据组成上述整数。根据此提出一种Logistic映射下混沌加密模型,如图2所示。

    图2 Logistic映射下混沌加密模型图   下载原图

    图3为完整的混沌加密、解密算法。为了加强混沌序列的任意性与数据加密速度,选取任意整数间隔M为5较符合现实需求,同时Xk使用Yk小数点后的第四、五、六位是最合适的,有助于提升所选明文的防破坏能力。根据实际训练可知,为了减少加密过中的冗余步骤,可以适当舍弃前两千个数字。

    图3 完整的混沌加密、解密算法   下载原图

    使用上述的混沌加密算法,在数字节融合机密数据时就不需要对样本信息做解密处理,只在移动网络最后sink节点中选取出数据加密结果。在Logistic映射下,只有破解了混沌密钥ki才能获取明文mi,同时ki是网络任意节点与sink节点的共有密钥。因此,使用混沌加密算法能够有效保证软件机密数据在传输过程中的安全。

    2.2.2 数据融合防丢失处理阶段

    假设使用t表示现在时间,用mi,t表示任意节点si的感知信息,且大小为5Byte。同时为了加强软件机密数据的安全性,让移动网络中每个节点都利用混沌加密算法得到密钥和哈希验证消息。数据融合阶段的感知信息形式可用下式进行表示:

    Kt=HM1(K,t) (6)

    ki,t=HM1(ki,t) (7)

    在式(6)与式(7)中,HM1(·)表示为哈希函数,使用的格式为SHA-1,并且得到Kt和ki,t的平均值为15Byte。同时,为了防止加密后的软件机密数据在后续传输中的丢失,要网络各节点的探测信息m上加入一组余数mi,如图4所示。

    图4 机密数据格式图   下载原图

    对机密数据进行加密,得到密文ci,t=E(di,K,ki,t,p),随后将其上传至下一节点,完成数据融合预处理阶段。同时p值的大小是根据Kt、ki,t的值决定的,Kt、ki,t值为15Byte,那么密文ci,t的值为15Byte。

    融合阶段使用的是同态加密函数,因此并不需要对融合的节点进行解密,可以直接得到数据密文,随后再将处理结果上传送到中心节点。设定融合节点Ai能够得到密文c1,t和c2,t,那么根据式(8)对数据同态加密融合结果做处理,即:

    c′t=c1,t+c2,tmodp (8)

    使用上述融合方法能够有效防止加密数据在传输过程中的丢失。

    2.2.3 机密数据加密传输防丢失验证阶段

    根据剩余原则对软件机密数据混沌加密后的防丢失传输效果评估。剩余原则表示如下:

    在数据中拥有N个素数pi>1,i∈{1,2,…,N},其乘积M=∏ipi,为了应对某整数集合{m1,m2,…,mN},在余数方程m=m1modp1,m=m2modp2,…,m=mNmodpN中,是[0,M-1]内的唯一有效解m,那么剩余原则公式可以表示为

    m=(∑i=1Νci×mi)(modΜ) (9)

    其中,ci是剩余原则的系数。假设Qi是公共原则系数,qi是Qi模pi的乘法逆比例元素,pi为剩余原则中的特征素数,即ci=Qi×qi,Qi=M/pi。使用式(10)进行表达为

    qi×Qi=1(modpi) (10)

    将式(10)带入,根据Sink节点计算mf,t的值,随后利用式(11)得出m′f,t,即

    mf,t´=(∑i=1Νmfi,t×qi×Qi)(modΜ) (11)

    其中,mf,t和m′f,t是数据进行防丢失处理前和处理后的评判依据。对比分析二者值的大小,由此评估软件机密数据在传输中是否出现丢失。若mf,t和m′f,t值相同,则数据未丢失;若mf,t>m′f,t那么判断该数据丢失几率较大,由此即可实现机密数据加密防丢失验证。随后利用式(8)对验证部分的加密数据再次融合,完成全部基于移动网关的软件机密数据传输防丢失处理。

    3 仿真研究

    为了该方法的有效性,将从软件机密数据传输的精准度、加密数据防丢失验证、机密数据加密保护效果测试三方面进行仿真对比实验。

    仿真在某学院完成,将PC设备当做移动网关,并使用移动终端对现场软件机密数据进行网上传输。使用的方法有GCM认证加密下数据防丢失(文献[3])、基于KEM框架的混合密钥算法(文献[4])以及基于移动网关的软件机密数据传输防丢失(混沌加密融合方法)三种。

    1)软件机密数据传输的精准度评估

    精准度是评估数据防丢失方法是否有效的基础,没有精准度,网络数据的防丢失处理就不具有任何意义,但是在现实生活中,软件数据传输丢失是难以避免的。实验中对实际节点数据和三种方法的数据传输结果进行对比分析,评估依据为衡量AR,AR=1时,为机密数据传输最理想状态,表示没有数据丢失。精准度对比仿真如表1所示。

    表1 AR衡量值对比 导出到EXCEL

    通过表1可知,文献[3]、文献[4]方法处理后,AR衡度较低,并且在节点数据查询周期较小条件下,二者方法的精准度差,虽然混沌加密融合方法较好,但在查询周期较小时,其精准度同样较低。随着周期逐渐拉大,混沌加密融合方法的AR已经基本保持在80%以上,当周期为30s时,达到90%,而文献[3]、文献[4]两种方法在后续实验中,精准度也一直较低,说明混沌加密融合方法能够最大程度保持软件机密数据传输的精准度。

    假设软件机密数据传输网络中有四个节点{N1,…,N4},在进行加密时,任意节点都预置p1=6,p2=8,p3=12,同时每个节点是探测信息m与其同余数m1,m2,m3的值。在实验过程中,利用式(11)求出m′f=212,若mf=m′f,证明数据未丢失;若融合结果dtamp中mtamp值为218,mtamp=218≠m′f,那么数据丢失。

    根据上述分析,测试本文混沌加密融合方法与文献[3]、文献[4]方法的mtamp值,结果如图5所示。

    通过图像可知,随着q值的增加,三种方法是P(q)也在逐步上升,但是混沌加密融合算法的P(q)值一直小于文献[3]、文献[4]方法,最高值仅为2%,由此表明本文所提方法机密数据加密保护能力最强。

    4 结论

    移动软件的机密数据包含用户的各种隐私信息,如何防止其传输丢失一直是现阶段研究的热点话题。为此提出基于移动网关的软件机密数据防丢失传输方法,通过混沌加密与同态数据融合两种算法加强数据安全性与完整性,并使用剩余原则验证该方法的可行性。通过仿真,证明该方法机密数据传输的精准度最高、加密数据丢失概率极小、机密数据加密保护能力强,拥有良好的鲁棒性,能够增强软件机密数据的安全性,有效防止加密数据的丢失,最大程度的保证用户隐私信息安全,可以广泛的应用于各类移动网络中。
