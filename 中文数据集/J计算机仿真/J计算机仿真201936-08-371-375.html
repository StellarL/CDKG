    摘    要：

    为了解决当前生命周期可控数据自动销毁时间开销较大、销毁率较低、销毁后数据恢复率较高的问题, 提出了基于数据折叠算法的生命周期可控数据自动销毁方法, 该方法在构建包括数据拥有者、计算机系统服务器、DHT网络和数据使用者数据自动销毁模型基础上, 加密生命周期可控数据明文, 采用随机抽样算法抽样数据加密密文, 将抽样后不完整密文上传给服务器;同时利用信任值评估模型评估DHT网络中各个节点的信任值, 并采用秘密共享算法处理数据加密密钥, 并将各个子密钥分发给评估信任值较高的节点上, 完成数据处理工作;在此基础上通过调用分布式文件系统接口将生命周期可控数据随机密文上传给服务器, 采用数据折叠算法对处理后的生命周期可控数据进行可控次数自动销毁操作。仿真结果证明, 提出方法能够实现生命周期可控数据自动销毁, 且降低了时间开销, 提高了销毁率, 同时降低了销毁后数据的恢复率。

    关键词：

    生命周期可控数据; 自动销毁; 数据折叠算法; 时间开销; 销毁率; 恢复率;

    Simulation Research on Automatic Destruction Method of Life Cycle Controllable Data

    CHANG Peng

    Taiyuan University, Computing Center

    Abstract：

    For the current life cycle controllable data, the automatic destruction time is large, the destruction rate is low, and the data recovery rate after destruction is high. Therefore, an automatic destruction method for life cycle controllable data based on data folding algorithm was proposed. Based on building the data automatic destruction model including data owner, computer system server, DHT network and data consumer, this method encrypted the life cycle controllable data plain text. Then, the random sampling algorithm was used to sample the data encrypted cipher-text. Moreover, the incomplete cipher-text was uploaded to the server. Meanwhile, the trust value evaluation model was used to evaluate the trust value of each node in DHT network, and the secret sharing algorithm was used to handle data encryption key. After that, each sub-key was distributed to the node with higher evaluation trust value, so as to complete the data processing. On this basis, the random cipher-text of life cycle controllable data was uploaded to the server by calling the distributed file system interface. Finally, the data folding algorithm was used to perform the automatic destruction with controllable number on the processed life cycle controllable data. Simulation results show that the proposed method can achieve the automatic destruction of life cycle controllable data and reduce the time overhead. Meanwhile, this method improves the destruction rate and reduces the recovery rate of data after destruction.

    Keyword：

    Life cycle controllable data; Automatic destruction; Data folding algorithm; Time overhead; Destruction rate; Restoration ratio;

    1 引言

    计算机技术的快速发展使得越来越多的企业、机构开始使用信息系统处理日常业务信息, 大量业务数据 (包括一些重要的文件、机密的数据) 被存放在计算机系统中, 一旦计算机系统被黑客等破译, 势必会造成这些机密数据的泄露, 给企业、机构等名誉和经济带来不可估量的损失[1,2]。数据安全作为计算机安全问题的核心问题受到了广泛关注, 数据自动销毁作为计算机系统数据安全的一个重要方面, 实现数据自动销毁的远程控制能够有效防止计算机系统服务器失控而造成的重要文件、机密数据泄露问题[3]。通常情况下, 计算机系统服务器中需要对以下几种数据进行自动销毁:

    ①服务器过期数据, 包括计算机系统服务器中长期没有任何用户访问需求的、超过生存时间戳的、由于删除不尽产生的数据包、浏览残留等;

    ②服务器中过多的数据备份, 通常情况下, 云存储都是冗余存储, 含有大量用户信息备份, 特别是对于那些存储时间较长的文档、图片、视频数据, 会产生过多冗余的、失效的数据备份;

    ③服务器中被恶意攻击的数据, 是指计算机系统服务器中那些正在遭受黑客攻击或者有可能受到黑客攻击的用户数据, 还有合法用户的非授权申请数据;

    ④计算机系统服务器中失效节点上存储的数据, 因为计算机系统硬件问题或者通信网络等问题很可能使得存储在失效节点上的用户数据发生泄漏, 给计算机系统服务器和信息安全带来危险。

    针对计算机系统服务器中存在上述几种生命周期可控数据, 业界专家学者提出了许多有效的数据自动销毁方法。比较典型的有文献[4]提出的基于移动Agent的生命周期可控数据自动销毁方法, 该方法通过采用移动Agent检测计算机系统服务器中生命周期可控数据, 将服务器中待上传密文数据的HASH值拼接在数据密文之后, 并以固定周期对数据进行双重检测, 判断服务器中数据是否被非法篡改;在此基础上, 通过预测生命周期可控数据受到攻击的可能性, 销毁那些已被非法篡改的数据或可能被非法篡改的数据。文献[5]提出了基于时间可控的生命周期可控数据自动销毁方法, 该方法通过采用函数变换方法处理计算机系统服务器中的元数据和加密处理后的密文数据;为了提高数据自动销毁的可控性, 定义了服务器中自动销毁数据对象, 使得服务器中任何过期的非法访问数据在触发数据重写程序时对自动销毁数据对象进行确定性永久删除, 完成服务器生命周期可控数据的自动销毁。文献[6]提出了基于同态hash函数的生命周期可控数据的自动销毁方法, 该方法采用同态hash函数生成和管理计算机系统服务器中生命周期可控数据的密钥, 保护数据完整性, 同时尽可能减少数据拥有者所需维护的密钥数量;在此基础上对加密后的数据做分块处理, 将所有密钥和部分数据密文存储在系统服务器中, 利用计算机网络动态变化特性将生命周期可控数据密钥在预设时间到达后自动从网络中彻底删除, 达到生命周期可控数据自动销毁的目的。

    为了解决上述方法在销毁生命周期可控数据时存在时间开销较大、销毁率较低、销毁后数据恢复率较高的缺点, 提出了基于数据折叠算法的生命周期可控数据自动销毁方法。

    2 生命周期可控数据自动销毁方法研究

    2.1 生命周期可控数据自动销毁模型构建

    生命周期可控数据自动销毁模型包括生命周期可控数据拥有者、基于分布式文件系统的计算机系统服务器、DHT网络和生命周期可控数据使用者四部分。其中, 生命周期可控数据拥有者拥有大量生命周期可控数据, 为了防止用户的个人隐私信息发生泄漏, 需要对用户个人信息做加密处理后再上传到计算机系统服务器中;基于分布式文件系统的计算机系统服务器能够根据用户要求对数据执行修改、删除、添加等操作, 为生命周期可控数据拥有者提供存储服务, 方便数据使用者下载、收藏、打印等;DHT网络主要用于存储生命周期可控数据加密子密钥的存储, 如果子密钥的生命周期超过预设时间戳, 系统节点自动删除过期密钥;生命周期可控数据使用者有权将数据拥有者的海量数据用作它途, 但必须严格遵守数据拥有者对生命周期可控数据的隐私保护要求, 不能将拥有者个人隐私信息泄露给他人[7]。

    2.2 生命周期可控数据的处理

    在完成生命周期可控数据自动销毁模型建模基础上, 采用随机抽样算法对生命周期可控数据加密密文进行抽样, 使得计算机系统服务器中的数据密文不完整, 能够有效抵御服务器数据生命可控周期内攻击者的密文破译;利用信任值评估模型评估DHT网络中各个节点的信任值, 能够有效避免恶意节点对生命周期可控数据加密子密钥的不定期删除、子密钥泄露以及子密钥伪造, 完成生命周期可控数据的处理工作[8,9]。具体步骤描述如下:

    1) 生命周期可控数据明文加密以及抽样处理

    假设Skey表示生命周期可控数据密钥;f表示生命周期可控数据密文抽样次数;b表示抽样数据量大小 (比特) ;n表示生命周期可控数据生成子密钥数量;t表示生命周期可控数据重构密钥门限值。计算机系统服务器数据拥有者在客户端输入安全参数k通过以下公式 (1) 对AES256对称加密算法进行初始化, 并利用初始化后的AES256对称加密算法将生命周期可控数据F加密成密文C, 可得:

    setup (k) → (Skey, f, b, n, t)     (1) AESEnc (F, Skey) → (C)     (2)

    根据上述计算, 采用以下公式 (3) 的密文抽样算法对生命周期可控数据完整密文C进行f次抽样缩减得到抽样密文C2、密文C2在计算机系统服务中的位置索引L, 以及不完整数据密文C1。

    DataExtract (C, b, f) → (C1, C2, L)     (3)

    在上述计算基础上, 利用分布式文件系统提供的接口将得到的不完整数据密文C1上传给计算机系统服务器, 在上传完成后, 服务器分布式文件系统将生命周期可控数据存储位置信息URL反馈给数据拥有者。

    2) 生命周期可控数据密钥分发与提取处理

    从DHT网络中任意选取m个节点作为候选节点, 服务器发送随机数据与m个候选节点交互, 根据候选节点反馈数据情况 (包括节点反馈数据是否及时、反馈数据是否正确以及反馈数据是否定期删除三个方面) , 将其中l个具有较高信任值节点的随机种子ID记录下来。以用户u与候选节点id交互为例, 假设fi (u, id) 代表第i次数据交互的信任评价值, 如果候选节点id满足节点反馈数据情况, 则fi (u, id) =1, 如果候选节点id全不满足节点反馈数据情况, 则fi (u, id) =0, 否则根据节点反馈数据满足情况在0～1范围内取值, 统计用户u与候选节点id进行l此交互的信任评价值, 根据网络节点原有信任评价值q可得候选节点id的局部信任评价值vuid, 具体计算公式如下:

    vuid= (∑i=1lfi (u, id) ) /l+q    (4)

    根据上述计算, 当l为0时, 说明用户u与候选节点id未发生交互, 此时vuid为0, 依次类推, 当κ个用户与候选节点id交互后, 可得候选节点id的全局信任评价值Vid为:

    Vid= (∑u=1κvuid) /κ    (5)

    根据上述计算, 采用以下公式 (6) 的秘密共享算法将生命周期可控数据密钥划分为n个子密钥, 子密钥根据ID分发到DHT网络中, 并将抽样密文C2、位置索引L、生命周期可控数据存储位置信息URL以及数据ID封装得到封装体E。

    SSShare (Skey) → ({Skey1, Skey2, ⋯, Skeyn})     (6) Encapsulate (C2, L, URL, ΙD) →E    (7)

    当数据使用者请求访问计算机系统服务器中的数据时, 先通过安全通信信道获取封装体E, 并采用以下公式 (8) 对其解封获得C2、L、URL以及ID, 按照以下公式 (9) 在数据生命可控周期内, 将提取到t个正确的数据子密钥通过秘密共享散发完成密钥重构。

    Decapsulat (E) → (C2, L, URL, ΙD)     (8) SSRecover ({Skey1, Skey2, ⋯, Skeyn}) → (Skey)     (9) Skey={Skey1, Skey2, ⋯, Skeyn}    (10)

    3) 生命可控周期数据密文合成与解密

    在上述计算基础上, 根据以下公式 (11) 和 (12) , 数据拥有者通过URL从计算机系统服务器中下载不完整密文C1并根据L将密文C2与C1合成组成生命周期可控数据完整密文。

    DataDownload (URL) → (C1)     (11) DataRecover (C1, C2, L) → (C)     (12)

    在此基础上, 利用DHT网络中存储的密钥Skey将密文解密, 获得生命周期可控数据明文F, 完成生命周期可控数据处理, 具体计算公式如下:

    AESDec (C, Skey) →F    (13)

    2.3 生命周期可控数据自动销毁

    1) 生命周期可控数据“首”定位

    在此基础上通过调用分布式文件系统接口将生命周期可控数据随机覆写密文上传给服务器, 通过获取待销毁原始目标数据序列{Farray_of_data{0}, N}的长度值N, 确定销毁目标数据序列的“首”位ListDatafirst0、“末”位ListDatalast0以及中间位置Data_Mid{0}, 以该中间位置将销毁目标数据序列划分为前半段和后半段两部分, 具体计算公式如下:

    param={array_of_data{0}}set→{ListDatafirst0, Data_Μid{0}, ListDatalast0}    (14)

    2) 生命周期可控数据“首”写入

    根据生命周期可控数据位一一对应规则分别对ListDatafirst0和ListDatalast0做模2加运算, 完成原始目标数据序列的首次覆写:

    {ListDatafirst0, Data_Μid{0}, ListDatalast0}mod2→{array_of_data{0}}    (15)

    在获取原始目标数据序列的上一次自动销毁过程中的中间数据位置基础上, 从该数据中间位置开始, 向待销毁原始目标数据序列的“首”位方向按照顺序依次查找第一个数据不为0的数据位, 如果该数据位不存在, 则完成数据折叠;如果存在该数据位, 则将该数据位作为新数据序列“末”位ListDatalastj, 新数据序列“首”位ListDatafirstj保持不变, 判断新数据序列“首”位与新数据序列“末”位是否相等, 如果相等, 则数据自动覆写结束, 否则转入以下步骤4) , 具体计算公式如下:

    {array_of_data{j}, Data_Μid{j}}seek→{ListDatafirstj, ListDatalastj}    (16)

    其中, j表示生命周期可控数据折叠次数。

    3) 生命周期可控数据“再”定位

    根据生命周期可控数据序列首位和新数据序列末位, 确定二者之间的中间数据位置, 作为此次数据覆写的中间数据位置, 该中间数据位置将待销毁数据序列首位和新数据序列末位之间的数据划分为前后两段, 具体计算方法如下:

    {ListDatafirstj, ListDatalastj}set→{Data_Μid{j}}    (17)

    4) 生命周期可控数据“再”写入

    将上述步骤 (17) 中生命周期可控数据首位和新数据序列末位按照数据位一一对应规则做模2加运算, 完成生命周期可控数据最终销毁, 具体计算公式如下:

    {ListDatafirstj, Data_Μid{j}, ListDatalastj}mod2→{array_of_data{j+1}}    (18)

    根据上述计算可知, 采用数据折叠的方法能够实现对生命周期可控数据的可控次数自动销毁操作[10]。

    4 结束语

    实现计算机系统服务器数据自动销毁的远程控制对于防止服务器失控而造成的重要文件、机密数据泄露问题具有重要现实意义, 提出了基于数据折叠算法的生命周期可控数据自动销毁方法, 该方法通过利用生命周期可控数据自身存储序列进行覆写实现数据自动销毁, 该方法无需事先生成数据覆写序列, 而是利用数据折叠算法进行数据覆写, 通过仿真测试对比, 充分证明了提出方法的有效性, 能够实现计算机系统服务器中过期数据的快速自动销毁, 同时降低了销毁后数据恢复率。
