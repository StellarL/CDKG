    摘    要：

    对复杂属性环境的数据识别有利于充分挖掘大数据包含的信息与价值,提出一种非关系型分布式大数据挖掘算法。首先,引入分布式的最大频繁项算法模型,在物理分散逻辑下,对非关系型数据应用分布式方法进行数据分析,利用建立候选频繁项的搜索条件降低数据检测次数,并采取划分投影法,计算各频繁项对应的分区编号,将其规划到不同的分区中,从而优化冗余频繁项,然后针对数据库中大数据的数据特征建立各个属性的关联度,根据Boosting聚类方法,把由弱聚类得到的局部模型在每次迭代过程中升级为全局模型,再对其进行区块划分,并利用划分质量重新调整迭代采样率,最终得到聚类结果,完成数据挖掘。通过仿真验证了非关系型分布式大数据挖掘具有显著的准确性、稳定性和实用性。

    关键词：

    复杂属性环境; 非关系型; 大数据; 聚类算法; 分布式最大频繁项算法;

    Complex Relational Environment Non Relational Distributed Large Data Mining Simulation

    ZOU Jing-zhao ZHAO Ning LU Cui-rong WU Lin

    HeBei University of Chinese Medicine Colledge of Economics & Management, Shijiazhuang Tiedao University

    Abstract：

    Data recognition for complex attribute environment is conducive to fully mining the information and value contained in big data. To solve this problem, a non-relational distributed large data mining algorithm is proposed. First, a distributed maximum frequent item algorithm model is introduced. In physical decentralized logic, data analysis is applied to non-relational data using distributed methods. The search condition of candidate frequent items is used to reduce the number of data detection. The partition projection method is used to calculate the partition numbers corresponding to each frequent item. We plan it to different partitions, so as to optimize redundant frequent items. Then, according to the data characteristics of the big data in the database, the correlation degree of each attribute is established. According to the Boosting clustering method, the local model obtained from weak clustering is upgraded to global model in each iteration process. Then we divide them into blocks. Finally, the clustering results are obtained by adjusting the iterative sampling rate by dividing the quality, and data mining is completed. The simulation results show that the non-relational distributed large data mining has significant accuracy, stability and practicability.

    Keyword：

    Complex attribute environment; Non-relational type; Big data; Clustering algorithm; Distributed maximum frequent item algorithm;

    1 引言

    当下社会,科学研究需要大量数据信息作为引导,商业决策与经济发展依赖于数据信息的支持,无论生产还是生活中,大数据技术与应用渗透到各行各业。因此,稳定、精确、适用的大数据挖掘方法成为当下研究的一个重要分支。

    数据量大、数据多样、价值密度低、需要处理速度快是现代大数据应用的主要特征。在数据分析领域,大数据挖掘技术越来越体现出其重要性,由于对大数据进行挖掘能够及时归纳并找到数据信息中包含的重点要素,从而总结出具有价值的信息,对科学探索、商业决策、知识库、医疗、军事等领域具有重大意义。

    2 复杂属性环境非关系型分布式大数据挖掘概述

    2.1 分布式大数据挖掘

    实际生产生活中用途最广,影响最大的数据算法有C4.5算法、k-means算法、Apriori算法、Em算法、PageRank算法、Adaboost算法、KNN算法、Navie Bayes算法、CART算法等,覆盖了分类、聚集、统计学习、关联分析和连接学习等数据识别与数据提取等研究领域。

    传统数据识别与提取算法由于大多采用集中式处理,即存在一个数据中心,大数据仓库模型中其余数据均连接至该数据中心,这种结构自身缺陷导致数据处理效率低下,因此数据挖掘算法受到很多限制。当多节点数据存储汇总到中心节点将耗费大量的带宽,并且汇总过程中可能存在着数据隐私泄露。由此,中心节点的性能成为数据挖掘发展的限制瓶颈。分布式数据挖掘将复杂的计算工作改为局部实现再汇总,有效地缓解存储、计算方面数据增多的压力,降低计算引起的数据挖掘瓶颈。特别是在负载过重属性环境下,采用分布式数据识别与提取能够获得更加显著的性能,由此在大数据识别挖掘方面拥有重要学术价值和使用价值。

    2.2 非关系型分布式大数据挖掘

    特征、位置、方向和长度是复杂属性环境下大数据的四个特征属性。为了更好的实现挖掘方法的准确性、稳定性和适用性,采取依据分散逻辑进行数据规划的非关系型分布式数据库进行数据挖掘,灵活、高效且经济适用。非关系型分布式大数据挖掘首先对大数据特征、位置、方向和长度四个属性关联度进行挖掘,然后再对各个属性的关联度进行聚类分析,从而得到挖掘结果。

    3 复杂属性环境非关系型分布式大数据挖掘关联规则与聚类方法

    3.1 分布式最大频繁项算法模型

    分布式的最大频繁项算法是在Spark 分布式处理框架基础上,结合 DMFIA 算法的优点进行改进,达到深度路径搜索和长度优先超集检验的目的。分布式最大频繁项算法首先产生候选项集合,然后根据前缀序号和长度两个指标将候选项进行排序,排序过程中优先选择前缀序号做处理,再利用长度序列递归计算,最后通过迭代筛选候选项不断缩小候选项集合的大小,同时降低挖掘次数,得出超集。分布式的最大频繁项算法应用FP-Growth算法生成最大频繁候选集,FP-Growth算法通过对事务集合进行筛查得到频繁项集合,同时利用事务集合进行数据过滤,从而把数据集内对应的各个事务映射到频繁项树。处理过程中统计出各个频繁项的频率,与其相应的节点指针,再经过递归计算搜索得到全部频繁项集合。该算法是在Apriori 算法基础上进行的,既减少了对内存的占用,同时也保留了事务中属性之间的关系。Fp-Growth 算法流程图如图1 所示。

    图1 FP-Growth 算法流程图   下载原图

    为了提高分布式最大频繁项算法大量冗余的频繁项子集读取效率,在数据规模超大导致单机所不能承载时,利用划分投影法,将候选项集合做分区处理,采用Partitioner计算分区,这样便实现了将记录分散至彼此不同的分区中,并使得不同处理节点中产生并列分区的候选项子集合。此时以逆序的方式对数据集合内所有事务的记录进行查询,搜索出处于不同分区中的事务记录,将其规划成一个候选项集合,并把它划分至此Item所处的Part分区内。数据投影示例如图2所示。

    图2 划分投影示例   下载原图

    为了降低分布式最大频繁项算法的时空复杂度,需要对数据滤波后的频繁项树做递归处理,依据从下至上的规则搜索出深度路径,形成间接候选频繁项集,对间接候选频繁项集进行循环的超集检验。

    划分投影到不同节点的分区并构建出各自的子频繁项树,将构建得到的子频繁项树存储在每个节点的内存上,采取键值对存储方式,设定支持度比较阈值,分别计算得到各个子频繁项树的支持度,并与比较阈值进行比较,利用递归计算对各个子频繁项树对应的支持度进行比较筛选,得到所有叶子节点至中心节点途中符合要求的节点集,同时将对应路径标记作为最大频繁项的候选项集合,由此,根据分区的事务记录候选项集合,最终得到一个全局的事务记录候选项集合,这个过程即为深度路径搜寻,其处理流程表示如下图3所示。

    图3 深度路径搜索的流程图   下载原图

    在每个分区中形成分区最大频繁候选项集之后,对该集合的每条记录进行一次升序排序,再按照该项集的长度进行二次升序排序, 流程图如图4所示。

    图4 长度优先检验的流程图   下载原图

    综上,分布式的最大频繁项算法分为四个流程:一次过滤、分区处理、形成候选频繁项集、分区合并。一次过滤形成频繁项集,使用频繁项集对原始数据集剪枝,对剪枝后的数据进行投影,划分成多个分区子数据集,形成各自的分区子树。对每个分区采取深度路径选择,将筛选出来的全部路径作为该分区的最大频繁选项集合。然后进行长度优先检验,得到分布式数据集,最后合并,再进行长度优先检验得出最终结果。

    3.2 基于Boosting技术的分布式聚类

    在聚类过程中引入Boosting技术,当进行迭代计算时,针对不同的子数据库,采取由弱聚类得到的局部模型在每次迭代过程中升级为全局模型,再对其进行区块划分,并根据划分质量重新调整迭代采样率,利用之前每次迭代的加权值,集成得到聚类结果。

    5 结束语

    对大数据信息提取挖掘具有重要的研究价值,目前数据挖掘算法对于复杂属性环境下的非关系型大数据挖掘效果不佳,为此本文提出了一种非关系型分布式大数据挖掘算法。该算法引入了分布式最大频繁项算法模型,建立候选频繁项的搜索条件,从物理分散层面对非关系型数据进行分析,并采取划分投影法,用以降低频繁项冗余,并利用Boosting提高聚类精度,通过对频繁项集合的迭代处理,最终得到聚类结果。仿真结果表明,本文方法对非关系型分布式大数据挖掘具有更好的准确性、稳定性和实用性。
