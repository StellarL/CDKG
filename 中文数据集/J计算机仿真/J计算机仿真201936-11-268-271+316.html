    摘    要：

    合理的分发协作任务是处理响应式Web前端缓存协作数据的基础,采用当前方法分发响应式Web前端缓存协作任务时,存在网络能耗高和目标检测率低的问题。将粒子群算法应用到响应式Web前端缓存协作任务的分发控制中,提出响应式Web前端缓存协作任务分发控制方法,通过响应式Web前端缓存协作任务的处理时间、访问时间和传输时间,获得执行时间的评价指标,根据响应式Web前端缓存协作任务的访问能耗、处理能耗和传输能耗得到网络能耗评价指标,结合网络能耗评价指标和执行时间评价指标构建响应式Web前端缓存协作任务分发控制模型,利用粒子群算法对响应式Web前端缓存协作任务分发控制模型进行求解,通过多次迭代获得最优的任务分发控制策略,完成响应式Web前端缓存协作任务的分发控制。仿真结果表明,所提方法的网络能耗低、目标检测率高。

    关键词：

    响应式环球网; 协作任务; 分发控制;

    Responsive Web Front End Cache Collaboration Task Distribution Control Simulation

    Ju Jie

    Zhengzhou University of Science&Technology

    Abstract：

    Reasonable distribution of collaboration tasks is the basis for processing responsive web front-end cache collaboration data. But the current method has high network energy consumption and low target detection rate. Therefore, the particle swarm optimization algorithm is applied to the distribution control for responsive web front-end cache collaborative task, and then the method to control the responsive web front-end caching collaborative task distribution is proposed. According to the handling time, access time and transmission time of the responsive web front-end cache collaborative task, the evaluation indexes of execution time were obtained. Based on the access energy consumption, processing energy consumption and transmission energy consumption of the responsive web front-end cache collaborative task, the evaluation indexes of network energy consumption were obtained. Moreover, the evaluation indexes of network energy consumption were combined with the evaluation indexes of execution time to construct the model to control the responsive web front-end cache cooperation task distribution. Then, the particle swarm optimization algorithm was used to solve this model. Finally, the optimal task distribution control strategy was obtained through multiple iterations. Thus, the control of the responsive web front-end cache collaboration task distribution was completed. Simulation results show that the proposed method has low network energy consumption and high target detection rate.

    Keyword：

    Responsive Web; Collaborative task; Distribution control;

    1 引言

    互联网技术在人们日常的工作、学习和生活中起到了关键的作用,成为人们生活中不可缺少的一部分[1]。人们越来越关注与网站之间的交互性以及资源在互联网中的利用效率。移动设备和技术的飞速发展,使手机成为人们上网的主要移动设备,是人们获取各种信息的主要途径,在当代社会中高效、快速的开发Web应用是Web前端工程师面临的重要问题[2,3]。在网络中存在大量具有计算能力和通信能力的节点,但单个节点的能力有限、覆盖范围小、测量精度低且功能单一,不能独立的完成分发任务的工作,只有多个节点之间在网络中协同工作,才可以完成任务的分发。响应式Web前端缓存协作受资源、规模和拓扑结构的影响较大,需要研究响应式Web前端缓存协作任务的分发控制方法,当前任务分发控制方法分发响应式Web前端缓存协作任务时,存在网络能耗高和目标检测率低的问题,需要研究并分析任务分发控制方法[4]。

    2017年提出的基于任务特性约束的任务分发控制方法根据图论的方法对任务分发问题进行建模,提在相关约束条件的基础上构建协作任务分发控制模型,通过分布式遗传算法求解协作任务分发控制模型,获得任务分配的最佳决策,完成响应式Web前端缓存协作任务的分发控制,该方法分发协作任务时,单个节点消耗的能量较高,存在网络能耗高的问题[5]。2018年提出基于目标信息估计的任务分发控制方法,该方法考虑任务执行能力和局部链式通信能力的约束,构建响应式Web前端缓存协作任务分配模型,在贝叶斯定理的基础上采用任务空间的离散不确定量和连续不确定量,通过一致性协调算法建立冲突调节规则,根据规则求解任务分配模型,实现响应式Web前端缓存协作任务的分发控制,该方法存在目标检测率低的问题[6]。2018年提出的基于Q学习的任务分发控制方法通过Q学习方法选择最优合簇时机,利用回报函数和Q学习函数获得最优簇首切换在并行阶段中的方案,根据目标特征标签,对目标信息做分离处理,完成响应式Web前端缓存协作任务的分发控制,该方法分发任务时存在节点能耗高的问题[7]。

    为了解决上述方法中存在的问题,将粒子群算法应用到任务分发控制中,提出响应式Web前端缓存协作任务分发控制方法。

    2 响应式Web前端缓存协作任务分发控制模型

    响应式Web前端缓存协作任务分发控制方法通过执行时间评价指标和网络能耗评价指标构建响应式Web前端缓存协作任务分发控制模型,具体步骤如下:

    1)执行时间

    响应式Web前端缓存协作任务执行时间包含处理时间、访问时间和传输时间。设ttrans代表的是传输时间;B代表的是无线网络带宽;Strans代表的是节点需要传输的数据量,传输时间ttrans的计算公式如下

    ttrans=StransB (1)

    设taccess代表的是访问时间,与数据包在网络中的大小有关。数据包的写入时间通常情况下与读取时间相同。得到访问时间taccess与存储速度V和访问量Saccess之间的关系如下

    taccess=2SaccessV (2)

    设tproc代表的是处理时间,不同应用中的处理时间都不相同。网络CPU占用率、数据处理时间、内存使用量和CPU类型会对中心节点在网络中处理数据所用的时间产生影响,可以计算中心节点在不同工作状态中完成应用所用的处理时间,通过神经网络进行训练,预测中心节点的处理时间tproc[8]。

    中心节点完成任务所用的最长时间为完成所有任务所用的时间T,时间T的计算公式如下

    Τ=max1<i<n[ttrans+taccess+tproc(i)] (3)

    设φTime(Si)代表的是相对执行时间节点Si的评价指标,其计算公式如下

    φΤime(Si)=ΤΤref (4)

    式中,Tref代表的是参考时间。

    2)网络能耗评价指标

    网络能耗主要包括三部分,分别是访问能耗、处理能耗和传输能耗[9]。

    设d0代表的是设定的标准距离;P0,trans代表的是标准距离d0对应的最小传输能耗。中心节点Si和中心节点Sj对应的空间距离di,j对传输能耗Pi,trans产生影响,传输能耗Pi,trans的计算公式如下

    Ρi,trans=di,j2d02×(4π)2βGtGrλ2×Ρ0,trans (5)

    式中,Gt代表的是中心节点Si在网络中对应的发射系数;Gr代表的是中心节点Sj在网络中对应的接收系数;λ代表的是通讯波长;β代表的是系统功耗因子。

    网络寿命是除了网络能耗的重要指标,用来评价响应式Web前端缓存协作任务的分发控制方法。使节点能量在网络运行过程中保持平衡,可以延长网络的平均寿命。通常情况下利用能量熵理论评价节点剩余能量在网络中的平均性。

    设Ekt代表的是中心节点Sk在t时刻的剩余能量;I(Sk)代表的是Ekt的比例关系,其表达式如下

    Ι(Sk)=log[1p(Ekt)] (6)

    设H(S')代表的是剩余能量Ekt在网络中的平均度,即网络剩余能量在t时刻的能量熵,其计算公式如下

    Η(S')=-∑kt∈Stp(Ekt)log[1p(Ekt)] (7)

    网络剩余能量随着能量熵H(S′)的增大,分布的越平均,网络寿命也随着能量熵H(S′)的增大变长。

    设φcost(Si)代表的是评价指标,用来评价节点Si在网络中的能耗,其表达式如下

    φcost(Si)=di,j2d02×Η(S´) (8)

    3)任务分发控制模型

    结合网络能耗和执行时间对响应式Web前端缓存协作任务分发控制的影响,构建响应式Web前端缓存协作任务分发控制模型M

    Μ=α×max1<i<n[φtime(Si)]+(1-α)×∑i=1nφcost(Si) (9)

    式中,α代表的是调节系数。

    3 基于粒子群算法的任务分发控制

    响应式Web前端缓存协作任务分发控制方法采用粒子群算法求解任务分发控制模型M,得到最优的任务分发控制策略,完成响应式Web前端缓存协作任务的分发控制,具体步骤如下:

    1)设N代表的是粒子群中存在的粒子总数,Pi=(y1i,y2i,…,ydi,…,y1i)代表的是粒子,初始化粒子,获得粒子当前的位置Pbesti,利用适应度函数得到适应度最小的粒子群体最优解Gbest。初始化每个粒子在搜索空间中的飞翔速度。

    2)设Xi=(x1i,x2i,…,xdi,…,xDi)代表的是粒子群中存在的第i个个体;Vi=(v1i,v2i,…,vdi,…,vDi)代表的是第i个个体在粒子群中的飞行速度;Pbesti=(p1i,p2i,…,pdi,…,pDi)代表的是个体在粒子群中的最优历史位置,通过评价函数得到;Gbest=(g1,g2,…,gd,…,gD)代表的是群体最优位置。通过下式更新粒子在搜索空间中的飞翔速度vdi(k+1)

    vdi(k+1)=ω×Vdi(k)+C1rand()×[Pdi(k)-xdi(k)]+C2rand()×[gd(k)-Xdi(k)] (10)

    式中,ω代表的是惯性权重,随着迭代次数的增加,惯性权重ω逐渐变低;C1代表的是加速常数,用来描述粒子的个体经验;C2代表的是加速常数,用来描述粒子的社会经验。

    3)通过下式更新当前粒子的位置

    Xdi(k+1)=Xdi(k)+Vdi(k+1) (11)

    Xdi(k+1)=Xdi(k+1)'+rl-2rl×rand() (12)

    式中,rl代表的是与惯性权重类似的参数,迭代次数越多,参数rl的值越小。

    4)构建惩罚函数E,惩罚函数由距离能量函数和碰撞能量函数构成,惩罚函数E的表达式如下

    E=ωcEc+ωIEI (13)

    式中,EI代表的是距离能量函数;Ec代表的是碰撞能量函数;ωc、ωI均代表的是常数系数。

    距离能量函数EI的计算公式如下

    EΙ=∑i=1Μ(Xi-1-xi)2+(yi-1-yi)2 (14)

    碰撞能量函数Ec的计算公式如下

    Ec=∑i=1Μ∑j=1ΝCij (15)

    将式(14)和式(15)带入式(13)中,将得到的惩罚函数作为适应度函数计算粒子在粒子群中的适应度[10]。如果粒子最优位置对应的适应度大于粒子当前位置对应的适应度,更新Pbesti。

    5)如果最优位置Gbest对应的适应度值高于个体最优位置对应的适应度值时,更新Gbest。

    6)回到2)步骤迭代,直到满足终止条件为止,得到响应式Web前端缓存协作任务分发控制模型M的最优解,完成响应式Web前端缓存协作任务分发的控制。

    4 实验结果与分析

    本次测试的实验平台为Simulink,操作系统为红旗Linux Desktop 4.1 plus。网络检测和能效是评价协作任务分发控制方法的主要指标,响应式Web前端缓存协作任务分发控制的主要目的是获取网络检测和能效之间的平衡,通过网络能耗衡量任务分发控制方法的能效,利用目标检测率检测网络检测性能。

    所有节点在一段时间内的通信能耗为总能耗。设EIX代表的是节点在无线通讯过程中接收数据时需要的能量,其计算公式如下

    EIX=k×Eelec (16)

    式中,k代表的是数据包长度;Eelec代表的是信道能量。

    设Etx代表的是节点传输数据包消耗的能量,其计算公式如下

    Etx=kEelec+d2kεamp (17)

    式中,d代表的是数据包在信道中传输的距离;εamp代表的是数据包在信道中停留消耗的能量。

    根据节点传送数据包和接受数据包消耗的能量,得到节点消耗的总能量Ez

    Ez=EIX+Etx (18)

    5 结束语

    无线网络中存在海量的节点,节点具有处理数据、采集信息和存储数据的功能,促进了无线网络的发展。任务分配控制是处理响应式Web前端缓存协作数据中的重要内容之一,具有重要意义。当前协作任务分发控制方法存在能耗高和目标检测率低的问题。将粒子群算法应用到协作任务分发控制中,提出响应式Web前端缓存协作任务分发控制方法,构建协作任务分发控制模型,并采用粒子群算法进行求解,获得分配响应式Web前端缓存协作任务的最优策略,解决了当前方法中存在的能耗高和目标检测率低的问题。
