    多用户网络拥塞中错误数据实时清理方法仿真

    济南大学泉城学院

    摘    要：

    针对当前网络数据清理方法存在准确性和查全性差的问题, 提出基于RAA的多用户网络拥塞中错误数据实时清理方法。将网络拥塞特性定义为网络延迟敏感性、网络吞吐量敏感性和网络同步突发性, 将拥塞原因定义为当多个客户端共同向一个服务器请求服务时, 服务器会产生大量数据流, 且链路网处理性能会比数据包的传输速率要低。依据网络拥塞特性与产生因素分析, 基于网络拥塞和数据包在该链路通过时延成正比理念, 估计出网络拥塞链路。在拥塞链路节点上采集数据样本之后, 先判断数据异常与否, 假设为正常数据, 则将该数据合并至中间聚集的结果集合中, 反之放至错误数据集合;网络中各节点合并本身全部子节点聚集结果集合、错误数据集合, 将合并结果传输至节点本身父节点;各节点将获取正常数据聚集结果与错误数据集合, 并将错误数据集合中需要清理的数据清理掉。实验结果表明, 该方法数据清理准确率和查全率均较高, 具有一定可靠性。

    关键词：

    网络拥塞; 错误数据; 清理;

    Simulation of error data real-time cleaning method in multi-user network congestion

    LI Xiao

    City College of Jinan University

    Abstract：

    Currently, the method to clean network data leads to low accuracy and recall performance. Therefore, a real-time method to clean the erroneous data in multi-user network congestion based on RAA was proposed. At first, this method defined the network congestion characteristics as the network delay sensitivity, network throughput sensitivity and network synchronous burstiness. And then, our method defined the congestion reason as a fact that when some clients sent the request services to a server together, the server generated massive data streams, and the processing performance of link network might be lower than the transmission rate of data packet. According to analysis of network congestion characteristics and generation factors, network congestion links were estimated based on the idea that network congestion data was proportional to the transit delay of data packets on this link. After collecting data samples on the congested link node, we firstly judged whether the data was abnormal. If it was normal data, we amalgamated the data into the intermediate result set, otherwise we put them in the error data set. Each node in the network merged all its sub-nodes and aggregated the result set and error data set, and then the merged result was transmitted to the parent node. After that, each node obtained the normal data aggregation result and the error data set. Finally, we cleaned the data that needs to be cleaned up in the error data set. Simulation results show that the proposed method has high data clearing accuracy and recall rate. Thus, this method is reliable.

    Keyword：

    Network congestion; Erroneous data; Cleanup;

    1 引言

    当前是一个信息数据爆炸式的时代。科学技术逐渐发展, 多用户已经成为各种类型网络的一种发展趋势[1]。当多个用户共同向服务器请求服务时, 会造成网络拥塞, 而在拥塞中存在错误数据, 将其中的错误数据清理掉, 对疏通数据传输信道和保障网络安全性有着非常重要的作用[2,3]。数据清理和清洗能够提升数据使用者满意度, 当前各领域相关研究成果众多, 但性能均有待完善。

    刘沅昆[4]等人针对配电网中的异常数据, 将现实中配电网数据当作样本, 先研究了配电网数据中的异常数据与丢失的数据产生原因, 再通过小样本数据开展算法研究, 对于配电网运行阻塞异常数据提出基于Kernel Smoothing的异常数据检测清洗法, 对于丢失数据提出基于Pearson相关系数和回归模型融合的丢失数据清洗处理法, 为验证所提方法有效性, 通过6台服务器搭建实验平台, 实验结果表明, 该方法具有实时性, 但对异常数据清洗不够彻底;朱会娟[5]等人对于当前数据清洗方法中存在的灵活性差的问题, 提出利用动态可配置规则清洗数据方法—DRDCM, 所提方法支持多种类型规则之间存在的复杂逻辑计算, 同时支持各种类型脏数据清洗修复, 该方法中包含数据异常检测、修复和转换, 有可配置与可扩展等特性, 过程中, 对DRDCM方法运行过程中的数据异常检测与修复定义和实现步骤进行描述, 然后分析了DRDCM方法能够支持各种规则类型和规则配置的行为, 最后, 实现DRDCM方法分析, 同时利用实际项目数据集对所提方法进行验证, 实验结果表明, 该方法能够高效检测出异常数据, 且能够将动态的可配置规则连续集成在多个数据源及各种不同领域, 同时该方法运行性能不会降低, 但存在数据清洗查全率较低的问题;潘玮[6]等人以关键词共现识别中数据清洗作为研究重点, 依据文献调研, 描述数据清洗概念与对象, 同时分析脏数据生成因素与产生的影响, 以此制定数据清洗步骤及策略, 实验结果表明, 该方法较为简单, 但数据清洗准确性较差。

    上述为电力数据、网络数据和关键词信息数据方面的清洗, 虽然造成几种类型数据异常、错误的原因不同, 但在清洗过程中均存在一定问题。在此, 针对多用户网络拥塞中的错误数据, 提出基于RAA的多用户网络拥塞中错误数据实时清理方法。

    2 多用户网络拥塞中错误数据实时清理

    2.1 网络拥塞特性与产生因素分析

    网络拥塞可大致划分成两类:网络延迟敏感性与吞吐敏感性, 其中, 网络延迟敏感的数据流量存在高同步突发性。

    网络延迟敏感性, 其对象为面向用户的密集型应用数据流量, 例如WEB检索和在线购物等, 其中还包含协调式集群程序更新与短信息, 它们均为网络延迟敏感的数据流量[7,8]。此类数据流量具有软实时性, 假设延迟超过预设阈值, 则会降低用户服务质量或者使集群状态变得不一致。

    网络吞吐量敏感性, 网络中有些后台数据流量呈现吞吐敏感特性, 其中包含大规模数据更新操作和倒排索引表等一系列操作。这些网络流量大多用于大规模数据的提取与组织中, 可以保障网络持续响应的质量。这些操作对于延迟并不具有敏感性, 只需要吞吐率较高。

    网络同步突发性, 网络中划分与聚合等模式应用过程中生成的数据流存在同步突发性, 如图1所示。从图1中可以看出, 因工作节点基本同时完成计算, 同时开始传输计算结果, 使得在比较短的时间内很多数据流共同汇聚至交换机同一接口, 耗尽了交换机内存或导致数据包丢失。即使数据包比较小, 这种现象也是存在的。当前网络数据交换机通常采用共享式浅缓冲框架, 缓冲会被消耗殆尽, 无法吸收同步突发, 错过时限。此种现象也可称为扇入突发现象。

    图1 网络数据同步突发   下载原图

    根据对网络数据流大小与并发性分析结果可知:延迟敏感的网络数据流, 其大小通常会小于1MB, 吞吐率敏感的数据流通常会大于1MB。网络中同步突发的数据流并发性取决于划分-聚合模式, 其中的位数约为一个机柜中服务器的数量。

    综上可知, 在多用户网络中, 吞吐率敏感数据流、网络延迟敏感数据量以及高突发性数据量共同存在。

    多用户网络拥塞详细因素可划分成以下几类:

    1) 当多个客户端共同向一个服务器请求服务时, 服务器会产生大量数据流, 不过链路网处理性能会比数据包的传输速率要低, 以此造成网络拥塞。

    2) 抢占带宽:网络数据中心和客户端间存在长数据流和短数据流及突发查询数据流。因这三种数据流一直存在交叉性传输情况, 由此有长数据流和短数据流共同抢占资源的状况[9,10]。

    3) 入队等待:因交换机缓冲区域的排列队伍比较长, 数据流会比较大, 造成队列太长溢出信道导致延时和丢包, 以此形成拥塞。

    2.2 拥塞链路检测

    依据2.1节网络拥塞特性与产生因素分析, 为了提升网络拥塞中错误数据清理查全率, 在各节点上, 针对一段时间Ttotal, 存在下列等式:

    Τtotal=Τb+Τbl+Τi    (1)

    式 (1) 中, Tb代表网络节点传输数据过程的全部时长, Tbl代表因其他数据流导致的平均阻塞时长, Ti代表节点持续空闲准备传输下个数据流量包的时间。

    动态保持Tb、Tbl和Ti均值, 能够估计出节点n上数据流量Fns, 计算式为:

    Fns=BsΤtotal    (2)

    即:

    Fns=Bs (Τb+Τbl+Τi)     (3)

    式 (2) 和式 (3) 得到的流量值是这段时间平均负载, 此节点上总数据传输速率Fn可表示为:

    Fn=∑sFns    (4)

    综合上述计算, 可得到带宽Cn的估计值:

    Cn=∑sBsΤb+Τbl    (5)

    在网络中可用带宽估计:针对节点上一个数据流s, 其最大支持速率Cns表达式为:

    Cns=Cn-∑Fns    (6)

    式 (6) 中, Fns代表已经存在的网络数据流量速率。

    网络端至端的可用带宽为由源节点至目的地节点路径中瓶颈值, 其中, Cavails=minn∈psCns能够当作各数据流配置速率的上界限。

    依据上述计算与分析, 估计拥塞链路:网络拥塞和数据包在该链路通过时延成正比。如果数据包来临与离开均符合M/M/1排队模型。网络节点n上拥塞Xn能够被估计成:

    Xn=1Cn-Fn    (7)

    依据式 (7) 计算, 新增加的数据流量ΔR导致的网络拥塞链路可表示为:

    ΔXn≈1 (Cn-Fn) 2ΔRn    (8)

    2.3 基于RAA的网络拥塞中错误数据清理

    为了更加准确地清理多用户网络拥塞中错误数据, 利用RAA算法定义链路中的错误数据, 并将其清理掉。

    上述中的RAA算法运行理念为:网络中各拥塞链路上的节点存储中间聚集结果和错误数据集合;节点采集数据样本之后, 先判断数据异常与否, 假设为正常数据, 则将该数据合并至中间聚集的结果集合中, 反之放至错误数据集合;网络中各节点合并本身全部子节点聚集结果集合、错误数据集合, 同时将合并之后的结果传输至节点本身父节点;最终各节点将获取正常数据聚集结果集合与错误数据集合, 将其中错误集合中需要清理的数据清理掉。

    综上, 多用户网络拥塞中错误数据清理详细过程如下:

    1) 处理网络拥塞链路ΔXn中叶子节点Xl

    对于存在拥塞的链路中叶子节点, 先判断本身采样所得数据正常与否, 假设正常, 则放至集合Al中;假设不是, 则放至集合Fl中, 将Al与Fl均上传至父节点。

    2) 处理拥塞链路ΔXn中间节点Xk

    ①Xk在接收到子节点的中间聚集集合和错误数据集合之后, 开始执行RAA聚集;

    ②Xk重复执行步骤1) , 一直到全部子节点数据均处理完毕, 并形成自身的Al与Fl;

    ③中间节点判断本身采样数据, 假设为正常数据, 则与Al合并;假设不是, 则与Fl合并;

    ④针对Fl中各元素Mt, 按顺序和Fl中位置坐标邻近的元素Ms开展相似性对比, 假设相似, 则将Mt与Ms从Fl中清理掉;

    ⑤将Fl和Al上传至自身的父节点。

    针对上述中链路错误数据识别:多用户网络拥塞中正常数据为存在趋势变化且有渐变趋势的数据, 而错误数据即为会使目前数据变化形式产生改变或能够使目前数据产生跳跃性的数据。为了精确清理错误数据, 定义如下向量关系:

    corri, j=bi (t) ⋅bj (t) |bi (t) |22+|bj (t) |22-bi (t) ⋅bj (t)     (9)

    式 (9) 中, corri, j代表多用户网络拥塞链路中错误数据集合, bi (t) 和bj (t) 代表两个读向量, 假设两者不存在相同趋势, 也不在一个范围内, 则corri, j趋近0;假设bi (t) 和bj (t) 为相同趋势时, 则corri, j趋近1。

    3 实验结果与分析

    为验证基于RAA的多用户网络拥塞中错误数据实时清理方法整体性能, 进行一次仿真实验。实验环境为:分布式集群实验平台, 每台电脑配置是一致的, 且均是32位计算机, centos6.4系统, 并有2GB内存, 实验数据取自于2018年11月1日至30日微博数据库。实验指标为:

    ①数据清理准确性

    ②数据清理查全性

    实验结果如下:

    图2 不同方法数据清理准确性对比   下载原图

    数据清理查准率是测试清理方法性能最为重要的指标之一, 可高效验证方法可行性。从图2中可以看出, 基于RAA的多用户网络拥塞中错误数据实时清理方法清理准确率曲线较为平稳, 均在90%以上。该方法为提升数据清理准确率, 在检测网络拥塞链路基础上, 利用RAA算法定义链路中的错误数据, 将网络中各拥塞链路上的节点数据划分为正常数据聚集集合和错误数据集合, 双重检测有效提升了错误数据清理准确性。

    由图3可知, 文献[5]方法数据清理查全率曲线虽然平稳, 但整体不高;文献[6]方法数据清理查全率曲线忽高忽低, 不具备可靠性;基于RAA的多用户网络拥塞中错误数据实时清理方法数据清理查全率曲线不仅较高, 且较为平稳。所提方法在分析网络拥塞特性与产生因素基础上, 估计得到新增加的数据流量导致的网络拥塞链路, 较为全面地考虑了造成网络拥塞的原因, 并就此检测到了拥塞链路, 为增强方法查全性提供了坚实支撑。

    综合上述实验结果可知, 基于RAA的多用户网络拥塞中错误数据实时清理方法无论是在清理精度还是完全性方面, 性能均优于传统方法, 可见该方法具有较好的鲁棒性与适用性。

    4 结束语

    网络正常运行情况下, 多用户造成的网络拥塞现象经常出现, 其中包含部分错误数据。为了使网络更好地服务于大众, 提出基于RAA的多用户网络拥塞中错误数据实时清理方法。利用网络拥塞特性与产生因素分析为拥塞链路检测奠定基础, 基于拥塞链路估计, 通过RAA算法定义拥塞链路中的错误数据, 并将其清理掉。经实验证明, 所提方法整体性能强, 能够应用至实例中。针对下一步研究提出建议为:网络拥塞链路中可能存在冗余数据, 未来可在错误数据清理前, 需要滤除冗余数据, 从而进一步提升错误数据清理性能。
