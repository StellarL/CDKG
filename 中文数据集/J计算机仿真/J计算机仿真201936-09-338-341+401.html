    摘    要：

    采用当前方法计算网络攻击图节点回流时,计算所用的时间较长,计算得到的节点回流与实际不符,存在计算效率低和准确率低的问题。提出基于置信度的网络攻击图节点回流建模方法,分析构成攻击图的元素,通过计算攻击者攻击网络时占有的网络资源的资源状态节点集合,通过节点构成的有向边集合,计算需要消耗对应的攻击成本及攻击行为在网络发生的条件概率,通过概率计算,得到节点置信度分布情况,并分析网络安全威胁在网络的传播路径,利用脆弱点扫描器扫描脆弱点在节点中的置信度,通过单个脆弱点置信度推导出节点在网络的剩余置信度,在威胁传播路径和攻击图构成元素分析结果的基础上构建攻击图节点回流模型。仿真结果表明,所提方法可在较短的时间内准确的计算出节点在网络的剩余置信度,验证所提方法的效率高、准确率高。

    关键词：

    置信度; 网络攻击图; 节点回流; 攻击成本;

    Reflux Modeling and Simulation of Network attack Graph Node Based on Confidence

    YANG Gai-zhen

    HuangGang Normal University

    Abstract：

    At present, when the method is used to calculate the network attack graph node reflux, the calculation efficiency and accuracy is low. Therefore, this paper presented a modeling method for network attack graph node reflux based on confidence level. Firstly, the elements that composing the attack graph were analyzed, and then the resource state node set of network resource occupied by the attacker in attacking the network was calculated. On this basis, the corresponding attack cost and the conditional probability of attack behavior on network were calculated through the directed edge set formed by nodes. Through the probability calculation, the distribution of confidence level of nodes was obtained. Meanwhile, the propagation path of network security threat was analyzed. In addition, the vulnerability scanner was used to scan the confidence degree of vulnerabilities in nodes. Based on the confidence degree of a single vulnerability, the residual confidence degree of node in the network was derived. Based on the analysis results of threat propagation path and elements of attack graph, the reflux model of attack graph nodes was constructed. Simulation results show that the proposed method can accurately calculate the residual confidence of node on network in short time. Thus, the efficiency and accuracy of the proposed method is high.

    Keyword：

    Confidence degree; Network attack graph; Node reflux; Attack cost;

    1 引言

    网络攻击技术的发展历程与计算机网络的发展历程一样,都是从简单逐渐发展为复杂的过程[1]。网络攻击在早期互联网的发展阶段中主要是无目的的攻击,包括利用操作系统漏洞和破解口令等方法,网络攻击之间的关联性较低[2]。网络攻击在当前社会中已经转变为复杂的过程,通常情况下在一次攻击中就存在着关联紧密的一组攻击,如痕迹清除、目标信息收集、开辟后门、弱点信息挖掘分析、攻击实施、攻击行为隐蔽和目标使用权限获取等[3]。攻击技术日益复杂促进了网络安全分析方法的发展,网络攻击图是网络安全分析的主要方法之一,可以得到为攻击目标攻击者选择的路径。通过计算机程序结合网络连接、综合攻击、主机和漏洞等因素可以模拟网络安全状态之间存在的可行变迁,形成完整的、间接的网络攻击图[4]。当前网络攻击图节点回流计算方法存在效率低和准确率低的问题,需要研究网络攻击图节点回流计算方法。

    王辉、鹿士凯、王银城[5]提出基于关联攻击图的置信度计算方法,该方法通过单步攻击、攻击证据和资源节点等参数构建关联攻击图,结合关联关系量化算法和贝叶斯理论推导出节点在关联攻击图中的可达概率,根据计算结果得到节点的剩余置信度,该方法计算[6]攻击图节点在网络中的剩余置信度时,耗时较高,存在效率低的问题。司健、陈鹏、顾宁平等人提出基于深度优先生成算法的网络攻击图节点置信度计算方法,该方法介绍了与网络攻击图相关的概念,分析了逆向生成算法在网络攻击图中的流程,根据分析结果利用基于区间树的规则匹配算法计算网络攻击图节点的置信度,该方法计算得到的结果与实际结果之间存在较大误差,准确率较低。张恒巍、余定坤、寇广等人提出基于状态约减的攻击图节点置信度计算方法,该方法基于目标网络特点和攻击者特点对比独立状态节点在攻击图中的权限,根据对比结果保留权限较高的节点,简约权限较低的节点,去除攻击图中的冗余攻击路径,计算节点回流,该方法中存在大量的算法,计算所用时间较长,存在效率低的问题[7]。

    综上所述,提出网络攻击图节点回流建模方法。

    2 网络攻击图威胁传播路径分析

    网络威胁是一种包含多个步骤的传播过程,基本资源状态在网络威胁传播的过程中会出现一系列的变化。网络状态受资源变化的影响,导致攻击者窃取系统一定级别的权限。攻击者通过窃取的权限可以获取更多的资源[8]。根据变化的基本资源状态网络管理员通常可以推断出威胁在网络中传播的概率较高的路径,根据推断结果采取相应的对策。

    2.1 攻击图

    网络攻击图可以清晰的表述出网络受到攻击时,各个威胁特征进行攻击的具体位置,也可以高效的分析出网络受威胁安全态势,所以对网络攻击图各个集合特征进行分析,显得尤为重要。具体的分析流程如下所示:

    设G代表的是网络攻击图,其表达式如下

    G={V,E,W,P,II} (1)

    其中攻击图中各参数的具体含义如下:

    1)V(G)代表的是非空有限的集合,是由资源状态节点构成的。其中节点反映的是攻击者攻击网络时占有的网络资源。相应节点在资源状态被改变和没有被改变时对应的值分别为True、False,集合V(G)的表达式如下

    V(G)={V0∪VG} (2)

    式中,V0代表的是初始节点构成的集合,用来描述攻击者在初始状态下占有的资源;VG代表的是目标节点构成的集合。

    2)E(G)⊆V×V是由节点构成的有向边集合。如果e1,2=<v1,v2>∈E代表的是节点v1指向节点v2的有向边,那么节点v2为节点v1的后件节点,节点v1为节点v2的前件节点。设e代表的是攻击步骤,在发生攻击时e的值为True,未发生攻击时e的值为False。∀e∈E(G),实施每次攻击步骤都需要消耗对应的攻击成本,设Con(v)、Pre分别代表的是节点v的后件节点集合和前件节点集合。

    3)W代表的是攻击权集合,∀w∈W和节点之间存在关联,通过二元组(h,m)描述。其中h代表的是选择攻击路径攻击节点vi时花费的攻击成本;m代表的是所有攻击路径路过节点vi花费的攻击成本。

    4)P代表的是条件概率,其表达式如下:

    P=(P1∪P2) (3)

    式中,P1代表的是攻击行为在网络发生的条件概率。当前件节点符合相应的条件时才会发生攻击行为,条件概率P1的表达式如下

    P1=(攻击行为发生|Pre(vi)满足条件)

    ={P2:Pre(vi),vi}→[0,1] (4)

    P2代表的是攻击步骤在网络成功的概率;攻击步骤只有在攻击行为发生时才能成功,即vi=True,条件概率P2的表达式如下

    P2=(vi=True|攻击行为发生)

    ={P2:攻击行为发生,vi=True}→[0,1] (5)

    5)II(G)代表的是节点在攻击图中的置信度分布,其表达式如下

    II(G)={π.v×v→[0,1]} (6)

    式中,π(vi)代表的是节点vi被攻击者占有的概率。

    设AND节点代表的是所有节点vi相应的前件节点之间存在的AND操作,节点vi的值只有当全部前件节点实施攻击行为成功后才能设置为True。设OR节点代表的是所有节点vi相应的前件节点之间存在的OR操作,节点vi的值只要存在一个前件节点实施攻击行为成功后就能够设置为True。

    通过上述分析可以采用脆弱点扫描攻击扫描网络的安全漏洞,根据扫描结果获得资源状态节点,分析节点之间在网络存在的依赖关系,标注有向边,得到主要结构。在目标网络状态的基础上估计安全威胁在网络的传播路径,得到网络攻击图,如图1所示。

    2.2 安全威胁的传播路径

    通过分析网络攻击图,获取攻击图中的威胁分布特征,在此基础上,分析网络威胁的传播路径。

    设Path代表的是起始节点和目标节点分别为V0、VG的资源状态在网络攻击图中的变迁序列,其表达式如下

    Path=V0→V1→…→VG (7)

    当变迁序列满足下述约束条件时为安全威胁在网络的传播路径:

    1)变迁Vi对应的下一变迁Vi+1一定是Vi在网络的后件节点,Vi一定是Vi+1在网络的前件节点[9]。

    2)目标节点、初始节点与节点集合之间的交集在变迁序列Vi+1中一定不是空集。

    3)变迁序列在网络的长度有限。

    对于所有符合约束条件的变迁序列,假设威胁在网络的传播路径如下

    V1→V3→V6→V9→V11 (8)

    j V1→V4→V7→V9→V11 (9)

    V2→V5→V8→V10→V11 (10)

    通过图1可知,攻击者通过上述三条路径中的节点可以达到目标,此时:

    如果不考虑节点之间存在的相关性和攻击成本,将节点在网络的值设置为True,攻击者可以通过上述三条路径中的多条或一条完成攻击。

    如果考虑攻击成本,被发现的概率较高,当成本Cost(e7,9)远远小于攻击步骤需要的成本Cost(e6,9)时,考虑成本攻击者会实施攻击步骤<v7,v9>,即攻击行为在网络的发生概率会受到攻击成本的影响,分别提高和降低了实施攻击路径(9)和攻击路径(8)的可能性。

    综上所述,可将威胁在网络的传播路径问题转变为求取攻击图节点回流的问题。

    3 节点回流建模

    根据网络攻击图构建过程和网络威胁在网络的传播路径分析结果构建节点回流模型,通过模型得到网络攻击图节点在网络的剩余置信度,具体过程如下:

    通过t种脆弱点扫描器扫描网络攻击图中节点Hk存在的脆弱点,将不同种类扫描器得到的扫描结果聚类构成集合Vs,其中s=1,2,…,t。合并由扫描器得到的扫描结果,并将合并结果作为节点Hk在攻击图中的置信度向量空间Θ

    Θ=∪s=1tVs={x1,x2,⋯,xm} (11)

    元素之间在脆弱点置信度向量空间Θ中是交叠的,将元素当做坐标轴,且坐标轴之间是互相垂直的,则置信度向量空间Θ为坐标系,在坐标系中存在m个坐标轴。

    设vs代表的是置信度向量空间Θ中存在的置信度向量,用来描述扫描攻击得到的扫描结果构成的结果集Vs,对不同种类扫描器对应的准确率做归一化处理,得到置信度向量vs的模rs,其表达式如下

    rs=||vs|| (12)

    设αvsxi代表的是坐标轴xi和置信度向量vs之间的方向角;cosαvsxi代表的是坐标轴xi和置信度向量vs的方向余弦。∀xi∈Θ,如果坐标轴xi不属于结果集Vs,存在cosαvsxi=0;如果坐标轴xi属于结果集Vs,则这些坐标轴与置信度向量的方向余弦都相同[10]。设向量vs={x1,x2,…,xu},其中u代表的是不同扫描器得到的扫描结果总数,得到方向余弦cosαvsxi的表达式

    cosαvsxi=1/u (13)

    设δ代表的是在各个坐标轴xi上置信度向量vs对应的攻防状态图分析值,其计算公式如下

    δ=rscosαvsxi (14)

    将相同坐标轴上不同置信度向量vs对应的攻防状态图分析值做求和处理,得到下式

    fxi=∑s=1trscosαvsxi (15)

    归一化处理所有求和结果fxi,根据处理结果构建节点回流模型f¯xi

    f¯xi=fxi/∑i=1mfxi (16)

    式中,f¯xi代表的是节点在攻击图中的剩余置信度。

    4 实验结果与分析

    在给定的实验环境中验证网络攻击图节点回流建模方法的整体有效性,网络环境由信任区域、隔离区域和Internet区域构成,其中隔离区域中存在域名服务器、邮件服务器和网络服务器,隔离区域位于完全可信的信任区域和不可信的Internet区域之间,本次测试的操作系统为Windows。分别采用网络攻击图节点回流建模方法(方法1)、基于关联攻击图的置信度计算方法(方法2)、基于深度优先生成算法的网络攻击图节点置信度计算方法(方法3)计算攻击图节点在网络中的剩余置信度,对比三种不同方法的耗时,测试结果如图2所示。

    5 结束语

    计算网络攻击图节点回流可以提高威胁在网络中传播路径预测的有效性,对网络进行高效的保护。通过本文所提方法可以很大程度上改善当前攻击图节点回流计算方法存在效率低和准确率低的问题,该方法可在较短的时间内准确的计算出攻击图节点的剩余置信图,解决了当前方法中存在的问题,为大规模网络的安全运行提供了有力的保障。
