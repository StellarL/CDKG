    摘    要：

    为了提高多链路即时通信网络的可靠性,需要对通信中交互数据异常点进行检测。针对传统的多链路即时交互数据异常点检测中,普遍存在着检测准确率较低、完成时间过长、成本消耗较大等问题,提出多链路即时通信中交互数据异常点检测方法。仅考虑时间序列变换特征,将K-均值聚类和粒子群优化算法相结合对交互异常数据进行聚类,取得聚类目标函数,利用粒子群对其进行求解,得到交互异常数据偏差函数,对多链路即时通信中的异常数据进行偏差统计,确定多链路即时通信中交互数据异常点,实现异常点的检测。实验结果表明,所提方法检测准确率较高、完成时间较短、成本消耗较低,具有一定的应用前景。

    关键词：

    多链路; 即时通信; 交互数据; 异常点检测;

    Simulation of Interactive Data Anomaly Detection in Multi-link Instant Communication

    ZHAO Lei

    Dongchang College, Liaocheng University

    Abstract：

    In order to improve the reliability of multi-link instant network, it is necessary to detect abnormal points of interactive data in communication. Traditionally, the detection accuracy is low, completion time is long and cost is high. Therefore, a method to detect the abnormal points of interactive data in multi-link instant communication was proposed. Only considering the characteristics of time series transformation, we combined K-means clustering with particle swarm optimization algorithm to cluster the interactive abnormal data, so as to get the clustering objective function which was solved by particle swarm optimization. Then the interactive abnormal data deviation function was obtained. Moreover, the deviation statistics was performed on abnormal data in multi-link instant communication, and the abnormal point of interactive data in the multi-link instant communication was determined. Thus, the detection for the abnormal point was achieved. Following conclusion can be drawn from experimental results. The proposed method has higher detection accuracy, shorter completion time and lower cost, so it has certain application prospects.

    Keyword：

    Multi-link; Instant communication; Interactive data; Abnormal point detection;

    1 引言

    随着互联网技术的快速发展,网络中一个出口链路已经无法满足网络数据流量的带宽需求,必须通过增加多个出口链路来扩大带宽,多链路可以有效提高网络出口的稳定性[1,2]。但是由于多链路即时通信中交互数据量较大,数据差异化较为明显,一旦出现数据异常点,就会影响到整个网络的安全运行,对多链路即时通信中的交互数据异常点进行有效检测已经成为通信领域当前阶段面临的主要难题[3,4]。

    目前,主要的多链路即时通信中交互数据异常点检测方法包括免疫神经网络交互数据异常点检测方法;隐马尔科夫模型的交互数据异常点检测方法;关联规则的交互数据异常点检测方法,其中经常使用的是关联规则的交互数据异常点检测方法[5]。在对多链路即时通信中交互数据异常点进行检测的过程中,由于多链路即时通信中交互数据量较大,并且存在着无序性,各个链路中没有统一的标准,利用传统方法进行交互数据异常点阈值设置时,不能形成一个统一的标准,导致多链路即时通信中交互数据异常点检测准确率和效率较低[6]。

    在实际即时通信中,不同链路集成的交互数据随着时间推移其属性变化通常是不同的,数据采集系统从多链路信道中读取的交互数据可能由于信道干扰、测量误差以及外界影响而掺杂部分异常数据。传统的基于经验模态分解和支持向量机相结合的数据异常点检测方法,虽然能够提高数据异常点的检测准确性,但是这个方法需要大量现场数据来充实数据库中评价标准,具有较低的实用性[7]。传统的基于不确定数据流异常点检测方法,虽然能够提高数据流异常点检测效率,但是却忽略了不确定数据流在计算中如何进行识别的问题,降低了数据异常点检测精度。

    针对上述情况,提出多链路即时通信中交互数据异常点检测方法。实验结果表明,所提方法检测准确率较高、完成时间较短、成本消耗较低,具有一定的应用前景。

    2 多链路即时通信中交互数据异常点检测方法

    2.1 获取异常数据偏差函数

    随着时间的不断推移和多链路即时通信中交互数据的连续变化,只靠基于分类的交互数据异常点检测技术在实际应用中是不够的,需要对其历史时间序列进行分类。

    采用以往的K-均值聚类方法进行数据聚类实现过程较为简单,局部搜索能力也较强,但是在多链路即时通信这种情况下,交互数据聚类效果过于依赖K值和聚类初始中心的选择,这样很容易造成局部最优的状态;而粒子群优化算法它自身带有收敛速度快和全局搜索能力较强的特点,将两种方法进行有效的结合,能够弥补传统方法的不足之处[8,9]。

    4 结束语

    基于多链路即时通信中交互数据的时序特征,结合K-均值聚类和粒子群优化算法检测即时通信阶段交互数据异常点,并通过模拟多链路即时通信交互数据验证了所提方法的可行性。与现有的即时通信中交互数据异常点检测方法相比,提出的多链路即时通信中交互数据异常点检测方法在一定程度上弥补了传统检测方法检测准确率和效率较低的不足。该方法在综合考虑不同链路即时通信过程中交互数据的时序特征的基础上,为交互异常数据偏差函数设计相应的检测方案,并通过计算偏差函数的最小值,来实现多链路即时通信中交互数据异常点检测。准确区分即时通信过程中交互数据偏差函数需要对其所在链路有充分的了解,因此,如何有效区分不同链路的通信协议及不同的时序特征是后续需要深入研究的课题。
