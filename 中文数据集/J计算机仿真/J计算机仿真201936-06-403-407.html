    摘    要：

    为了增强移动应用数据库的数据信息完整性, 提升数据利用价值, 需要对数据库中的隐蔽数据进行挖掘与定位。针对现有数据定位方法中存在的定位结果误差较大、耗时较长等问题, 提出一种基于数据流的移动应用数据库中隐蔽数据高效定位方法。根据移动应用数据库隐蔽数据密度分析结果, 利用增量子空间数据挖掘算法计算数据流之间的相关度以及数据流遍历节点的相关系数, 依据计算结果获取数据库中的隐蔽数据;采用数据融合异构异质定位方法, 将得到的隐蔽数据信息转换到相同的区域内进行信息融合, 结合蜂窝融合定位算法, 实现移动应用数据库中隐蔽数据高效定位。实验结果表明, 所提数据定位方法的定位误差与定位耗时明显低于实验对比方法, 验证了所提方法的鲁棒性。

    关键词：

    应用数据库; 隐蔽数据; 数据定位; 增量子空间; 数据流;

    Abstract：

    In order to enhance the integrity of data information in mobile application database and improve the data utilization value, it is necessary to mine and locate the hidden data in database. This article puts forward an efficient method to locate hidden data in mobile application database based on data flow. According to the density analysis result of hidden data in mobile application database, the incremental subspace data mining algorithm was used to calculate the relevance among data streams and the correlation coefficient of data flow to traverse nodes. According to the calculation result, the hidden data in database was obtained. Then, the heterogeneous localization method of data fusion was used to convert the obtained hidden data information into the same area, so as to fuse information. Combined with the cellular fusion location algorithm, the efficient location of hidden data in mobile application database was achieved. Simulation results show that the location error and location time consumption of the proposed data location method is significantly lower than that of comparison method. Meanwhile, the robustness of the proposed method is verified.

    1 引言

    数据库技术是当前信息有效管理与利用的重点技术手段, 移动应用数据库的开发与应用, 为社会生产与生活提供了更便利的信息资源获取方式。数据库能够实现各类信息资源的高效整合, 并根据信息间的差异与关联性进行多类别存储, 信息冗余度低且便于查询与共享[1,2]。数据库中数据采集、录入与共享过程中, 受到人为或系统因素影响, 会产生部分数据隐藏情况, 造成部分信息缺失, 降低数据的利用价值[3]。数据库中隐蔽数据的挖掘与定位是当前大数据领域内的重点内容, 受到了很多相关专家与学者的高度重视, 目前已有一些较好的数据定位方法[4]。

    文献[5]提出一种基于遗传算法的目标数据定位方法。结合数据搜索工程思想构建目标数据的染色体编码方式, 并计算各染色体的适应度值, 利用自适应遗传算法搜索数据库内的高适应度数据, 完成目标数据定位。该方法具有一定的实用性, 定位误差率较低, 但定位过程能耗较高。文献[6]提出一种基于数据流控制的数据定位方法。该方法通过对数据流进行分析, 将原始数据与移动数据相结合, 采用卡尔曼滤波算法对数据进行定位, 该方法定位准确度较高, 但是定位所需时间过长。文献[7]提出一种基于卷积神经网络的目标数据定位方法。首先对数据库中的数据进行特征提取, 获取不同位置数据的特征值, 对不同位置数据信息进行SVM训练, 提取数据中所包含信息, 根据不同层次的信息融合结果, 确定目标数据位置。该方法的定位过程耗时较短, 但定位结果的误差较大。

    针对上述方法中存在的问题, 提出一种基于数据流的移动应用数据库中隐蔽数据高效定位方法, 并通过与其它方法的对比实验, 验证了所提出的隐蔽数据定位方法的性能优越性。

    2 移动应用数据库中隐蔽数据定位原理

    在对移动应用数据库中隐蔽数据进行定位前, 利用相关子空间刻画局部数据集的数据元素分布特征, 并根据集合中数据元素的密度、标准距离与异常因子的计算结果, 结合高斯误差函数计算数据的离群概率, 综合K近邻算法与高斯误差函数[8]完成隐蔽数据提取, 进而实现数据定位。具体过程如下所述:

    设移动应用数据库中某一d维数据集为DS, 数据属性集FS={A1, A2, …, Ad}, xij代表第i个数据元素obji在第j项属性上的对应值, 其中i=1, 2, …, n, j=1, 2, …, d。则对于任一子空间S⊆FS与数据元素o∈DS的近邻元素, 若S代表相关子空间, 则N (o, S) 为非均匀分布;否则, N (o, S) 为均匀分布。

    不均匀分布的子空间能够反映出隐蔽数据所含信息, 即通过相关子空间可以获取隐蔽数据的信息, 子空间度量与寻找是对应数据集中隐蔽数据挖掘与定位的关。

    对于数据元素o∈DS, 设存在相关数据集S1∈DS且o∈S1, 元素o在数据集S1中的概率分布距离为pd (o, S1) , 则pd (o, S1) 包含于以o为中心的球体中, 球体半径与元素o相对于集合S1的概率分布距离相等。通过集合中的各数据元素相对于集合的概率分布距离可估计集合S1中的数据元素密度值, 计算式为

    pdens (o, S1) =1pd (o, S1)     (1)

    其中, pdens (o, S1) 表示集合中数据元素的密度值。设S∈S1, 且元素o到子空间S的距离集合趋近于高斯分布, 则o与S的距离近似于标准差的计算过程, 计算公式为

    δ (o, S) =∑o∈Spd (o, S1) 2|S1|    (2)

    其中, δ (o, S) 代表o与S的距离值, |S1|代表集合S1中的数据元素量。

    根据局部数据集的子空间定义向量来刻画属性维度的局部稀疏性差异, 并获取局部数据集合中的隐蔽数据, 设obj的子空间定义向量维度为vi, obj的j维属性相对的局部稀疏因子为λij, 数据元素s与o在相关子空间S内的欧式距离为dS (o, s) 。则obj在S中的归一化距离δS (obj, LDS (obj) ) 的计算式如下

    δS (obj, LDS (obj) ) =∑o∈LDS (obj) dS (obj, o) 2|LDS (obj) |= (∑j=1dvi⋅λij)     (3)

    在obj的S空间中, obj的临近元素loc～obj∈LDS (obj) , 若loc～obj为第m个数据元素, 则其相对于LDS (loc～obj) 的归一化距离为

    δS (loc∼obj, LDS (loc∼obj) ) = (∑j=1dvi⋅λmj)     (4)

    计算obj在S中的概率异常因子PLOF, 计算公式为

    ΡLΟFS (obj) =δS (obj, LDS (obj) ) Eo∈LDS (obj) δS (o) -1    (5)

    式中, E代表LDS集合中数据元素的平均值, δS (o) 代表标准化处理后的δ (o, S) 值。

    在obj的相关子空间S内, 近邻元素loc～obj∈LDS (obj) 的概率局部异常因子计算式为

    ΡLΟFS (loc∼obj) =δS (loc∼obj, LDS (loc∼obj) ) Eo∈LDS (loc∼obj) δS (o, LDS (o) ) -1    (6)

    对于异常因子PLOF, 结合高斯误差函数来描述数据元素的离群概率, 进而获取隐蔽数据元素, 元素离群度计算式为

    Factor (obj) =max{0, erf (ΡLΟFS (obj) 2Eo∈LDS (obj) ∪obj[ΡLΟFS (o) ]2) }    (7)

    其中, erf ( ) 代表高斯误差函数计算。通过上述计算得到局部数据集中的隐蔽数据元素离群度后, 结合LSH策略与K近邻算法, 获取对应数据集合中的隐蔽数据, 进而实现移动应用数据中隐蔽数据的定位。

    综上所述, 即为数据库中隐蔽数据定位的原理, 得到隐蔽数据的定位结果准确度较高, 但计算过程复杂且计算量较大。

    3 移动应用数据库中隐蔽数据高效定位方法

    3.1 数据库内隐蔽数据挖掘

    通过对移动应用数据库隐蔽数据密度进行分析, 利用增量子空间数据挖掘方法[9]来获取隐藏数据相关度与数据流经过节点的相关系数, 完成数据库中隐蔽数据挖掘。具体过程如下:

    假设, 移动应用数据库中由m个数据构成, 第k个数据的密度为φ (k) , 已经知道的需要挖掘的隐蔽数据流i′的大小为σ (i) , 则隐蔽数据流的特征集为D (i) , 给出隐蔽数据流与移动应用数据库的相关度L (k, i′) 计算公式为

    L (k, i′) =exp[ (∑i=1m (φ (j) -φ (k) ) ) 1+σ (i) ×aD (i) ]    (8)

    式中, a代表隐蔽数据的相似度因子, 隐蔽数据的相似度因子与移动应用数据库的平均数据流大小存在一定的关系, 假设, 第k个数据有n个数据流, 则隐蔽数据的相似度因子为

    α={1  |∑k=1nσ (k) n-σ (i) |>Τ0.5 |∑k=1nσ (k) n-σ (i) |<Τ0  |∑k=1nσ (k) n-σ (i) |=Τ    (9)

    当隐蔽数据流与移动应用数据库的相关度最大时, 隐蔽数据流在数据库中的可能性就最大, 但是当数据库中隐蔽数据流相关度区分不大时, 对隐蔽数据挖掘就会越困难。采用多增量空间隐蔽数据挖掘法来挖掘出隐蔽数据流所在的路段。

    假设, 当隐蔽数据流经过k个路段时, 利用下式给出, 隐蔽数据流经过路段时的痕迹系数G (k, i) 为

    G (k, i) =ln (D (i) Μk+σ (i) Μk) ×exp (1+δ (k) )     (10)

    式中, Mk代表路段上隐蔽数据节点数量, δ (k) 代表路段上两个隐蔽数据节点之间的方向系数。

    通过式 (10) 可以确定隐蔽数据是否经过此路段, 当可以确定隐蔽数据经过路段时, 根据计算隐蔽数据经过节点和所需时间的相关系数来确定隐蔽数据流位于路段中的哪个节点, 利用式 (11) 给出, 相关系数表达式为

    3.2 基于数据流的隐蔽数据高效定位方法

    以3.1小节获取的隐蔽数据信息为基础, 利用隐蔽数据的融合异构异质定位将信息转换到相同的区域内, 引入融合信息理论与蜂窝定位算法[10], 完成移动应用数据库中隐蔽数据高效定位。具体过程如下:

    设定数据基站1的坐标位置为 (0, 0) , 移动台的位置设定为 (500, 500) , AOA的测量值设定为θ1, TOA的测量值设定为τ1, 则利用式 (13) 给出如下线性方程组

    {τ1=x2+y2θ1=arctan (yx)     (13)

    传统算法主要是根据线与线之间的交叉位确定目标位置, 在传统算法的基础上, 结合参数置信度, 将传统测量参数的线, 扩展为可行区域, 将线与线的交叉转化为区域交叉, 在此基础上引入其它传感器的测量参数, 可以有效缩小移动区域。通过测量可知, 获取足够多的测量参数, 就可以更加精确移动台的位置, 即

    {τ1-Δτ≤x2+y2≤τ1+Δτθ1-Δθ≤arctan (yx) ≤θ1+Δθ    (14)

    综上所述, 得到一种改进后的蜂窝融合定位算法, 下面给出遗传算法的具体步骤:

    1) 设定迭代次数, 将所有染色体进行初始化。

    2) 分别计算不同染色体的适应值。

    3) 根据获取的适应度将染色体进行遗传或变异。

    4) 检查当前迭代次数是否是最大迭代次数, 如果是, 将所有染色体的结果进行平均值计算获取定位结果, 否则, 返回步骤1) 。

    以下给出遗传算法适应度值eval (Xi) 计算式为

    eval (Xi) =∑k=1sλkδk    (15)

    式中, s代表特征参数的维数, λk代表第k个特征参数的信度系数, 为取值在[0, 1]之间的随机数, δk代表所在位置区域适应度。

    将不同位置的区域进行叠加, 尽可能融合更多的定位信息, 完成移动应用数据库中隐蔽数据高效定位

    Δa=λkeval (Xi) * (τ1+Δτ)     (16)

    5 结束语

    在移动应用数据库中数据定位的研究过程中, 针对当前定位方法存在的不足之处, 提出一种基于数据流的移动应用数据库中隐蔽数据高效定位方法。通过数据密度分析与数据流遍历节点的相关系数计算, 完成数据库中隐蔽数据的初步挖掘;综合改进的蜂窝融合定位算法, 实现隐蔽数据的定位。实验结果表明, 所提出数据定位方法的定位误差率较小, 定位时间较短, 且能耗较低。
