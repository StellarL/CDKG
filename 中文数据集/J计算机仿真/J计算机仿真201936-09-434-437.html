    摘    要：

    针对当前数据流推荐存在安全性差和精确度低的问题,提出基于频繁项集的电商多用户数据流混合推荐方法。将蚁群算法与属性相关分析融合,将蚁群收敛所至路径判断为存在安全隐患的路径,对路径中存在安全隐患的数据进行判断,确定最终异常值并去除。基于数据流安全分析,引入增量挖掘算法,采用次频繁项索引表更新频繁树,并利用压缩FP-树与矩阵技术对新频繁树的频繁模式进行挖掘。根据数据流频繁模式挖掘,将不同数据流分类至相应主题组。基于频繁项集计算多用户访问相似程度,同时找到用户最近邻平时访问数据。结合主题抽取、最近邻访问及多用户共同兴趣相似度计算,将最符合用户的数据流推荐给用户,实现用户数据流混合推荐。实验结果表明,上述方法推荐过程安全性能好,且推荐结果准确度高,是一种切实可行的数据推荐方法。

    关键词：

    频繁项集; 多用户; 数据流; 推荐;

    Multi-User Data Stream Hybrid Recommendation Simulation Based on Frequent Itemsets

    WANG Pei-pei HU Wei-wei SUN Li-na

    Minsheng College, Henan University

    Abstract：

    At present, the data stream recommendation leads to low security and low precision. Therefore, this paper focuses on a method of hybrid recommendation for e-commerce multi-user data stream based on frequent item set. The ant colony algorithm was integrated with the attribute correlation analysis. And then, the path of ant colony convergence was judged as the path with security risk. The data with security risk in the path was judged, so that the final abnormal value was determined and removed. Based on the data flow security analysis, the incremental mining algorithm was introduced and the frequent tree was updated by the sub-frequent index table. Meanwhile, the compressed FP-tree and the matrix technology were used to mine the frequent pattern of new frequent tree. According to the frequent pattern mining of data flow, different data flow was classified into corresponding topic groups. Based on frequent item sets, the degree of similarity of multi-user access was calculated and the average access data of nearest neighbors to was found at the same time. Combined with topic extraction, nearest neighbor access and multi-user common interest similarity calculation, the data flow that best met user's need was recommended. Thus, the hybrid recommendation for user data flow was achieved. Simulation results show that the proposed method has good safety performance of recommendation process and high accuracy of recommendation result. This method is a feasible method of data recommendation.

    Keyword：

    Frequent item set; Multi-user; Data flow; Recommendation;

    1 引言

    互联网不断普及,电商在大众日常生活中逐渐占据了很重要的位置。但相关网站有海量商品数据信息,顾客在其中寻找感兴趣的信息比较困难[1]。此种情况下,数据流推荐应运而生,已经成为电商发展中的重要探讨内容。

    电子商务推荐为电商迅猛发展下的产物。从理论方面来讲,电商推荐存在的学术价值很高,就实际应用方面而言,研究电商推荐能够为企业带来非常高的经济价值[2,3]。电子商务发展中必不可少的即为顾客这一元素,能够为顾客提供更满意的服务,就可以占得先机,提高自身价值。由此可见,电商多用户数据流推荐研究十分必要。

    王斐玉[4]等人针对Web挖掘网页推荐相关技术在网络中的应用,将其与BP神经网络相融合,并应用至电商网页推荐中。过程中,对用户检索的Web日志实行相应处理,并筛选出5种具有代表性的特征。将所得特征当作BP神经网络输入数据,通过聚类识别出最终的网页优先级。基于优先级将排列在靠前位置的推荐给各用户。肖会敏[5]等人指出随着电商不断发展,其中的信息过载现象成为了该行业研究重点,具有个性化的推荐也随之成为了该领域的热点内容。利用协同过滤技术实现电商数据个性化推荐过程中,在协同过滤技术中引入时间指标与位置指标,利用遗忘函数对过滤算法进行优化,实现最终的数据推荐。艾丹祥[6]等人认为数据推荐是电商发展的关键途径,在C2C环境下,商品供应者为多个,提出一种三维推荐方法。考虑买卖双方和商品三者间存在的相关性,为用户推荐相应数据。过程中,依据卖家特征对卖家之间的相似程度进行计算,利用相似程度计算结果与销售间存在的关系补充三维推荐中评分数据。基于推导出的评分值对卖家相似程度进行计算,找到具有相似产品爱好的最近邻。利用三维预测模型,对卖家与商品组合进行评分,依据评分为买家推荐数据。

    电商发展过程中,多用户数据流推荐必不可少。上述相关研究成果在推荐过程中对安全方面考虑比较少,存在安全隐患,且推荐准确性也不是很强。为此,提出频繁项集的多用户数据流混合推荐方法。

    2 基于频繁项集的多用户数据流混合推荐

    2.1 数据流异常检测

    为了保障数据流推荐安全性,需要先将存在安全隐患的数据清除掉[7,8]。利用蚁群算法对数据流异常进行检测,详细过程如下:

    1)设定蚁群算法运行相关参数,其中包含蚂蚁数量和信息素挥发系数等。

    2)对路径信息素参量进行初始化,同时设计创建空表S。

    3)将相同数量蚂蚁均衡分布在DG头节点。

    4)每只蚂蚁基于路径选择规则在DG上筛选出一条路径L。

    5)对路径L上的信息素进行更新。

    6)转至步骤4),一直到收敛或满足算法停止条件。

    7)选择信息素比设定阈值大的全部路径,并获取路径上各节点存在安全隐患的数据,依照要求保存或者修改S。

    上述步骤中:

    1)步骤2)中,DG上各边ei(1≤i<E_number)的信息素可表示为式(1)

    γi=1E_number (1)

    式中,E_number代表DG中边数量。

    2)步骤2)中,S的表达式为

    S=(T,A,V,M) (2)

    式中,T代表蚂蚁遍历路径元组ID,A代表目标属性名称,V代表元组中包含的目标属性值,M代表元素中A的安全隐患值。

    3)步骤4)中,设置节点v,蚂蚁选取邻近边ei概率可利用式(3)计算

    Ρi=γiηi∑γiηi (3)

    式中,ηi代表ei的启发因子值,其表达式为

    ηi=1su(ei),∀ei∈Ι (4)

    式中,I代表v下一条邻接边集合,su(ei)代表边(vh,vk)支持度。由式(4)可知,边(vh,vk)频繁程度越低,ei启发因子值就越大,选取该条路径概率也就越大。

    4)步骤5)中,最近经过的路径L中各边信息素可利用式(5)更新,剩余边信息素可利用式(6)更新。

    γi(t+1)=(1-ρ)γi(t)+γi(t)/Q,∀ei∈L (5)

    γi(t+1)=γi(t)ηi∑γi,∀ei∉L (6)

    式中,ρ代表信息素的挥发率,该值可高效控制蚁群收敛速度。Q值表达式为

    Q=∑su(ei)γi(t+1),ei∈L (7)

    其中,Q值越小,则表示在L上剩余的信息素就越多。

    5)步骤7)中,在最新选取的路径L中,设定元组t中包括路径L中全部顶点。对L中各属性安全隐患值O进行计算,即计算O(Ai,t),Ai∈A,再得到最小O(Ak,t)值,将与所得值相应的元组有关数据保存至S中。在表S中查询是否存在s:(Ak,s(T))=(Ak,t),s(A)=Ak,s(V)≠t(Ak),假设存在s同时有O(Ak,t)小于s(M),那么删除s。

    综合上述内容,将蚁群算法与属性相关分析有效融合,将蚁群收敛所至路径判断为存在安全隐患的路径。对路径中的安全隐患数据进行判断,确定最终异常值并去除。

    2.2 频繁项集模式挖掘

    以2.1节异常数据检测清除为基础,对数据流项集模式进行挖掘。

    定义如下符号内容:D代表原数据流数据库,T′代表新增数据项,U代表更新之后数据库,I′代表数据项,d′代表D数据数量,t′代表T′数据数量,SD(I′)代表I′在D中计数,ST′(I′)代表I′在T′中计数,SU(I′)代表I′在U中计数。c′代表上一周期扫描数据库处理过的新数据数量,I-I代表更新T′处理的数据项集合,B-I代表更新D处理的数据项集合,R-I代表判断是否具备频繁特性的数据项集合,P-T代表原数据库中需要做一定处理的数据集合。

    将频繁项集模式挖掘输入定义为:异常检测后的数据库,原频繁模式树和相应项头表,频繁阈值上下限Su、Sl,原数据库次频繁项表以及新增项。

    将频繁项集模式挖掘输出定义为:更新之后数据库呈现出的频繁模式。

    频繁项集模式挖掘详细过程如下:

    1)利用2.1节得到数据流安全值。

    2)对新增项进行扫描,得到全部数据项和计数。

    3)依据阈值上下限将新增数据集合中数据项划分成三部分:频繁数据项、次频繁数据项与非频繁数据项。

    4)针对在原数据库中为频繁项的数据进行如下操作:

    ①计算SU(I′):

    SU(I′)=SD(I′)+ST′(I′) (8)

    ②假设SU(I′)/(d′+t′+c′)≥Su,设定SD(I′)=SU(I′),同时将I′引入至I-I数组中;假设Sl≤SU(I′)/(d′+t′+c′)<Su,在项头表中将I′删掉,设定SD(I′)=SU(I′),将I′引入至次频繁项列表;反之在项头表中将I′删掉。

    5)针对原数据中的次频繁项I′,进行以下操作:

    ①计算SU(I′)=SD(I′)+ST′(I′)。

    ②假设SU(I′)/(d′+t′+c′)≥Su,将I′引入至I-I和B-I数组,同时将索引表中与I′相应的数据标识引入至P-T中,保持P-T中元素具有唯一与顺序两大特性;假设Sl≤SU(I′)/(d′+t′+c′)<Su,设定SD(I′)=SU(I′),对次频繁项计数进行更新;反之,将I′从次频繁项列表剔除。

    6)针对原数据库中的非频繁项,即数据流频繁性不强的,但在新增数据中为频繁项或者次频繁项的I′,将其引入至R-I。

    7)假设t′+c′小于设定阈值f′或R-I是空集,终止该步骤;反之,进行如下操作:

    ①重新对数据库进行扫描,获取R-I中的I′在原数据库计数SD(I′)。

    ②计算SU(I′)=SD(I′)+ST′(I′)。

    ③假设SU(I′)/(d′+t′+c′)≥Su,则将I′引入至I-I和B-I中,同时将扫描的整个过程中得到的数据标识引入至P-T中;假设Sl≤SU(I′)/(d′+t′+c′)<Su,设定SD(I′)=SU(I′),并将与I′相应的标识引入至次频繁列表;反之,忽略I′。

    8)将B-I中数据项依照计数做降序排列操作,加入至项头表尾端。

    9)I′为B-I中数据项,P-T中数据项为元数据库中需要进行处理的数据,针对这些数据假设频繁模式树中不存在相对应分支,那么将I′插入至该分支尾端,同时设定计数为1;反之,对节点I′计数做加1操作。

    10)针对包含I-I中I′的新增数据,处理的方式如步骤9)所示,同时更新与次频繁项相应的数据。

    11)假设t′+c′大于预设阈值f′,设定d′=d′+t′+c′,c′=0;反之,c′=c′+t′。

    12)获取新频繁项集中的两个频繁项间计数,将以此构造出的计数集合利用矩阵Aϑ表示。

    13)最后利用压缩FP-树与矩阵技术对新频繁树的频繁模式进行挖掘。

    2.3 多用户数据流混合推荐

    为了有效提升推荐结果的精确度,在数据频繁模式挖掘基础上,结合主题抽取与多用户共同兴趣相似度计算,将最符合用户的数据流推荐给用户。基于2.2节挖掘出的数据流频繁模式,对于频繁项集簇中全部候选项,最优主题词不仅要具备很高的代表性,还要具有区别于其它数据的可辨别性。

    式中,wi代表单词tki属性权重值,tfi代表tki在高质量数据集合中词频,supk(i)代表tki在簇中支持度,(Key-I′)/|I′i|代表含有tki的频繁项集数量,|C″i|代表tki簇的数量。

    利用对各数据簇Ck中各单词tki重要程度打分,就能够抽取出各簇中分值为前g个数据当作该簇主题词。

    假设T″k代表簇Ck中一组主题词,其表达式为

    T″k={tki1,L,tks} (11)

    那么任意数据和主题词组间相似程度可利用式(12)计算

    sim(Ιi´,dk‴)=∑j=ikiwj⋅tfjk (12)

    式中,tfjk代表词在数据集合中出现的次数。

    根据式(12)的计算,各数据将被分至相应主题中。

    假设现有两个用户Ua和Ub,这两个用户均具备高度相关相同频繁访问路径,该路径越长,则表示用户背景越雷同,也就是两者存在的共同兴趣数据越多[9,10]。综上,基于频繁项集的用户访问相似程度表达式为:

    sim(Ua,Ub)=sim(Ιi´,dk‴)⊕DLmax(Ua,Ub)min(semaxUa,semaxUb)+Sco(tki)Τk˝ (13)

    计算出用户访问相似程度之后,即能够找到用户最近邻平时访问数据,联合主题抽取,即可实现多用户数据流混合推荐。

    3 实验结果与分析

    为检测所提方法的性能,设置实验环境为:实验设备内存为480MB,CPU为Pentium,IV为2.93GHz,实验平台操作系统选取的是Windows XP,利用VC++6.0实现所提方法。

    将淘宝网中服饰相关数据当作实验数据。采用爬虫体系通过设置关键词,爬取2018年9月至2019年1月相关数据。作为一个数据集合,每条数据相关信息均带有自身标签,以此验证推荐性能。其中,随机抽取了30名淘宝用户参与实验。实验以推荐过程安全性和推荐结果准确性为测试指标完成。

    推荐结果准确性越强,用户满意度越高。从图2中可以看出,不同数量用户下所提方法推荐结果准确率整体很高,最高约为98%。该方法以提升推荐结果的精确度为目的,不仅对数据流频繁模式进行了挖掘,还有效结合了数据流主题抽取与多用户共同兴趣相似度计算,实现了多用户数据流混合推荐,综合考虑了多方因素,高效提升了推荐结果准确率。

    4 结束语

    个性化推荐是社会各行各业不可或缺的重要研究内容。为了更好地服务于大众,提出基于频繁项集的多用户数据流混合推荐方法。利用异常数据检测,为数据安全推荐奠定基础,结合数据流频繁模式挖掘、用户访问相似程度计算等实现数据流混合推荐。经实验证明,本文所提方法性能相较传统方法更为完善,鲁棒性强。
