    摘    要：

    为了保证数据库中数据价值与数据安全, 防止数据及其特征被恶意篡改, 针对现有数据特征防篡改方法中存在的防篡改攻击时间较短, 加密与解密过程耗时较长等问题, 提出基于加密验证的数据库中分布式大数据关键特征防篡改方法。根据分布式数据特征子集差异度与单项特征类间差异度计算结果, 设定分布式大数据特征评判准则。结合基于稀疏分数的数据特征提取方法, 与范数最小化方法完成分布式大数据关键特征提取。植入监控代码获取关键特征的实时状态, 通过特征函数控制流的完整性来检验数据特征的完整性, 并选择适当的密钥生成函数对数据特征进行加密, 利用解密验证实现数据关键特征防篡改保护。实验结果表明, 相比于实验对比方法, 所提方法的防篡改攻击时间明显增加, 同时加密与解密的时间成本更低。

    关键词：

    数据库; 分布式; 数据特征; 特征提取; 防篡改;

    Anti-Tampering Simulation of Key Features of Distributed Big Data in Database

    CHENG Guang-de

    Pass College, Chongqing Technology and Business University

    Abstract：

    In order to ensure the data value and data security in database and prevent the data and features to be tampered maliciously, this paper puts forward a tamper-resistant method for key features of distributed big data in database based on encryption verification. According to the computing results of the difference degree among distributed data feature subsets and the difference degree among single feature classes, we set the evaluation criterion of distributed big data feature. Combined the data feature extraction method based on sparse fraction with the norm minimization, we completed the extraction for key feature of distributed big data. Moreover, we implanted the monitoring code, so as to obtain the real-time status of key feature. Based on the integrity of feature function control flow, we verified the integrity of data feature, and then we selected the appropriate key function to encrypt the data feature. Finally, we used the decryption verification to achieve the tamper-resistant protection for data key feature. Simulation results show that the tamper-resistance attack time of the proposed method is significantly increased. Meanwhile, the time cost of encryption and decryption is lower.

    Keyword：

    Database; Distributed; Data feature; Feature extraction; Tamper resistance;

    1 引言

    大数据是多来源的多元实时数据组, 分布式大数据是利用分布式网络存储技术将所采集的数据分散为多部分存储, 便于数据库中的数据管理, 同时降低了数据存储成本, 提高了数据查询效率。随着云时代的发展, 以及云技术的不断进步, 大数据的应用越来越广泛, 数据安全问题备受关注[1,2]。数据篡改是指对数据库中数据进行非法添加、删减等, 造成数据破坏, 导致数据中包含的信息发生变化, 原有价值降低或丧失[3]。数据特征是指某一数据或一类数据与其内含信息的抽象结果, 是区别于其它数据的特点, 数据关键特征是数据的价值体现。数据库中分布式大数据关键特征防篡改问题, 是当前大数据与云技术领域的重点和难点, 受到了相关专家与学者的高度重视, 已出现了一些较好的数据特征防篡改方法[4,5]。

    王岩等人提出了基于混淆算法加密的数据特征防篡改方法[6]。利用密钥与同余方程生成原始数据特征代码所需的不透明谓词, 采用混沌映射对其进行重新构造, 利用重构不透明谓词对压扁控制流算法进行融合优化, 通过混淆后算法增加控制流的复杂度, 提升密钥破解难度, 抵御篡改攻击。该方法的防篡改攻击效果较好, 但加密与解密过程耗时较长。杜宗福提出了基于数据耦合优化的特征自适应加密防篡改方法[7]。结合信息交互方式与瀑布流思想优化数据库中低混沌程度的数据特征, 提升后续计算的收敛速度, 降低时间成本;根据数据阶数, 将微分方程与加密算法进行整合, 进一步提升加密计算的自适应收敛速度, 实现快速加密, 防止数据特征被篡改。张震宇等人提出基于隐藏字符水印的数据特征防篡改方法[8]。为防止数据库中数据特征被篡改, 在保持原有数据结构的基础上, 利用编码与替换技术将所设定的水印信息转换为隐藏字符, 将其嵌入到指定数据特征中, 利用水印中的Hash信息保护数据特征。该方法的数据特征防篡改效果较好, 但水印编码、转换过程较为复杂, 耗时较长, 效率较低。

    针对当前方法中存在的问题, 提出基于加密验证的数据库中分布式大数据关键特征防篡改方法, 通过实验与当前方法进行对比, 验证了所提方法的性能。

    2 数据库中分布式大数据关键特征防篡改方法

    2.1 分布式数据特征评判准则

    在对数据库中分布式大数据进行关键特征提取前, 需要设定分布式数据特征评判准则, 用于评价特征的重要程度, 并判断对应特征的类间辨识能力。具体过程如下:

    对于数据库中分布式大数据, 将所有数据映射到多维空间Rm中, 其中, m>0代表空间维数;数据集样本量为n′, 对应的训练集为{ (xk, yk) |xk∈Rm, yk∈{1, ⋯, l}}, 其中, l≥2, k=1, …, n′, 第j类样本量为nj, 给出包含i (i=1, …, m) 个特征的分布式数据特征子集差异度DFSi的计算式如下

    DFSi=∑j=1l|x¯i (j) -x¯i|2∑j=1l1nj-1∑k=1nj|xk, i (j) -x¯i (j) |2    (1)

    式中, i代表含有i个特征的特征子集在全部数据集内的均值矢量, xi (j) 代表含有i个特征的特征子集在编号为j的样本数据集中的均值矢量;xk, i (j) 代表第j个样本集内第k个数据相对i个特征的特征值矢量。式 (1) 中, 分子代表l个数据集中各集合对应含有i个特征的特征子集的数据中心矢量, 与全部样本集对应i个特征的中心矢量距离平方和, 该值越大, 说明数据集类间特征差异越大;分母代表各类数据集对应含有i个特征的特征子集的内部方差, 该值越小, 说明对应数据集内的数据特征差异越小;DFSi的值越大, 说明含有i个特征的特征子集对于数据集特征的类别分辨能力越强。

    当i=1时, DFSi转换为单项特征的数据集类间差异度量准则, 转换后的公式为

    DFS′=∑j=1l|x¯ (j) -x¯|2∑j=1l1nj-1∑k=1nj|xk (j) -x¯ (j) |2    (2)

    其中, 代表对应特征在全部数据集中的平均值, (j) 代表该特征相对于第j类数据集的平均值, k (j) 代表第j个数据集中编号为k的数据点对应的该项特征值, DFS′越大代表该项特征的分辨能力越强, 数据特征的类间区分效果越好。

    2.2 分布式数据关键特征提取

    根据2.1节得到的分布式数据特征评判准则, 结合基于稀疏分数的数据特征提取方法[9], 获取数据库中分布式大数据的关键特征。所采用特征提取方法与分数方程法近似, 通过过滤方式完成数据特征选择, 计算各维度特征对应的稀疏分数值, 并根据稀疏分数值的大小选取数据特征。数据关键特征提取的具体过程如下。

    根据稀疏表示思想, 利用L1范数最小化方法, 对数据库中的分布式大数据集合进行优化, 获得各数据相对于整个数据集的稀疏表示重构系数。设{Xi}i=1n′为分布式数据集合, 且满足Xi∈Rm, 该数据集合的数据向量与对应矩阵X=[X1, X2, …, Xn′]∈Rm×n′的各列一一对应。为了计算各数据向量i的稀疏表示重构系数矩阵si, 将其转换为求解数据间L1范数最小化的线性规划问题, 对应的计算公式如下

    Y=minsi|si|s.t.X→i=X′si    (3)

    式中, X′代表X删减第i列数据向量后的数据矩阵, si=[si1, …, sii-1, 0, sii+1, …, sin′]T为一个n′维系数向量, sij (j≠i) 代表数据集中第j个数据向量j在重构i中的贡献。据此得到数据集稀疏表示的重构矩阵S= (ij) n′×n′, S=[1, 2, …, n′], 各列向量i为j对应的稀疏表示重构系数。

    得到重构系数后, 计算数据集初始数据的各特征及重构特征间的差异值, 并累加全部数据样本的重构差异, 获取数据集各特征的稀疏表示留存性能, 差异值越小, 说明特征留存性能越好。据此, 得到稀疏分数目标函数式S (r) 为

    S (r) =∑i=1n (xir- (Xs˜i) r) 2Var (X (r:) )     (4)

    式中, 分母为计算全部数据集的r维特征xir与对应重构特征 (Xi) r的累加差异值, Var (X (r:) ) 代表数据集r维特征的方差值。根据上式计算得到的S (r) 值, 对各特征在全部数据集上的稀疏表示留存性能按照由小到大的顺序进行排列, 选择S (r) 较小的特征集, 完成数据库中分布式大数据的关键特征提取。

    通过引入稀疏表示思想, 利用稀疏分数对数据集进行数据特征提取。若某一特征的重构差异值小于另一特征的重构差异值, 表明该特征的稀疏表示留存性能更好, 将其作为数据的关键特征。数据库中分布式大数据关键特征提取的流程如图1所示。

    2.3 数据关键特征防篡改方法

    非法篡改, 是指对数据、软件等进行非法变动与修改, 实质上是通过改变控制流来变更数据特征、软件程序等。监测控制流的变化可以测定数据特征是否发生篡改, 数据的不同特征可利用不同函数进行区分, 并通过检验函数控制流的完整性来判断数据特征的完整性。

    通过在数据中植入监控代码来实现防篡改机制, 利用监控代码实时获取数据库中分布式大数据关键特征的状态, 在适当位置植入监控代码不会改变数据的整体结构。控制流可通过原数据库获取, 也可利用静态分析法从二进制文件中获取;采用函数级控制流图描述数据特征的正常状态[10]。

    函数执行序列即为分布式数据关键特征的实际状态, 为了得到数据的状态信息, 将监控代码植入各数据特征中。同时在函数入口植入监控代码用于执行跳转功能, 将对应的函数名传递至完整性检验函数数组, 通过该数组将各函数按执行顺序排列保存, 用于数据特征完整性检验。监控代码执行完成后, 执行函数, 以此来确保监测力度。获取数据特征函数控制流图, 通过函数运行信息检验数据特征是否发生篡改。各监控代码共享数组部分, 按顺序接收、存储各监控代码传递的函数运行信息。

    为了增强防篡改机制的抗攻击性, 对数据各特征函数进行加密, 若要篡改数据特征, 需要获取各函数的解密密钥对各特征函数进行解密, 对各函数采取分别传输与存储策略。

    假设f (x) 为[0, 1]内的特征函数, 多项式函数Bn (x) 的描述式如下:

    Bn (x) =Bn (f (x) ) =∑k=0nf (kn) (kn) (1-x) n-kxk    (5)

    上式中, n为正整数, (kn) 代表多项式的n次因子, 计算式为:

    (kn) =n!k! (n-k) !    (6)

    根据式 (5) 、 (6) 可得:

    Bn (x) =Bn (f (x) ) =∑k=0nf (kn) Bkn (x)     (7)

    式中, Bkn (x) 代表基函数。

    依据各部分加密方式的差异, 选取适当的密钥生成函数, 并使其满足以下条件:

    Κ2i=Fi (Ui, Ri, Ηi (CF) )     (8)

    其中, K2i和Fi分别代表第i个特征的解密密钥和密钥生成函数, Ui代表分解用户码的第i个值, Ri代表分解注册码的第i个值, Hi (CF) 代表第i个特征函数运行信息生成的散列值, CF代表数组。

    生成的密钥由多个加密代码元素组成, 各元素的解密密钥由隐藏于数据特征中密钥生成函数计算所得, 在数据特征函数运行中, 根据各特征的顺序, 前一个特征函数正常运行才能解密后一个特征函数, 若某一数据关键特征被改动, 则特征函数无法正常运行, 并致使解密密钥的生成结果错误, 无法正常解密。同时, 多种加密方式构造的多个差异化密钥生成函数, 提高了验证密钥的破译难度, 加强了数据库中分布式大数据特征函数防篡改的保护力度。

    3 实验结果与分析

    为了验证所提基于加密验证的数据库中分布式大数据关键特征防篡改方法的综合有效性, 需要进行一次实验, 实验平台系统为WindowsXP系统, 处理器频率1.8GHz, 存储空间64GB, ROM内存6GB, 编程语言为Java语言, 实验样本为数据库中随机选取的分布式数据集合。

    采用所提数据关键特征防篡改方法、基于混淆算法加密的数据特征防篡改方法与基于数据耦合优化的特征自适应加密防篡改方法进行对比实验, 实验结果中, M1代表所提基于加密验证的数据库中分布式大数据关键特征防篡改方法, M2代表基于混淆算法加密数据特征防篡改方法, M3代表基于数据耦合优化的特征自适应加密防篡改方法。

    4 结束语

    计算机与网络技术的不断发展, 推动了信息技术的进步, 同时也带来了数据安全隐患, 数据篡改攻击是一种常见的数据安全问题。在研究现有方法的基础上, 针对其中的弊端, 提出基于加密验证的数据库中分布式大数据关键特征防篡改方法。设定分布式数据特征评判准则, 结合稀疏分数思想与范数最小化方法进行数据库中分布式大数据关键特征提取, 提高了特征提取的精准度;在数据特征函数中植入监控代码, 并利用加密算法生成特征函数密钥, 通过解密验证增加对数据特征的安全保护, 实现数据关键特征防篡改。
