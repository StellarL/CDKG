    基于混沌映射的云存储动态密钥种子分配仿真

    王铁滨

    摘    要：

    针对现有的动态密钥种子分配方法, 存在密钥种子分配安全性较差、平均能耗较大、速度较慢等问题, 提出基于混沌映射的云存储动态密钥种子分配方法, 该方法首先需要构建一个基于混沌序列的动态密钥模型, 然后将构建的动态密钥模型分成网络密钥种子和会话密钥种子, 最后通过动态密钥种子部署、动态密钥种子安全簇建立、数据传输和密钥更新四个阶段, 实现对云存储动态密钥种子的分配。仿真结果表明, 所提方法能够在高安全性、低平均能耗的情况下, 实现对动态密钥种子的分配, 且分配速度较快, 能够满足对动态密钥种子分配速度的需求。

    关键词：

    混沌映射; 云存储; 动态密钥; 种子分配;

    作者简介： 王铁滨 (1974-) , 女 (汉族) , 黑龙江绥化人, 硕士, 副教授, 研究方向:大数据管理。;

    收稿日期：2019-01-04

    基金： 黑龙江省省属高等学校基本业务费多学科交叉研究项目 (KY YWF10236180201);

    Cloud Storage Dynamic Key Seed Allocation Simulation Based on Chaotic Map

    WANG Tie-bin

    Suihua University

    Abstract：

    Due to poor security of key seed distribution, high average energy consumption and slow speed of current methods, this paper puts forward a method to allocate the cloud storage dynamic key seeds based on chaotic mapping. Firstly, this method needed to construct a dynamic key model based on chaotic sequence, and then the constructed dynamic key model was divided into the network key seed and session key seed. Through the dynamic key seed deployment, the establishment of dynamic key seed security cluster, data transmission and key update, the allocation of cloud storage dynamic key seeds was achieved. Simulation results show that the proposed method can realize the allocation of dynamic key seeds with high security and low average energy consumption. Meanwhile, the allocation speed is fast and the requirement of dynamic key seed allocation speed can be satisfied.

    Keyword：

    Chaotic map; Cloud storage; Dynamic key; Seed allocation;

    1 引言

    云存储是指利用应用软件将Internet中类型多样的数据存储设备集合起来协同工作, 在工作过程中为使用者提供数据存储及访问功能的系统。当云计算系统的运算核心和处理核心为大量数据存储和数据管理时[1,2], 系统中需要配置大量数据存储设备, 采用云存储方式的用户能够在任意时间、任意地点, 利用任意能够连接Internet的设备连接到云上, 方便使用者对网络数据进行存取[3]。由于网络技术与计算机技术的快速发展, 使得云存储系统中数据的安全性大大下降, 所以数据加密与密钥分配方法得到了普及。针对现有的动态密钥种子分配方法, 存在的安全性较差的问题, 专家们提出了如下几种密钥种子分配方法。

    文献[4]提出基于SSLVPN的动态密钥种子分配方法, 该方法利用哈希函数对安全性较低的网络数据进行保密增强操作, 通过该保密增强操作完成了对动态密钥种子的分配, 但这种动态密钥种子分配方法虽然增强了网络数据的安全性, 但是安全性增强效果仍不能满足用户需求。文献

    [5]提出利用保密增强的方式来实现安全密钥的分配。这一过程利用快速傅立叶反变换调制出复用子载波, 通过协方差矩阵描绘出密钥的分配性能, 实现对动态密钥种子的分配, 但这种方法的密钥分配速度较慢, 不能满足用户对密钥种子分配速度的要求。文献[6]提出基于正交频分复用的动态密钥种子分配方法, 该方法利用量子密钥种子分配手段解决了密钥种子在分配过程中存在的安全隐患问题, 减少对协议的修改, 实现对动态密钥种子分配, 但这种方法的成本开销较大, 不能满足用户对密钥种子分配低成本开销需求。

    针对上述三种动态密钥种子分配方法存在的密钥种子分配安全性较差、能耗较大、速度较慢等问题做出了改进, 提出基于混沌映射的云存储动态密钥种子分配方法。

    2 基于混沌映射的动态密钥种子分配方法

    2.1 基于混沌序列的动态密钥模型构建

    利用混沌系统中的变量和耦合方程生成虚拟吸引子, 设定一个置换矩阵和明文信息, 将明文信息随机发送到某个虚拟空间中, 形成密文, 通过不可转换函数, 完成对动态网络密钥模型的构建。

    混沌又称为浑沌, 是大自然中存在的一种自然现象。混沌理论是上个世纪的三个重大发现之一, 起初是一门用于研究奇异函数和图形的数学理论[7]。

    一维混沌映射的离散形式为:

    x (n+1) =cos (4arccos (x (n) ) )     (1)

    其中, -1≤x (0) ≤1。由此一维映射生成的混沌序列{x (n) }全部位于[-1, 1]上。

    随着迭代过程的累积, 指数被起始节点分离, 这将导致其混沌序列混乱。这一性质使得该一维混沌映射与由安全密钥生成的序列相匹配。

    将混沌序列信号{x (t) }利用不可逆转换函数转变为二进制序列流{Sm (t) }。

    如果采用实值, 混沌随机序列经过计算后会变为伪随机序列或者周期序列[8]。因为我有限精度的效应, 直接采用迭代生成的序列会存在如下问题:

    1) p容易被破解

    迭代生成的混沌序列, 当得到处于相同分段的两个任意点对, 就能够确定p, 如式 (3) 所示。

    p= (x (t′) -x (t) ) / (F (x, p) -x (t+1) )     (3)

    2) 在有限精度下实现, 容易陷入周期循环。

    在迭代过程中, 由于参数p相同, 所以x (t) 必将产生同样的x (t+1) 。

    因为有限精度效应, x (t) 的不同值数量有限, 所以有迭代次数n存在, 致使x (n) 与n-1次迭代中的某个输出x (i) 相同, 0<i<n。这种情况出现后混沌序列会陷入到周期循环中。因为n和i与p和初始状态相关, 所以不能确定最小周期的大小是否能够满足要求。

    1) 分配部署阶段

    ①创建簇内密钥种子列表

    综上所述, 采用所提方法后, 网络信息的安全性得到了极大的改善, 且密钥种子分配用时较短、平均能耗较低。

    4 结论

    针对传统基于SSLVPN的动态密钥种子分配方法、基于保密增强的动态密钥种子分配方法和基于正交频分复用的动态密钥种子分配方法存在动态密钥种子分配安全性较差、平均能耗较高、动态密钥种子分配速度较慢等问题, 提出了基于混沌映射的云存储动态密钥种子分配方法。实验结果表明, 采用所提方法能够满足对高动态密钥种子分配安全性、低能耗动态密钥种子分配以及低动态密钥种子分配用时的需求。接下来, 为了使所提动态密钥种子分配方法在某些情况下仍然可以保持在最佳状态, 需要针对动态密钥种子分配安全性进行了更为深入的研究。
