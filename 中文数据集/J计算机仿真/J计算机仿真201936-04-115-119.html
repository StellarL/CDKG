    摘    要：

    当前移动终端验证方法存在安全性差、存储空间占用严重的问题, 提出移动终端中密码锁短信动态口令公开验证方法。依据挑战码和应答方式构建动态口令公开验证模型, 分析了模型中口令卡组成部分的显示单元、CPU控制单元、时钟单元和通信单元。依据所建模型, 着重分析CPU控制单元中异常检测:评估流量分布均值置信区间识别正常流量, 并在一个时间窗口范围内获取数据和评估所得值比较情况, 如果差值在置信范围, 则表示流量正常。如果差值超过置信区间, 则认为流量异常, 发出报警, 以此使动态口令公开验证时避开异常。基于异常避让, 给出动态口令验证方案:用户登陆时, 服务器端会生成随机数, 将该随机数发送给客户端, 客户端接收到随机数后通过口令信息与该随机数实行运算操作, 生成一个登录信息, 传送到服务器端, 公开验证系统收到客户端的登陆口令之后验算用户合法性, 实现密码锁短信动态口令公开验证。实验结果表明, 上述方法安全性强, 缓解了终端空间占用情况。

    关键词：

    移动终端; 密码锁; 短信动态口令; 验证;

    Password Lock SMS Dynamic Password Public Verification Simulation in Mobile Terminal

    FANG Kai-bin QIN Zhi-wu YANG Chao-yu WANG Chao-kai

    Shenzhen Customs Guangdong University of Technology

    Abstract：

    Security of current authentication method of mobile terminal is poor. Meanwhile, occupation of storage space is serious. Therefore, a public authentication method of note dynamic password of coded lock in the mobile terminal is presented in this article. Firstly, our research built public authentication model of dynamic password according to challenge code and response mode and analyzed display unit, control unit of CPU, clock unit, and communication unit of constituent part of password card in the model. According to the model, anomaly detection in the control unit of CPU was analyzed emphatically. Mean value confidence interval of flow distribution was evaluated to recognize normal flow and comparison situation of data and evaluation value was acquired in a time window range. If difference value was in confidence interval, the flow was normal. If the difference value was beyond the confidence interval, the flow was abnormal. Meanwhile, alarm was sent. Thus, the dynamic password avoided abnormity during public authentication. Based on the abnormity avoiding, authentication scheme of the dynamic password was provided. Random number was generated in server side and was sent to client side during user landing. Client side carried out computation operation via password information and the random number after receiving the number and generated landing information, then sent it to the server side. Public authentication system received landing password of the client side and checked validity of the user. Thus, the public authentication was achieved. Simulation results show that the method has excellent security and can remit occupation of storage space.

    Keyword：

    Mobile terminal; Coded lock; Note dynamic password; Authentication;

    1 引言

    信息技术不断发展, 移动终端安全问题越来越严重, 各种攻击无孔不入[1]。身份认证作为实现通信双方身份识别的一种手段, 是安全体系的首道防线, 在数据信息安全方面有着十分重要的作用[2,3]。当前比较常用的移动终端认证方法, 安全性和扩展性不是十分理想, 无法满足大众需求。鉴于移动网络安全性问题的重要性, 急需一种易实现、安全性强的身份认证方法, 来有效解决当前移动终端身份认证面临的问题。

    安迪[4]等人提出基于指纹与口令的移动终端密码认证方案。该方案运行的核心理念为:将密文存储在服务端, 当用户登录时, 通过手机辅助产生私钥, 以此对注册阶段产生的密文进行解密而生成认证的密钥。在私钥产生时需要输入口令与指纹, 使用者在服务端输入口令实现盲化, 然后再和手机交互。实验结果表明, 此方案具有一定安全性, 但指纹生成会加大手机存储压力。戚湧[5]等人以保障移动终端安全为目的, 利用移动云计算实现移动终端安全认证。过程中, 依据多跳接入构建最佳带宽分配策略D2MSMC网络覆盖拓展机制, 进而优化无线拓扑结构。D2MSMC算法可保证将认证服务器服务质量达到最大化, 以保障移动终端的安全认证过程顺利。实验结果表明, 所提方法稳定性较好, 但存在认证过程安全性较差的问题。张鑫[6]等人提出移动终端网络可信匿名验证协议, 当移动终端接入网络并开始身份认证与平台认证时, 在可信网络连接结构下, 设计可信漫游认证与切换认证详细过程。在认证过程中, 通过移动终端设置的预存假名及相应公私钥完成使用者匿名隐私保护。实验结果表明, 该协议运行过程复杂度较低, 但存在安全性较差的问题。

    上述关于移动终端安全认证的方案与协议存在安全性差与存储空间占用率高的问题, 可实践性不是十分理想。为此, 提出移动终端中密码锁短信动态口令公开验证方法, 并分析其可行性。

    2 移动终端中密码锁短信动态口令公开验证

    2.1 密码锁短信动态口令公开验证模型

    在此, 选择异步口令技术实现移动终端中密码锁短信动态口令公开验证。此种技术一般利用挑战码和应答相呼应的方式, 服务器会随机生成一个挑战码, 使用者收到挑战码之后, 将挑战码传输至口令牌, 令牌将该挑战码当作种子, 生成口令。其中, 口令牌与服务器无需同步数据, 不仅缓解了移动终端存储空间压力, 还能高效解决令牌失步问题, 提升了短信动态口令公开验证模型的可靠程度[7,8]。此种方式下, 每次生成的口令均为新随机生成的挑战码, 所以不会受到小数的攻击, 以此增强安全性。

    图1中的口令卡中包含显示单元、CPU控制单元、时钟单元和通信单元。其结构如图2所示。

    显示单元:其主要应用至生成若干位数字动态口令。另外, 还其中含包含五条计时线, 每相隔10s就由上至下按顺序消隐一条计时线。经上述设计, 能够使用户把握口令变化时间, 保障了用户操作过程中的连续性与方便性, 同时增强了动态口令公开验证正确性[9,10]。

    CPU控制单元:该单元主要用来生成与实现动态口令。其为动态口令卡运算与控制的核心, 主要功能为:基于时间与密钥, 调用算法得到动态口令;监控时间, 提供时间指标应用至生成口令与控制计时条变化;通过温度补偿机制, 以保障计时准确性。其中还包含密码短信动态口令公开验证过程中异常检测子单元, 通过异常检测可使验证时避开攻击, 进一步增强验证安全性, 详细过程如2.2节所示。

    时钟单元:该单元用在时间生成与时钟稳定性控制方面。其作为CPU实现晶振频率补偿的根基, 能够保障晶振计时具有稳定性与精准性。

    通信单元:该单元应用在动态口令卡和外部通信方面, 主要功能为系统运行过程中, 写入初始时间及灌入密钥。

    2.2 密码锁短信动态口令公开验证异常检测

    为了增强动态口令公开验证时的安全性, 需要检测验证过程中的异常, 使实施验证时能够准确避开异常攻击, 以此提升验证过程安全程度[11,12]。在此, 利用滑动时间窗检测异常, 详细过程如下:

    依据历史记录, 能够获取流量稳定分布。由此利用评估流量分布均值置信区间判断正常流量, 并在一个时间窗口范围内获取数据和评估所得值比较情况, 如果差值在置信范围, 则表示为正常情况;如果差值超过置信区间, 则认为异常, 即存在异常, 发出报警, 使动态口令公开验证时避开异常。详细过程如下所示:

    1) 确定时间窗口大小和样本大小。依据流量历史记录, 选择近期n个流量值当作样本。

    2) 设置置信水平1-α, 也就是给定α值, 基于α和n的确定, 查询t分布表, 获取tα/2 (n-1) 值。

    3) 依据样本值得到样本均值与置信区间:

    其中, 样本均值计算公式为

    X¯=∑i=1nXin    (1)

    样本标准差为

    S2=1n-1∑i=1n (Xi-X¯) 2    (2)

    根据式 (2) 得到S值。根据所得的S值可得到X¯±S/ntα/2 (n-1) , 以此获取置信区间α取值范围

    α=[X¯-S/ntα2 (n-1) , X¯+S/ntα2 (n-1) ]    (3)

    4) 选择实际流量值, 将其和置信区间相比, 如果其在置信区间内, 则表示为正常流量;假设高于置信区间的上界限, 则表示为异常流量, 并进行异常报警;假设低于置信区间的下界限, 则表示可能为其它方面故障。

    5) 将原来的流量样本中最远的一个值消除, 其余值向前移动一个位置, 且最后一个位置通过新读取的流量进行填补, 形成新流量样本。

    6) 迭代上述步骤3) 至步骤5) , 进行下一流量异常检测。

    2.3 密码锁短信动态口令公开验证关键技术分析

    在密码锁短信动态口令公开验证模型中最为关键的步骤即为验证环节。在用户登陆时, 服务器端会生成随机数, 并将该随机数发送给客户端, 客户端接收到随机数后通过口令信息与该随机数实行运算操作, 生成一个登录信息, 传送到服务器端, 公开验证系统收到客户端的登陆口令之后开始验算用户合法性, 以此提升登陆程序安全性。详细来讲, 该系统主要组成部分为服务器与客户端口令运算程序。服务器端向客户端传输一个随机数, 客户端收到该数据之后, 通过程序将该数据与用户口令经单向函数运算生成具有动态性和实时性的登陆口令信息, 随着用户ID传输至认证服务器, 服务器依据不同用户自身ID, 由秘密数据区域选择该用户密码, 开始对接收到的口令信息及随机数进行同样的计算, 对比计算结果和客户端口令信息, 完成最终验证。

    移动终端作为当前发展较为广泛的应用之一, 在大众生活和工作中占据着十分重要的地位, 提出移动终端中密码锁短信动态口令公开验证方法。通过构建动态口令公开验证模型和分析验证过程的关键技术完成课题研究, 并通过实验证明了所提方法具有很强地鲁棒性。对于下一步研究提出的建议为:移动终端中密码锁验证技术不断进步, 可将应用较多的静态验证方法与动态验证方法相结合, 增强身份验证性能。
