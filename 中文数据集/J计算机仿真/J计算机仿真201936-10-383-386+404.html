    摘    要：

    针对传统的图书馆公共资源大数据分层存储方法存在存储效率较低、执行开销较大、能耗消耗较高等问题,提出一种基于云存储环境下的公共资源大数据分层存储方法。利用组织群对大规模数据进行处理,构建云存储环境下公共资源大数据的并行处理模型,通过贝叶斯的分类原理,找到能够并行的点,根据MapReduce平台实现基于朴素贝叶斯的数据分类。在上述基础上,引入扩展哈希编码方式动态向外扩展存储节点,将内层节点的数据进行备份。通过多阈值级存储方法,使数据能够均匀的存储在多个节点,实现云存储环境下公共资源大数据分层存储。实现结果表明,所提方法能够保证对图书馆的公共资源大数据进行高效率、低成本的有效存储。

    关键词：

    云存储环境下; 公共资源; 大数据分层存储;

    Hierarchical Storage Simulation of Public Resources Big Data in Cloud Storage Environment

    LI Zhen-bo

    Library,YingKou Institute of Technology

    Abstract：

    Due to low storage efficiency, high execution cost and high energy consumption of traditional method, this article presented a tiered storage method of public resource big data based on cloud storage environment. Firstly, we used the organization group to process large-scale data, so as to construct the parallel processing model of public resource big data in the cloud storage environment. Secondly, we found the parallel points through Bayesian classification principle. Based on MapReduce platform, we realized data classification based on Naive Bayes. On this basis, we introduced the extended hash coding method to dynamically expand the storage node. After that, we backed up the data of inner node, so that the data could be uniformly stored in multiple nodes through the multilevel threshold storage method. Thus, we achieved the tiered storage of public resource big data in cloud storage environment. Simulation results show that the proposed method can ensure effective storage for library public resource big data with high efficiency and low cost.

    Keyword：

    Cloud storage environment; Public resource; Big data tiered storage;

    1 引言

    大数据时代的到来,使图书馆数据资源种类增加,数据海量增长,传统的公共资源大数据分层存储系统已经无法满足当前海量、复杂大数据的存储需求[1,2]。随着数据总量的不断增长,传统的网络传输带宽[3],造成了数据存储的瓶颈期,很大程度上影响了数据的存储效率,导致其成本较高[4]。当前,需要研究一种新的数据存储方法,实现图书馆公共资源大数据的高效分层存储[5]。

    目前,国内外已有相关学者给出了较好的研究,文献[6]提出一种基于共享策略的公共资源大数据分层存储方法,引入一种No SQL共享策略,对系统进行重构,并着重针对数据存储进行量化,该方法具有一定的实用性,但是执行开销较高。文献[7]提出一种基于节点分类的公共资源大数据分层存储方法,引入节点分类的数据存储策略,将节点进行划分,划分为“边缘”类节点与“核心”类节点,根据节点的位置进行数据分布,获取最优位置,实现公共资源大数据分层存储,该方法时延较短,但是消耗的能耗较多。

    为了解决上述方法存在的存储效率较低、能耗以及运行成本较高等问题,提出了一种基于云存储环境下的公共资源大数据分层存储方法。

    2 基于云存储环境下的公共资源大数据分层存储方法

    2.2 公共资源大数据分层存储方法

    在上述研究的基础上,通过扩展哈希编码方法对存储节点进行动态扩展,增加存储节点。引入多阈值级别方法,将数据均匀的分布到各个存储节点,实现云存储环境下的公共资源大数据分层存储,具体过程如下:

    在任意一个网格内,当传感器节点感知到数据后,生成相应的网格消息,即

    Date=[(xi-x0)/d]   (12)

    将消息发送到相应的网格内,计算IDE节点

    针对公共资源大数据的相关特征,利用下式计算N值的最大节点

    Ν=p×cosθi+Ei   (14)

    式中,p代表在当前节点跳到下一节点i的距离,θi代表当前节点与下一个节点构成的夹角,Ei代表当前节点的剩余能量。

    在求解N值的过程中,数据传输的欧式距离p×cos θi与节点的剩余能量Ei之间存在一定的差异,为了能够让两者之间保持平衡,需要加入调整数量级参数β。假设调节权重的参数为α,利用式(15)给出修正后的表达式
