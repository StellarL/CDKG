    基于模糊反馈的共享网络远程数据控制仿真

    摘    要：

    共享网络数据利用率和用户满意度的提升现实意义重大,使得共享网络远程数据控制变得异常重要。针对当前相关研究成果存在容错性较差与控制误差较大的问题,提出基于模糊反馈的共享网络远程数据控制方法。综合考虑网络中的数据描述与管理、数据管理使用的框架、动态数据发现与不同使用者对于同一个数据的个性化追求等数据控制影响因素,融合系统误差和容错性以及QoC,结合集中式和分布式数据控制模型,利用层次式控制模型设计一个共享网络下模糊反馈调度器,通过模糊最大优先级调度方式动态式调节网络远程数据传输。利用归一化控制质量评价指标作为数据控制性能衡量准则,保障控制性能。实验结果表明,上述方法控制精度高,容错性强。在接下来研究中该方法可起到一定借鉴作用。

    关键词：

    模糊反馈; 共享网络; 数据控制; 远程数据传输;

    Remote Data Control Simulation of Shared Network Based on Fuzzy Feedback

    SUN Chao

    Jilin Engineering Normal University

    Abstract：

    Due to poor fault tolerance and large control error of current method, this article proposed a method to control the shared network remote data based on fuzzy feedback. Based on the comprehensive consideration for data control influence factors such as the data description and management in network, the framework for data management, dynamic data discovery and the individualized pursuance for the same data, we integrated the systematic error with fault tolerance and QoC. And then, we combined centralized data control model with the distributed data control model. Moreover, we used the hierarchical control model to design a fuzzy feedback scheduler based on the shared network, which dynamically adjusted the network remote data transmission by fuzzy maximum priority scheduling. Finally, we used the normalized control quality evaluation index as a criterion to measure the data control performance. Thus, the control performance could be guaranteed. Simulation results show that the proposed method has high control accuracy and strong fault tolerance. This method can provide the reference to the next research.

    Keyword：

    Fuzzy feedback; Shared network; Data control; Remote data transmission;

    1 引言

    计算机性能日益提升,网络通信技术也随之迅速发展,用户需求逐渐趋向大规模和多样化,这就需要将各种存储资源通过共享网络远程传输给各用户[1,2]。数据资源控制是共享网络远程数据利用的核心部分。针对网络条件下资源进行控制是个比较复杂的问题,且共享网络具有异构性与分布性以及自治性等特点,促使着传统数据控制模式必须进行优化[3]。由于共享网络业务不断扩展,造成服务质量成为衡量网络优良的重要决定因素。在时代不断变化下,网络服务功能不仅仅是速度和性价比,还有用户满意程度等方面的对比。

    相关技术日益革新,有关研究学者提出了很多优秀成果。庞晓丹[4]等人针对移动通信网络中的数据控制问题,接入一个控制层级别切片调度器,以此使各个运营商间能够按照一定比例实现资源共享。其中,运营商是切片所有者,能够基于不同服务等级协议设置资源比例。对比固定切片和NVS结构下切片数据资源块利用率,实行仿真模拟。与固定切片方式进行对比,NVS法在各种切片之间用户互为隔离的状态下有更好的信息利用率。另外,基于实时业务与非实时业务的时延差异构建业务切片,选择相对应的资源调度法实现数据控制。许荣斌[5]等人以在有限时间处理大规模业务调度为研究对象,利用DAG对数据资源控制模型进行构建,实现多DAG共享资源调度。过程中,将相对严格程度与宽松度相结合对DAG时序松紧程度进行衡量,确定多DAG任务执行的紧急程度。根据紧急程度对数据资源调度控制进行优化,并针对具备时序约束的各DAG任务实施调度。针对调度过载问题,通过有关方案选择性舍弃一些任务。熊余[6]等人指出网络资源控制过程中存在时延等问题,提出将业务分区应用至资源调度中,以实现数据控制。基于网络负载情况对网络单元组波长数量进行实时调整,完成资源共享,同时动态性地分解子周期。构建混合资源调度法,凭在线实现波长空隙填补,凭离线保证资源调度,从而实现网络共享数据控制。

    上述相关研究成果虽然具备一定优越性,但存在容错性较差与控制误差较大等方面的问题。为此,提出基于模糊反馈的共享网络远程数据控制方法。

    2 共享网络远程数据控制研究

    共享网络远程数据控制过程分为网络数据控制模式分析和模糊反馈下数据控制两部分。

    2.1 网络数据控制模式

    2.1.1 数据控制影响因素分析

    共享网络中的资源种类非常多,且功能各有各的特点,资源分布在不同管理域中,归不同组织所有及操作,在使用方案及安全机制方面也大有不同[7,8]。综合考虑共享网络远程数据的特性,对数据合理控制时需要注意以下方面:

    1)网络中的数据描述与管理。因数据种类繁多,数据状态和所在位置均会随网络变化而变化。由此,如何利用统一方式实现网络数据描述与分类,是数据控制中需要考虑的问题。

    2)数据管理使用的框架。共享网络是由不同管理方案的自治域构成的,如何基于该特点选取适当的体系框架,又使其对数据的提供方与使用方呈现出透明化等问题也十分重要。

    3)动态数据发现与不同使用者对于同一个数据的个性化追求。该方面体现在两个点:其一,要对某数据进行定位,其必须具备位移全局标识,但提供服务的数据位置可能会发生改变,数据控制过程中需要对该问题予以重视。其二,因使用者需要的服务质量存在很大差异,数据模型与检索机制需要灵活,同时具备可扩展性,能够尽可能地满足使用者的各种需求。

    4)数据控制如何支持异构体系间互操作,并屏蔽来自不同自治域存在的差异性。综合考虑当前共享网络远程数据控制技术,在接下来的研究中必须兼容异构性。

    2.1.2 数据控制模式分类

    融合当前相关研究,可将共享网络远程数据控制模式分为以下几个类别:

    1)集中式数据控制模型

    数据控制中仅存在一个负责全局数据控制的调度器,因其基于数据需求者与数据提供方的能力与约束进行全局范围数据调度控制。由于使用该模型的数据控制系统管辖的数据量有限,因此,调度器可以得到数据控制系统中全部数据的详细信息,以此能够更好地权衡数据需求者实际需求与数据提供方的能力及约束,尽可能地兼顾两方面利益,并给出全局数据调度优化方案。

    2)分布式数据控制模型

    分布式数据控制模型中,各个高性能节点的重要性都是平等存在的。此种控制模式因没有全局控制结构,所有节点均为对等形式,所以具备很强的可扩展性能与容错性能,且其中的任何节点发生异常均不会对整个控制过程造成过大影响。因所有节点地位均平等,由此,在调度前存在一个权限认证环节,利用其确定此节点有权对共享网络远程数据进行访问。

    3)层次化数据控制模型

    层次化数据控制模式具备两层或者更多层的调度体系。针对两层体系而言,位于上层的数据调度控制体系称作全局调度模式,位于下层的数据调度控制体系称作局部调度模式[9,10]。此种结构优越性体现在容易以全局范围实现数据调度控制方面,所处高层的调度体系能够对底层体系控制起到决定性作用。图1为共享网络远程数据控制的一般结构。

    图1 共享网络远程数据控制的一般结构   下载原图

    由图1可知,共享网络远程数据控制一般包含三个阶段:数据发现和生成一个可行的数据列表;数据采集与选择;数据调度任务执行。

    2.2 模糊反馈下数据控制

    2.2.1 模糊反馈结构

    在基于模糊反馈的共享网络远程数据控制中,将2.1节分布式和集中式相结合,即得到分层式控制方法,充分考虑到系统误差和容错性以及QoC等实现共享网络远程数据控制。图2为多回路模糊反馈结构,在模糊控制中设定整个网络,紧急数据与控制回路数据均为可调度的数据;非实时节点与紧急节点数据均服从泊松分布规律。

    2.2.2 模糊反馈数据调度器设计

    基于模糊反馈的共享网络远程数据控制中的调度器,在专家语言与实时数据相结合基础上,获取一个高效动态优先控制策略。将模糊反馈调度器定义为控制回路产生的误差e(k)与误差差分ec(k)=e(k)-e(k-1),通过模糊化操作转换为E、EC,两者模糊集合和为{ΝB,ΝS,ΖE,ΡS,ΡB},输出变量DP模糊集为{S,Μ,B}。为了降低计算复杂度,对输入变量E、EC论域进行归一化操作,其二维的模糊控制器输入与输出隶属度分布函数与控制规则示意图如图3所示。

    图3 E、EC、DP隶属度分布函数   下载原图

    综上,控制规则可表示为:由此可得

    R=∪n,mEn⊗ECm⊗DΡnm   (1)

    其中R隶属度函数表达式为

    μR(e,ec,dp)=μEn(e)∧μECm(ec)∧μDΡnm(dp)   (2)

    式中,μ代表数据控制传输;e、ec、dp分别为E、EC、DP归一化后的表示形式。综上,输出DP基于最大和最小模糊推理结合的规则可表示为

    DΡ=(E⊗EC)⊕R   (3)

    其中,DP隶属度函数表达式为

    μDΡ(dp)=μR(e,ec,dp)∧μE(e)∧μEC(ec)   (4)

    以上表达式中,∧代表极值运算,⨂、⊕代表模糊算子。其中,模糊算子的计算公式可利用式(5)确定

    dp=∑μDΡ(dpi)⋅dpi∑μDΡ(dpi)   (5)

    其中,dpi代表模糊反馈数据,满足DPi(dpi)=1。

    通过以上模糊推理,每个回路的优先级配置表征了系统误差与误差差分之间存在的关联程度,是对系统响应最为合适的描述。合理选取输入与输出论域元素和数量,基于上述运算得到检索项目能够减小数据控制误差。

    共享网络远程数据中的模糊反馈调度与控制过程中,当网络节点传输数据,也会将数据传输给反馈调度器,由此在共享网络中通过一点和多点通信就会变得很方便,也会增强数据控制效果。反馈调度器一直存储各回路中最新的信息数据,利用模糊反馈法实现控制回路的优先级动态分配。其能够通过事件驱动或者是时间驱动实现回路优先级别刷新,事件驱动能够根据最大采样回路流量包的接收当作事件的激励原动力,将最大采样周期的1.5倍当作时间驱动刷新的规律周期。

    共享网络远程控制回路QoC能够通过误差泛函积分进行评估与衡量,将其控制系统瞬时误差值e(t)当作泛函积分进行评估。在此,采用比较通用的IAE指标对其进行衡量,其中第i个控制回路IAE表达式为

    ΙAEi=ωi∫0∞|e(t)|dp   (6)

    式(6),ωi代表基于各控制回路重要程度所赋予的权重系数,通过离散化可得

    ΙAEi(e:seq〈Γi(t)〉)=ωi∑t|e(t)|hi   (7)

    其中,hi代表控制回路采样周期,IAEi(e:seq〈Γi(t)〉)代表网络随机组织时延序列下生成的绝对误差相加值。为了更好地反应共享网络远程数据控制质量,定义一个归一化控制质量评价指标QoC,该值越大则表示控制质量越好,反之越差。

    3 实验结果与分析

    实验环境:综合考虑控制回路和优先级节点等,实验平台搭建在MATLAB/Truetime1.5上。实验假设条件如下:

    1)共享网络中的数据链路层使用的是IEEE802.1p。

    2)数据调度的间隔是150ms,且数据的长度是100bit。

    3)控制回路的数据长度是900bit。非实时数据平均达到速率是110包/s,剩余策略服从随机分布规律。

    根据上述实验环境,将数据控制误差和容错性作为验证基于模糊反馈的共享网络远程数据控制方法的有效指标。图4为数据控制后的不同时刻下,本文方法的归一化控制质量评价指标QoC结果。

    图4 不同时刻归一化控制质量评价指标   下载原图

    分析图4可知,在不同时刻下,本文方法的归一化控制质量评价指标值均较大。在上述研究中共享网络远程数据的模糊反馈调度与控制考虑了网络节点传输数据时,也会将数据传输给反馈调度器,进而在共享网络中通过一点和多点通信就会变得很方便,这也就增强了数据控制质量。

    为了验证本文方法的有效性,采用文献[4]、文献[5]和本文方法对网络远程数据控制误差进行对比分析。对比结果如图5所示。

    图5 不同研究成果控制误差对比   下载原图

    根据图5可知,控制误差在一定程度上决定了用户满意度,误差越小则用户满意度越高,误差越大则用户满意度越低。文献成果控制误差曲线波动较大且整体较高,而本文方法的网络远程数据控制误差值在不同实验次数下不超过0.1。本文方法的网络远程数据控制误差比另外两种方法的控制误差低,说明本文方法具有较好的控制精度。

    为了进一步验证本文方法的有效性,采用文献[4]、文献[5]和本文方法对网络远程数据控制容错性进行对比分析。对比结果如图6所示。

    容错性越高表示运行效率越高,由图6可明显看出本文方法的共享网络远程数据控制容错系数最大,有效增强了控制过程的容错性。

    4 结束语

    为了给用户提供更加优质的服务,结合集中式和分布式数据调度模式,构建模糊反馈下数据调度器,通过该调度器实现共享网络远程数据控制。通过仿真证明了所提方法的可行性。在此基础上,对下一步的研究提出以下建议:共享网络环境下存在资源预约和计费等相关问题,如何将该方面与数据控制相结合可当作接下来的研究重点。
