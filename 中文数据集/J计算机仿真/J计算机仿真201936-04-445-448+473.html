    摘    要：

    传统调度方法没有考虑网络节点剩余能量问题, 造成节点过早死亡, 网络拥塞严重, 平均调度延时较长, 收敛速度较慢等问题。针对上述问题, 提出了基于改进蚁群算法的大规模异构传感通信大数据智能调度方法。方法采用四元组形式描述异构传感通信大数据智能调度模型, 给出了模型成立的3点假设条件, 并建立了异构传感通信大数据智能调度成本模型。基于调度模型和调度成本模型模型, 通过计算节点剩余能量和实时更新网络节点信息素浓度的方式改进原有蚁群算法, 结合定向扩散路由算法, 在调度任务执行源节点与目标节点之间建立多条通信路径, 均衡各个节点能量消耗, 实现大规模异构传感通信大数据智能调度。仿真结果表明, 所提方法具有避免网络局部拥塞, 减小平均调度延时, 加快收敛速度, 延长节点生存寿命的优点。

    关键词：

    大规模; 异构; 传感通信大数据; 智能; 调度;

    Large Scale Heterogeneous Sensor Communication Large Data Intelligent Scheduling Simulation

    WANG Chao

    Center for Information and Educational Technology, Southwest University for nationalities

    Abstract：

    Due to premature death of node, serious network congestion, long average scheduling delay, and slow convergence speed of traditional scheduling method, a new intelligent scheduling method of big data in large scale heterogeneous sensor communication based on the modified ant colony algorithm was presented in this article. Firstly, intelligent scheduling model was described via quadruple form. Three assumed conditions for the succeed model were provided and cost model of the intelligent scheduling was built. Based on the scheduling model and cost model, original ant colony algorithm was modified via calculating residual energy of node and updating pheromone concentration. Directed diffusion routing algorithm was integrated to build multiple communication path between source node of scheduling task execution and goal node. Energy consumption of each node was balanced. Thus, the intelligent scheduling was achieved. Simulation results show that the method can avoid network congestion and decrease long average scheduling delay. Meanwhile, it can accelerate convergence speed and prolong node lifetime.

    Keyword：

    Large scale; Heterogeneous; Sensor communication; Big data; Intelligent; Scheduling;

    1 引言

    大数据是与自然资源、人力资源同等重要的战略资源, 是一个国家数字主权的重要体现。大数据时代的到来意味着国家层面的竞争力将部分体现在一个国家拥有的大数据规模、数据活性以及对数据的解译和应用能力[1]。随着云计算和大数据的推广, 以及互联网应用的快速普及, 网络中的共享资源急剧增加, 异构数据种类和来源也不断增加, 研究一种有效的大规模异构传感通信大数据调度方法对实现资源的整合与统一服务, 提高计算资源整体的使用效率, 降低服务成本, 加速异构传感通信数据的传输效率具有重要现实意义, 成为了国内外许多专家学者的热议话题[2]。

    学者薛涛和马腾在文献[3]中提出基于资源最大综合利用率的多源异构大数据跨源调度方法, 该方法在充分考虑网络中多源异构数据权重比的基础上, 采用基于权重最大资源利用率算法分配多源异构数据, 实现了多源异构大数据跨源调度, 但该方法存在调度延时较长的缺点;齐平、王福成等人在文献[4]中提出基于图模型的多源异构大数据跨源调度方法, 该方法将用户对多源异构大数据的需求与云服务商供给的最优分配问题转换为构造和求解多源异构大数据跨源调度最小费用最大流图的目标函数问题, 实现了多源异构大数据跨源调度, 但该方法收敛速度较慢。

    为了避免上述调度方法没有考虑网络节点剩余能量问题, 造成节点过早死亡, 网络拥塞严重, 平均调度延时较长, 收敛速度较慢等问题, 提出了基于改进蚁群算法的大规模异构传感通信大数据智能调度方法。

    2 异构传感通信大数据智能调度方法研究

    2) 大数据智能调度成本模型

    大规模异构传感通信大数据调度过程中, 系统服务提供商关注的除了调度任务完成时间, 还有调度成本问题。研究将系统服务器作为大规模异构传感通信大数据智能调度单位, 用户调度任务需求根据系统服务器实际数量配置。根据系统服务器的性能差异, 用户调度任务在系统服务器上的调度执行时间也有所差异, 选择适合的调度方法不仅能够减少系统服务提供商的调度成本, 而且能够增加企业收益, 根据Google Compute Engine模式, 制定系统服务提供商执行异构传感通信大数据智能调度任务的单位时间成本pj[8], 计算公式如下

    pj=pi1×αi+pi2×βi+pi3×γi    (7)

    式中, pi1、pi2、pi3分别代表单位时间内系统服务器单位内核、内存以及磁盘空间成本。则可得每个系统服务器执行异构数据调度任务的单位时间成本为:

    Ρ={p1, p2, ⋯, pm}    (8)

    在上述计算基础上, 针对用户的某一异构数据智能调度任务, 系统服务商需要承担的服务器成本为

    pj= (pi1×αi+pi2×βi+pi3×γi) ×max1≤i≤n{sij⋅xjei}=pj×max1≤i≤n{sij⋅xjei}    (9)

    Ρ=∑j=1mpj×max1≤i≤n{sij⋅xjei}    (10)

    根据上述计算可得异构传感通信大数据智能调度目标函数minP, 即调度成本最低目标。

    原有蚁群算法没有充分考虑网络节点 (即系统服务器) 的剩余能量, 使得大规模异构传感通信大数据调度过程中蚂蚁都通过信息素浓度较高的通信链路到达目标节点, 导致个别节点过早死亡, 减小了整个网络的生存寿命。为了解决这一问题, 研究通过计算节点剩余能量以均衡网络节点能量消耗[9], 计算公式如下

    ηsu=eudsu    (13)

    式中, eu代表节点u的剩余能量信息;dsu表示节点s和u之间的通信距离, 由此说明, 蚂蚁k下一跳选择访问节点u的概率Pk (s, u) 的取值决定因素除了节点信息素浓度值外, 还受到蚂蚁下一跳择访问节点的剩余能量影响, 此时, 蚁群对于大规模异构传感通信大数据智能调度路径的选择会随着节点剩余能量的变化而变化。采用以下式 (14) 实时更新网络节点信息素浓度τ (s, u) new

    interest= (type, interval, rect, timestamp, expiresat, energy)     (16)

    根据上式, 当目标节点nj从邻居节点ni接收到异构传感通信大数据调度兴趣消息时, 将节点ni添加到本身梯度表中, 并获取节点ni的剩余能量信息, 根据上述式 (14) 更新网络节点信息素浓度, 并将该兴趣消息广播到各个邻居节点, 直到兴趣消息洪泛到调度任务执行源节点ns。

    3) 多路径构建

    调度任务执行源节点ns收到异构传感通信大数据智能调度兴趣消息后, 按照式 (11) 的选择概率Pk (s, u) 带着异构数据包的前向蚂蚁从节点ns出发, 建立多路径[10], 并依据以下规则转发探测数据包, 实现大规模异构传感通信大数据智能调度:

    ①检查蚂蚁是否来过该节点, 如果来过, 则停止转发, 相反, 则计算该节点是否比蚂蚁访问的上一个节点距离智能调度目标节点更近, 同时距离节点ns更远, 如果是, 则将该节点做标识;

    ②根据选择概率计算公式, 选择节点nj作为数据包转发的下一跳节点, 获取节点ni的剩余能量信息和通信链路带宽信息;

    ③如果邻居节点拒绝转发探测数据包, 则转回第②步重新选择新的邻居节点;

    ④如果蚂蚁梯度表中包含的所有邻居节点均拒绝转发该探测数据包, 则节点ni将该探测数据包丢弃并将其从蚂蚁梯度表中彻底删除;

    根据上述规则, 当种群中的所有蚂蚁完成一次完整遍历后, 从任务调度源节点到目标节点的多路径即建立完成。

    观察图3 (a) 的实验结果可以发现, 采用文献[3]方法求解源异构大数据调度完成时间目标函数时, 迭代到第30次才能收敛到最优值, 文献[4]方法也需要迭代执行20次才能收敛到最优值, 提出方法在迭代到第10代时就成功收敛到了最优解;观察图3 (b) 的实验结果可以发现, 采用文献[3]方法求解源异构大数据调度完成时间目标函数时, 迭代到第25次才能收敛到最优值, 文献[4]方法甚至需要迭代执行40次以后才能收敛到最优值, 提出方法在迭代到第10代左右时就完全收敛了, 这是由于提出方法实时更新了网络节点信息素浓度, 还在启发因子中引入了节点剩余能量, 同时建立了多路径, 有效均衡了系统服务器节点调度任务, 延长了系统服务器节点生存寿命, 加速收敛的同时, 避免了对比方法容易陷入局部最优解的问题, 实验结果充分证明了所提方法具有明显优越性。

    4 结束语

    研究提出的基于改进蚁群算法的大规模异构传感通信大数据智能调度方法, 在定向扩散路由协议基础上, 充分利用蚁群算法的自适应性和动态寻优优点, 在异构传感通信大数据智能调度任务执行源节点和目标节点之间建立了多条通信链路, 较好地均衡了系统服务器节点能量消耗, 避免了节点过早死亡, 实现了异构传感通信大数据智能调度, 为今后网络资源共享和资源分配提供了有效参考。
