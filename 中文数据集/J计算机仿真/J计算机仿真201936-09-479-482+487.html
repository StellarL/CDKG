    摘    要：

    采用传统的方法进行网络资源调度时,难以确定网络切片资源调度指标的权重,存在调度精度低、运行时间较长和资源丢包率较高等问题。为解决上述问题,提出基于概率模型的网络切片资源优先度调度方法,采用贝叶斯网络表示概率模型图,根据模型图得到模型观测变量、部分变量以及变量的后验分布。并在上述基础上,创建概率模型,根据概率模型计算出网络切片资源优先度。利用调度方法对任务集优先度进行调度。仿真结果证明,所提方法资源丢包率较低,调度效率较高,为更好地提升网络工作效率提供了较大的帮助。

    关键词：

    概率模型; 网络切片资源; 资源优先度; 调度;

    Network Slice Resource Priority Scheduling Simulation Based on Probability Model

    ZHOU Bing

    Xinlian College,Henan Normal University

    Abstract：

    In traditional method, it is difficult to determine the weight of network slice resource scheduling index during the network resource scheduling. The scheduling precision is low, the running time is long and packet loss rate is high. Therefore, a scheduling method of network slicing resource priority based on probabilistic model was proposed. Firstly, Bayesian network was used to represent the probability model diagram. According to the model diagram, the posterior distribution of observed variables, partial variables and variables was obtained. On this basis, the probability model was built. According to the probability model, the resource priority of network slice resource was calculated. Finally, the priority of task set was scheduled by the scheduling method. Simulation results show that the proposed method has lower resource packet loss rate and higher scheduling efficiency, which provides a great help for improving the network work efficiency.

    Keyword：

    Probability model; Network slice resource; Resource priority; Scheduling;

    1 引言

    网络切片是一种特定的虚拟化形式,允许多个逻辑网络在共享的物理网络基础设施上运行。网络切片概念的关键优势在于它提供了一个端到端的虚拟网络,不仅包括网络,还包括计算和存储功能[1]。在5G网络中,网络将进一步被抽象为“网络切片”,这种连接服务是通过许多定制软件实现的功能定义。

    网络切片资源优先度计算是实施网络切片资源优化调度的重要前提条件,但是目前现存的调度方法在调度决策中仍未考虑需求点重要度的区分问题,认为不同需求点同等重要,使得最终的资源调度要求[2]。在这种情况下,如何根据网络切片资源的优先度进行优化调度成为该领域急需解决的主要任务,而根据概率模型可以计算出网络切片资源的优先度,并兼顾多方面的目标要求,重点地安排调度任务,是解决上述问题的根本途径,同时引起了很多学者的关注[3]。由于网络切片资源的优先度调度对网络资源管理具有深远的发展意义,因此受到很多相关学者和专家的重视,出现了很多较好的研究成果。

    文献[4]提出针对网络切片资源的随机优先级调度(REDP)方法。该方法根据网络资源数据流的优先级和网络拥塞程度对资源错失数量进行调整,降低网络拥塞程度和数据传输时系统能耗;创建两条网络切片资源中继网络编码的马尔科夫链模型,推导出资源丢包率和时延。该方法可有效减低资源错失数量,但其在运行时需操作的步骤较多,存在资源错失数量大、运行时间过长的问题。文献[5]提出基于模糊聚类理论的优先级调度(RCA)方法,将具有相似属性的网络切片资源进行模糊聚类,根据设计的算法计算出网络切片资源的优先度,对资源优先度进行合理调度进而提高网络通信经济效益。运行该方法进行网络切片资源优先度调度时,忽略了网络中随机不确定因素,导致优先度调度结果不准确、调度跨度大等问题。文献[6]方法提出一种基于调度任务时间的网络切片资源动态优先度调度方法,该方法首先考虑调度时间不确定性,计算出于任务调度顺序;然后利用剪枝对不确定的调度任务执行时间,对经过简化以后的调度任务进行排列组合,计算出该组合的解,即任务执行几率;最后以上述为基础创建任务初始执行优先顺序,依据创建的顺序、调度任务完成时间和空余时间,依次对网络切片资源分派优先度,最终实现对网络切片资源完成优先度调度。采用该方法进行资源调度时,由于计算任务优先度过程中,易受时间的影响,导致调度跨度较大,存在平均队长过长等问题。

    针对上述方法中存在的问题,提出一种基于概率模型的网络切片资源优先度调度方法。仿真证明,所提方法较为合理,且调度过程中信息丢失率较低、调度准确性较高,为更好地提高网络资源管理工作效率提供了有力的依据。

    2 网络切片资源的优先度调度方法

    2.1 基于概率模型选取优先度

    所提方法将网络切片资源的优先度输入到概率模型中,得出最佳网络切片资源优先度。

    2.2 任务优先度调度方法设计

    将概率模型选取的优先度与任务优先级调度进行结合,设T={T1,T2,...,Tn}为网络切片资源的优先度调度任务集,采用五个单元表示Tj=〈IDj,aj,Lj,vj,Dj〉。其中IDj表示优先度调度任务Tj的标识;aj表示优先度调度任务领取的时间;Lj表示优先度调度任务的长度,单位为百万指令(MI);vj表示为优先度调度任务的价值;Dj表示为优先度调度任务的完成时间期限。

    设R={R1,R2,...,Rm}为网络切片资源集,其中单个资源采用三个单元组表示为Rj=〈IDj,MIPSj〉。其中,IDj表示为网络切片资源的序号;MIPSj表示网络切片资源rj的处理效率。

    根据上述对网络切片资源优先度调度任务集的描述,资源任务tj的信息密度VDj采用式(14)进行表示

    VDj=vjLj (14)

    VDj的大小和优先调度任务成正比例关系,VDj值变大,调度任务的优先级也随之变得越高。

    设任务时间为t,根据式(3)可知资源优先调度任务Tj的剩余操作时间Sj,Sj的计算公式如下

    Sj=Dj-t (15)

    优先调度任务剩余操作时间Sj越小,表示调度任务时间越紧迫。

    依据调度任务剩余操作时间Sj和任务信息密度VDj,可按照式(3)和式(4)对调度任务的优先度Pj进行设定,计算公式如下

    Ρj=VDjSj (16)

    式中:Pj用来表示调度任务Tj在剩余操作时间Sj内的信息密度,该时间内调度任务信息率和资源优先级成正比例关系,信息率越大,资源优先级则越高。根据任务优先度进行网络切片资源调度时,对优先级较高的资源优先进行调度,这样使得信息量大且时间紧迫的网络切片资源优先得到处理。

    3 仿真证明

    为了验证提出基于概率模型的网络切片资源优先度调度有效性,需要进行仿真。在系统为Win10专业版,内存为8GB,CPU为3.8GHz的环境下搭建仿真平台。实验数据主要来源于通信网络资源库记录,资源库中包含了大量的数据信息。

    分别采用所提调度方法与REDP方法、RCA方法对网络切片资源进行对比实验。选取任务平均队长作为算法性能对比指标,该值为常数,利用式(17)计算任务平均队长

    Lw=∑Lj/t (17)

    其中,任务平均队长Lw为在参数不变的情况下进行多次实验得到的均值,为常数。

    4 结束语

    针对传统方法进行网络切片资源优先度调度时,存在调度效率低等问题,提出一种基于概率模型的网络切片资源优先度调度的方法。该方法在进行优先度调度时,按照网络切片资源的概率递减的顺序来计算优先度,不仅可准确地计算出资源优先度,还提高了整体运行效率。仿真证明,所提方法在调度跨度、平均队长和资源丢失数量方面相比现存调度方法都有明显的改进,同时保证了资源的完整度,可为通信工程管理工作提供帮助。
