    摘    要：

    研究一种有效的数据资源快速存储方法, 可以减少丢包率, 保证存储的稳定性, 提高数据存储速度, 具备一定的应用价值。针对传统的非结构化数据资源快速存储模式存在能耗均衡性较差、存储延时较长和丢包率较高等问题, 提出基于海量存储云调制的非结构化数据资源快速存储方法, 该方法将多类别非结构化数据资源集合作为输入, 对特征要素进行阈值处理, 输出大规模非结构化数据资源关联模型。根据关联模型得到数据资源储存适应的关联强度, 将强度与频宽占用概率进行非结构化数据资源储存控制, 调节非结构化数据的粒度, 依照最小传输粒度频宽梯度传输, 加快大规模非结构化数据资源快速存储。通过实验结果验证, 上述方法可降低存储延时、减少存储丢包率、保证存储能耗的均衡性, 提高大规模非结构化数据资源的存储效率。

    关键词：

    非结构化数据; 快速存储; 存储延时; 丢包率; 均衡性;

    Simulation of Large-Scale Unstructured Data Resource Fast Storage Method

    ZHU Lin LANG Qian-wen HU Jin-bei

    School of Electronic & Communication Engineering, Guiyang University

    Abstract：

    At present, it is necessary to research an effective method for fast storage of data resource. In this way, the packet loss rate can be reduced, and the stability of storage can be guaranteed, and the data storage speed can be improved. Therefore, this article focuses on a fast storage method for unstructured data resource based on massive storage cloud modulation. This method took the set of multi-class unstructured data resource as the input, and then the method performed threshold processing on the feature elements, and thus to output the large-scale unstructured data resource association model. According to the association model, the correlation intensity of data resource storage adaptation was obtained. Moreover, unstructured data resource storage control was performed on intensity and bandwidth occupation probability. Finally, the granularity of unstructured data was adjusted. According to the minimum transmission granularity bandwidth gradient, unstructured data was transmitted. Thus, the rapid storage for large-scale unstructured data resource was accelerated. Simulation results prove that the proposed method can reduce the storage delay, decrease the storage packet loss rate, and ensure the equilibrium of storage energy consumption. Meanwhile, this method can improve the storage efficiency of large-scale unstructured data resource.

    Keyword：

    Unstructured data; Fast storage; Storage delay; Packet loss rate; Balance;

    1 引言

    互联网技术的快速发展, 带动了Internet技术的不断提升, 使非结构化数据的数量日益增加, 用于管理结构化数据关联数据库的局限性逐渐展现出来[1]。继而, 数据库技术相应进入基于网络应用的非结构化数据库时代。数据资源储存为人们生活带来很大便利的同时, 也使存储技术面临着很大的压力。非结构化数据体积较大、格式多样化, 增加速度较快, 需要设计一个具备使用性较强、扩展性较大的存储系统作为快速储存的总目标[2]。伴随着数据资源爆发式的增加, 为结构化数据存储而存在的存储系统, 已经无法满足大规模非结构化数据资源储存的需求。在非结构化数据资源储存过程中, 由于非结构化数据资源储存规模较大, 导致原有存储系统存储丢包率较高、速度较慢以及能耗均衡性较差等问题, 需要研究一种有效的非结构化数据资源快速储存方法, 可以减少丢包率, 保证存储的稳定性, 提高数据存储速度, 具备一定的应用价值。

    文献[3]提出一种基于邻接链表的非结构化数据资源快速储存方法, 通过分析目标信息, 获取目标信息中的非结构化数据, 建立邻接链表非结构化数据结构, 实现非结构化数据资源快速储存。该方法存在数据存储延时的问题。文献[4]提出一种基于面向SaaS应用的非结构化数据资源快速储存方法, 该方法通过对数据资源的分类定义, 设定非结构化数据资源存储计划的模式化语义和数据存储策略, 再利用非结构化数据资源存储计划的数据访问算法对非结构化数据进行分类存储, 实现了非结构化数据资源快速储存。该方法存在存储数据的能耗均衡性较差的缺点。文献[5]提出一种基于异构监控的非结构化数据资源快速储存方法, 以基于反范式化, 运用统一异构数据表现方式兼容非结构化数据资源, 经过区分标签, 非结构化数据压缩、数据缓存预热、索引等机制完成了非结构化数据资源快速储存。该方法存在存储丢包率较高的缺点。

    综上所述, 提出一种基于海量存储云调制的非结构化数据资源快速存储方法, 通过仿真测试证明, 该方法可以降低丢包率、减小存储延时, 提高能耗的均衡性, 保证存储速度。

    2.2 大规模非结构化数据资源快速存储方法

    基于上述建立大规模非结构化数据资源关联模型, 提出了基于海量存储云调制的非结构化数据资源快速存储方法, 能够在考虑非结构化数据存储关联强度的情况下, 当计算得到的一阶距是负数时, 利用最小传输粒度超频宽梯度, 对大规模非结构化数据存储出现数据拥塞现象进行调节, 具备存储分流的作用, 同时加快了大规模非结构化数据存储速度, 实现大规模非结构化数据资源快速存储[9,10]。

    1) 非结构化数据资源储存控制

    根据关联模型中的数据资源储存适应的关联强度V (Ux, Uy) 和频宽占用概率O (Ux, Uy) 进行非结构化数据资源储存控制。

    在云网络中, 非结构化数据储存都有一个单独的储存节点, 不同的节点数据资源的关联强度均会不尽相同, 可运用数据资源储存适应的关联强度V (Ux, Uy) 和频宽占用概率O (Ux, Uy) 进行传输分配。kτ表示实时非结构化数据流接收的单独储存节点, α表示泊松分布的非结构化数据流服从指数, 同时数据资源储存关联强度可以满足同等指数分布[11]。则满足非结构化数据关联强度V (Ux, Uy) 的一阶距条件的计算公式如下

    F[V (Ux, Uy) ]=α-F|V (Ux, Uy) |α2F|V (Ux, Uy) 2|-F|V (Ux, Uy) |    (10)

    式中, F|V (Ux, Uy) |表示V (Ux, Uy) 的期望, 与F|V (Ux, Uy) 2|表示V (Ux, Uy) 的二阶期望。

    数据资源储存关联强度O (Ux, Uy) 的一阶距F[Ο (Ux, Uy) ]与V (Ux, Uy) 的F[V (Ux, Uy) ]达到线性分比例关系的具体表达式如下

    F[Ο (Ux, Uy) ]=α2F|Ο (Ux, Uy) 2|-F|Ο (Ux, Uy) |α-F|Ο (Ux, Uy) |    (11)

    当式 (10) 计算得到的一阶距是正数时, 全部非结构化数据资源快速储存关联属于正向关系, 当计算得到的一阶距是负数时, 就需要运用一定的机制, 调节非结构化数据的粒度, 可以增强非结构化数据资源传输速度, 完成速度控制[12]。

    3 仿真分析

    为了证明基于海量存储云调制的非结构化数据资源快速存储方法的有效性, 将文献[3]方法、文献[4]方法和文献[5]方法与基于海量存储云调制的非结构化数据资源快速存储方法作比较, 主要选取存储延时、存储丢包率和存储能耗的平衡性作为评价指标进行仿真测试。

    实验主要配置是Intel Xeon E3-1231 v3;主频为3.4GHz, 内存为32GB, 2TB硬盘。Win7 64位系统, 运用Java编辑语言C++7.0的环境下构建测试平台。

    1) 数据资源储存延时

    仿真中, 选用100个节点功率的Hasoop集群, 20个主要节点NameNode, JobTracker和另外80个节点功率, 运用文献[3]方法与基于海量存储云调制的非结构化数据资源快速存储方法进行数据资源储存延时时间 (ms) 对比, 测试结果如图1所示。

    图1 两种快速存储方法的存储延时对比   下载原图

    分析图1可知, 在节点功率不断上升的同时, 存储时间也在不断的提高, 原因是由于节点功率的增加, 导致非结构化数据流量突然增多, 数据拥塞现象不断发生。当运用文献[3]方法时, 非结构化数据存储延时间较长, 并且波动幅度较大, 存储速度较慢, 并不能更好的缓解数据的拥塞现象, 当运用基于海量存储云调制的非结构化数据资源快速存储方法时, 非结构化数据存储延时时间较短, 波动幅度较低, 储存速度较快, 说明运用所提方法与文献[3]方法相比, 能够有效的减少存储时间, 保证数据传输的稳定性, 加快非结构化数据存储速度。

    4 结束语

    伴随着因特网技术的迅速发展, 非结构化数据的数量日益增加, 导致非结构化数据的规模不断扩大, 原有的存储技术已经不能满足人们的存储需求, 为了提高大规模非结构化数据资源的存储速度, 提出一种海量存储云调制的非结构化数据资源快速存储方法, 能够在考虑非结构化数据存储关联强度的情况下, 当计算得到的一阶距是负数时, 利用最小传输粒度超频宽梯度, 对大规模非结构化数据存储出现数据拥塞现象进行调节, 具备存储分流的作用, 同时加快了大规模非结构化数据存储速度。实现大规模非结构化数据资源快速存储。通过仿真表明, 所提快速存储方法保证了快速存储的均衡性, 将存储延时降低到最小, 具有一定的时效性。
