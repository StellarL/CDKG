    基于脆弱特征导向的移动端软件性能测试仿真

    摘    要：

    当前移动端软件性能测试方法在进行移动端软件性能测试时,测试所用的时间较长,得到的测试结果与软件实际性能不符,存在测试效率低和测试结果准确率低的问题。提出基于脆弱特征导向的移动端软件性能测试方法,利用错误传播分析方法寻找移动端软件中存在的脆弱点,用挖掘网络关键节点的问题代替挖掘移动端软件脆弱点的问题,利用局部极值实现移动端软件脆弱点的挖掘,并将脆弱点中的敏感字节作为脆弱点的特征。在通用弱点评价体系的基础上计算移动端软件的脆弱性影响和脆弱性可利用性,根据计算结果结合脆弱点特征完成移动端软件性能的测试。仿真结果表明,所提方法的测试效率高、测试结果准确率高。

    关键词：

    脆弱特征导向; 移动端软件; 性能测试;

    Mobile Terminal Software Performance Test Simulation Based on Fragile Feature Orientation

    DING Qun

    School of Information Engineering Zhujiang College, South China Agricultural University

    Abstract：

    Currently, the test time is too long and the test result is inconsistent with the actual performance of software. Meanwhile, the test efficiency is low and the test accuracy is low. This paper focuses on a method to test mobile software performance based on fragile feature orientation. At first, we used the error propagation analysis method to find the vulnerable points in mobile software, and then we used the problem of mining the key network nodes to replace the problem of mining the vulnerable points in mobile software. In addition, we used local extremum to mine vulnerable points in mobile software, and took sensitive bytes in vulnerable points as the features of vulnerable points. Based on the general vulnerable point evaluation system, we calculated the influence and availability of vulnerability of mobile software. According to the calculation results, we completed the test of performance of mobile software by combining with the characteristics of vulnerable points. Simulation results prove that the proposed method has high efficiency of test and high accuracy of test results.

    Keyword：

    Fragile feature orientation; Mobile software; Performance test;

    1 引言

    移动端数量以及应用类型的不断增加,人们利用移动端软件买卖商品、拓展业务、娱乐休闲和亲友交流等活动的频率越来越高[1]。使用移动端软件的用户数量和用户需求不断攀升,移动端软件的升级也逐渐加大了对外开放的程度,增加了移动端软件性能和构架的复杂性[2]。当代社会是一个高效率时代,影响效率的关键点在应用交付链中的类型逐渐繁杂,面对上述状况,现有移动端软件正在进行升级,用户操作过程中的不稳定性和系统的复杂性同样对移动端软件提出了更高的要求。业务信息化程度不断提高以及业务流程运转之后,业务流程更容易被用户掌握,且移动端软件的多种性能开始觉醒。经调查发现,存在百分之七十左右的用户希望在移动端打开网页的速度与电脑上一样快[3]。为了保证移动端软件的多元化和逻辑功能的复杂化,同时满足用户对移动端软件的需求,保障移动端软件启动时间短、耗电少、软件安装时间短、消耗流量少和响应速度快等特点,需要测试移动端软件的性能,根据测试结果提升移动端软件的整体性能。当前移动端软件性能测试方法存在测试效率低和测试结果准确率低的问题,需要研究移动端软件性能测试方法[4]。

    吴佳、曾惟如、陈瀚霖等利用K-means方法利用移动端软件的特征参数建立移动端软件的观测状态,根据观测状态构建马尔可夫模型,得到移动端软件内部状态和观测状态之间的关系,采用指数平滑法预测移动端软件特征参数,根据特征参数完成移动端软件的测试[5]。姜腊林、彭霞、熊兵通过分析移动端软件的消息处理时间和到达时间,在排队论的基础上结合分析结果构建移动端软件性能测试模型,推导平均逗留时间、等待时间、队列长度和等待时间等性能参数,完成移动端软件的性能测试[6]。向敏、何川、田力等人选取移动端软件性能的测试指标,利用模糊处理函数对指标做模糊归一化处理,在模糊神经网络模型中引入指标数据,完成移动端软件性能的测试[7]。

    以上三种方法均存在测试效率低和测试结果准确率低的问题,因此提出基于脆弱特征导向的移动端软件性能测试方法。

    2 基于最优化分析方法的移动端软件脆弱点挖掘

    基于脆弱特征导向的移动端软件性能测试方法采用最优化分析方法挖掘移动端软件中存在的脆弱点,将挖掘脆弱点当做寻找全局最优的问题。

    设D代表的是要考察的移动端软件数据集合,其表达式如下

    D={D1,D2,…} (1)

    将移动端软件受不同错误的影响程度作为评价标准,构建评价标准集合ED,其表达式如下

    ED={ED1,ED2,…} (2)

    设Eth代表的是判决门限,在ED(i)≥Eth的基础上选取移动端软件中对应的信号。在通常情况下移动端软件的复杂性较高,简单的模型难以描述移动端软件。节点参数度是一个变量,可以描述移动端软件的局部性质,基于脆弱特征导向的移动端软件性能测试方法采用节点参数度描述移动端软件的性质,通过局部极值点挖掘移动端软件的脆弱点,具体步骤如下

    1)生成错误传递矩阵MPEPSA→B:根据数据流在移动端软件中的交互关系,得到移动端软件中存在的错误传递矩阵,移动端软件错误传递网络中存在的局部信息由错误传递矩阵组成。通过渗透率PEPSi,kA→B得到错误传递矩阵MPEPSA→B。渗透率PEPSi,kA→B的计算公式如下

    ΡEΡSi,kA→B=1Ν∑lΡEΡ(Ι)i,kA→B (3)

    式中,N代表的是数据在移动端软件中的二进制位数;I代表的是1～N范围内的整数。PEP(I)i,kA→B代表的是第I位的错误渗透率。

    2)进行数据加权处理后生成度矩阵MDI。计算错误传递矩阵的总和,得到所有元素在移动端软件中的度,其计算公式如下:

    MDI=∑MPEPSI→I+1(,:) (4)

    式中,矩阵MDI由输入数据在第I个模块中的度组成,度矩阵MDI中存在的第k个元素即为模块I中存在的第k个数据对应的度MDI(k);(,:)代表的是按行求取错误传递矩阵MPEPSI→I+1的总和。上式计算得到的节点度是信号渗透率对节点度进行加权处理的结果,是一种加权节点度。

    3)比特级脆弱点在移动端软件中的识别:计算不同数据位在移动端软件中的渗透率,根据计算结果进行排序,渗透率越高,数据位就越有可能流入移动端软件中,增加了移动端软件的脆弱性[8]。

    4)模块级脆弱点在移动端软件中的识别:统计模块在移动端软件中的平均节点度,即计算模块度矩阵在移动端软件中的均值。模块在移动端软件中的平均节点度越高,移动端软件受模块的影响越大。

    5)信号级脆弱点在移动端软件中的识别:利用节点度识别移动端软件中存在的信号脆弱点,通过节点在移动端软件中的局部特性筛选节点,在移动端软件中寻找到满足要求的脆弱点。通过设置合理的判决门限,使虚景概率和漏检概率保持均衡[9]。模块在移动端软件中的平均渗透率可以表示模块的重要程度和对移动端软件的影响程度。通过平均渗透率对各模块中信号对应的筛选系数进行加权,计算公式如下

    ξ=k⋅∑iDi×(Ρi∑iΡi+Di∑iDi)2 (5)

    式中,Di代表的是模块i中存在的总数据量;Pi代表的是模块i在移动端软件中的平均渗透率;k代表的是权重系数;ξ代表的是加权处理后的筛选系数。通过筛选系数ξ挖掘出移端软件中存在的脆弱点。

    通过二分查找算法提取脆弱点中存在的敏感字节,作为脆弱点的特征,具体步骤如下:

    1)设Size代表的是输入程序的文件大小;Low=0代表的是文件偏移量,对其做初始化处理;High代表的是上限,其表达式如下:

    High=Size (6)

    设Low代表的是下限,在上限High和下限Low中确定敏感字节的位置。

    2)设置终止条件:如果满足下式,输出上限High和下限Low:

    Low>High-3 (7)

    3)设Mid代表的是上限High和下限Low的中间位置,其计算公式如下

    Μid=Low+Ηigh2 (8)

    确定敏感字节所在的位置,提取敏感字节作为脆弱点的特征t。

    3 基于脆弱特征导向的移动端软件性能测试

    基于脆弱特征导向的移动端软件性能测试方法根据通用弱点评价体系中的评分因素结合脆弱点的特征测试移动端软件的性能,测试方法包括脆弱性影响和脆弱性可利用性两部分,其中脆弱性可利用性中存在确认程度、攻击途径、修正措施、攻击复杂度、利用代码和认证;脆弱性影响中存在目标移动端软件脆弱点安全需求和安全需求破坏性以及脆弱点在软件中的依赖度。在上述分析的基础上,进行脆弱性可利用性以及脆弱性影响两方面进行分析。

    3.1 脆弱性可利用性

    脆弱性可利用性是攻击者利用目标移动端软件中存在的脆弱性发起攻击时采取的攻击手段[10]。脆弱性可利用性通常描述的是攻击者利用脆弱性攻击移动端软件的困难程度。攻击者利用移动端软件中某个脆弱性对移动端软件发起攻击的可能性能够通过脆弱性可利用性计算得到。

    脆弱性可利用性通常情况下包含确认程度RC、攻击途径AV、修正措施RL、攻击复杂度AC、利用代码E和认证AU。

    攻击途径AV:攻击者通过脆弱性攻击移动端软件时,发起的攻击可能是本地的,可也能是远程的。

    攻击复杂度AC:攻击者通过脆弱性攻击移动端软件时,攻击的复杂度可以分为三个级别,分别是低级、中级和高级。

    认证AU:攻击者触发移动端软件的脆弱性有可能会需要认证。

    利用代码E:观察通过脆弱性发起的攻击中是否存在可利用代码,如果存在可利用代码,则攻击者有很大的可能会利用移动端软件中存在的脆弱性发起攻击。

    修正措施RL和确认程度RC:通过脆弱性是否通过确认以及是否做了补丁措施进行取值。

    设CR代表的是脆弱性可利用值,通过确认程度RC、攻击途径AV、修正措施RL、攻击复杂度AC、利用代码E和认证AU计算得到,其计算公式如下:

    CR=AV·AC·AU·E·RL·RC (9)

    3.2 脆弱性影响

    攻击者通过目标移动端软件中存在的脆弱性攻击移动端软件后,移动端软件的状态会出现变化,移动端软件受脆弱性攻击产生的影响程度可以通过脆弱性影响比较得到,即通过脆弱性影响描述移动端软件受攻击而出现的移动端软件破坏程度。

    脆弱性影响包括目标移动端软件脆弱点的依赖度VD、安全需求和安全需求破坏性SD,其中目标移动端软件的安全需求包括可用性AR、完整性IR、机密性CR。

    可用性AR、完整性IR、机密性CR:根据实际中对目标移动端软件的需求,有的对目标移动端软件可用性要求很高,有的对目标移动端软件机密性要求很高。基于脆弱特征导向的移动端软件性能测试方法提取的移动端软件安全需求通过一组权重值进行描述,表示了移动端软件对可用性、完整性和机密性的要求。

    安全需求破坏性SD:通过SD表示不同安全需求受脆弱性影响而出现的移动端软件破坏程度。基于脆弱特征导向的移动端软件性能测试方法通过需求破坏向量表示脆弱性导致的破坏程度。需求破坏向量是通过移动端软件的可用性、完整性和机密性受脆弱性影响而生成的具有破坏倾向的向量。

    脆弱点的依赖度VD:通过脆弱点依赖度描述不同脆弱点在移动端软件中的重要性。脆弱点依赖度又可以描述为脆弱点节点在脆弱点依赖图中对应的出度,节点出度代表的是其它脆弱点在移动端软件中对脆弱点的依赖度。移动端软件中依赖某脆弱点的脆弱点总数随着节点出度的增大而增加。

    其它脆弱性影响的因素都在区间[0,1]内取值,所以基于脆弱特征导向的移动端软件性能测试方法计算移动端软件的脆弱性影响值时,归一化处理脆弱点在移动端软件脆弱点依赖图中的依赖度。设n代表的是移动端软件中存在的脆弱点数目;设置系数x,如果脆弱点在移动端软件中依赖度为n-1时,系数x的值为1。

    通过可用性AR、完整性IR、机密性CR、安全需求破坏性SD、脆弱点的依赖度VD以及系数x得到脆弱性影响值YX的计算公式

    YX=(CR,ΙR,AR)⋅SD⋅VDx (10)

    结合移动端软件脆弱点的特征t、脆弱性可利用值CR和脆弱性影响值YX得到移动端软件的性能测试结果JG,测试结果为移动端软件性能的评分,在区间[0,100]内取值

    JG=(CR+YX)2t (11)

    当前移动端软件在性能方面都存在一些问题,通过测试移动端软件的性能可以提高移动端软件的整体质量。针对当前移动端软件性能测试方法存在测试效率低和测试结果准确率低的问题,提出基于脆弱特征导向的移动端软件性能测试方法,可在较短的时间内准确的测试移动端软件的性能,为移动端软件的进一步发展奠定了基础。
