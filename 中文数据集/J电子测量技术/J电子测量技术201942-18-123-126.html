    虚拟仪器软件执行时间测试与评估方法

    余学锋 刘辉 翟巧丽

    中国人民解放军63870部队

    摘    要：

    针对虚拟仪器动态响应时间评估需求,提出了虚拟仪器软件执行时间测试的新方法。为了实现软件执行时间测量,开发了Lab VIEW时间测量功能。通过执行过程的重复循环以及测量数据的统计分析,有效解决了程序执行时间短和Lab VIEW时间测量精度有限的问题。对虚拟仪器执行时间测试方法实验结果表明,新方法可用于虚拟仪器动态响应时间评估。

    关键词：

    虚拟仪器; 程序执行时间; 实时系统; 测试与评估;

    作者简介： 余学锋,硕士,高级工程师,主要研究方向为兵器试验与测试技术。E-mail:yxfyd@163.com;

    收稿日期：2019-03-26

    Test and assessment methodology of software execution time in the virtual measurement systems

    Yu Xuefeng Liu Hui Zhai Qiaoli

    Unit 63870,PLA

    Abstract：

    Deal with the assess requirements for virtual instrumentation dynamic response time, This paper presents the new approach to the test and assessment of the software execution time in the virtual instrumentation. To determine the times of the VI software part execution, the Lab VIEW timer function is exploited. By repeating examined operations in a loop, and using statistical analysis to determine the time of a single operation, the question of very short time of the tested operations and the limited accuracy of the Lab VIEW timer is effectively solved. The experimental results show that virtual instrument execution time test new method can be used to evaluate the dynamic response time of virtual instrumentation.

    Keyword：

    virtual instrumentation; program execution time; real time systems; test and assessment;

    在常规武器鉴定试验中,虚拟仪器技术已经广泛应用于各类主测设备中。长期以来,对于虚拟仪器的性能评价更注重于测量结果不确定度以及软件可靠性等。由于虚拟仪器与通用仪器最大的不同是它是软件仪器,因此应用程序的执行时间也是其关键参数。特别是对于实时测量环境,不仅对测量结果的准确性有要求,同时对其运行时间也有约束。当硬件配置和操作系统选定情况下,虚拟仪器的实时性很大程度上依赖于软件执行时间。特别当测试需求的动态响应成为关键因素时,优化程序执行时间就显得非常重要。

    目前在虚拟仪器中广泛采用的软件开发平台是Lab VIEW语言,它是一种图形化语言(虚拟集成开发环境)。Lab VIEW的程序是数据流驱动的,相较于传统的文本编程语言如C、Java、VB等,LabVIEW中它每一项执行功能都是以图标形式表述(例如快速傅里叶变化FFT图标),而连接图标之间的数据流控制着程序的执行次序,其图标所需的执行程序指令周期数无法获知,因此不能采用以往的计算指令周期数的方法。

    关于程序执行时间的测量评估,文献[1]对Lab VIEW程序循环结构运行速度进行了讨论,文献[2]给出了Linux下代码运行时间的高精度测量,文献[3]分析了云应用程序执行时间预估方法,文献[4,5,6,7]给出了实时系统最坏执行时间分析方法等,但主要是用于C、Java、VB等顺序执行结构类语言或虚拟机申请在云资源调度中的执行时间估计,均不涉及Lab VIEW程序执行时间的测量。

    为此,本文提出了一种新的Lab VIEW程序执行时间分析测量方法,该方法可在图形化软件环境及并行结构下获取执行时间,并给出软件执行时间的分布情况,它可以为软件稳定运行的临近点的设计以及软件开发的实时响应测试需求提供帮助。

    1 虚拟仪器执行时间测试原理

    基于Lab VIEW平台的虚拟仪器系统软件可以分解为多个软件功能模块,比如数据处理模块,频谱分析模块。以实时频谱分析仪为例,就包括波形显示、信号发生、加窗、滤波、谱分析等功能模块[8,9,10]。而每个软件模块又可分解为若干个基本软件单元,以单一图标显示。对于实时测量而言,在软件设计过程中,一方面要了解整个程序执行时间,以及分析各功能模块执行时间。另一方面,通过执行时间测试,了解受操作系统、硬件配置及运行环境影响,软件执行时间的分布情况以及最坏执行时间等。由于大多数软件执行时间都非常短,在微秒级甚至纳秒级别,对执行时间进行单次测量精度不能保证。需要通过在执行时间进行多倍循环的基础上应用统计方法来实现[11,12,13]。因此,对虚拟仪器程序执行时间测量时,采用了P次循环运行方式,在P次循环中分别设置了“内循环”和“外循环”模式进行。

    在“内循环”模式下,每次程序运行中,先对程序中的各功能模块(S个)进行P次循环运行,分别测量各功能模块P次循环后的执行时间tpink(k=1,2,…,S;i=1,2,…,P),将各模块测量值相加得到整个程序的执行时间tin,然后再按上述测量过程重复N次,通过统计学方法得到程序执行时间的平均值t¯in和标准偏差σin。通过内循环方式进行软件执行时间测试,可以分析软件中各功能模块执行时间的分布,并分析这些模块与软件总的执行时间的关系,对改善软件实时性提供帮助。

    在“外循环”模式下,每次程序运行中,先对整个程序进行P次循环运行,测量经过P次循环后的程序执行时间TPout,然后再按上述测量过程重复N次,通过统计学方法得到程序执行时间的平均值t¯out和标准偏差σout。通过“外循环”方式进行软件执行时间测试,可以分析虚拟仪器软件运行与计算机操作系统软硬件的相关性,以及运行环境变化对软件运行时间的影响。内、外循环模式下软件执行时间测量过程及算法流程如图1～图4所示。

    可以看出,软件执行时间测量及优化有下几个步骤:首先根据实际的运行软件原型实时性使用要求,确定软件执行时间测量精度和允许执行时间最大值。然后,对所测试的软件中各功能模块进行分类并测量每个模块的平均执行时间,找出对执行时间贡献较大的功能模块并进行设计优化。最后,再对整个软件完成任务所需要的执行时间进行测量,并与软件设计运行时间理论值进行比较,直到满足实时性要求为止。

    2 虚拟仪器执行时间测试与验证

    在Lab VIEW环境下软件执行时间的测量主要有两种方式,一种为直接测量方式,即直接在任务中插入时间测试软件代码,运用编程工具本身提供的时间测量软件模块,获取测试任务执行的开始时间和结束时间。在Lab VIEW编程环境下,时间测量模块有Tick Count库函数,它可以提供毫秒级的计时。还有时间戳计数器Timestamp库函数,大约可以精确到微秒。另一种为外部测量方式,即编制专门的时间测量程序,通过设置的外部测试点进行测试,时间测试程序监控和跟踪任务软件执行过程,记下任务的开始时间和结束时间,即可获取任务软件运行时间[14,15]。本文对时间测量模块中的时间戳计数器Timestamp库函数进行开发,编制了专门的时间测量程序,实现软件执行时间测量。

    在程序执行时间测量中,通过Lab VIEW编程内置专门的时间测量程序,嵌入被测试软件中,使其可以对整体软件或部分模块重复运行的时间进行测量,并采用统计方法确定程序单次运行时间。所有执行时间测量数据的概率分布均假设为高斯分布,测量结果验证采用对已知执行时间的软件进行测量,执行时间的理论设计值作为参考值与测量值进行比较。软件执行时间测量值分别存储在矩阵Win和矩阵Wout中。

    从实际测量中发现,Pin(Pout)循环次数的选择对执行时间测量结果特别是标准偏差有较大影响,为了确保测量结果的重复性,Pin(Pout)至少应>30次,但Pin(Pout)过多对测量质量的提升有限,循环次数达100次就基本可以满足要求。Pin(Pout)与执行时间测量标准偏差关系如图5所示。

    本文设计的执行时间测试方法,采用的“内循环”方式可以分析软件中各功能模块执行时间的分布,而“外循环”方式可以分析虚拟仪器软件运行与计算机操作系统软硬件的相关性。通过多次迭代循环测量,可有效消除计算机硬件及操作系统对软件执行时间测量结果的影响。同时采用统计学方法处理得到的执行时间平均值更趋近于实际值,而标准偏差可以反映程序执行时间的稳定性。

    3 结 论

    基于Lab VIEW平台的虚拟仪器系统软件执行时间不仅能测量而且可以进行设计优化,通过开发Lab VIEW语言中自带的时间测量模块,嵌入到实际程序中随需随用,为软件开发设计者理解软件算法、数据结构差异对软件执行时间的影响提供帮助。

    本文给出的软件执行时间是Lab VIEW编程语言运行时间,不包括数据采集卡上数据采集时间。随着计算机硬件技术的发展,软件执行时间也将越来越短,因此开发出可对采集时间进行估计以及更高精度的执行时间测量模块将是未来研究的方向。
