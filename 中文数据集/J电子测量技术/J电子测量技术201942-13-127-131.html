    极化码的K-best译码算法

    许虎 张煜

    中国电子科技集团公司第四十一研究所 电子信息测试技术安徽省重点实验室

    摘    要：

    基于K-best算法和球形译码算法, 提出了一种占用极少内存的K-best译码算法, 该算法解决了短码情况下SC译码算法性能不佳和SCL译码算法硬件实现占用内存过多的问题, 改进了SD时间复杂度不固定和译码性能过于依赖初始半径选择的问题, 可以通过调整K大小在纠错性能和复杂性之间提供良好的折衷, 具有固定时间复杂度, 并且不涉及半径。仿真结果表明, 目标算法与SC算法和SCL算法的译码性能几乎相同, 但仅占SCL算法48%的内存。

    关键词：

    5G NR; Polar码; 级联码; 球形译码; K-best译码;

    K-best decoding of polar codes

    Xu Hu Zhang Yu

    The 41ST Institute of CETC Electronic Measurement Technology Key Laboratory of Anhui Province

    Abstract：

    A polarization code decoding algorithm based on K-best is proposed, which solves the poor performance and SC decoding algorithm in short code cases, the hardware implementation of the SCL decoding algorithm consumes too much memory. It also improves the problem of the sphere of the spherical decoding, the time complexity and the decoding performance too dependent on the initial radius selection. The proposed algorithm is based on k-best algorithm and spherical decoding, and can provide a good tradeoff between error correction performance and complexity by adjusting K size, which has fixed time complexity and does not involve a radius. The simulation results show that the proposed algorithm is almost the same as that of SC algorithm and SCL algorithm, but it occupies 48% of the memory of SCL.

    Keyword：

    5G new radio; polar codes; concatenated codes; sphere decoding algorithm; K-best decoding algorithm;

    2009年Arikan[1]提出了极化码, 并且通过数学方法证明了当码长无限长时其性能可以达到香农极限。极化码是一种结构性与迭代性极强的信道编码技术, 一经提出就在国际上引起广泛的关注, 2016年, Polar码被采纳为5G eMBB场景的控制信道编码方案[2]。

    Arikan提出的针对极化码的连续消除 (successive cancellation, SC) 译码算法, 对长码有很好的纠错性能, 但是对于中短码长译码性能有显著的降低。为了克服这个问题, 文献[3]提出了 (successive cancellation list, SCL) 译码算法, 特别是在冗余循环校验 (cyclic redundancy check, CRC) 辅助下的SCL的译码性能几乎可以与最大似然 (maximum likelihood, ML) 译码相当, 但同时SCL译码引入搜索宽度为L的路径列表, 因此算法复杂度相对于SC译码也增加L倍。文献[4]提出分段添加多个CRC辅助SCL译码减少列表大小和误帧率。文献[5]提出了堆栈SC (SC stack, SCS) 译码, SCS解码器的时间复杂度比SCL的时间复杂度要低得多, 并且在高信噪比情况下可以非常接近SCL。但是算法的实现仍然需要大量的存储器来存储由解码器生成的中间值。

    基于此, ML译码算法, 例如球形译码 (sphere decoding, SD) 被应用于极化码[6]。ML译码纠错性能最佳, 但是复杂性非常高, 以至于仅适用于非常短的极化码。SD算法虽然可以在一定程度降低ML译码算法的复杂度, 但是O (N3) 的复杂度使其只能应用于短码。此外SD算法有2个重要缺点, 1) SD算法时间复杂度依赖于信道状态, 因此时间复杂度是可变的, 难以保证解码器纠错性能和吞吐量;2) SD依赖于球体半径的选择, 取决于通道特性。选择不理想的半径会导致纠错性能下降。

    本文提出基于K-best的极化码译码算法。该算法是前向搜索的过程, 每一层节点的计算相互独立, 可并行执行, 因此具有固定的时间复杂度, 且不涉及球体半径的选择, 而且可以通过参数K调整纠错性能 (直到ML界限) 和复杂度之间的平衡。

    1 极化码

    极化码是一种结构性与迭代性极强的信道编码技术[7], 其核心理论是信道极化, 主要包括信道联合过程和信道分裂过程。

    1.1 信道极化

    信道联合:对已知的二进制离散无记忆信道W进行N次迭代复制WN:XN→YN, N=2n, 并对复制所得信道进行递推方式组合。WN和WN之间的转移概率关系为:

    WΝ (y1Ν|u1Ν) =WΝ (y1Ν|u1ΝGΝ)           (1)

    信道分裂:将WN恢复成N个相互独立的子信道WΝ (i) {WΝ (i) :1≤i≤N}的过程, WΝi:XN→YN×Xi-1, 定义其转移概率为:

    WΝ (i) (y1Ν, u1i-1|ui) =Ρ (y1Ν, u1i-1, ui) Ρ (ui) =∑ui+1Ν∈XΝ-i12Ν-1WΝ (y1Ν|u1Ν)           (2)

    根据Polar Code编码结构, 式 (2) 可由奇序分裂信道的转移概率和偶序分裂信道的转移概率2个递归式得到。对任意n≥0, N=2n, 1≤i≤N/2有:

    WΝ (2i-1) (y1Ν, u12i-2|u2i-1) =∑u2i12WΝ/2 (i) (y1Ν/2, u1, o2i-2⊕u1, e2i-2|u2i-1⊕u2i) ⋅WΝ/2 (i) (yΝ/2+1Ν, u1, e2i-2|u2i)           (3) WΝ (2i) (y1Ν, u12i-1|u2i) =12WΝ/2 (i) (y1Ν/2, u1, o2i-2⊕u1, e2i-2|u2i-1⊕u2i) ⋅WΝ/2 (i) (yΝ/2+1Ν, u1, e2i-2|u2i)           (4)

    如图1所示, 是码长N=4 096的信道极化仿真。根据仿真结果, 可以看出部分比特信道的信道容量成两极分化。据此可以选择信道容量趋于1的比特信道传输信息比特, 作为信息位;信道容量趋于0的比特信道传输固定比特, 作为冻结位。

    1.2 极化码编码

    极化码编码[8]过程为x1Ν=u1Ν·BNG⨂n, 其中向量uN1= (u1, u2, u3, …, uN) 是经过信道极化后选择的结果, BN是排序矩阵, 将十进制序号i∈{1, 2, 3…, N}按二进制表示为 (i-1) → (bn, bn-1, …, bn) ;然后将该二进制序列反序, 得到 (b1, b2, …, bn) ;以b1为最高位转化为十进制表示为 (b1, b2, …, bn) → (j-1) , G⨂n表示对矩阵的n次克罗内克积。由于比特排序仅在硬件实现方面具有数据访问优势[9,10], 因此本文放弃排序矩阵BN, 并考虑自然索引顺序中的输入位, 以简化算法表示。因此, x1Ν=u1Ν·G⨂n。

    如图2所示为码长为8的极化码编码和传输模型[11], 其中M是调制, s0Ν-1=[s0, s1, …, sN-1]是调制后的向量, W是比特信道, y0Ν-1=[y0, y1, …, yN-1]是接收向量, 通常表示为y0Ν-1=s0Ν-1+v, 其中v是通过W传输时信号的噪声。ML算法是遍历所有路径, 然后选则欧式距离 (euclidean distance, ED) 最小的一条作为译码结果, 计算如下。

    u^0Ν-1=argmins0Ν-1∥y0Ν-1-s0Ν-1∥2=argminu0Ν-1∥y0Ν-1- (10Ν-1-2u0Ν-1G⊗n) ∥2          (5)

    式中:10Ν-1是长度为N的全1向量。

    2 SD译码

    SD算法[12,13]在保证性能的同时, 通过压缩搜索范围降低计算复杂度, 搜索空间是以接收向量y1Ν-1为球心, r为半径的超球体内, 在搜索的同时减小搜索半径, 从而降低复杂度。式 (6) 表示根据ED来选择候选者的路径度量值。

    ΡΜ (u1Ν-1) =∑j=iΝ-1|yj-sj|2=∑j=iΝ-1|yj- (1-2∑k=jΝ-1gkjuk) -|yj||2          (6)

    式中:gkj是GN第k行第j列的元素。

    假定一个码长为N的极化码, 调制方式为BPSK。如图3所示, 在SD算法中, 构造一个N层的译码树。利用极化码生成矩阵GN的下三角结构, 译码从uN-1开始, 然后分别计算0和1两种情况下yN-1和sN-1之间的ED, 并与半径r比较。如果某一层欧氏距离大于半径r, 则不再继续向下搜索, 转向该层的下一个信号继续搜索。若某条路径一直到u0的ED和满足小于半径r, 则保存该路径, 并将半径更新为该路径的ED, 继续下一路径的搜索, 直到搜索完所有路径, 最终存储的路径信息作为译码输出。

    3 K-best译码算法

    SD可以用较少的计算量达到ML的译码性能。但是可变的时间复杂度, 使得SD算法有不可预知的解码延迟 (除了访问所有路径的上限外) 。另外SD的性能过于依赖半径的选择, 半径过大译码复杂度增加, 半径过小译码性能将会大大降低[14]。为了克服这些问题, 提出基于K-best的极化码译码算法。

    定义PEDi为第i层的部分欧式距离 (partial euclidean distance, PED)

    ΡEDi=|yi- (1-2∑k=Ν-iΝ-1gkiuk) |2          (7)

    算法利用极化码生成矩阵GN的下三角结构, 译码从uN-1开始, 按广度优先的方法对路径进行扩展;每一层向下一层扩展时, 选择当前层中具有较小PM的K条。当路径没有扩展到达叶节点而超过搜索宽度, 则按照PM从大到小的排列保留PM小的K条路径。重复这个过程直到到达叶节点, 然后选取PM最小路径作为译码结果。第i层的第k条路径的PM为:

    ΡΜ0k=0ΡΜik={ΡΜi-1k, if (Ν-i) ∈AcΡΜi-1k+ΡEDi, if (Ν-i) ∈A          (8)

    式中:A代表极化码的信息位;Ac为冻结位。目标译码算法有固定时间复杂度, 每次都有一个新比特被估计。由于存在K组码字候选, 所以每次新的位估计产生2 K组候选路径, 其中路径度量值大K条路径需要丢弃。译码过程不涉及半径。如图4所示为K-best译码搜索树K=2, 码长为4, u2为冻结比特。其中黑色代表保留路径, 灰色为删除路径。

    图4 K-best译码搜索树, K=2   下载原图

    K-best译码算法具有比SC和SCL算法更低的内存需求。对于SC译码需要内存为

    (2N-1) QLLR+N-1 (9)

    其中2N-1个内存比特用来存储对数似然比 (LLR) 的值, 将LLR量化为QLLR。剩下的N-1存储部分和。SCL译码需要内存为:

    (N+ (N-1) L) QLLR+LQLLR+ (2N-1) L (10)

    其中:L为列表宽度, 需要 (2N-1) L内存比特存储部分和, 需要 (N+ (N-1) L) QLLR比特存储LLR。QPM用于量化路径度量值, 则需要LQPM比特存储路径度量值。对于SC和SCL译码需要接受到所有矢量才能开始译码, 而K- best译码每次只需要一个LLR。不需要分配LLR内存。只需要对候选路径和相关路径的度量值进行存储, 因此K-best算法所需的内存为:

    LQPM+NL (11)

    5 结 论

    本文从工程实现的角度, 提出了一种快速稳定、易于实现的基于K-best极化码译码算法。算法核心采用每次固定大小的候选路径被保留, 而其它路径被丢弃, 避免了SD可变的时间复杂度和译码性能依赖于半径值选择的缺点。通过对提出的算法进行仿真分析, 结果显示, 在对于极化码短码, 提出的算法有着与SC和SCL译码算法相近的译码性能, 且空间复杂度更低, 大大减少了运算复杂度和硬件要求。该算法已经应用于国家科技重大专项项目“增强移动宽带5G终端模拟器研发”开发中。
