    智能表数据远传系统性能优化研究

    严军 王迎

    上海大学通信与信息工程学院

    摘    要：

    随着物联网技术的发展,远传智能表不断推广应用,业务数据快速增长,导致服务器端有大量并发请求接入,系统并发性能严重影响数据远传系统的稳定运行。根据远传智能表的数据特点,故提出了一种结合远传终端和服务器端的性能优化方案,并设计了相关的通信机制和协议。该方案中针对远传终端,提出一种分时上传算法,平滑系统高峰时期并发数。而在服务器端,采用多服务器协同处理的方式,将系统并发请求轮流分发至多台解析服务器上处理,提高系统服务器端软件吞吐量和处理能力。测试表明,本文提出的优化方案使系统接收到的并发请求均匀分布,并发处理能力有较大提升。

    关键词：

    分时上传算法; 通信协议; 高并发; 无线通信; 服务器;

    Research on performance optimization of smart meter data remote transmission system

    Yan Jun Wang Ying

    Communication & Information Department of Shanghai University

    Abstract：

    With the development of the Internet of Things technology, the remote smart meter is continuously promoted and applied, and the business data is rapidly growing, resulting in a large number of concurrent requests for access on the server side. The system concurrent performance seriously affects the stable operation of the data remote transmission system. According to the data characteristics of the remote transmission smart meter, a performance optimization scheme combining the remote terminal and the server is proposed, and the related communication mechanism and protocol are designed. In the scheme, a time-sharing algorithm is proposed for the remote terminal, which smooths the number of concurrent periods in the peak period of the system. On the server side, the multi-server collaborative processing method is used to distribute the system concurrent requests to multiple resolution servers for processing, thereby improving the system server software throughput and processing capability. Tests show that the optimization scheme proposed in this paper makes the concurrent requests received by the system evenly distributed, and the concurrent processing capability is greatly improved.

    Keyword：

    time-sharing algorithm; communication protocol; high concurrency; wireless communication; server;

    随着物联网技术[1]的发展,行业信息化建设的不断深入,无线远程抄表技术[2,3]替代了传统的挨家挨户的人工上门抄表方式,能够大大提升抄表效率,符合智慧城市的发展潮流。近年来,基于3G、4G[4]等公网无线通信方式的智能表(水、电、气)在我国各省市覆盖率逐渐提高,导致智能表的业务数据量急剧增长,这对智能表数据远传系统性能提出了挑战[5]。

    数据远传系统采用C/S架构,远传智能表将采集数据按照通信协议[6,7,8]上传至服务器端,服务器端可以下发命令给智能表远程进行功能设置,因此要求系统能够全天不间断地稳定运行,其中关键问题是,系统面对大量并发请求能否及时处理并快速响应[9,10,11,12]。目前,智能表数据采用基于时间相关性的差分消零数据压缩算法[13]进行压缩,提高传输效率。同时为了数据安全,使用AES加密算法[14,15]处理后进行传输。

    针对上述问题,根据远传智能表数据的特点,从远传终端和服务器端两个角度考虑,提出一种优化方案,可以根据实际情况远程调整智能表数据上传时刻,提高服务器端的请求处理能力和并发性能,从而优化整个系统的性能。

    1 数据远传系统架构与协议设计

    1.1 系统架构

    数据远传系统包括智能表远传终端、公网和服务器端组成,服务器端包括通信服务器、解析服务器和数据库服务器,如图1所示。智能表远传终端通过3G、4G等公网无线通信方式与服务器端进行交互,智能表主动上传数据,服务器端接收和处理上传数据,同时可以下发命令对智能表进行参数调整、功能设置等操作。

    2 远传终端分时上传算法设计

    2.1 分时上传算法原理

    智能表厂商在生产智能表时,按照各家习惯设置智能表数据上传时刻,智能表上传时刻分布没有规律,导致服务器端在某些时刻并发请求数很高,而其余时间并发请求数较低。因此,根据智能表上传时刻可远程调整的特点,设计一个分时上传算法,根据智能表编号计算每个智能表的上传时刻,将所有智能表上传时刻均匀分布在一天内,自动平滑服务器的高峰并发请求数量。

    由于基站忙闲情况和传输数据长度在一定程度上会影响服务器端处理并发数据的时间,所以根据上传数据处理时长划分忙闲时间段,空闲的时间段可多分配数据上传的智能表数量,每个时间段内智能表数据上传时刻均匀分布。智能表有不同种类,其上传数据有长有短,因此优先将上传数据量大的智能表上传时刻分配在空闲时间段。

    首先按照智能表类型统计所有活跃的智能表数量,按照算法参数将一天24 h划分n个时间段,以h为单位,计算公式为:

    {Si=wi⋅A∑i=1nwi=1,n∈[1,24]         (1)

    式中:wi表示第i个时间段分配智能表数量的权重系数,A表示智能表总数量,Si表示第i个时间段内智能表数量。通过式(1)可将智能表数量平均分配给每个小时,也可以按照忙闲时间段设置不同的权重系数来划分智能表数量,权重系数计算规则为公式:

    wi={1n-x,Τsi-t>ts1n,|Τsi-t|<ts1n+x,Τsi+t<ts         (2)

    式中:Tsi为第i个时间段内历史上传数据平均处理时长,ts为该日上传数据总平均处理时长,x和t为阈值,t可以调整忙闲时间段的划分。

    然后将第i个时间段内的智能表从零开始编号,表示为Num,上传时刻由时分各1字节组成,则小时h和分钟m计算公式为:

    h=(Num modT)+H (3)

    m=ΝumΤmod 60 (4)

    式中:T表示该时间段时长,H表示前i-1个时间段总时长,单位都为小时。mod表示取余,⎣」表示向下取整。通过上述公式可以将每台智能表的上传时刻均匀分布至该时间段内,精度为1 min。

    3 服务器端多服务器协同处理方式

    3.1 多服务器协同处理方式原理

    随着智能表数量的增加,数据越来越多,单台服务器的资源无法满足需求,因此系统实行业务分离,分为数据通信层、业务处理层和数据服务层,针对不同业务的特点分配不同配置的服务器,如图3所示。

    图3 多服务器协同处理原理   下载原图

    通信服务器负责接收所有请求,并将符合条件的数据帧分发至多台解析服务器,解析服务器按照通信协议解析数据帧,并回复相关应答包,数据库服务器负责提供读写数据服务。系统各层之间的耦合度降低,系统稳定性提高,由多台解析服务器协同处理客户端请求,提高了系统的并发能力,并且抽象出数据服务层,简化了数据获取过程,提高了数据处理效率。

    数据远传系统服务器端功能主要是接收、处理、存储智能表上传的经过加密、压缩的数据,最后发送应答数据包,完成一个完整的请求处理过程,其处理流程如图4所示。

    图4 智能表上传数据处理流程   下载原图

    3.2 请求分配原理

    考虑到业务处理层的解析服务器群软硬件配置相同,每个请求的区别主要在于数据包的长短,每个请求处理时间相差不大,因此采用轮询法。轮询法原理是把通信服务器接收的客户端请求按顺序轮流分配给4台解析服务器,如此循环往复,分配流程如图5所示。并且由于采用分时上传算法,智能表上传数据的时间间隔比较均匀,4台解析服务器接收的请求时间间隔相差不大,不会造成解析服务器负载不平衡的情况。

    4 系统性能测试

    4.1 测试背景

    本系统采用C#进行编程,运行在.NET平台上,采用super socket框架,数据库采用oracle,测试某区域内所有活跃的智能表。将数据远传系统安装在云平台上,不间断运行若干天,观察数据上传时刻和并发情况。

    结合远传终端分时上传算法和服务器端多服务器协同处理方式,进行系统优化测试。图6选取了优化前某3天上传表数据来展示,图中可以看出上传时刻分布不均匀,并且智能表数量不断增加,5月25日有4 285台,6月2日有4 523台,7月3日有5 111台。

    优化后的系统并发连接数分布如图7所示,总智能表数量增加至6 296台,系统总并发连接数为6 296,单服务器并发连接数为1 574,可以看出优化后的系统请求均匀分配至4台解析服务器处理,请求时间均匀分散在一天内。系统吞吐量较优化前有较大提高,能够完全接收和处理所有请求,系统并发性能得到提升。

    5 结 论

    本文针对数据远传系统高并发性能的需求,根据智能表上传数据的特点,提出一种优化方案,采用分时上传算法均匀分配远传终端数据上传时刻,服务器端采用多服务器协同处理的方式,数据远传系统实行业务分离,将请求分发给多台服务器协同处理,提高系统处理效率。经过测试表明,本文提出的优化方案能够有效提升数据远传系统性能。
