    摘    要：

    为了提高云构架数据中心网络的流量预测能力, 提出一种基于分簇聚类的网络流量分类调度模型。采用小波熵特征提取方法进行云构架数据中心网络流量预测, 对云构架数据中心网络的流量数据进行了比特序列流模型构建, 采用统计回归分析方法对云构架数据中心网络流量数据进行全样本分段线性规划分析, 采用多尺度小波分解方法提取小波熵这一重要的非线性特征, 根据特征提取结果采用分簇聚类方法进行流量分类调度, 实现云构架数据中心网络的流量预测。仿真结果表明, 采用该方法进行网络流量分类调度的均衡性较好, 流量传输的丢包率较低, 避免了网络拥堵。

    关键词：

    分簇聚类; 网络流量; 分类调度; 云构架; 数据中心网络;

    Research on network traffic classification and scheduling model based on clustering

    Zhang Chuangji

    Guangzhou Huali Science and Technology Vocational College

    Abstract：

    In order to improve the traffic prediction ability of cloud architecture data center network, a network traffic classification scheduling model based on clustering is proposed. A cloud data center network traffic prediction model based on wavelet entropy feature extraction is proposed. The traffic data of cloud data center network are constructed by bit sequence flow model, and the traffic data of cloud data center network are analyzed by using the statistical regression analysis method, which is based on the whole sample piecewise linear programming. Wavelet entropy, an important nonlinear feature, is extracted by multi-scale wavelet decomposition method. According to the feature extraction results, traffic classification and scheduling are carried out by clustering method, and traffic prediction of data center network in cloud architecture is realized. The simulation results show that the proposed method has good equilibrium and high accuracy in traffic prediction and avoids network congestion.

    Keyword：

    clustering; network traffic; classification scheduling; cloud architecture; data center network;

    随着网络信息技术的发展, 通过云构架方式进行数据中心网络构架设计, 实现网络流量的云存储和云计算, 设计云构架数据中心网络, 提高网络的流量集成转发和控制能力, 从而提高网络的数据收发质量和自适应控制能力[1]。为了保障云构架数据中心网络的稳定可靠性运行, 需要对云构架数据中心网络的流量进行准确调度和预测, 在动态网络环境下进行数据中心网络的流量分簇调度和预测, 提高网络的QoS保障能力和服务质量, 结合流量控制和综合调度方法, 避免网络拥塞, 提高网络性能, 结合动态规划设计和信道均衡控制, 实现云构架下的数据中心网络流量优化控制和预测方法, 在提高网络的业务等级和传输带宽方面具有重要意义[2,3,4]。本文提出一种基于分簇聚类的网络流量分类调度模型。首先进行数据中心网络流量模型分析, 然后进行流量比特序列的特征提取和聚类处理, 根据聚类结果进行流量分类调度, 最后进行仿真实验分析, 展示了本文方法在提高网络流量分类调度和预测控制能力方面的优越性能。

    1 流量比特序列流模型构建和特征分析

    1.1 云构架数据中心网络流量的比特序列流模型构建

    云构架数据中心网络的流量数据为一组非线性时间序列, 可以采用非线性特征比特序列流分析方法进行流量的分类调度和预测, 在对云构架数据中心网络流量数据进行建模阶段, 采用比特序列流模型构建方法, 对云构架数据中心网络的超流量运行情况进行特征分析[5]。设有M个全方向性监测的云构架数据中心网络流量数据, 给定监测的云构架数据中心网络流量数据表示为:

    U={U1, U2, …, UN} (1)

    式中:Ui为高频分量为d维的网络传输控制带宽, 各个网络流量分布式调度的数据集扩展带宽为Ui。结合正态相关性调节方法, 构建云构架数据中心网络流量采集的节点分布模型如图1所示。

    1.2 云构架数据中心网络流量的统计回归分析

    在上述比特序列流模型构建的基础上, 采用非线性时间序列分析方法进行数据中心网络流量数据的特征提取和预测[10]。结合关联规则调度方法, 在m维云构架数据中心网络流量数据序列中, 得到网络流量的m维矢量为:

    X (n) ={x (n) , x (n+τ) , …, x (n+ (m-1) τ) }n=1, 2, …, N (7)

    在云架构网络中流量分布序列为均匀正态分布模型, 在重组的特征空间中取的任意一点Xn, 对超过队列拥塞标记阈值进行非线性重组[11], 取得流量序列的最近邻点表示为Xη (n) , 定义Rmn为Xn与Xη (n) 两点之间的距离, 以i为横坐标, j为纵坐标, 用拥塞控制方法进行云构架数据中心网络流量数据的关联规则挖掘, 得到流量序列的特征聚类中心分布为:

    Rmn=‖Xη (n) -Xn‖2 (m) (8)

    用平均互信息算法计算数据中心网络流量重构空间的嵌入维数, 随着m增加到m+1, 得到最优化云构架数据中心网络流量数据的滑动平均窗口为:

    R (m+1) n=‖Xη (n) -Xn‖2 (m+1) (9)

    计算云构架数据中心网络流量时间序列的几何不变量, 得到的si= (xi, xi+τ, …, xi+ (m-1) τ) T称为局部嵌入空间状态矢量, 当R (m+1) n>Rmn时, 认为是数据中心网络流量的统计信息特征点投影, 采用统计回归分析方法对云构架数据中心网络流量数据进行全样本分段线性规划, 结合线性规划模型进行流量均衡调度[12,13,14]。

    2 网络流量的分类调度和预测模型

    2.1 网络流量特征的分簇聚类模型

    在对云构架数据中心网络的流量数据进行了比特序列流模型构建的基础上, 进行流量分类调度模型设计, 本文提出一种基于分簇聚类的网络流量分类调度模型。采用多尺度小波分解方法提取小波熵这一重要的非线性特征, 结合Lyapunov指数分析[15], 在重组的状态空间中, 云构架数据中心网络的流量数据分布的邻域矩阵:

    Bxi= (δxi (j1) , δxi (j2) , …, δxi (jNb) ) T (10)

    由于流量调度受到拥塞丢包和流延迟变大的影响, 设置流延迟控制器, 计算Jxi (1) , 计算数据中心网络流量时间序列的概率置信度区间为{δxi+1 (jk) =xjk+1-xi+1|k∈1, ⋯, Νb}, 从而得到不同业务流之间的隔离邻域矩阵:

    Bxi (1) = (δxi+1 (j1) , δxi+1 (j2) , …, δxi+1 (jNb) ) T (11)

    由于ε足够小, 数据流分配的阈值满足:

    Bxi (1) =Bxi (Jxi (1) ) T (12)

    设B†xi为Bxi的不同业务流的小波分解尺度, 数据中心网络流量采样时刻xi和xj的距离δij=‖xi-xj‖, 当‖xi-xj‖<ε时, 得分簇聚类的统计特征函数为:

    (Jxi (1) ) T=B†xiBxi (1) (13)

    式中:Rtol=15。 通过统计回归分析, 采用多尺度小波分解方法提取小波熵特征量, 对特征提取结果进行分簇聚类分析, 结合聚类分析结果进行网络流量的分量调度和均衡控制设计。

    4 结 论

    在动态网络环境下进行数据中心网络的流量分簇调度和预测, 提高网络的QoS保障能力和服务质量, 避免网络拥塞, 提高网络性能, 本文提出一种采用小波熵特征提取的云构架数据中心网络流量预测模型。采用非线性时间序列分析方法进行数据中心网络流量数据的特征提取和预测, 结合关联规则调度方法, 采用全样本分段线性规划分析方法实现流量的分类调度和预测控制。本文方法进行流量调度的均衡性较好, 降低了流量调度的丢包率, 提高了网络传输性能。
