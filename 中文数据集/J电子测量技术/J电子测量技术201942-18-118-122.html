    摘    要：

    为提升Flash存储器中的数据质量水平,达到SOC芯片对大容量数据高速率、可靠性的存储需求,首先介绍了嵌入式Flash高速存储器的应用性能及基本构成,并透过FLASH存储器中数据坏块和位交换故障对数据存储质量的影响性,引入基于阈值控制的损耗均衡算法,将损耗均分至不同的物理块上,控制坏块发生率;同时,融合ECC校验算法对发生位交换故障的数据进行修正。研究发现,损耗均衡算法通过将各数据块的擦除次数达到近似均衡,延长了Flash存储器的使用寿命,而ECC通过写校验码和读校验码的异或运算,便可实现对错误数据的纠正,两种算法的流程简单、可操作性强,可对Flash存储器数据质量进行有效控制。

    关键词：

    FLASH存储器; 数据质量控制; 损耗均衡算法; ECC校验算法;

    Analysis of data quality control method for embedded high-speed memory

    Guo Jie

    Computer Academy Of Xi'An Aeronautical University

    Abstract：

    In order to improve the data quality level in Flash memory and meet the requirement of SOC chip for large-capacity data high-speed and reliability storage, this paper first introduced the application performance and basic structure of embedded Flash high-speed memory, and through the influence of bad data blocks and bit exchange faults in FLASH memory on data storage quality, a loss equalization algorithm based on threshold control was introduced, which divided the loss into different physical blocks to control the occurrence rate of bad blocks. At the same time, the data of bit-switching faults were corrected by combining ECC verification algorithm. It is found that the loss equalization algorithm can extend the life of Flash memory by approximating the erasure times of each data block. ECC can correct the wrong data by writing and reading the exclusive or operation of the check codes. The two algorithms have simple flow and strong operability, and can effectively control the data quality of Flash memory.

    Keyword：

    FLASH memory; data quality control; loss equalization algorithm; ECC verification algorithm;

    系统级芯片(system on chip,SOC,是一个有专用目标的集成电路)是各种智能化终端的核心设备,随着集成化水平的提高,SOC芯片中嵌入式高速存储器的面积逐渐增大,嵌入式存储器中数据的数量不断增多,存储器中数据质量的水平直接影响物联网终端的智能化水平。目前嵌入式高速存储器主要采用嵌入式快闪存储器(FLASH memory, FLASH存储器),随着FLASH存储器中数据数量的增多,使FLASH存储器产生坏块区和位交换故障次数增多,严重影响FLASH存储器中的数据质量水平。为提升FLASH存储器中数据质量水平,徐永刚基于NAND Flash的嵌入式图像记录技术研究了坏块管理方法,通过位索引对坏块进行标记,利用坏块标记表和滑动窗口技术实现无效块的匹配、查询和替换功能;胡红对基于 FPGA 硬件实现的坏块管理方法进行了研究,该方法利用FPGA片内RAM实现对坏块信息的保存,用顺序查询和跳过的方法对坏块进行坏块管理;但是这两种方法都不适用于随机访问寻址并与上层FTL管理算法结合的存储器;为此本文根据损耗均衡算法[1,2]和ECC校验算法[3,4,5](即数据更正算法)对FLASH存储器中的数据质量进行控制,为嵌入式高速存储器数据质量控制研究提供参考。

    1 FLASH存储器简介

    随着嵌入式高速存储器不断发展,目前FLASH存储器已成为嵌入式高速存储器的主流设备,FLASH存储器是一种非易失性存储器[6,7]。FLASH存储器物理特性与一般嵌入式存储器有根本性的差异,一般嵌入式存储器如同步动态随机存取存储器(synchronous dynamic random access memory,SDRAM)、倍速率同步随机动态处理器(double data rate,DDR)和串行数据传输模式的随机存储器(Rambus DRAM,RDRAM)存储器都属于挥发性存储器,当对存储器停止电流供应,其中的数据便无法保持,每次智能终端开机都需重新载入数据[8];而FLASH存储器在没有电流供应的条件下也能够长久地保持数据,其存储特性相当于硬盘,是目前广泛应用的高速存储器。

    基于NAND(即NOT AND与非关系)型和NOR(即NOT OR或非关系)型两种存储单元构成的FLASH存储器分别称为NAND FLASH存储器、NOR FLASH存储器。其中存储单元以页或块单位,一张数据页中包含多个数据位,若干数据页则组成数据块[9]。FLASH存储器的存储单元为三端器件包含源极、栅极和漏极。存储单元通过在栅极与硅衬底之间设置二氧化硅绝缘层,使浮置栅极中的电荷不被泄漏,提升了存储单元的电荷保持能力[10]。FLASH存储器是一种电压控制型器件。存储单元结构如图1所示。

    图中S表示源极,CG表示栅极,D表示漏极,P表示硅衬底,FG表示浮置栅极。其中,NAND型FLASH存储器通过电子隧道效应进行数据擦除和写入操作的,电子隧道效应是指当能量低于所面对势垒的能量时电子贯穿通过该势垒的量子效应,FLASH存储器写入数据就是电流穿过浮置栅极与硅衬底之间的二氧化硅绝缘层对浮置栅极进行充电的过程,而擦除数据过程就是浮置栅极的放电过程[11]。NOR型FLASH存储器数据擦除操作也是通过电子隧道效应完成的,但在写入数据时则是采用热电子注入方式,即电流通过浮置栅极流到源极的过程。

    2 影响FLASH存储器中数据质量因素分析

    FLASH存储器是一种使用寿命长、处理速度快和数据存储可靠性高的嵌入式高速存储器,存储器中数据质量水平较高,但随着FLASH存储器中数据数量的不断增加,FLASH存储器会产生数据坏块和位交换故障,影响存储器中数据质量,为控制存储器中的数据质量,对这两种故障进行了分析。

    2.1 存储器中坏块的产生对数据质量的影响分析

    FLASH存储器根据数据写入前先擦除历史数据的特性进行数据更新存储,即在对FLASH存储器中的数据进行更新时,需要先将数据预存储的数据块进行擦除操作,擦除该数据块上的历史数据后,然后在同一数据块内写入之前准备存入的数据。目前,FLASH存储器数据块可被擦除次数受到限制,如上文中NAND型FLASH存储器的可被擦除次数上限为一般为1×105次,而NOR型FLASH存储器可被擦除次数则只有1×104次[12]。当FLASH存储器中数据块可被擦除次数超过限制次数,此数据块将成为坏块,导致坏块中的数据不会随着数据更新而更新或者在数据读取时无法被读取,造成存储数据丢失,降低了FLASH存储器中的数据质量;同时,当FLASH存储器频繁对某些数据块进行擦除操作,将导致这些数据块提前成为坏块,大大降低FLASH存储器的使用寿命,同时严重影响FLASH存储器中的数据质量水平。

    为说明坏块产生的原因,以在flash存储器建立FAT16文件系统为例,需要存储的文件包为8 MB,文件包分块大小为2 kB,需要占用4 k个块。在NAND型Flash存储器中,文件将被分为不同的文件簇进行存储,每个文件簇的大小是2 KB,并且在存储文件时会生成文件记录表。FLASH存储器中存入一个8 MB的文件则需要占用4 096个文件簇,为控制数据质量,每存入一个文件簇,文件记录表就更新一次,当整个8 MB的文件存储完成时,文件记录表则需要更新4 096次;若文件记录表在固定数据块中存放,依照每块的擦除寿命10万次计算,当这个8 MB文件更新25次时,文件记录表所在的数据块已经被擦除了10万多次,超过限定次数成为坏块[13]。

    2.2 存储器中位交换现象对数据质量的影响分析

    FLASH存储器中最小的数据存储单位为数据位(即bit),而数据存储的采用的是二进制方式,即在数据位中写入“0”或“1”。但在不稳定因素干扰的情况下会出现数据位中“0”变“1”或者“1”变为“0”的跳变现象,称为位交换现象[14]。当FLASH存储器发生位交换现象,此数据位所在数据块内数据信息发生变化将影响FLASH存储器中数据质量的可靠性水平。不稳定因素包括漂移效应、写操作干扰效应和读操作干扰效应。FLASH存储器的存储单元是电压控制器件,随着存储单元电压变化导致数据位发生位交换现象称为漂移效应;写操作干扰是指FLASH存储器对数据块或数据页进行写入操作时,引起其他数据页或数据块中发生位交换现象;读操作干扰是指FLASH存储器对数据页或数据块进行读取操作时,数据块内数据位发生位交换的现象。

    3 FLASH存储器中数据质量的控制方法

    针对上文中分析影响FLASH存储器中数据质量的因素,为控制FLASH存储器中数据质量的水平,分别对数据坏块和位交换现象的解决方法进行了分析。

    3.1 损耗均衡算法分析

    损耗均衡算法通过减少FLASH存储器中数据坏块的产生数量和速度,进而使FLASH存储器中的数据块中的数据更加准确,不会因数据块变为坏块导致数据丢失,提升了存储器中数据质量水平。

    损耗均衡算法即FLASH存储器中各个数据块的被擦除次数近似均衡,但是损耗均衡算法并不是强制使各个数据块被擦除此达到近似均衡,若强制使各数据块的擦除次数达到近似均衡,会带来额外的擦除操作,增加了数据坏块产生的可能性。损耗均衡是指是在满足相应均衡条件的前提下,减少数据块总的被擦除次数。损耗均衡分为动态损耗均衡和静态损耗均衡两种方式。动态损耗均衡是指FLASH存储器在进行数据存储更新时,将动态数据(即经常被更新的数据)写入擦除次数最小的或未被使用空闲数据块内;静态损耗均衡是指将静态数据(即更新次数少的数据或者是只读数据)强制迁移至已被多次擦除的数据块中去,使各个数据块被擦除次数达到近似均衡,从而提高存储器中的数据质量水平。

    为在简化算法的同时,实现较好的损耗均衡效果,创新性的引入了基于阈值控制的损耗均衡算法,首先依据块内数据的状态,将参与映射的所有物理块分为4类,如表1所示。

    表1 导出到EXCEL


    状态    		内涵界定

    空闲块    		该块已经被擦除,全部页面为空闲状态,可写入数据

    有效块    		该块内某些页面已经写入数据,但写入数据仍有效

    垃圾块    		该块内为无效数据,仅能进行擦除操作

    坏块    		通常不做任何操作

    为实现损耗均衡,数据存储时应以擦除次数小的块为先,静态数据存储在有效块内,较少更新,需要将其迁移出去,将所有块的擦除次数控制在一个数量级内,阈值设为TH,在达到该阈值控制条件时,从存储器有效块中选取最小、最大擦除次数的块,分别记为A、B,两块擦除次数的差值为[15]:

    NumErase(B)-NumErase(A)>TH (1)

    若该差值大于阈值,则表示A块中存储着静态文件,较少更新,此时,启动阈值控制,将B块内的数据迁移至空闲块内,擦除B块,用于存储A块内转移来的数据,将A块标定为垃圾快。该方法的关键是数据迁移发生在静态文件块和擦除次数最大的块之间,在擦除次数最大块中存储静态文件,可降低擦除次数最大块的擦除频率,降低坏块的发生数量和速度。该损耗均衡算法阈值的选取过大,则不能取得很好的损耗均衡效果;擦除阈值选取过小,则静态数据会被频繁迁移,增加了其他数据块的被擦除次数,不利于存储器中数据质量的控制。

    设定NAVD Flash使用寿命为:

    使用寿命=(C-CS)×(1-ρ)FStyp×1fw         (2)

    式中:E为块可擦写次数的最大值;C为芯片容量,CS为已存储文件所占的空间;ρ为额外消耗系数;FStyp为存储文件平均大小;fw为文件的存储频率。

    设定NAND Flash存储器的容量为64 M,ρ为0.7,以损耗均衡算法,将损耗均分至所有的物理块,每天写入一个8 MB大小的文件, 则根据式(2)可得该存储器的理想寿命为[16]:

    理想寿命=容量×总擦除次数×ρ每天写入字节数=64ΜB×100000×0.78ΜB=560000(天)

    可见,基于阈值控制的损耗均衡算法将个快的擦除次数限定在一个数量级中,实现了损耗均衡,降低了坏块的发生率,可有效延长flash存储器的使用寿命。

    3.2 ECC校验算法分析

    根据上文可知存储器中的位交换现象也是严重影响数据质量的重要因素,下文根据ECC校验算法通过对发生位交换现象的数据位进行错误数据更正,提高了存储器中的数据质量水平。

    差错检测和修正(error checking and correction,ECC)校验算法通过对存储数据编解码处理,并对存储器当中数据的进行检错和纠错功能,实现存储器中数据质量的控制。ECC校验算法是根据奇偶校验算法发展而来的,存储器中的数据块是各数据位以矩阵方式组成,利用数据位矩阵中的行和列的奇数偶数信息生成ECC校验码。ECC校验算法根据校验码可检测并纠正发生位交换现象的错误数据。ECC校验码分为写校验码和读校验码,FLASH存储器存入数据时将对存入的数据生成校验码并跟随数据存入空闲区,此校验码称为写校验码;FLASH存储器读取数据时同样会生成一个校验码称为读校验码,并根据ECC校验算法对写校验码和读校验码进行比对,并根据比对结果进行数据更正。

    例如FLASH存储器中某数据页中将存入2 048字节的数据,则根据ECC校验算法生成4字节的写校验码并存入数据页空闲区,当FLASH存储器对此数据读取时再生成一个4字节的读校验码,并利用ECC校验算法规则将写校验码和读校验码进行异或运算。若输出结果0,则表示此数据页没有发生位交换现象;若输出结果的矩阵中有14个1,则此数据页中发生位交换现象,并对错误数据位进行纠正,提高存储器中的数据质量;若输出结果的矩阵中有1个1,则表示空闲区的写校验码发生错误;若出现其他结果,则表示发生无法更正的位交换现象,数据可靠性降低[17]。ECC校验算法流程如图2所示。

    4 算法仿真和结果分析

    为对本文研究方法进行验证,通过基于Zynq的嵌入式存储器算法验证平台来实现本方案,平台通过FPGA硬件平台进行搭建,通过信号连接线将算法验证平台和FLASH存储器连接起来实现对存储器的控制,通过USB端口将算法验证平台连接至计算机实现数据结果分析处理,并通过在验证平台中开发设计CVI测试软件实现对存储器的读写测试,实现了对损耗均衡算法和ECC校验算法的验证。

    5 结 论

    根据对FLASH存储器中影响数据质量控制的因素进行分析,得出存储器中数据坏块产生的数量和速度,以及存储器中发生位交换现象是影响存储器中数据质量的主要因素;针对主要因素,采用损耗均衡算法对数据块被擦除次数进行控制,降低因数据坏块的产生导致数据丢失的可能性;采用ECC校验算法对数据进行检测与更正,减少因位交换现象导致的数据错乱。通过上述两种方法可以使存储器中数据质量水平提升,是嵌入式高速存储器数据质量控制的重要方法。
