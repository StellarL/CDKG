    摘    要：

    由于传统故障检测技术缺少状态评估机制,造成检测误差较大,提出了基于非线性状态估计的长时延丢包网络故障检测技术可降低检测误差.根据检测原理,设计非线性网络故障状态估计流程,以此构建长时延故障状态评估机制,根据评估结果分析长时延信号特征;设定合理分析规则,构建丢包故障状态评估机制,结合服从伯努利分布信息,分析丢包信号特征.根据信号特征分析结果完成对故障的检测.由实验结果可知,该技术检测误差最低达到0.1,能够满足实际误差指标约束条件.

    关键词：

    非线性状态; 估计; 长时延; 丢包; 网络故障; 检测;

    Fault detection algorithm for long delay packet loss network based on nonlinear state estimation

    WANG Yu

    Information Construction and Network Security Office,Northeastern University

    Abstract：

    Due to the lack of state evaluation mechanism in traditional fault detection technology and the large detection error, a fault detection technology based on nonlinear state estimation and long delay packet loss network is proposed, which can reduce detection error. According to the detection principle, the design of nonlinear network fault state estimation process, so as to construct a long delay fault condition assessment mechanism, according to the analysis of long time delay signal feature evaluation results; set reasonable analysis rules, construct the packet loss fault state evaluation mechanism, combined with obeys Bernoulli distribution information, analysis of packet loss signal characteristics. The fault detection is completed according to the result of signal characteristic analysis. The experimental results show that the detection error of the technology is up to 0.1, and it can satisfy the constraints of the actual error index.

    Keyword：

    nonlinear state; estimation; long delay; packet loss; network fault; detection;

    1 引 言

    针对复杂网络状态进行故障检测,需了解每一个网络节点信息,但是对于非线性的网络结构,所有节点信息并不全面,因此需要利用网络中已知的信息对未知节点的信息进行状态估计.通常情况下,将复杂网络分为离散型和连续型,连续型的网络是在理想状态下实现的,而离散型的网络是在现实生活中存在的.由于网络在传输过程中产生长时延和丢包现象,导致自身传输受到影响,为此对于网络长时延和丢包故障检测是具有现实意义的[1].目前有关网络状态估计信息都只满足线性网络结构,但是对于离散型的网络结构来说,必须考虑与实际情况相关的非线性指标要求.采用传统检测技术存在检测精准度低的问题,无法满足实际误差指标要求,为此,提出了基于非线性状态估计的长时延丢包网络故障检测技术.

    2 长时延丢包的非线性网络状态估计故障检测

    2.1 检测原理

    面对网络故障发生的数据属性变换,可利用故障检测技术对数据进行实时的检测[2].针对多种网络参数的辨识和校正,需依据正常状态下,网络设备运行时的参数进行误差分析,以此作为网络故障是否发生的依据,提前采取保护出错,避免因网络故障造成的不必要麻烦,具体检测原理如下所示:

    设正常状态下的网络设备运行参数集,提取网络状态向量函数;设置网络数据集,并在已知状态向量函数上进行可逆操作;分析状态检测网络故障传输信号的特性,利用该特性研究网络故障发生后的状态情况;探求网络状态与检测机构协调性,在信号特性函数中输入具体函数值,使用具有非线性的网络来描述.

    根据该原理,分别对网络长时延和网络丢包问题进行分析,实现网络故障的检测,进而保障网络能够可靠运行.

    2.2 长时延网络故障信号特性统计分析

    在信号传输过程中,由于受到网络带宽局限性和随机数据流影响,造成信号传输出现长时延,即为网络诱导时延[3].该时延的出现会造成网络传输性能较差,严重影响了信号传输的可靠性,为此针对具有长时延问题的网络故障进行分析.

    充分考虑系统运行状态,将过程输出和故障信号全部作为故障检测的依据,分析出现长时延的网络结构如下所示:

    {q(k+1)=Aq(k)+Bh(k)+Cf(k)p(k)=Dq(k)+Eh(k)         (1)

    式中,q(k)为正常网络运行状态;p(k)为运行结果输出;h(k)为扰动输出;f(k)为网络故障信号;A、B、C、D、E为网络维数常数矩阵.

    网络在传输过程中会出现长时延问题,为此需采用非线性状态估计方法检测传感器到达故障检测滤波器中信号传输情况.设网络时延为α(k),那么输出信号可表示为:

    p′(k)=Dq(k-α(k))+Eh(k-α(k))+Ff(k)         (2)

    式中,p′(k)为有效时延过程输出;0<α¯<α(k)<α¯,其中α¯为有效时延的下界,α¯为有效有效时延的上界;F为网络维数常数矩阵.对于在有长时延下的非线性网络故障状态估计流程设计如图1所示:

    图1 有长时延的非线性网络故障估计流程   下载原图

    根据图1中出现的网络时延构造如公式(3)的非线性网络故障状态评估机制:

    {q′(k+1)=Aq′(k)+Bg(k)+L(p′¯(k)-p′(k))p′(k)=Dq′(k)r(k)=Μ(p¯(k)-p′(k))         (3)

    式中,q′(k)为网络运行状态估计情况;p′(k)为出现时延网络输出;g(k)为评估输出;r(k)为出现时延的残差信号;L、M分别为网络故障检测的增益矩阵和出现残差信号的增益矩阵[4].

    充分考虑非线性网络故障估计过程中存在的通路时延情况,满足网络分布序列,统计的信号特性为:

    p(β(k)=0)=R{β(k)}=β¯p(β(k)=1)=1-R{β(k)}=1-β¯         (4)

    网络中的检测信号与运行信号随机时延分别通过式(2)中的α(k)和式(4)中的β(k)来表示,其中α(k)=1表示网络运行所接收到的检测信号,β(k)=1表示网络运行所接收到的运行信号,此时p¯(k)=p(k-1),g(k)=g´(k-1),说明该网络存在较长时延;α(k)=0和β(k)=0分别表示网络没有接收检测信号和运行信号,此时p¯(k)=p(k),g(k)=g´(k),说明该网络无时延.

    对长时延问题描述,采用非线性状态估计方法检测传感器到达故障检测滤波器中信号传输情况.根据该情况设计非线性网络故障状态估计流程,设置网络故障状态评估机制,分析通路时延情况,并统计信号特性,以此检测网络长时延故障情况[5].

    2.3 丢包网络故障检测研究

    在数据传输过程中,由于受到网络拥塞,连接中断等影响,导致数据包传输丢失现象出现[6].该丢包的出现会造成网络传输性能变差,严重影响了数据包传输的可靠性,为此针对出现数据包丢失的网络故障进行分析[7].

    在网络丢包环境中,充分考虑系统运行状态,采用非线性状态评估方法可描述丢包的网络结构,如下所示:

    {x(k′+1)=A′x(k′)+B′u(k′)+C′f(k′)y(k′)=D′x(k′)         (5)

    式中,x(k′)为正常网络运行状态;y(k′)为运行结果输出;u(k′)为输入信号;f(k′)为网络故障信号;A′、B′、C′、D′为网络维数常数矩阵[8].

    网络出现随机丢包现象时,输出结果为:

    y(k′)=α′(k′)E′x(k′)         (6)

    式中,y(k′)为网络检测输出结果;E′为网络维数常数矩阵.

    非线性网络故障状态评估模型如公式(7):

    {x(k′+1)=A′x′(k′)+B′g¯(k′)+L′(y′¯(k′)     -y′(k′))y′(k′)=D′g′(k′)         (7)

    式中,x′(k′)为非线性网络运行状态估计向量;g¯(k′)为出现丢包的网络输入;g′(k′)为无丢包现象的网络输入; L′、M′分别为网络故障检测的观测器增益矩阵和控制器增益矩阵[9].

    引入变量τ(k′),使故障状态评估模型服从伯努利分布信息,得到统计信号特性为:

    Ρrob(τ(k′)=1)=R′{τ(k′)}:=τΡrob(τ(k′)=0)=1-R′{τ(k′)}:=1-τVar(τ(k′))=R′(τ(k′)-τ)2=(1-τ)τ=τ2         (8)

    式(6)中的α′(k′)和公式(8)中的τ(k′)分别表示传感器向控制器传送和控制器向执行器传送出现的丢包现象.当α′(k′)=1表示传感器向控制器成功传输数据包,α′(k′)=0表示传感器向控制器传输数据包时出现丢包现象;当τ(k′)=1表示控制器向执行器成功传输数据包,τ(k′)=0表示控制器向执行器传输数据包时出现丢包现象.

    根据上述描述,需遵循如下检测原则:如果误差绝对值小于输出结果阈值时,那么网络为正常运行状态;如果误差绝对值大于等于输出结果阈值时,那么网络出现长时延丢包现象.根据该原则能够设计故障补偿机制,完成网络系统的容错控制,对于故障检测过程中的容错现象进行有效补偿.

    3 验证分析

    为了证明提出的非线性状态估计方法在网络故障检测时的有效性,以网络故障检测的误差作为实验标准,将传统技术与非线性状态估计技术对网络故障检测的误差进行实验对比.选择某单位网络故障检测中的数据变量,取自2017年10月1日12:00至10月2日17:00时间段的100组数据作为实验样本,依据以往历史记录获取正常状态下的数据,设置采样频率为每分钟采样10次,将数据集进行标准化处理.

    3.1 长时延故障实验结果与分析

    如果信号在发送与接收过程中出现延迟,那么整个网络都将受到延迟的支配.通常情况下,出现网络长时延的主要原因是主机与服务器之间的路由跳数过多,导致传输速度过快,由路由器转发的数据包处理出现延迟比较明显;网络宽带数量不能满足实际需求,容易造成大量的数据丢失,从而出现响应时延.

    针对网络有无长时延故障情况,分析残差信号走势,如图2所示.

    图2 有无长时延故障残差信号   下载原图

    由图2可知:当步长的取值范围为(150～250)时,无故障情况的残差信号的走向都呈现上下波动状态,而有故障情况的残差信号在大于1.0的位置呈上下波动状态;由此可看出,当故障发生时,残差信号有明显变化,当故障发生在140个步长之后,故障信号被检测出来.

    为了验证采用非线性状态估计技术对网络长时延检测的可靠性,将传统技术与该技术对网络故障检测的误差进行对比,结果如图3所示.

    图3 两种技术网络长时延故障检测误差对比结果   下载原图

    由图3可知:当步长范围为(0～130)时,采用传统技术检测的数值,当步长范围为(130～150)时,采用传统技术检测存在的误差最大值为0.7,超过了控制限值,最小误差为0.25,在允许误差范围内.采用非线性状态估计技术检测存在的误差最大值为0.75,超过了控制限值,最小误差为0.1,未超过控制限值;当步长范围为(150～220)时,采用传统技术对网络故障进行检测时,大部分数据超过了控制限值,而采用非线性状态估计技术进行检测时全部超出了控制限值;当步长范围为(220～400)时,采用传统技术检测的误差达到了最大值0.75,而采用非线性状态估计技术检测误差均未超过控制限值.由此可知,采用非线性状态估计技术对网络出现长时延问题故障进行检测时,误差较小.

    3.2 丢包故障实验结果与分析

    数据是以数据包为单位在网络中进行传输的,不同数据包都具有不同的数据帧,因此无论网络环境多好,数据都不是以线性状态传输的,在传输过程中,总会出现空洞现象.由于线路故障、病毒攻击等因素,导致网络传输出现数据包丢失现象,如果线路情况较差,那么数据包丢失量将会大大增大.

    针对网络有无丢包故障情况,分析残差信号走势,如图4所示.

    图4 有无丢包故障残差信号   下载原图

    由图4可知:当步长的取值范围为(0～11)时,有无故障情况的残差信号的走向都呈现上下波动状态,且波动范围为(-0.4～0.4);当步长的取值范围为(11～48)时,无故障情况的残差信号走向也呈现上下波动状态,但幅度较大,最大值为0.5.有故障情况的残差信号所呈现的上下波动程度较大,最大值达到2.4,最小值达到-1.9;当步长的取值范围为(48～80)时,有无故障情况的残差信号的走向都呈现上下波动状态,且波动范围为(-1.0～0.5).由此可看出,当故障发生时,残差信号有明显变化,当故障发生在11个步长之后,故障信号被检测出来.

    为了验证采用非线性状态估计技术对网络丢包的可靠性,将传统技术与该技术对网络故障检测的误差进行对比,结果如图5所示.

    图5 两种技术网络丢包故障检测误差对比结果   下载原图

    由图5可知:当步长范围为(0～18)时,采用传统技术检测的数值误差均超过控制限值,而采用非线性状态估计技术检测误差均未超过控制限值;当步长范围为(18～21)、(49～51)时,采用非线性状态估计技术检测误差超过限值;当步长范围为(50～60)时,采用传统技术检测的误差达到了最大值0.9.当步长为40时,采用非线性状态估计技术检测误差达到了最小值0.2.由此可知,采用非线性状态估计技术对网络出现丢包问题故障进行检测时,误差较小.

    3.3 实验结论

    根据上述实验内容可得出实验结论:采用非线性状态估计方法对网络故障进行检测是有效的.

    针对上述内容进行总结:提出的非线性状态估计方法对网络故障进行检测时,误差较小,能够满足既定的误差约束条件.

    4 结束语

    虽然针对网络故障的检测已经取得了较多研究成果,但是大多数只针对具有线性的网络结构,网络中存在的长时延、丢包问题,大大限制了网络故障检测的精准度.为了提高网络使用的安全性,研究网络在传输过程中出现的长时延和丢包情况下的非线性状态估计问题是具有必要性的.对于网络研究所涉及的学科较多,但是有关网络控制的研究却相对较少,从历史资料中提取出的相关知识具有不确定性,导致故障检测难度增加,所以需要深入研究.
