    网络安全数据可视化融合的分析方法

    张杰 傅文博

    山西大同大学计算机与网络工程学院

    摘    要：

    文章提出一种基于时间序列的网络安全数据可视化融合分析方法, 利用网络数据之间的关联性进行特征选择和一致性过滤, 保留原始特征的同时消除冗余特征;通过网络数据特征之间的信息熵计算, 利用INTERACT方法实现归一化处理;通过时间序列分析法对处理后的网络数据进行可视化融合分析, 利用切比雪夫法对网络数据包进行验证, 判断是否存在入侵行为, 实现网络安全数据的可视化融合.实验结果表明, 通过对检测阈值的调控能够提高检测率, 随着数据规模的不断扩大, 所提方法的检测率较为平稳, 且误报率相比其他方法更低.

    关键词：

    网络; 安全数据; 可视化融合; 分析;

    Analysis method of visual fusion of network security data

    ZHANG Jie FU Wen-bo

    School of Computer and Network Engineering, Shanxi Datong University

    Abstract：

    In this paper a visual fusion analysis method for network security data based on time series is proposed. The correlation between network data is used for feature selection and consistency filtering. The original features are preserved while the redundant features are eliminated. The information entropy between network data features is proposed. Calculation, using the INTERACT method to achieve normalization processing; through the time series analysis method for visual fusion analysis of the processed network data, using the Chebyshev method to verify the network packets to determine whether there is intrusion behavior, to achieve network security data Visual fusion. The experimental results show that the detection rate can be improved by regulating the detection threshold. With the continuous expansion of data size, the detection rate of the proposed method is relatively stable, and the false alarm rate is lower than other methods.

    Keyword：

    The internet; safety data; visual fusion; analysis;

    1 引言

    网络应用的迅速发展对网络安全提出了更高的要求[1].随着人们对网络安全需求的不断增加, 在分析网络安全数据时网络安全分析人员遇到了许多新的难题.文献[2]提出一种基于网络异常检测的分析方法, 利用转移学习方法对熟知的网络进行实验检验, 来验证网络中是否存在异常入侵行为进行数据可视化的分析.但是这些检测方法有两种缺点, 一是检测率低, 二是误报率高.

    根据上述方法所存在的缺点, 提出了基于时间序列的网络安全数据可视化融合分析方法.

    2 网络安全数据可视化的研究和分析方法

    2.1 网络安全数据的预处理

    由于筛选方法挑选的特征只取决于统计数据培训结果, 比较而言成本预估低的算法让它对于类似KDDCup99的整个数据包有较高效率[3,4].

    (1) 基于启发式评估函数的网络数据特征提取

    利用关联的启发式评估函数对网络数据进行特征选择, 提取出与分类紧密相关且网络数据之间不具有强关联性的单个特征组成最优特征子集, 消除网络数据的冗余特征, 实现网络数据的有效特征提取.具体描述公式如下:

    Μs=krcfk+k (k-1) rff          (1)

    式中, rcf和rff分别表示网络数据特征与所属类别和网络数据特征与联系强弱的平均值;Ms代表一个数据包含k个网络特征子集的引导式的重要性;krcf代表网络数据特征与类别接触反映出来的强弱值;k+k (k-1) rff代表在特征处理中有多少多余的网络数据.

    (2) 原始网络数据的信息熵计算

    原始网络数据是基于SU方法的数据信息相对不确定性.此性质是讲述x、y中原始信息的利润IG和熵H比值的度量.计算公式得出:

    SU (x, y) =2ΙG (xy) Η (x) +Η (y) -krcf          (2)

    ΙG (xy) =Η (y) +Η (X) -Η (x, y)           (3)

    除了SU, INTERACT还包含原始属性一致性.网络特征属性的一致性能反映出此项网络特征对一致性的影响达到的标准[5].此方法通常在查找数据特征中使用, 且彼此之间互相影响.由两部分组成:一是上述网络特征都以不确定性的降序排列, 二是从后面启动网络数据特征序列, 分别评估网络数据特征, 假设阈值高于网络数据特征的一致性, 则阈值就不会被使用, 相反阈值被使用, 得到去除冗余数据的特征, 公式如下:

    X'=IG (xy) -Μs          (4)

    2.2 基于时间序列的网络安全数据可视化融合分析

    网络数据的时间序列是监管检测过程中按照时间先后顺序得到的.用x (i) 表示监管检测数据, 则x (1) , ⋯, x (i) , ⋯表示每个时间段内K次检测的时间序列定为一个周期, 划分出每个周期运行的时间段, x (i, j) 代表检测时第j个时段内入侵行为发生的概率.Μ (j) 表示在第j个时段内这个时间发生的平均次数, 则发生平均次数的表达式为:

    Μ (j) =1Κ∑i=1kX (i, j) +y          (5)

    K+1次检测之后, 用Μ (j) 表示第j个时间段内事件 (数据包的接收率或数据包到达的时间间隔) 的预期值.用x (Κ+1, j) 代表第j个时刻真实所发生事件的次数, 则得到的表达式如下:

    z (j) =|x (Κ+1, j-Μ (j) ) |Μ (j)           (6)

    根据时间序列的网络安全数据建立时间模型, 设置一个数据的阈值用δ表示, 当阈值偏差z (j) 大于且等于δ1时, 即可得知入侵事件大概会在什么时间段发生入侵.

    通过多项式对时间序列划分出可用的阈值, 对入侵事件分析预测.在时间序列网络安全数据Y中取值, D代表领域布置, A代表节点收集, m代表网络数据长度d代表数据的维度, W代表数据窗口, ω代表流动窗口 (ω<<m) [6].检测时间序列的网络数据可视化过程表达式为:

    c0=1ω∑j=1mf (tj) p0 (tj) =1ω∑ωf (tj) Ζj          (7) ci=2ω∑j=1ωf (tj) pi (tj)           (8)

    类似序列:S、R的时间序列数据分别为C、D.假设 (C、D) <ε, 则称时间序列S和R相似.对时间序列S来说, 假如历史数据的窗口中, 与它类似的时间序列similar (S, R) 表示数量小于一定的比例, 那么时间序列S称为离群时间序列, 即:

    |{R|similar (S, R) , S, R∈W|}+∑j=1mf (tj) ||{Η|Η∈W}|          (9)

    根据网络数据特征的预处理结果对网络数据进行熵计算, 时间序列内数据预处理到达的时间和网络数据接收成功的概率称为其特征, 从而为网络入侵的判断打好基础.网络数据的时间序列方法可被网络入侵检测过程使用[7,8].则其方法如下:

    对某个时间段的限流开始检测;z (j) 设为j时间的差值;t设为无异常时时间序列的个数, δ1设为阈值的时间序列检测, S设为时间序列在j时间内的数据运行结果, δ2设为阈值与时间序列的近似值.

    在规定范围内检测到网络数据集合, 警惕对网络的入侵.选择各种时刻所对应的时间序列, 设δ1为阈值, 设z (j) 为时间差值, 根据以下公式进行网络安全数据可视化融合分析:

    z (j) <δ1 (10)

    z (j) ≥δ1 (11)

    S≥δ1 (12)

    S≥δ2 (13)

    如果满足上述公式 (10) , 则该测量时间序列在规定范围内发送给正常的行为库, 从而对正常数据进行放行;如果满足上式 (11) , 则说明j时间内可能存在入侵, 时间序列内的网络数据可能存在偏差, 通过分析来判断j时间内的时间序列S和阈值δ2.式 (12) 成功验证, 开始分析相对偏差, 式 (13) 成功验证, 则立刻将时间序列S设为离散时间序列, 从而实现数据的入侵行为检测, 报警系统启动.

    3 实验及结果分析

    3.1 实验环境与设计

    在Corei3 PC机 (2G内存) 的WindowsXP环境下, 利用网络测试基地, 对主要内容进行分析.收集一组网络数据.依次选择不同的网络数据进行实验, 数据规模可以设置为10组、50组、80组、120组、200组.

    把误报率和检测率作为主要衡量标准, 开始验证误报率和检测率的变化趋势并对检测方法进行评价, 检测率代表样本数与样本总数之比, 误报率代表被误判的正常样本数与正常样本总数之比.

    3.2 检测阈值对不同时间序列分析方法功能的影响

    针对检测阈对不同时间序列分析的基本特征, 设定几个数据大小不同的参数, 得出δ1∈[1.2, 5.2]的选值开始实验, 对参数δ1的取值分析并探讨, 分析δ1是否会影响时间序列分析方法, 检验结果可以看出, 随着δ1取值的不停增大, 时间序列分析方法的检测率适当降低, 当δ1>4时, 检测率快速下降, 而当δ1<3时, 它的误报率偏高.

    为了更详细地比较参数δ1对时间序列分析方法性能的影响, 我们主要对上述五种不同模式的数据集进行实验.在实验中得知, 取值的大小影响样本集的变化, 同时检测率、误报率也产生了应有的变化.实验结果表明, 数据集用S1S2S3S4及S5表示, 本次实验主要选值δ1分别取2、2.5、3、3.5这四个值开始验证.如表1、表2所示.

    3.3 数据集的规模对方法性能的影响

    根据上面实验的参数δ1对时间序列分析方法性能的影响, 取阈值为3当做参考, 比较文献[2]和时间序列分析两种方法对5个不同规模的实验样本进行多次实验.

    图1表示误报率的对比.横坐标分别用S1—S5来表示, 纵坐标对照的是检测率和误报率, 黑色矩形代表文献[2]方法, 白色则代表时间序列分析方法.

    图1 两种方法误报率的对比   下载原图

    随着数据集规模的不断增大, 时间序列分析方法的检测率逐渐升高, 误报率相对减少, 由于样本数的增加, 正常与异常数据点具有显著的区分.这时, 不正常区域范围里的数据点比正常的数据点的数量少了很多, 由此说明文献[2]方法可以很轻松辨识大多数的异常值.但时间序列分析方法的检测率一直都比文献[2]方法的检测率高, 还有误报率比文献[2]方法低.是由于时间序列分析方法在文献[2]方法的基础上建立的, 通过阈值的调整, 让这个方法能更精准的判断, 可以辨识到文献[4]方法不能辨识的异常值.

    4 结束语

    提出了基于时间序列分析的网络安全数据可视化融合方法, 简单说明了分析人员对网络数据的应对, 并主要从选择特征和网络安全数据可视化这两个方面进行验证, 利用时间序列分析方法来判断网络情况, 对检测率和误报率进行了实验, 实验中能够根据阈值的变化完成调整, 提升了时间序列分析方法的检测率, 并使误报率降低.
