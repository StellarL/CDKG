    摘    要：

    在通信过程中, 由于信道的干扰, 会使信息出错, 通常会使用纠错码来提高传输数据的正确率.在纠错码中级联码以优秀的性能被广泛应用.而LDPC码作为接近香农限的特别好码受到越来越多的应用, 所以对LDPC码编解码算法的研究成为了热点.本文以DVB-S2协议的前向纠错系统中LDPC码为研究背景, 对LDPC码译码算法最小和算法 (Minimum Sum Algorithm, MSA) 、偏移最小和算法 (Offset Minimum Sum Algorithm, OMSA) 及归一化最小和算法 (Normalized Minimum Sum Slgorithm, NMSA) 进行了介绍.本文提出了一种最小和算法的实现方法, 并且用于归一化最小和算法和偏移最小和算法的实现, 之后通过对迭代次数和误码率的对比, 选择出偏移量为0.1, 迭代次数为20次的的偏移最小和算法作为最优算法来实现LDPC码译码器设计.

    关键词：

    DVB-S2; LDPC; 最小和算法; 偏移最小和算法;

    Abstract：

    In the communication process, due to the interference of the channel, the information be erroneous.Error correction code is usually used to improve the accuracy of the transmitted data. Concatenated codes as a kind of error correction codes are widely used due to their excellent performance, and LDPC codes are increasingly used as a good code that is close to the Shannon limit. Therefore, the research on LDPC codec algorithms has become a hot topic This dissertation takes the LDPC code in forward error correction system of DVB-S2 protocol as the research background, and introduces the minimum sum algorithm, offset minimum sum algorithm and normalized minimum sum algorithm of LDPC code decoding. In this paper, a minimum sum algorithm implementation method is proposed, and it is used to the implementation of normalized minimum sum algorithm and the offset minimum sum algorithm. After comparing the number of iterations and the error rate, the offset parameter is set to 0.1 and the number of iterations is set to 20, and the algorithm is used as the optimal algorithm to implement the LDPC decoder design.

    1 引言

    由于信息经过信道之后会受到噪声影响而出现错误, 所以在现代数字通信系统中前向纠错码受到了广泛应用, 而优秀的级联码相对与一种纠错码可以表现出更加优秀的性能, 可以进一步降低误码率, 级联码性能的理想的情况是逼近香农极限[1].在第二代地面数字电视广播标准 (Digital Video Broadcasting Satellite-2nd Generation, DVB-S2) 中也采用了级联码纠错技术.在DVB-S2中应用了外码BCH (Bose Chaudhurl Hocquenghem) 和内码低密度奇偶校验 (Low Density Parity Check, LDPC ) 码相级联的结构[2,3].

    在LDPC码译码算法中, Tanner提出了和积算法 (Sum-Product-Algorithm, SPA) 和最小和算法 (Min-Sum-Algorithm, MSA) 两种信息传递算法.在之后人们对于LDPC码的译码算法进行了更加深入的研究[4].近些年, 国内外学者对研究LDPC码译码算法和设计译码器进行了广泛的关注.比如Dale E.Hocevar于2004年提出了基于行的分层译码算法, 该算法所需的迭代次数很少, 相比并行译码算法减小了一倍[5].但是应用到DVB-S2协议中的LDPC的译码算法普遍都是最小和算法, 比如国防科技大学的张波涛等人于2009年, 在IEEE上提出了一种DVB-S2标准的LDPC码译码器, 该译码器是基于最小和算法和RMP (Row Message Passing) 的译码顺序的, 当其码率为9/10, 时钟为320MHz时, 译码速率可达998Mbit/s[6].本文对DVB-S2协议中的LDPC码译码算法进行了研究, 提出了一种关于最小和算法与其改进算法:归一化最小和算法 (Normalized Minimum Sum Algorithm) 与偏移最小和算法 (Offset Minimum Sum Algorithm) 的实现方式, 并对比了它们的性能.发现当偏移最小和算法偏移量为0.1时算法最优.

    2 最小和算法与其改进算法

    LDPC码的校验矩阵是稀疏矩阵, 因此有很多高效的译码算法.其中最常用的译码算法是BP算法及其改进衍生出的算法, BP算法最常见的有三种:基于概率的BP算法、对数域BP算法及最小和算法.概率域的译码算法其运算量非常大, 易造成算法具有不稳定性.对数域BP算法有tanh (x) 和tanh-1 (x) 两个函数, 计算复杂度较高.

    L (U) =logΡU (u=0) ΡU (u=1)     (1)

    L (xn|yn) =log (Ρ (xn=0|yn) ) (Ρ (xn=1|yn) )     (2)

    归一化最小和算法和偏移最小和算法只是在最小和算法的基础上进行了修改, 修改的是校验节点更新的步骤.

    MSA其实估计的幅值要大于实际值, 所以可以适当的对最小和算法进行改进来逼近对数域BP算法的性能.而本节所讲述的算法就是以MSA估计的幅值乘以一个0到1之间的因子, 从而更加逼近真实值.在本节讲述的算法中对于校验节点的计算如公式 (10) 所示, 其他迭代步骤与最小和算法相同[8,9].

    3 算法实现过程

    最小和算法解码程序总体框图即main函数流程图如图1所示.可以看到译码的第一步是进行编码, 利用随机函数产生信息位序列, 然后按照上一小节中所提到的DVB-S2协议中给出的LDPC码的编码方法对信息序列进行编码.编码之后对码字进行添加噪声, 然后将添加噪声后的似然比信息作为迭代中传输的数据进行解码, 解码之后输出一些参考信息.

    图1 最小和算法解码程序总体框图   下载原图

    在主函数执行过程中调用了一些子函数, 下面将会一一介绍其功能.在程序中包括有get_P, check, ms2_decode, top_dec_include_noise, generate_codeword和initms2、iterms2几个函数.

    get_P函数:该函数有两个功能.一是用来编码时产生校验位然后作为码字进行添加噪声解码等.二是在每一次迭代之后用硬判决之后的信息位进行编码与添加噪声之前的校验位对比.在generate_codeword和check调用.因为在编码中已经介绍该函数的流程图, 在此不再介绍.

    check函数:每一次迭代之后用硬判决之后的信息位进行编码与添加噪声之前的校验位对比, 如果相同则译码成功, 如果不相同则继续迭代.可以看到该函数的传入值有code_word[N], 所以只需要将新的硬判决的值进行编码与code_word[N]对比即可.函数比较简单所以在此不给出流程图.

    initms2函数:用来初始化迭代参数并进行第一次硬判决, 将N个似然比信息依次初始化给H矩阵的N列, 每列1的位置初始化相同的值, 0的位置不需要初始化.

    iterms2函数:迭代主体函数, 完成译码算法主要工作.其流程图如图2所示.可以看到第一步是定义迭代中需要的存储器即数组来存储迭代过程中的中间变量.然后进行校验节点更新, 校验节点更新的操作是找到每个校验节点所连接的变量节点传输过来的最小值、次小值、最小值的位置以及所有该校验节点相邻节点传输过来的负值个数.下一步是更新校验节点传给变量节点的值, 更新规则是之前传输最小和的线更新成次小值其它更新为最小值, 符号信息由本身的符号与之前计算的所有负值的符号共同决定.接下来是更新变量节点, 实际上是更新传输到校验节点的值, 其传输值为:所有与该变量节点连接的校验节点传输过来的值+初始似然比-该校验节点上一步传输给该变量节点的值.最后是进行硬判决, 所有与该变量节点连接的校验节点传输过来的值+初始似然比为该变量节点的硬判决参考值, 硬判决需要判断这个值为0还是为1.至此结束一次迭代.

    图2 解码主体函数iterms2流程图   下载原图

    ms2_decode函数:顶层的解码函数, 该函数流程图如图3所示:先调用initms2函数进行迭代初始化, 然后调用check函数检测译码是否成功.然后判断是否达到最大迭代次数或者是解码成功, 如果是则结束译码, 如果否则调用iterms2函数继续迭代.

    图3 ms2_decode顶层的解码函数流程图   下载原图

    generate_codeword函数用随机数产生信息位序列再通过调用get_P函数来获得校验位, 与之前的随机数产生的信息位一起组成码字序列.

    top_dec_include_noise函数用来添添加噪声声.添加噪声的过程在信道建模已经介绍, 在此不再介绍.

    归一化最小和算法与偏移最小和的实现过程是类似的, 只是在校验节点更新处有稍微不同.因为最小和算法对于校验节点更新处的概率进行了过估计.所以需要减小最小值来逼近标准值.归一化最小和算法是在最小和算法中的最小值基础上, 乘以一个因子.偏移最小和算法是在最小值基础上减去一个值.两者都对最小和算法进行了修正, 两个因子可以通过仿真来决定取值, 从而找到更加逼近标准值的解.

    本文是基于DVB-S2协议中LDPC码的译码器实现上, 提出了一种最小和算法的实现方法, 基于该方法实现了归一化最小和算法与偏移最小和算法.并对三种算法的实现结果进行了对比, 发现在迭代次数为20次的基础上当偏移化最小和偏移系数为0.1时算法最优, 优于最小和算法与归一化最小和算法.
