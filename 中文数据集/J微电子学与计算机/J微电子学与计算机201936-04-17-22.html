    摘    要：

    FinFET作为现在最前沿的晶体管技术, 在嵌入式存储器上得到了广泛应用.但是, 新的工艺也带来了新的缺陷.Synopsys公司通过故障建模及测试发现:FinFET存储器比平面存储器对动态故障更敏感.而经典的测试算法仅仅针对静态故障.目前关于动态故障的测试算法极少, 并且复杂度很高.因此, 本文提出了一种改进的动态故障测试算法, 该算法能够覆盖所有连续两次敏化操作的单单元动态故障.

    An Efficient Dynamic Fault Test Algorithm for FinFET Single Memory Cell

    SANG Sheng-Nan ZHANG Li-Jun ZHENG Jian-Bin PENG Zeng-Fa

    Soochow University Megacores Technology Co.LTD

    Abstract：

    FinFET as the most advanced transistor technology, has been widely used in embedded memory. However, the new process also brings new defects. Synopsys found through fault modeling and testing that FinFET memory is more sensitive to dynamic faults than planar memory. The classic test algorithm is only for static faults. At present, there are few test algorithms for dynamic faults, and the complexity is very high. Therefore, an improved dynamic fault test algorithm is proposed in this paper. This algorithm can cover all single-cell dynamic faults in two successive sensitization operations.

    Keyword：

    SRAM; march algorithm; dynamic fault; fault primitive; MBIST;

    1 引言

    随着半导体技术的不断发展, FinFET 通过多栅结构有效的抑制沟道效应和制作工艺的良好兼容性, 在嵌入式存储器上得到了广泛应用.但是, FinFET存储器也带来了新的设计复杂性、缺陷覆盖和良率挑战.Synopsys关于FinFET工艺故障建模的部分普遍结论是: (1) FinFET存储器比平面存储器对动态故障更敏感; (2) FinFET存储器对制程变异故障更稳定; (3) 静态单单元和耦合故障在两种存储器中均很常见[1].

    常用的测试算法如MATS、March C、March C+、March C-等都仅适用于静态故障测试.但对于动态故障的研究, 国内外都尚处于起步阶段.目前关于动态故障测试算法的研究成果有:Said Hamdioui提出了RAW1 (13N) 和RAW (26N) [2]两种算法, 这两种算法仅能覆盖部分单单元和双单元动态故障.Benso提出了一种复杂度为100N的March算法[3], 该算法主要针对连续两次敏化操作的单单元故障及耦合故障.Harutunyan将上述100N算法优化成复杂度为70N的March MD2算法, 且没有改变故障覆盖率, 同时, 还提出March MD1a (33N) 和March MD1b (33N) 两种用以检测所有的连续两次敏化操作的单单元故障的算法[4].

    本文在March MD1a (33N) 和March MD1b (33N) 两种算法的基础上进行该进, 提出一种复杂度为25N的测试算法且保证故障覆盖率不变, 并应用于16 nm存储器芯片的自动测试电路中.

    2 FinFET技术简介

    2.1 FinFET器件

    鳍式场效应晶体管 (Fin Field-Effect Transistor, FinFET) 是由美籍华人科学家胡正明教授提出的一种MOS结构, 其中的Fin在构造上类似鱼鳍, 所以称为“鳍式”.FinFET是三维结构, 也称三栅晶体管, 如图1所示.沟道是由金属栅极包裹的薄鳍片, 沿源漏方向鳍片的长度为沟道长度.这种结构增强了栅极对沟道的控制能力, 从而降低了漏电流, 抑制短沟道效应.在工艺制造时, 通过短沟道隔离 (Shallow Trench Isolation, STI) 决定鳍片的高度 (Hfin) [5].鳍片的宽度 (Wfin) 决定了一些电学特性, 晶体管开启电流与关断电流的比值及阈值电压及随着鳍片宽度的增加而减小, 这表明更短的鳍片宽度, 可以更好地控制沟道电流[6].

    2.2 FinFET特殊缺陷

    与平面工艺相比, FinFET表现出了一些特殊的缺陷, 如图2所示, FinFET主要有五种特殊缺陷[7]:

    (a) 鳍片开路:鳍片完全开路和电阻开路.

    (b) 栅极开路:栅极完全开路和电阻开路.

    (c) 鳍片粘连:源极和漏极完全短路和电阻短路.

    (d) 栅极-鳍片短路:栅极和鳍片完全短路和电阻短路.

    (e) 制程变异:FinFET参数值发生变化.

    3 故障原语及故障模型

    故障原语 (Fault Primitive, FP) [8]表示对存储器进行某一操作时, 存储器预期行为和所观察到的行为之间的差异, 记为FP= (S/F/R) .S表示敏化测试单元故障行为的状态或操作, F表示故障单元的状态或行为, R表示存储单元的输出值.

    3.1 静态故障

    静态故障表示最多对存储单元进行一次操作而导致的故障, 用故障原语表示即S∈{0, 1, 0w0, 0w1, 1w0, 1w1, 1r1, r0, r1}, 分别表示存储单元状态0、状态1、0写0、0写1、1写0、1写1、读0、读1.F∈{0, 1, ↑, ↓}, ↑ (↓) 表示上升 (下降) 传输.R∈{0, 1, -}, 当S不是读操作时, R为-.耦合故障的故障原语表示为FP= (Sa:Sv/F/R) , Sa (Sv) 表示敏化施主 (受主) 单元的状态或操作, 此处应注意到, 若Sa为一个状态, 则Sv可能是一个状态或操作;若Sa是一个操作, 那么Sv只能是一个状态.

    静态故障可分为单单元故障及耦合故障, 单单元故障包括:地址译码故障 (Address-Decoder Fault , AF/ADF) 、固定故障 (Stuck-At Fault, SAF) 、转换故障 (Transition Fault, TF) 、写干扰故障 (Write Disturb Fault, WDF) 、读破坏故障 (Read Destructive Fault, RDF) 、欺骗性读破坏故障 (Deceptive Read Destructive Fault, DRDF) 、错误读故障 (Incorrect Read Fault, IRF) [9], 其中地址译码故障和固定故障由于表现形式一致, 可以统一为状态故障 (State Fault, SF) .考虑到算法复杂度的问题, 当前的算法主要研究相邻单元耦合故障, 其故障类型主要包括:反向耦合故障 (Inversion Coupling Fault, CFin) 、幂等耦合故障 (Idempotent Coupling Fault, CFid) 、状态耦合故障 (State Coupling Fault, CFs) .以上故障的功能故障模型 (functional fault model, FFM) 及故障原语如表1所示.

    3.2 动态故障

    动态故障是指对单个存储单元至少进行一次操作而导致的故障, 同静态故障一样, 动态故障可分为单单元动态故障及相邻单元动态耦合故障, 本文仅对单单元动态故障进行研究.与静态故障的故障原语不同的是, 在单单元动态故障的故障原语中, S∈{0r0r0, 0r0w0, 0r0w1, 1r1r1, 1r1w0, 1r1w1, 0w0w0, 0w1w1, 1w0w0, 1w1w1, 0w0w1, 0w1w0, 1w0w1, 1w1w0, 0w0r0, 0w1r1, 1w0r0, 1w1r1 }.单单元动态故障主要包括:动态读破坏故障 (Dynamic Read Destructive Fault, dRDF) 、动态欺骗性读破坏故障 (Dynamic Deceptive Read Destructive Fault, dDRDF) 、动态错误读故障 (Dynamic Incorrect Read Fault, dIRF) 、动态转换故障 (Dynamic Transition Fault, dTF) 、动态写破坏故障 ( Dynamic Write Destructive Fault, dWDF) .其功能故障模型及故障原语如表2所示.

    4 几种动态故障测试算法

    March算法 (齐步法) 是在 1982 年 ITC 国际测试会议上推出的[10], March算法具有线性的时间复杂度, 即测试时间与存储容量成正比, 并且具有较高的故障覆盖率, 是目前使用比较广泛和成熟的 RAM 测试算法之一.每种March算法都包含多个March元素, 每个March元素都由地址变化顺序、读写操作和操作数据三部分组成.⇑表示按地址升序进行操作, ⇓表示按地址降序进行操作, e表示操作的地址顺序可以是升序, 也可以是降序.

    March RAW1算法是基于先写后读来检测单单元动态故障的, 仅能覆盖部分dRDF、dDRDF及dIRF.March RAW算法是在March RAW1算法的基础上进行拓展, 使其能覆盖部分的动态耦合故障.March MDIa算法能覆盖所有单单元动态故障.March MDIb算法与March MDIa算法没有本质区别, 仅仅是将第三个和第八个March元素中的r0操作放到前一个March元素中, 没有改变故障覆盖率.

    8 结束语

    本文基于故障原语介绍了SRAM静态故障及单单元动态故障.通过研究几种动态故障测试算法, 提出了一种新的动态故障测试算法, 该算法与March MDIa和March MDIb对单单元动态故障的覆盖率都达到100%, 但复杂度却比March MDIa和March MDIb算法降低了约24%, 然后设计了MBIST电路, 对March DS算法进行了仿真验证, 结果表明该算法可以覆盖SRAM单存储单元动态故障, 最终将设计的MBIST电路应用于16 nm存储器测试中, 测试结果表明, 本文提出的新的动态故障测试算法能够有效探测到FinFET工艺中可能出现的新型的单存储单元的动态故障.
