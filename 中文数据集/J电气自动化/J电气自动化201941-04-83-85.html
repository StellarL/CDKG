    变电站历史数据分布式存储系统设计与实现

    任浩 闫卫国 王强 陈建 郭凌旭

    中国电力科学研究院(南京) 国网天津市电力公司

    摘    要：

    目前, 变电站接入的数据类型越来越多, 存在存储分散, 结构和访问方式差异大, 以及没有建立统一的存储和访问接口等问题。针对以上问题, 提出一种基于Key-Value非关系型数据库的“Redis代理机+SSDB存储集群”的变电站历史数据分布式存储的实现方式, 包括Redis代理服务器和SSDB存储集群, 在代理服务器端通过历史数据统一导入工具, 将不同厂家的变电站SCADA历史数据和非结构化历史数据统一存储在Key-Value网络化缓存库中, 再通过代理服务程序, 采用一致性哈希分片算法将其分布式转存至存储集群中, 对于存储集群的每个存储子机通过LevelDB存储引擎将缓存数据存入磁盘文件中, 实现对历史数据的分布式永久性存储。经实验室测试证明, 开发出的变电站历史数据分布式存储系统的功能需求和性能指标完全达到预期目标。

    关键词：

    非关系型数据库; 非结构化数据; Redis; SSDB; 分布式存储;

    Design and Implementation of a Distributed Storage System for Historical Substation Data

    Ren Hao Yan Weiguo Wang Qiang Chen Jian Guo Lingxu

    China Electric Power Research Institute (Nanjing) State Grid Tianjin Electric Power Corporation

    Abstract：

    At present, so many types of data are connected to substations that there appear such problems as dispersed storage, large difference in the respect of structure and access mode, and no existence of unified storage and access interface. In view of above problems, based on the Key-Value non-relational database, a method for realizing distributed storage of historical substation data, namely, Redis agent + SSDB storage cluster was presented in this paper. The system contained a Redis proxy server and a SSDB storage cluster. At the side of the proxy server, historical SCADA data and unstructured historical data of substations from different manufacturers were all stored in the Key-Value networked cache library with the help of a unified import tool for historical data. Then, through use of proxy service program, cached data was transferred to the storage cluster in the consistent Hash fragment algorithm. For each storage sub-machine of the storage cluster, the cached data was stored in the disk file through the LevelDB storage engine to realize distributed permanent storage of historical data. Laboratory tests proved that the functional requirement and performance indexes of the developed distributed data storage system for historical substation data reached the expected goal.

    Keyword：

    non-relational database; unstructured data; Redis; SSDB; distributed storage;

    0 引 言

    随着智能电网的建设, 电网规模越来越大, 智能电网、智能变电站等应用研究不断深入, 系统面对的采集点和接入的数据类型越来越多, 实现了稳态、动态、暂态以及各种设备参数、定值等各类型数据的采集, 不仅包括“四遥”等结构化数据, 还包括波形、模型和图形等非结构化数据, 由于不同类型数据存储分散、结构和访问方式存在差异, 因此没有建立统一的存储和访问接口。不论从写入速度还是查询效率, 采用常规的关系数据库来存储这些非结构化、大量的历史信息都很难满足应用需求, 因此阻碍了高级应用智能化水平的提高[1,2,3]。

    为解决此问题, 本文提出一种基于Key-Value非关系型数据库的变电站历史数据分布式存储方法, 采用基于Redis缓存库和SSDB持久化存储结合的方式, 充分利用Redis读写速度快和SSDB存储容量大的优势[4,5], 基于该方法设计的存储系统包括Redis代理服务器和SSDB存储集群, 实现对历史数据的永久性存储, 解决缓存库对存储容量的限制, 满足对不同厂家不同类型数据统一存储的要求。

    1 关键技术研究

    1.1 多厂家数据统一映射

    非结构化数据是以文件方式读入Redis缓存中, 图形文件虽然为各厂家私有格式文件, 但在“大运行”体系要求下, 变电站需具备远程浏览功能, 主流监控厂家均具备CIM/G文件转换功能, 同时故障录波和动态数据文件格式都遵循现有国网规范, 因此对于非结构化数据无需转化, 本文只关注结构化数据的通用化映射。

    关于结构化数据, 各厂家数据格式差异较大, 为便于后续统一分析, 需要将其映射到通用数据格式, 映射规则是剔除掉原始数据格式中冗余的属性信息, 必须的私有数据属性需要映射为相同的属性名称和数据内容。比如遥信登录表, 对于遥信对象的描述, 南瑞科技采用别名yx_alias, 四方电气采用digid32, 通用化数据格式采用yx_name, 需要借助实时库定义的遥信定义表, 通过关联映射将yx_alias、digid32转化为yx_name, 从而实现各厂家数据格式的统一映射。

    1.2 数据模型设计及序列化

    考虑到后续查询的便利性, 延续关系型数据库表设计理念, Redis缓存库采用HashMap数据类型进行存储, key标识表名称, field由主键域或几个域的组合组成, 对应一条记录的唯一性标识, value对应该条记录内容。由于Hash类型的数据内容是以字符串形式填充的, 而字符串存储本质是字节流, 对于结构化数据及非结构化文本文件, 需要将其序列化为字节流方可存储Redis缓存库中, 考虑到protobuf序列化工具的轻量高效性[6], 采用其将结构化数据转化为字节流, 对于非结构化文本文件采用fstream读入文件内容的字节流。

    对于结构化数据, 在通用历史数据结构设计的基础上, 按照protobuf数据类型映射表, 将历史数据表结构转化为protobuf消息对象模型, 再通过protoc工具由proto文件生成xxx.pb.h和xxx.pb.cc文件。通过文件解析和pb的set接口生成pb消息对象集, 再通过pb的序列化接口SerializeToArray将pb消息对象集序列化为字节流存入Redis缓存库, 其序列化过程如图1所示。对于非结构化数据, 主要包括图形、图元、故障录波和动态数据文件, key标识文件类型如“PIC/ICON/COMTRADE/PMU”, field采用文件名称“file_name”, value采用fstream读入文件内容的字节流。

    2 系统架构设计

    本文设计的Key-Value非关系型数据库的变电站历史数据分布式存储系统架构设计如图2所示, 该系统数据来源于不同厂家的变电站一体化监控系统的历史数据, 包括结构化数据 (SCADA历史库数据、部分SCADA实时库数据) 和非结构化数据 (图形、录波、序列化数据) , 考虑到部署、配置方便性, 以及对站端影响最小等因素, 采用离线文件导入/导出方式, 对于后续增加的历史数据, 可通过站内离线工具获取再增量式导入至分布式存储系统中。

    该分布式存储系统包括Redis代理服务器和SSDB存储集群。Redis代理服务器安装Redis数据库软件并运行Redis服务, 并在其环境下开发支持多厂家历史数据统一导入工具、支持多厂家历史数据统一浏览工具、Redis代理服务程序;SSDB存储集群有多个存储服务器, 每个存储服务器安装SSDB数据库软件并运行SSDB服务, 采用LevelDB作为存储引擎, 将Key-Value数据存入磁盘文件中。其中, 历史数据统一导入和浏览工具依赖于Redis的操作库hisdataop, 此操作库是将Redis客户端C语言库hiredis的操作接口redisCommand, 按照数据类型及操作类别封装成增加、删除、更新和查询等具体操作的一系列函数接口。

    3 程序设计与实现

    3.1 历史数据统一导入工具

    遍历实时库映射表文件集, 并对每个映射表文件进行解析, 获取对应映射表Map数据, 然后遍历历史记录表文件集, 并对每个历史记录表文件进行解析, 获取对应原始历史记录表数据, 利用映射表Map数据集关联查询, 将原始历史记录表数据转化为通用历史记录数据, 并采用Protobuf序列化接口SerializeToArray将通用历史记录数据序列化为字节流存入Redis数据库中。

    3.2 历史数据统一浏览工具

    读取XML文件形成主界面左侧树形结构, 通过树形控件层层引导点击最后一层表结点, 调用redis读取接口获取该表的字节流队列, 然后采用protobuf反序列化接口将字节流转化为结构化数据, 并通过表格控件展示各属性域内容。支持历史数据表记录内容的展示, 可在界面上对历史数据记录进行增删改操作, 并同步至历史数据库中, 同时支持大容量数据表记录的分页显示和文本导出功能。

    3.3 Redis代理服务程序

    读取分布式存储配置文件, 获取各SSDB存储节点的IP地址和节点名称, 通过哈希算法计算出各节点名称对应的哈希值;获取指令操作对象的key值, 通过哈希运算计算出key值对应的哈希值;将节点名称的哈希值和操作对象key的哈希值排列在0-2^32-1的圆环区间上, 从数据映射到的位置开始顺时针查找, 将数据映射到找到的第一个SSDB存储节点上;如果接收到的是读取指令, 利用上述映射规则, 从该SSDB存储节点读取数据存入Redis库中;如果接收到的是写入指令, 利用上述映射规则, 将Redis数据写入该SSDB节点中。

    4 测试结果

    以两台SSDB存储服务器为例来验证该分布式存储系统的性能指标。首先验证分布式、持久化存储功能, 在Redis代理服务端导入12张历史数据表, 每张表30条记录, 经存储服务端测试, 结果有4张表数据存储在第一台SSDB存储服务器上, 8张表数据存储在第二台SSDB存储服务器上, 分别重启两台SSDB存储服务器, 重启后仍然能查询到重启前存在的数据;其次验证非结构化数据存储功能, 以录波装置PT2201在2001年03月12日04时10分06秒897毫秒所录制的波形文件PT2201_RCD_00062_20010312_041006_897.hdr为例, 在Redis代理服务器端导入该录波文件, 经Redis读取接口测试, 可反读出该录波文件;最后进行存储容量及顺序读取速度的测试, 以事故信息表为例, 在Redis代理服务器端导入1 000 030条事故信息表记录, 经读取接口测试结果, 读取所有记录耗费149秒, 平均读取速度为6 711条记录/秒, 单张表容量超过100万条记录, 如图3所示。

    5 结束语

    本文提出了一种基于Key-Value非关系型数据库的变电站历史数据分布式存储的实现方案, 该系统包括Redis代理服务器和SSDB存储集群, 在代理服务器端通过支持多厂家多业务历史数据统一导入工具, 将不同厂家的变电站一体化监控系统的SCADA历史数据和非结构化历史数据统一存储在Key-Value网络化缓存库中, 再通过代理服务程序将其分布式转存至存储集群中, 对于存储集群的每个存储子机通过LevelDB存储引擎将缓存数据存入磁盘文件中, 实现对历史数据的分布式永久性存储。经实验室测试证明, 开发出的变电站历史数据分布式存储系统的功能需求和性能指标完全达到预期目标。
