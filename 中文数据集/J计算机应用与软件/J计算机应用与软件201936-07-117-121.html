    摘    要：

    为了推进土地执法监察信息化, 以ArcGIS API for JavaScript和ArcGIS Runtime SDK for Android为WebGIS和移动GIS开发框架, 基于REST服务, 构建Web端土地管理系统和移动执法巡查系统。该系统支持国土数据可视化、案件查询、轨迹记录、案件审查等功能, 解决国土执法监察中数据传输慢、处理不及时、缺少监管等问题。充分利用移动GIS的移动性、便携性、实时性特点和WebGIS的跨平台多用户特点, 建立土地执法监察GIS系统, 将执法数据进行高效整合和服务共享, 为提高土地执法监察效率提供技术支持。

    DESIGN AND IMPLEMENTATION OF LAND LAW ENFORCEMENT SUPERVISION SYSTEM BASED ON GIS

    Xiao Xin Fang Chaoyang

    School of Geography and Environment, Jiangxi Normal University Key Laboratory of Poyang Lake Wetland and Watershed Research, Ministry of Education

    Abstract：

    In order to promote the informatization of land law enforcement and supervision, ArcGIS API for JavaScript and ArcGIS Runtime SDK for Android were used as the development framework of WebGIS and mobile GIS. Based on REST service, we constructed the land management system and mobile law enforcement inspection system on the Web side. The system supported the visualization of land data, case inquiry, track record, case review and other functions, and solved the problems of slow data transmission, slow processing and lack of supervision in land enforcement and supervision. We make full use of the mobility, portability and real-time characteristics of mobile GIS and the cross-platform and multi-user characteristics of WebGIS to establish the land law enforcement and supervision GIS system, effectively integrate and share the law enforcement data and provide technical support for improving the efficiency of land law enforcement and supervision.

    Keyword：

    Mobile GIS; WebGIS; Land enforcement; Law enforcement supervision;

    自然资源部发布的《2017中国土地矿产海洋资源统计公报》显示2017年发现土地违法案件7.52万件, 涉及土地面积2.98万公顷, 同比增长1.6%和10.7%, 立案查处违法用地案件4.81万件, 涉及土地面积2.36万公顷, 同比增长2.2%和8.9%, 收回土地1 939.3公顷。国家不断地强调土地保护的重要性, 持续加强相关立法, 加大对土地违法案件的查处和惩罚力度, 但是土地违法并没有收敛, 各地土地违法现象都非常严重[1,2]。因此国土资源管理部门应该充分利用信息化技术, 实时动态监测土地利用情况, 及时处理国土资源利用相关问题, 促进合理利用土地。

    现阶段我国在土地执法监察领域, 各个环节主要还是依靠人工作业来完成, 并通过纸质图作为数据信息的媒介来沟通, 需要投入大量人力物力。传统的土地执法体系存在着以下问题[3]:巡察工作量大, 巡查人员工作效率低;巡察人员监督难, 考核难;土地违法案件上报和移交不及时, 处理不规范;违法用地统计分析处理困难, 案件反馈不及时。因此有必要创建一个信息化系统, 让土地执法监察全过程信息化, 以加强土地实时监测、动态评估以及精准监管能力。随着互联网和空间信息技术快速发展, 使得GIS强大的地理数据采集、数据管理和空间分析能力更有效地发挥。葛小三等[4]在分析企业在物流信息管理中存在的问题后, 构建了基于移动GIS的移动物流信息管理系统;为解决传统的国土资源核查工作无法及时、准确、方便地核查企业的土地资源等信息, 王卫红等[5]设计并研制基于移动GIS的地块采集系统;朱巧玉等[6]针对矿业权管理中数据、结构、关系复杂的特点, 设计了基于WebGIS与工作流的矿业权管理系统。本文在过去研究的基础上, 充分利用移动GIS的移动性、便携性、实时性特点和WebGIS的跨平台多用户特点, 结合移动技术和Web技术, 将GIS技术应用于土地执法监察, 设计并开发了执法巡查移动APP和监察管理Web系统, 大大提高土地资源执法部门的效率和管理水平。

    1 研究方法与架构

    根据《土地违法案件查处办法》[7], 土地违法案件查处主要分为以下四个阶段:

    受理和立案阶段:国土资源管理部门应当及时受理上级相关部门交办、其他部门移送或者转办和群众举报的土地违法案件。群众可以通过口头或书面的形式举报违法用地案件, 相关部门应当详细记录, 并要求举报人签名盖章。如果举报人不愿意透露真实姓名, 土地执法部门要尊重举报人的意愿, 对举报人信息严格保密。如果受理案件符合立案的条件, 则应当填写呈批表, 立案生效;如果受理案件属于重大案件, 则应当上报给上级国土资源管理部门备案。

    调查和处理阶段:案件立案后, 国土资源管理部门应当及时指派专员处理案件, 专员在调查取证时必须至少有两人在场。收集的证据可以是:物证、书证、视听材料、证人证言、当事人陈述、调查笔录和勘察笔录、鉴定结论等有效材料。

    执行阶段:国土资源管理部门向违法用地当事人和相关负责人送达《土地违法案件行政处罚决定书》和《土地侵权行为处理决定书》等相关文件。如果单位或个人在违规用地区域继续施工, 国土资源部门应该立即查封相关建筑, 及时阻止违法行为。

    结案阶段:案件专员处理案件完后, 填写《土地违法案件结案报告》, 上交后经国土资源管理部门相应的主管领导签字批准后即可结案。专员对结案后的案件应该整理归档。

    结合土地执法监察业务要求, 系统主要分为应用层、服务层、数据层、支撑层。内业管理系统采用B/S架构, 服务层以ArcGIS系列软件为GIS平台, 完成GIS数据的集中存储以及各种空间分析功能的执行、数据计算。ArcGIS Server作为发布企业级GIS应用程序的综合服务平台, 提供基于REST风格的Web服务和Web应用。移动端巡查系统采用Android操作系统, 通过无线通信网络与服务端进行交互, 并通过移动智能终端实现数据展示、数据采集、数据查询与空间分析等功能。系统总体架构如图1所示。

    图1 系统架构图   下载原图

    应用层:通过可视化的用户界面, 为巡查人员和管理人员提供数据采集和案件管理等服务。本系统的巡查客户端选用了目前最为流行的Android智能手机系统, 结合ESRI公司的ArcGIS API for Android框架开发完成。ArcGIS API for Android能够通过使用Java程序语言构建多种地图应用程序, 这些应用程序可以调用ArcGIS Server发布的地图服务、要素服务、地理编码服务、地理处理服务和其他自定义服务, 并将它们部署到Android系统上。在管理端主要应用ArcGIS API For JavaScript框架, 该API结合了现代Web技术和强大的地理空间处理功能, 使开发者能够创建高性能的地图应用程序和更智能化的地理数据可视化。

    服务层:服务层是执法监察系统的核心, 用于处理和执行应用层发过来的数据请求, 由GIS服务部分和Web应用服务部分构成。GIS服务部分用于处理客户端对空间数据的请求, 传统的Web应用服务器无法处理空间数据的请求, 因此需要专门的GIS服务器来处理空间数据请求。本系统选择ArcGIS Server作为GIS服务器来发布和管理GIS服务, 主要涉及的服务有地图服务、要素服务、地理编码服务以及网络分析服务等。Web服务器主要用于处理一些基本的土地执法检查的业务数据和系统数据, 如案卷信息、立案信息、系统用户信息等。通过ArcGIS Web Adaptor将ArcGIS Server和Web服务器整合到一起, 这样方便站点统一管理。

    数据层:数据层是执法监察系统的物质基础, 是空间数据、属性数据、业务数据的存放场所。数据层存储的基础数据有铅山县不同比例尺 (1∶500, 1∶1 000, 1∶10 000等) 的影像地图数据和矢量地图数据、地理编码数据、专题图层数据、土地利用现状数据等。通过调用服务层提供的服务, 执法人员在应用层根据角色权限, 对数据进行更新。考虑到本系统需存储大量空间数据, 必须使用空间数据库, 所以选择微软的SQL Server作为数据库。

    支撑层:支撑环境是土地执法监察经营示范应用系统建设、运行和服务的必备条件。为保障项目顺利实施以及框架建成后的长期稳定运行, 要建立健全框架建设和应用的支撑环境。

    2 关键技术

    2.1 移动GIS技术

    移动GIS是对动态世界的时空模拟, 能克服传统GIS对移动对象模拟表达能力的不足[8]。无线通信技术的发展和移动硬件能力的提升是促成移动GIS的产生和发展的重要原因。移动GIS不但拥有传统地理信息系统的空间数据处理功能和空间分析功能, 而且还结合了嵌入式技术、移动互联网技术、GPRS技术和定位技术, 充分集成了这些技术的优点。通过移动GIS技术, 执法人员能够将电子化的数据随身携带, 在任何地点、任何时候都能查询到任何需要的数据, 完全达到3A (anytime, anywhere, anything) 的效果[9]。传统野外工作流程主要以纸制文档为沟通媒介, 然后在室内对成果进行数字化。这个过程导致野外和室内工作重复, 并且野外手写记录数据难免会出现错误, 室内数字化人员和野外工作人员对获得的数据理解也有可能不一致, 这也会导致错误。而移动GIS可以先约定好输入规范, 外业人员只能按规范录入数据, 并且数据可以直接数字化到地理数据库, 这样避免了内外业工作重复、节约成本、提高了数据的准确度, 从而大大提高了工作效率。

    2.2 WebGIS技术

    WebGIS是快速发展的互联网和不断升级的地理信息系统相结合的产物, 理论上可以实现传统GIS具有的所有功能, 为实现地理数据共享、GIS互操作提供了最佳解决途径[10]。WebGIS是一种分布式的地理信息系统, 这个系统中至少包含一个服务器和一个客户端, 客户端一般指浏览器, 但也可以是其他应用程序。GIS与互联网的结合, 赋予了WebGIS许多优点:

    (1) 异地使用:任何地方, 任何地点, 只要有网络和浏览器就可以使用全世界的WebGIS, 避免了在每台设备上装庞大的GIS软件。土地执法监察涉及多部门多人联动工作, 不同设备上都能使用系统提高了工作效率, 节约成本。

    (2) 多用户:传统桌面GIS在某个时间只能单人使用。只要软硬件性能支持, WebGIS可以支持很多用户同时使用, 具有很强的扩展能力。

    (3) 跨平台:Web浏览器是WebGIS的主要运行环境, 不管什么操作系统都有Web浏览器, 所以WebGIS可以运行在不同的系统上, 相较于传统桌面GIS, 具有很强的跨平台性。

    (4) 更新统一:如果要对传统桌面GIS更新, 必须每台电脑下载并安装更新, 而同一个WebGIS一般部署在相同的服务器, 管理人员只要对服务器进行更新就可以对所有客户端实现更新。

    在土地执法监察系统中, WebGIS可以提供在线的GIS服务, 例如地图可视化、案件追踪、案件空间相关性、执法人员分配、地理围栏等GIS功能。这些功能结合土地执法相关业务逻辑和WebGIS异地使用、多用户、跨平台、更新统一等特性可以大大提高工作质量和效率, 节省办公成本。

    3 系统实现

    如图2所示, 基于GIS的土地执法监察系统主要分三端:移动端、管理端、公众端。在移动端, 基于Android手机实现土地储备经营管理数据的地图浏览、巡查上报、综合查询、统计分析、轨迹跟踪功能。在管理端, 基于Web技术, 实现对公众和执法人员上传的举报信息进行受理、立案、审核以及结案, 向巡查人员安排巡查任务, 向公众发布公开信息, 以及对整个系统的管理。公众端可以查看管理端发布的执法公告, 以及举报违法用地活动。

    3.1 管理端功能实现

    基于GIS的土地执法检查系统管理端采用B/S架构, 服务层以ArcGIS系列软件为GIS平台, 完成GIS数据的集中存储以及各种空间分析功能的执行、数据计算。ArcGIS Server作为发布企业级GIS应用程序的综合服务平台, 提供基于REST风格的Web服务和Web应用。管理端主要实现的功能如图3所示, 管理端界面如图4所示。

    图3 管理端功能模块   下载原图

    1) 执法档案:

    跟踪违法案件处理模块从受理、立案、调查取证、审理、执行、到结案全程, 并提供案件管理、案件查询、案件报表统计等管理分析功能, 实现土地执法监察工作流程中的证据采集、案情分析、案件统计和各种表格输出。结合移动端动态巡查, 利用GIS的空间分析功能, 对违法主体、违法面积、违法地类和是否占用农田等各种信息要素进行整合, 得出分析结果, 从而形成一条完整的证据链。

    2) 执法一张图:

    执法一张图页面功能有地图的页面移动、地图放大缩小、地图页面切换、关键字查询、显示土地信息、距离面积测量等GIS基本功能, 还有跟业务相关的历史案件定位、案件信息展示等功能。

    3) 巡查轨迹:

    该功能是案件查处模块下下记录执法人员巡查得到的相关数据, 了解巡查人员巡查范围、巡查地点, 通过地图的方式展示。巡查列表中提供所有巡查人员每次执法的记录, 该记录以最新时间从前往后排序。记录有巡查人员、巡查时间、巡查编号、巡查相关备注等基础信息。点击其中一条巡查记录可以查看更加具体的信息, 包含巡查开始时间, 巡查结束时间以及巡查的地理信息在地图上的标注等。

    4) 执法统计:

    该功能模块主要对历史案件做一些定性和定量的统计分析, 主要包括:案件统计模块, 该模块记录不同状态下的案件数目;案件来源统计, 该模块用雷达图形式来比较不同来源的土地违法案件;违法性质统计, 该模块用饼状图比较不同违法性质案件所占比例;违法趋势, 该模块运用时间轴, 模拟不同时间违法情况。

    5) 系统管理:

    该模块下包括人员管理和日志管理。人员管理可以添加、删除、审核执法人员, 对执法人员的出勤和工作进行考察以及分配。该模块也可以通过地图的形式可视化查看外业执法人员的执勤轨迹、执勤记录等信息。日志管理主要记录系统运行、操作、备份等相关信息。

    3.2 移动端功能实现

    移动端巡查系统采用Android操作系统, 基于ArcGIS API For Android框架开发, 通过无线通信网络与服务端进行交互。基于移动GIS的数据展示、数据采集、数据查询与空间分析等能力, 移动端主要实现的功能如图5所示。移动端界面如图6所示。

    图5 移动端界面   下载原图

    1) 电子地图:

    移动执法端集成管辖区域的电子地图, 可以查看不同比例尺下管辖区域地图, 以及包括地名地址搜索 (支持语音输入) 、图层切换 (卫星底图和矢量底图) 、指北针、长度和面积测量、GPS定位、放大、缩小、平移等基础地图功能。

    2) 执法监察:

    执法人员在执勤时, 可以记录执勤时间和执勤轨迹, 点击终止执勤后, 执勤信息将上传到后台, 从而实现对执法人员的监督。执法人员可以选择已经立案的案件进行调查, 系统将记录调查轨迹, 通过地理围栏技术, 到案发地点可以上传证明文件, 文件可以是图片、视频、音频、文本等。

    3) 案件举报:

    执法人员在执勤过程中发现违法用地案件可以实时定点举报, 填写举报信息表, 系统将自动定位案发地点和记录举报人等信息。

    4) 自由巡查:

    自由巡查模块下执法人员可以随时记录自己的轨迹信息, 并把自己的巡查轨迹进行上传。轨迹可以随时随地回放, 便于执法人员判别执法位置, 避免执法盲区, 提高执法效率。

    5) 案件查询:

    系统提供两种案件查询方式, 第一种通过选择时间段查询, 系统以表格和地图标注形式显示该时间段的土地违法案件;第二种通过空间位置查询, 在地图上绘制多边形区域, 地图自动显示该区域所有的违法案件。

    4 结 语

    基于GIS的土地执法监察系统的设计与实现建立在移动GIS、WebGIS相关技术和国土资源执法监察相关业务知识的基础之上, 结合互联网技术、GPS技术等强大功能, 构建了一个移动化、智能化的国土资源执法监察系统。本系统从实用性出发, 设计和构建了土地执法监察系统外业巡查移动APP和内业管理Web系统, 探索了一条新的土地执法模式。系统以简洁但逻辑性强的界面设计和齐全的功能开发解决了以往土地执法监管难、流程长、反馈不及时等痛点, 为实现地理信息系统技术在国土资源巡查和监管中应用做出了创新的尝试。
