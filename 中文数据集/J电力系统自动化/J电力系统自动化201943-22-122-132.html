    基于图数据库的电网CIM/E模型构建及网络拓扑

    摘    要：

    电力系统规模日益庞大、运行调整更加频繁,对分析计算实时性提出了更高要求。图数据库是近年来兴起的一种源于互联网海量数据并行处理的新型数据库,其数据模型可直观表达电网拓扑结构并易实现并行化遍历查询。首先,从数据模型和数据查询等方面介绍了图数据库的特点,分析了将其应用于大规模电力系统分析计算的潜在优势;其次,面向完整性、一致性以及高效性等准则,提出了基于图数据库并遵循CIM/E标准的电网数据模型设计方法,研发了数据模型装载工具。最后,在电网图数据库模型基础上,实现了一种并行化的电力网络拓扑分析算法。对实际大规模省级电网的计算结果表明,所提方法可显著提高拓扑搜索效率。

    图数据库; 图计算; 网络建模; 网络拓扑; CIM/E;

    Graph Database Based Construction and Network Topology of CIM/E for Power Grid

    HUANG Hua DAI Jiangpeng WANY Yi SHAN Xin ZHAI Mingyu LIU Guangyi

    NARI Group Corporation ( State Grid Electric Power Research Institute) NARI Technology Co.,Ltd. State Key Laboratory of Smart Grid Protection and Control Global Energy Interconnection Institute

    Abstract：

    Power system is becoming larger and larger,and the operation adjustment is more and more frequent,which puts higher requirements on the real-time performance of analysis and calculation. The graph database is a new database that originated from the parallel processing of massive data in internet. The data model can visually express the topology of the grid and easily realize parallel traversal query. Firstly,the characteristics of graph database are introduced from the aspects of data model and data query,and the potential advantages of its application in large-scale power system analysis and calculation are analyzed. Secondly,oriented to the criteria for integrity,consistency and efficiency,a power grid model design method based on graph datebase and following CIM/E standard is developed. Finally,based on graph database model of power grid,a parallel power grid topology analysis algorithm is implemented. The calculation results of the actual large-scale provincial power grid show that the proposed method can significantly improve the topology search efficiency.

    Keyword：

    graph database; graph computation; power grid model; network topology; CIM/E;

    随着经济技术发展,电力系统一方面通过高电压、大容量和长距离的输电网络实现数千公里范围内的高度互联,使得互联大电网的范围不断扩大[1,2];另一方面新能源机组、可调节负荷、电动汽车等接入电网[3,4,5,6,7,8,9,10],使得接入的可调节设备海量增长并增加了运行方式不确定性。因此,电力系统规模日益庞大,运行调整更加频繁,对分析控制的准确性和实时性提出了更高要求。

    调度控制系统是大电网运行控制的重要中枢。物理电网的发展要求调度控制系统具有“全、快、准”等特征[11],能够基于跨调控中心的全局模型进行快速准确计算[12]。目前调度控制系统的运行分析周期一般为5 min,而新能源间歇性可能使1 min内风电、光伏出力剧烈波动,因此提高安全分析软件的实时性、缩短调度控制系统分析计算周期对电网安全运行具有十分重要的意义。

    调度控制系统的发展与测量控制以及计算机技术的进步密切相关[13,14,15]。其中,数据库管理系统是调度控制系统的核心,一般包括商用数据库和实时数据库。商用数据库用来存放非实时和非经常访问的数据,一般采用成熟的关系型数据库;实时数据库可采用关系型、层次型或对象型的数据模型,一般采用内存数据库方式实现[16]。随着电网监控范围的不断扩展和海量内存、多核CPU/GPU以及并行计算等计算机软硬件技术的进步,调度控制系统的实时数据库及相关应用亦从单机集中式扩展为集群分布式,以满足电网调控系统对大规模实时数据的处理性能要求。文献[17]设计了一种基于集群管理的电网调度分布式实时数据库,以厂站为单元、依据电网运行分区等进行数据分片,并以分片为基础实现分布式的实时数据访问。文献[18]提出了一种新型分布式数据采集与监控(SCADA)系统的体系架构,基于分布式实时数据管理技术实现了数据采集处理及拓扑着色分析等功能。文献[19]提出一种基于实时电网动态分区及多调度控制中心的大电网分解协调状态估计方法,多个子区域独立进行状态估计然后进行整合。综合来看,文献[17,18,19]采用了多机分布式架构,以分片或子区域为单元分配计算任务并进行协同,并行粒度为分区级,且处理数据采集等任务时,分片或厂站之间数据交互和协同计算较少。

    图数据库(graph database,GDB)是近年来兴起的一种用于互联网海量数据并行分析处理的技术,其最初的概念来源于图论[20,21]。由于电力系统网络拓扑结构可用图来直观表达且易实现并行化访问,因此将图数据库应用于电力系统分析有望提高大规模电力系统海量数据处理的计算效率[22]。文献[23]基于图数据库研究电网拓扑分析,验证了图数据库相对关系数据库的优势;但其电网图数据模型将开关定义为顶点,在模型构建方面还可进一步优化;此外,其拓扑面向电网运行管理,主要分析物理节点之间的连通性,未涉及电力系统分析使用的计算母线、电气岛等基础计算模型。

    CIM/E模型是中国国家电网有限公司提出的一种描述电力系统模型的规范[24,25],相对于CIM/XML标准可优化对网络拓扑关系的描述。电力系统模型构建及网络拓扑是调度控制系统最基础应用,本文遵循CIM/E规范构建基于图数据库的电网模型,并通过网络拓扑验证了图数据库和图计算技术的优势。

    1 图数据库简介

    图数据库由基于图结构的数据集合以及基于图遍历的计算引擎组成,下文将从数据模型以及数据查询等方面介绍其特点并分析其应用于电力系统的潜在优势。

    1.1 数据模型

    与传统数据库不同,图数据库基于图论理论,采用节点(vertex)和边(edge)表达数据模型[20]。数据属性存储在节点和边中,数据间的关系通过节点之间的边进行描述。

    图数据库与传统数据库对于实体的描述具有共通性,例如无论关系型、层次型或面向对象型数据库对于并联补偿器的描述,都可以采用额定电压、额定容量、无功量测等属性集,不同之处在于对实体之间关联的描述和处理,如表1所示。

    其中,关系型数据库在电力系统中应用最为广泛,具有简单、易维护、用户界面友好等优点,但对电力系统拓扑结构的描述比较困难;层次型数据库较符合实际电力系统的分层分级关系,容易描述电力系统拓扑结构,目前许多电力系统分析应用软件基于层次型数据库实现;面向对象数据库易于与面向对象的软件工程结合,但实现比较复杂,目前还没有成熟产品[16]。图数据库的数据模型与电力系统网络结构则具有自然的相似性,可对电力系统拓扑结构等进行直观表达,并更易扩展。

    1.2 数据查询访问

    数据查询包括一组用于访问数据的程序接口以及相关的支撑服务,是数据库管理系统的计算引擎。图数据库的数据模型中,每个顶点或边均可具备计算功能,既是数据存储单元也是动态计算单元。对图结构的数据集合进行查询也称为图遍历或图计算。基于图中各顶点独立进行局部计算并通过边进行信息交换,易于实现并行化的图计算。目前,Pregel[26]等并行图计算框架主要采用Valiant等提出的整体同步并行(bulk synchronous parallel,BSP)计算模型[27,28]。

    BSP模型通过一系列全局超步(SuperStep)进行逐次迭代。如图1所示,一个超步主要包括以下3个组件。①局部计算:每个参与的处理器都有自身的计算任务,不同处理器计算任务异步并且独立;②通信:各处理器相互交换数据;③栅栏同步(barrier synchronization):当一个处理器遇到“路障”(或栅栏),会等到其他所有处理器完成它们的计算步骤。每一次同步也是一个超步的完成和下一个超步的开始。

    2 基于图数据库的CIM/E模型构建

    基于图数据库建立电网模型是将图计算技术应用于电力系统分析计算的基础。其基本原则是遵循CIM/E标准,保证电网模型的完整性、一致性。同时,适应图数据库特点以尽量提高模型访问效率。

    2.1 CIM/E模型数据描述规范

    CIM/E是在IEC 61970-301电力系统公共信息模型(CIM)的基础上,为解决CIM/XML描述电网拓扑时的效率问题而开发的一种新型高效的电力系统模型数据描述规范[24,25]。其以类似关系表的形式描述了电力系统基准电压、厂站等26类对象。以并联补偿器类(含并联电容器、电抗器)为例,其对应的CIM/E文件如附录A图A1所示。其中,并联补偿器类对象表包括(mRID,name等)11个域,其中既包含描述基本特性的属性(mRID,name等)、描述拓扑的属性(I_node),也包含描述关联的属性(VoltageLevel和Substation等)。

    2.2 数据模式设计

    图数据库数据模式设计主要内容是根据应用场景,定义顶点、边的类型以及相关属性(attribute)。模式设计的准则如下。

    准则1:完整性和一致性,能完整描述电力系统对象特性,并与已有定义尽量保持一致。

    准则2:高效性,适应图数据库特性以尽量提高数据访问效率。

    2.2.1 准则1

    CIM/E模型在公共信息模型基础上发展而来,对电网设备的数据描述具有通用性,因此以CIM/E为基于图数据库的电力系统模型构建基础范本,在图数据库中将CIM/E对象类建模为顶点、将对象类之间的关联建模为边,可满足完整性和一致性准则。

    图数据库数据模型可直观描述电力系统拓扑属性,下文将结合模型系统的拓扑结构,对设计准则进行进一步阐述。如图2所示,该模型系统包含3个厂站,各厂站内部由母线(BusBar)、开关(断路器)、刀闸(隔离开关)、发电机(G)、负荷(LD)等设备构成,各厂站之间通过交流线路(LN)连接。

    图2 模型系统拓扑结构   下载原图

    Fig.2 Topologic structure of power system model

    基于准则1,在图数据库中将母线、开关、刀闸、发电机、负荷、线路等电力系统设备定义为顶点、各设备之间的连接线定义为边。此外,为建立设备之间的拓扑关联,将CIM/E模型中描述设备拓扑关系的物理节点属性定义为顶点。满足准则1的图数据库模型如附录A图A2所示。

    2.2.2 准则2

    准则2与目前图数据库以顶点为中心、基于图遍历进行访问查询的设计理念相关。遵循该设计理念,图数据库进行数据访问查询的特点如下。

    1)可直接对顶点进行访问但不能直接对边进行访问,对边的访问需要从顶点出发进行遍历查询。

    2)对图的完全遍历所需时间与经过的跳数(从起始顶点遍历到目标顶点所经过的边个数)成正比,即图的顶点数和边数越多,访问查询所需时间越长。因此,需要在满足准则1的基础上,对图数据库中的顶点和边进行调整和优化。

    由图2可见,开关、刀闸数量是变压器、母线、发电机、负荷、线路等设备数量的数倍。在附录A图A2的基础上,如将开关、刀闸在图数据库中的模型从顶点修改为边,则图数据库中顶点和相应的边数量将会大幅减少,因此图遍历的跳数亦将相应减少并可提高图数据库查询访问效率。优化后的图数据库模型如附录B图B1所示,该模型可同时满足准则1和准则2。

    除拓扑关联外,CIM/E模型还描述了设备其他关联,例如同步发电机类所含的基准电压标识、电压等级标识、厂站标识等关联。与拓扑关联类似,首先在图数据库中建立基准电压、电压等级、厂站等类型的顶点,然后通过同步发电机类顶点到基准电压等类型顶点的边描述上述关联。

    2.3 数据模型装载

    数据模型装载将CIM/E文件解析成面向图数据库的顶点和边数据集,并存储到图数据库。

    根据数据模型定义,从CIM/E文件可直接抽取出区域、基准电压、厂站、电压等级、母线段、同步发电机、交流线路、交流线段、交流线端点、直流线、直流线端点、换流器、负荷、变压器、变压器绕组、变压器分接头类型、并联补偿器、串联补偿器、间隔等19类顶点的数据集,每类顶点的数据集形成独立数据文件。关于物理节点类的顶点,由于其在CIM/E交换文件中并无对应的对象类,因此扫描CIM/E文件中所有含物理节点属性的对象,读取该属性形成物理节点数据的并集。

    形成各顶点的数据集后,读取对象的引用属性,在源对象与目标对象之间形成边。以附录A图A1中并联补偿器对象为例,并联补偿器与物理节点之间的边以节点对(mRID,I_node)表示,3个并联补偿器到物理节点之间的边分别为(117938015996739703,44999748349852),(117938015996739704,44999748349853)以及(117938015996739705,44999748349884)。依次扫描CIM/E文件所有对象,可按引用属性类别形成所有表示各类边的节点对,并将节点对按边类型存储为边数据集。开关、刀闸等2类边数据集与上述边略有差异,该类型边的源对象和目标对象均为物理节点。

    形成顶点和边的数据集文件后,图数据库启动数据装载任务。各顶点的数据集装载可以并发进行,提升了数据装载速度。

    3 基于图数据库的网络拓扑

    网络拓扑是根据开关、刀闸状态将电网物理节点(Node)模型转化为计算母线(Bus)模型的过程[13]。在已构建的电网图数据库模型基础上,结合图划分与BSP模型实现了一种并行化的电力网络拓扑分析方法。

    3.1 图划分与网络拓扑

    如图2所示,电力网络由发电机、负荷、母线、变压器、线路、开关、刀闸等元件以及物理连接节点构成。其图数据库模型用附录B图B1表示,记为图G=(V,E)。其中V是G的顶点集合,E是G的边集合,且顶点和边均赋予属性。

    图划分问题的目标是将G划分为更小的分量。给定G=(V,E)和正整数p,找到V的子集V1,V2,…,Vp,使得i≠j时,∪i=1pVi=V且Vi∩Vj=∅。

    任意集合{Vi⊆V[1]≤i≤p}称为p路分区,如图3所示。每个Vi是分区的一部分,称为G的子图。当子图中任意2个顶点均可通过一条或多条边相互连接,同时任何顶点都不与子图外的其他顶点相连接,则称该子图为G的一个连通分量。针对图3进行划分,如果删去子图间的连接边,则可形成G的连通分量,如图4所示。

    图4 连通分量   下载原图

    电力系统网络拓扑一般分为2个阶段,第1阶段形成计算母线,第2阶段形成电气岛。其中计算母线是通过处于闭合状态的开关、刀闸连接起来的物理节点集合,而电气岛则是通过变压器、线路等支路元件连接起来的计算母线集合。计算母线和电气岛均可视为不同阶段图划分所形成的子图。因此,网络拓扑可转化为满足一定约束的分阶段图划分问题。

    3.2 并行网络拓扑算法

    传统网络拓扑搜索类算法一般采用深度优先[13]或广度优先法[29],在程序实现上多采用串行方式或厂站、分区级别的并行方式[18,30,31]。应用图划分技术求解网络拓扑的基本策略是通过搜索连通分量自动形成子图。连通分量搜索亦可采用深度优先或广度优先法,但处理大规模连通分量时运行缓慢[32]。着眼于子图连通性分析,基于图划分技术和BSP模型,本文实现了一种细粒度的并行化网络拓扑方法,以下将分2个阶段介绍。

    3.2.1 计算母线搜索

    计算母线是G的子图,子图内各顶点可通过闭合的开关、刀闸所在边连通,且不能跨越变压器、线路等支路元件。以附录B图B1为例,变电站2中开关CB10,CB11,CB12,CB13均处于闭合状态,在G上基于BSP模型进行连通性搜索形成计算母线,其过程如图5所示。顶点CN9,CN10,CN11和CN12的初始物理节点标识(BID)分别为9,10,11和12。

    图5 基于BSP模型的计算母线搜索   下载原图

    Fig.5 Computation bus search based on BSP model

    具体步骤如下。

    步骤1:初始化,给每个节点类顶点赋予全网唯一的标识BID。

    步骤2:发送初始消息,各节点类顶点判断出射边(指从该顶点出发指向相邻顶点的有向边)类型和状态,如果出射边为开关、刀闸且处于闭合状态,则沿该出射边发送一条包含BID值的消息,同时各顶点转入休眠状态。

    步骤3:接收处理消息,各节点类顶点如果接收到其他顶点发送的消息,则解析得到BID;如果收到的BID小于当前顶点自身BID,则更新当前顶点BID且该顶点转为激活状态,同时沿出射边发送新的BID。

    步骤4:判断顶点状态,如果所有顶点均处于休眠状态,则转步骤5;否则,返回步骤3。

    步骤5:在图数据库中增加计算母线类顶点。

    步骤6:结束。

    其中,每个顶点均具备比较自身与相邻顶点BID大小并以较小值更新BID的计算功能。为每个顶点分配独立的处理器工作进程或线程,各顶点独立进行局部计算并通过边发送信息到相邻顶点。通过处理器主控进程或线程实现各顶点信息同步,判断所有顶点状态及控制迭代过程。其中,每次迭代称为1个超步。

    通过3次迭代,各顶点BID均更新为最小值9,表明这些顶点通过开关、刀闸连通成子图,属于同一计算母线。

    计算母线搜索过程可从G中所有节点类顶点同时启动,因此该过程是一种更细粒度的节点级并行方式。所有变电站计算母线搜索过程结束后,图数据库中形成计算母线模型。

    3.2.2 电气岛搜索

    建立计算母线后,通过变压器、线路等支路元件将其连通,具体步骤如下。

    步骤1:从变压器、线路类顶点出发,沿出射边搜索其两端的节点类顶点,并进而查找节点类顶点各自对应的计算母线类顶点。

    步骤2:在图数据库中增加对应计算支路(BCH)类型的边,该边的两侧顶点为步骤1中获得的计算母线。

    通过上述步骤,形成计算母线和计算支路构成的计算母线模型图,如图6所示。电气岛是由计算支路连接起来的计算母线集合,也是计算母线模型图的子图。

    5 结语

    图数据库数据模型可直观表达电网拓扑结构并易实现并行化遍历查询,具有应用于大规模电力系统分析计算的潜在优势。本文提出了基于图数据库并遵循CIM/E标准的电网数据模型设计方法,研发了数据模型装载工具。采用该方法建立的电网图数据库模型具有一致性和完整性好、访问效率高等优点。在电网图数据库模型基础上,实现了一种基于图划分和BSP模型的并行化网络拓扑分析算法。与传统网络拓扑算法相比,本文所提算法是一种细粒度的节点级并行算法。某省级大规模电网的实际测试验证表明,本文模型和算法可显著加快网络拓扑分析速度,为电网在线分析等提供毫秒级的计算模型,从而提高大电网实时计算整体效率。

    目前本文重点研究了电网一次设备的图数据库建模及其拓扑。面向电网泛在物联,构建电网一次设备与二次设备融合、包含地理信息及用能数据的图数据库模型是进一步的研究方向。
