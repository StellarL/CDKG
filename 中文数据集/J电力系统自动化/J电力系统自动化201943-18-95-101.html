    摘    要：

    针对当前电网调控系统运行对在线分析提出更高时效性的需求,将流式数据处理、分布式实时数据库、内存数据网格等技术进行综合研究,提出了一种优化在线分析速率的系统架构,并基于该架构建立了一套面向快速响应的电网在线分析系统。通过该系统可有效缩短在线分析从数据采集到完成分析的整体时间,从而能够快速跟踪电网运行方式变化。

    关键词：

    流式数据处理; 分布式实时数据库; 内存数据网格; 快速响应;

    Architecture of Online Analysis System for Power Grid Oriented to Rapid Response

    ZHAO Lin ZHOU Mike FAN Zelong FENG Donghao YUAN Qihai

    NARI Group Corporation (State Grid Electric Power Research Institute) (Beijing) Beijing Kedong Electric Power Control System Co., Ltd.

    Abstract：

    Aiming at the high time-efficient demand for online analysis in the operation of the current regulatory system for power grid, the technologies of streaming data processing, distributed real-time database and in-memory data grid are comprehensively studied. System architecture for optimizing the online analysis rate is proposed and a set of online analysis system for power grid oriented to rapid response based on the above architecture is established. The system can effectively shorten the overall time of online analysis from data acquisition to completion analysis, so that it could quickly track the changes of grid operation mode.

    Keyword：

    streaming data processing; distributed real-time database; in-memory data grid; rapid response;

    0 引言

    近年来,特高压交直流混联大电网发展迅速,电网运行的整体性特征愈加明显,对全局态势感知和风险防控能力提出了更高的要求。随着可再生能源相关技术迅速发展[1,2],风电、光伏以及直流的大规模投产,电网的电力电子化特征愈发凸显,电网稳定形态愈加复杂。新能源发电的间歇性与波动性以及以分布式电源、储能、电动汽车等为代表的新型负荷的双向性与冲击性,造成电网正常运行的潮流波动幅度和频率显著加大。为了保障大电网的稳定运行,有必要加快数据分析速率,加强电网运行风险预警,尽早发现事故隐患并加以避免;在发生故障时能够快速提供故障分析结果;故障紧急处理后能够进一步快速分析,防止连锁故障发生。在现有基础上进一步提升电网在线分析实时性,必将促进电网自动控制功能的进一步丰富与深化,继而为提升电网调控系统的整体自动控制能力和智能化控制水平奠定基础,因此面向电网全局分析的快速性已成为未来电网调控系统应用功能的最主要特征之一[3]。

    目前大型互联电网调控系统中应用的在线分析系统为准实时系统[4],通过拓扑分析及状态估计为各个应用提供基础数据。文献[5]研究了电网在线安全稳定分析的分解协同计算,基于该高性能计算模式设计了分布式并行在线计算服务应用场景和状态估计计算流程。文献[6] 研究了电网在线安全稳定分析的应用模式,结合三种应用模式能够对电网安全稳定情况进行判断并提供决策。文献[7]基于电网当前运行方式,利用调度操作调整、调度计划、新能源预报和负荷预测数据,在短周期内快速评估电网安全状态及变化趋势。上述文献主要从高级应用层面展开研究,然而在线计算耗时既取决于高级应用计算自身耗时,也取决于数据准备阶段的数据准备耗时,本文将数据准备阶段的数据准备耗时划分为三个部分,分别为数据传输、数据处理、电网模型搭建,基于这三部分对在线计算整个流程加速方法进行研究。目前在整个数据处理流程中,从数据采集、逐级转发、状态估计、数据分发、分析预警、裕度分析到最终形成辅助决策需要约6～9 min,其中基础数据采集的时间时效性和一致性较差,数据处理速率存在瓶颈,导致数据处理结果上送较慢,基础平台对外提供电网状态估计结果的耗时较长,电网在线分析的实时性难以满足现代电力系统在线调控运行的时效性要求。

    本文提出一种优化在线分析速度的电网在线分析系统架构,从数据实时传输、分布式并行处理和内存数据网格[8]等方面展开研究,并基于该架构实现了快速电网静态安全分析。在对各个环节进行优化后,在线分析速度的提升能够更加有效地跟踪电网运行方式变化。

    1 系统架构设计主要思想

    为了能够有效缩短从数据采集到完成分析的整体时间,文中电网在线分析系统架构通过数据实时传输的方式提升数据准备阶段的采集和传输效率,进而优化数据的时效性和一致性。通过分布式并行处理的方式提升数据处理阶段的处理效率,进而缩短获取数据耗时。通过内存数据网格将状态估计后的母线/支路模型以对象的方式进行存取,提升状态估计与后续应用的数据获取速率,并通过基于内存数据网格的分布式并行计算提升静态安全分析等应用的计算效率。

    电网在线分析系统架构由数据采集、分布式数据采集与监视控制(SCADA)系统、分布式实时数据库、内存数据网格构成,其结构如图1所示。

    图1 面向快速响应的电网在线分析系统架构   下载原图

    Fig.1 Architecture of power grid online analysis system oriented to rapid response

    数据采集包括数据直采和数据转发两部分,其中数据直采的数据是从子站直接采集获得数据,数据转发的数据是由各级调控系统接收子站采集数据后经过广域数据总线层层转发共享的数据。数据采集模块接收子站直采数据和各级调控系统转发的数据,其中直采数据以规约的方式传输,转发数据以事件的方式传输。

    分布式SCADA用于接收数据采集模块转发的采集数据,并将接收到的数据处理后存入分布式实时数据库中。

    分布式实时数据库按照传统表结构存储电网运行数据,并在此基础上生成电网物理模型[9]。为了保证模型的准确性,采用了周期获取和事件订阅相结合的数据获取方式,计时器以较长的时间周期从分布式实时数据库中以全网数据方式获取电网的物理模型,经过全网拓扑分析后生成以对象形式建模的电网模型,周期之间的变化遥测、遥信数据以事件订阅的方式从SCADA即时获取,如果是遥信数据,则进行局部拓扑。在每次状态估计后根据计算结果对该模型进行修改。

    在需要进行分析计算时,以该电网对象模型为基础,克隆对应的对象模型并储存至内存数据网格,相对于表形式的关系数据,内存数据网格中存储的对象型数据更容易生成复杂的数据模型,譬如图中所示调度员潮流对象模型、短路电流计算模型、静态安全分析对象模型等,然后以算法配合得到计算结果。

    随着一体化进程不断推进,各级调控系统对数据的实时性和可靠性提出了更高要求,但量测数据、SCADA处理数据在各级调控系统之间转发时仍采用存储后转发的方式,相对于传统的批量式数据处理,流式处理的方式可以在数据产生后即时触发处理,相对于缓存后定期进行批量处理的数据处理方式可满足更高的实时性要求[10]。本文提出的在线分析系统架构使用流式数据处理方式取代了原有的批量数据处理方式。

    在线分析系统的数据流程如图2所示,电力系统运行过程中不断生成量测数据,数据经过流式数据处理后,生成分析电网对象模型并存入内存数据网格,并随电网运行方式变化而对应的进行修改,进一步克隆出适合于调度员潮流、静态安全分析、短路电流等应用的母线/支路模型对象,进而触发应用计算得出结果。

    2 流式数据处理

    2.1 数据实时传输

    随着电网互联互通的快速发展,电网运行中产生的数据量越来越大,非直采数据需要在多级调控系统存储后再层层转发共享,并且按时间周期进行批处理发送的方式导致获取的实时量测数据存在一定的时延,直采数据与转发数据在时间上差距较大(约10 s),给网络分析等后续计算的准确性带来较大影响。沿用之前的周期批量上传的方式已经逐渐难以满足调控系统对数据实时性的要求。

    图2 在线分析系统数据流程   下载原图

    Fig.2 Data flow of online analysis system

    本节提出一种基于流式思想的数据实时传输模型,从数据采集开始,将数据作为一个稳定的流,在各个接收和处理环节之间传输,对比批处理方式,避免了数据多次存储再批量转发所造成的时间损耗。由于调控系统数据来源主要为通过电力调度数据网、专线网络或其他规约通信方式的主子站数据采集,存在实时持续不断到达、到达次序独立、数据来源众多且格式复杂的特点,不落地存储的即时触发式数据实时传输模型能够更好地满足时效性要求。

    数据实时传输模型如图3所示,下级调度机构(省调、地调)的主站系统作为调管范围内子站数据的汇集点,上级调控系统(国调、分调)通过调度数据网以事件的方式向下级调控系统订阅数据。下级调控系统解析上级调控系统所订阅的量测数据信息,在正式通信开始之前,根据测点描述进行信息匹配,确定下级调控系统量测数据与上级调控系统量测数据的对应关系。下级调控系统通过IEC 104规约通信对子站的量测数据进行采集,经过规约解析将采集报文转换成事件,并根据下级调控系统量测数据与上级调控系统量测数据的对应关系,将该事件转换为上级调控系统可识别的事件,通过基于调度数据网的广域数据总线[11]以事件驱动的方式将数据转发至上级调控系统。

    图3 数据实时传输模型   下载原图

    Fig.3 Real-time data transmission model

    实现数据实时传输的核心在于提供通用的广域数据传输手段以保证数据传输速率,并提供广域数据冗余选择以保障稳定性。为了提高数据传输的实时性、稳定性,打破规约采集的局限,该模型以广域数据总线为基础,为整个数据传输流程的各个环节提供统一、高效的广域数据传输和共享机制;实时数据接入某一个汇集点(省调、地调控系统)后,即可实现在广域范围内数据共享。

    目前遥测数据因为采集数据量较大,采取下级调控系统SCADA处理后周期上送的方式,即SCADA处理遥测数据后先进行数据入库处理,再通过周期性数据筛选,最后对变化数据进行转发。相较于非直采数据需要在多级调控系统的数据库中存储后再层层转发的数据传输方式,通过广域数据总线有助于提升数据传输效率;上级调控系统的SCADA处理模块与下级调控系统的SCADA处理模块获取采集数据的时延仅为数据网络传输的延时。如图3所示,整个数据处理流程耗时包括上下级数据采集耗时、上下级调控系统之间网络传输耗时、上级SCADA处理耗时、数据发送至状态估计耗时。在数据传输速率提高、降低数据网络传输时延的前提下,可以通过缩小子站实时量测数据的变化区间来提高数据采集频率,进而将子站与主站通过规约进行实时数据传输的周期缩短。根据两地距离远近与网络介质不同,消耗时间也不同,在千兆光纤环境下,每1 000 km数据传输耗时仅几十毫秒,以中国直线最远距离估算,网络传输时延可控制在0.5 s以内;数据采集和SCADA处理对于单个事件耗时在分布式机制下可控制在毫秒级;数据从SCADA发送到状态估计为事件驱动,耗时为局域网内数据传输时延,基本可以忽略不计。因此如图3所示,理论上将子站实时数据传输的周期缩短到0.5 s以内时,可将上级调控系统状态估计获取的SCADA数据的时差控制在1 s以内,进而状态估计提供更加理想的电网实时数据断面。

    2.2 分布式数据处理

    在调控系统中,传统的SCADA主要处理开关量、功率和电压等量测数据,主备冗余式架构的SCADA便能够应对当下的数据规模的处理需求。但在面对电网运行中生成的海量数据,例如通过数据实时传输模型提高数据传输效率、满足了数据传输的实时性要求后随之增大的传输数据量,SCADA也应提高数据处理效率以应对更大的数据规模的处理需求,从而避免产生因数据传输量过大导致数据无法及时处理的问题。

    文献[12]提出了一种分布式SCADA技术,通过划分处理数据对象的方法将数据分片,分别存储在不同节点上,由主分片对外提供访问。该方案从应用级实现了SCADA功能的分布。本文提出一种分布式SCADA模型,依托基础平台支撑能力,即以分布式实时库为基础实现SCADA功能的分布。如图4所示,SCADA各节点通过总线可获取所有分区的数据,再根据分区策略对数据进行过滤,在各个SCADA节点均只处理分区策略划定范围的数据(某一区域的实时数据),在完成处理计算后通过分布式实时库应用程序编程接口(API)将结果提交给分布式实时数据库并存储。数据存储后通过分布式实时数据库的数据冗余机制同步到其他节点,以确保当某个SCADA节点故障时其工作可迅速由其他SCADA节点接替。数据冗余机制由分布式实时数据库支撑实现,无须在SCADA再次进行数据冗余备用,因此各个节点的SCADA不存在重复数据处理工作,更大程度地节省了计算资源,保证了数据处理效率,且数据处理集群可以根据实际需要进行快速扩张。

    图4 分布式SCADA模型   下载原图

    Fig.4 Distributed SCADA model

    该模型中的分布式实时库具有以下基本特征。

    1)数据分布、逻辑统一,支持应用的透明访问,即应用无须获知分布式实时数据库的数据分布特性,便可通过分布式实时数据库API访问全网实时数据。对于自动发电控制(AGC)、特殊计算、网络拓扑等需要全网实时数据的应用可以运行于分布式实时库的任意一个节点。目前根据应用自身特性,AGC、网络拓扑主要采用一主多备机制,特殊计算采用分布式处理机制。

    2)数据冗余机制,在数据存储完成后,自动将其同步到分布在其他节点的冗余分片中,冗余分片的同步存在毫秒级时延,通过在内存中记录主分片的操作日志,即时传输至备分片操作日志,然后在备分片上重放操作日志来保证主备分片的数据一致性。

    3)负载均衡方式支撑高效、并发访问。

    4)弹性扩展,随着节点数的增减调整数据分布。

    通过分布式实时数据库将各个节点的数据分片信息以分区任务的方式告知各个节点的SCADA模块,每个节点的SCADA模块仅处理自身收到的分区数据。这样对于每个节点的SCADA模块,每次访问分布式实时数据库时,可以保证绝大部分时间是在本节点读取或者存储,减少网络读写操作,有效提高了数据读写效率。

    分布式数据处理有利于解决当前大电网背景下的海量实时数据造成的调控系统实时数据处理的瓶颈问题,提升数据处理效率和可靠性。

    3 电网建模及快速计算

    电力系统高级应用分析计算的基础是电网分析模型,以往电网分析模型的建立方式是从关系型实时库中或文件中取出数据,经过状态估计后形成电网物理模型[13],在高级应用自身进程空间中通过网络拓扑计算,然后进行下一步的仿真计算分析。本文提出并实现了基于内存数据网格的分布式并行计算,仿真分析类应用可以依靠底层平台的支撑进行更高效的计算,其特点如下。

    1)采用键/值方式存储,母线/支路模型可以作为一个对象直接存储到内存数据网格中,内存数据网格具有动态可扩展性、高可用性以及对象数据模型的良好支持性。

    2)对象型电网分析模型,利用内存数据网格技术提供数据亲和功能,以所有数据为整体或者部分变化数据进行索引并克隆,高效地实现在分布式环境中的多份克隆数据,其中在初次克隆时以所有数据为整体进行克隆,初次克隆完成后的后续循环计算中以变化数据进行克隆。节点由内存数据网格管理,电网分析模型在计算机集群的分布、冗余备份以及同步自动完成。对象可以克隆出多份,并利用内存数据网格均匀地分布到多个节点后展开并行计算。

    3)高级应用的算法根据数据亲和原则“贴住”存储在内存网格中的母线/支路模型,即将算法嵌入存储于内存数据网格中的对象直接进行计算,并尽可能使用并行计算。

    如图5所示,计时器从SCADA周期性获取电网物理模型,周期之间的变化遥信、遥测数据通过事件订阅的方式即时获取并更新电网物理模型,经过拓扑分析后生成的对象型电网分析模型,然后基于对象型电网分析模型对电网数据进行状态估计,以对象形式保存于内存数据网格中。保存于内存数据网格中的电网分析模型支持3种级别的并行计算。

    图5 电网分析模型对象及其快速计算示意图   下载原图

    Fig.5 Schematic diagram of power grid analysis model object and its rapid calculation

    1)多节点级别并行计算,内存数据网格的多个节点可以分别对其存储的对象型电网分析模型并行处理。如图5所示,节点1基于对象型电网分析模型对一部分故障集实施静态安全分析,节点2同时基于同步的电网分析模型对剩余故障实施静态安全分析。

    2)多案例级别并行计算,克隆多份对象型电网分析模型,并在内存数据网格的同一节点中存储,多个电网分析模型可以分别嵌入算法同时进行仿真计算。如图5所示,节点1中对电网分析模型克隆1实施静态安全分析,对电网分析模型克隆2实施短路电流计算。

    3)算法级别并行计算,基于电网仿真分析与算法解耦的实施方法,以对象型电网分析模型为参数创建分析算法对象,在算法本身逻辑允许的前提下,电网分析模型支持算法级别并行处理。如图5所示,以对象型电网分析模型为参数(建立映射关系)创建静态安全分析算法,静态安全分析算法依据电网分析模型生成N-1故障集。故障分析对象包含2个参数:电网分析模型、故障集中的单个故障对象,基于这种方法创建故障分析对象的集合。从算法层面出发,这种故障分析对象可以独立处理,静态安全分析算法将故障分析对象集合分组送入服务器的线程池并行处理。

    4 实验验证

    采用本架构的电网在线分析系统已经过实验室验证,部分功能已在网省级调控系统中得到应用部署。

    在面向快速响应的电网在线分析系统中,主要优化效果如下。

    1)通过数据实时传输,缩短数据转发的时延。监听并接收事件,并通过广域数据总线向上级调控系统发送。实时数据采集后,1 s之内便可经由广域网络传输至需要数据的调控系统,而原有处理模式需耗时10 s左右。

    2)通过分布式数据处理,提升SCADA数据处理效率,实验室环境中采用6台服务器组成分布式SCADA。

    测试用的电网模型规模为45万遥测点、44万遥信点,通过大量发送变化遥测和变化遥信事件,保证系统正常处理的情况下得出效率对比如表1所示。采用分布式SCADA后实时数据处理能力基本达到原SCADA处理能力的6倍。

    5 结语

    调控系统运行对全局态势感知和风险防控能力存在更高时效性的需求,结合当前调控系统在线分析的特点,建立了面向快速响应的电网在线分析系统架构。利用该架构可优化电网在线分析速度。通过实验验证表明,采用该架构,在线分析响应速度和实时性方面取得实质性的突破,从而能够更加快速地跟踪电网运行方式变化,使得电网在线分析能够更有效地发挥预测和预警的作用,为改善现有在线趋势分析、预想运行方式的执行机制提供了可能。

    未来可以研究与本文架构一致的分布式状态估计算法与结果分发服务技术,进一步提升在线分析响应速度。研究基于灵活配置定义的电网在线分析模型的在线配置管理技术,灵活地满足多样的在线分析应用基础支撑需求。研究具有自适应与弹性扩张能力的电网分析模型的在线调度技术,充分发挥分布式并行计算的性能优势。
