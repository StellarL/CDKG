    摘    要：

    提出概率统计中区间估计的四个涉及统计计算思维的问题, 并与启发式教学相融合, 设置了区间估计的问题情境, 从理论上分析最短区间长度与参数函数及其分布之间的线性与非线性关系;并通过多个例题及实验设计对上述问题进行计算求解和直观展示, 以期达到融合统计计算思维与启发式教学的效果.

    Calculation Problems and Teaching Thinking of Interval Estimation

    Lv Shu-long Liu Wen-li Xue Mei-yu

    College of Mathematics and Computer Science, Fuzhou University

    Abstract：

    Four questions about the thinking of statistical calculation in the interval estimation of probability and statistics are provided. Combining the heuristic teaching, it sets the problem situation of interval estimation firstly and analyzes the linear and nonlinear relationship between the shortest interval length and parameter function and probability distribution in theory. Then several examples and experiments are carried out to solve the above problems and show them intuitively, so as to achieve the effect of heuristic teaching and statistic calculation thinking.

    Keyword：

    Received： 2018-07-24

    统计学专业实验教学对于提高统计学科的教学与科研水平有重要意义[1], 既可加深对统计概念和理论的理解, 又能提升统计的应用.参数估计是统计学中理论性和实践性融合的一个重要内容, 特别是区间估计, 其应用领域极为广泛.在教学设计上, 区间估计一般针对的是总体分布中的未知参数[2,3], 重在介绍区间估计的基本思想和方法, 即区间估计要求在一定的可信度保证下, 使得估计区间的平均长度最短.关于最短区间的存在性、唯一性和求解等问题的理论研究较多, 如文献[4,5,6,7,8]等.文献[9]探讨了面向计算思维的启发式教学, 并将虚拟实验引入计算机基础教学中.总所周知, 启发式教学是一种激发学生积极思考并强调主动实践的教学理念, 它对于思维的培养至关重要.区间估计的相关内容经过设置问题情境、问题求解和实验设计, 可以巧妙地实现统计思维与计算思维的融合, 一方面提升数理统计课程的层次感和趣味性, 另一方面对培养学生的统计计算思维也能提供一种有益的尝试.

    1 问题提出

    区间估计的一般形式可表述为:设X1, X2, …, Xn是总体X～F (x;θ) 的简单随机样本, g (θ;X1, X2, …, Xn) 为θ和样本的实值函数, 且g (θ;X1, X2, …, Xn) ～G (x) , 其中G (x) 是与θ无关的已知分布.为方便起见, 不妨将函数g (θ;X1, X2, …, Xn) 记为g (θ) .则关于θ的置信度为1-α的区间估计由P (G1≤g (θ) ≤G2) =1-α, 解得

    Ρ (θ^1≤θ≤θ^2) =1-α. (1)

    其中θ^1, θ^2都是样本X1, X2, …, Xn的函数.按照区间估计的思想, E (θ^2-θ^1) 最短时估计的精度最高.在教学中, 结合计算思维能力[9,10]的培养, 对于 (1) 式, 以下四个问题适合采用启发式教学:

    (i) 为何习惯取G1, G2为分布G (x) 的双侧分位点G1-α/2和Gα/2?

    (ii) 是否存在G1, G2, 使得E (θ^2-θ^1) 达到最小? 若存在, 该如何求解?

    (iii) 如果区间G2-G1达到最短, 是否可以保证E (θ^2-θ^1) 达到最短?

    (iv) 是否有其他方法能导出θ的置信度为1-α的区间估计?它与常规方法有什么关系?

    其中, 分位点定义采用上侧分位点, 即P (G>Gα) =α.

    针对上述问题, 从教学和思维渐进的视角, 结合理论和实验对统计计算思维进行探讨.

    2 以问题解析强化统计思维训练

    经典统计大都是模型驱动[11]的, 模型驱动奠定了统计计算思维的基础.因此, 从区间估计的定义及 (1) 式, 在教学中可引导学生从随机变量的分布、分位点等方面的知识入手, 从理论上对上述问题进行逻辑推导和归纳总结, 强化模型驱动的思维训练.比如在 (1) 式中, 可以讨论g (θ) 为θ的线性函数时对区间估计的精度是否有影响?分布G (x) 的密度函数对称时对区间估计的精度是否有影响?等等.

    2.1g (θ) 是θ的线性函数

    定理1 若g (θ) 是θ的线性函数, 且g (θ) ～G (x) , 若存在G1, G2, 使得P (G1≤g (θ) ≤G2) =1-α且G2-G1达到最短, 则由 (1) 式导出的θ的区间估计 (θ^1, θ^2) 可保证其区间长度最短.

    证 不妨设g (θ) =Aθ+B, 其中A>0, 则由P (G1≤Aθ+B≤G2) =1-α, 得

    Ρ (G1-BA≤θ≤G2-BA) =1-α. (2)

    则θ的区间估计为 (θ^1, θ^2) = (G1-BA, G2-BA) , 其区间长度为θ^2-θ^1=G2-G1A.由于G2-G1达到最短且A为常数, 故θ^2-θ^1也达到最短.当A<0时, 同理可证.

    有了上述理论证明, 即可引导学生对满足该定理条件的各类区间估计进行梳理, 总结满足上述定理条件的同类型的区间估计.显然单正态总体均值的区间估计, 双正态总体均值差和方差比的区间估计, 指数分布参数的区间估计都是符合的.

    2.2g (θ) 是θ的非线性函数

    若g (θ) 是θ的非线性函数, 且g (θ) ～G (x) , 而由 (1) 式导出的θ的区间估计 (θ^1, θ^2) 是G1, G2的非线性函数, 其区间长度也是G1, G2的非线性函数.因此即使存在G1, G2, 使得

    P (G1≤g (θ) ≤G2) =1-α

    且G2-G1达到最短, 也难以判断θ^2-θ^1可达到最短, 第3部分的实验也表明了这个结论.

    同理, 可总结满足上述情况的区间估计, 如单正态总体方差的区间估计、泊松分布参数的区间估计、二项分布参数的区间估计等.

    2.3 关于双侧分位点

    定理2 当分布G (x) 的密度函数恰好是偶函数 (即关于y轴对称) 时, 其双侧分位点G1-α/2和Gα/2是满足P (G1≤g (θ) ≤G2) =1-α且保证区间G2-G1长度达到最短, 其中G1-α/2=-Gα/2.

    基于定理1和定理2, 单正态总体均值的区间估计、双正态总体均值差和方差比的区间估计, 都可以由双侧分位点G1-α/2和Gα/2按照 (1) 式直接导出, 并能保证估计 (θ^1, θ^2) 的区间长度最短.

    2.4 通过非参数方法解决参数的区间估计

    对于问题 (iv) , 作为思维、方法和计算的扩展, 可以启发学生另辟蹊径, 从非参数角度解决参数问题, 顺势引入随机模拟和Bootstrap方法[12].从参数θ的估计量构造出发, 结合Bootstrap自助法产生足够数量的基于子样本的估计量, 再利用区间估计定义获得参数θ的区间估计, 从而为参数区间估计开辟一种通用的不依赖与分布假定的估计方法.当然, 作为教学探索, 可以深入研究非参数方法和参数方法的异同点等.本问题的实验设计和说明见第3部分的实验4.

    2.5 模型驱动向问题驱动、计算驱动的转变

    上述的解析, 在一定程度上解释了提出的四个问题, 但是在非线性或非对称分布中, 问题 (i) 还没有彻底解决.问题 (ii) 更需要给出具体的算法设计以便得到最佳的区间估计, 而问题 (iii) 还缺少一个教学上直观的展示.前三个问题的焦点集中在E (θ^2-θ^1) 达到最小, 而这明显又是个期望最小问题, 体现了模型驱动和计算驱动的思维.第四个问题侧重从数据角度出发, 构造求解问题的方法, 体现了问题驱动和计算驱动的思维.众所周知, 统计理论和方法的发展都是由问题驱动产生的.在大数据时代, 问题驱动和计算驱动并发影响着统计的发展, 文献[11]给出了类似的观点.在实际的统计教学与人才培养中, 融合计算思维的统计计算思维更应该积极转变模式, 特别是把计算驱动的统计思维融入教学当中, 构建起融合问题驱动、模型驱动、随机模拟、机器学习和人工智能相结合的新型教学模式.

    3 以实验设计及问题求解提升统计计算水平

    实验1 最短区间存在时, G1, G2的求解并与双侧分位点G1-α/2和Gα/2比较.

    Step1 构造函数g (θ) ～G (x) , 按 (1) 式得到区间估计 (θ^1, θ^2) 及区间长度θ^2-θ^1;

    Step2 以min (θ^2-θ^1) 为目标函数, 以 (1) 式为约束条件, 通过极值优化, 得到最佳的G1, G2;

    Step3 通过分布G (x) 的双侧分位点G1-α/2和Gα/2, 得到常规区间估计的长度.并与Step2中得到的G1, G2比较.

    例1 以指数分布的参数λ的区间估计为例, 构造g (λ) =2nX¯λ∼χ2 (2n) , 显然g (λ) 是关于λ的线性函数, 而分布G (x) 是非对称的.由 (1) 式可得

    Ρ (G1≤2nX¯λ≤G2) =Ρ (G12nX¯≤λ≤G22nX¯) =1-α,

    从而区间估计为 (G12nX¯, G22nX¯) , 其长度为G2-G12nX¯, 因此求最短估计区间等价于求min (G2-G1) .

    fmin1=function (G1, n, level) #构造目标函数

    { p1=pchisq (G1, 2*n)

    if (p1+1-level<1) { G2=qchisq (p1+1-level, 2*n) ; return (G2-G1) }

    #确保概率不能超过1, 得到区间长度G2-G1

    else return (10000000)

    }

    min1.interval=function (n, level=0.05) #利用优化函数, 求解最短区间

    { out=optimize (fmin, c (n/2, 2*n) , n, level) ;

    best=c (out$minimum, qchisq (1-level+pchisq (out$minimum, 2*n) , 2*n) )

    return (c (best, best[2]- best[1]) ) #返回 (G1, G2, G2-G1)

    }

    min1.quantile=function (n, level=0.05) #给出双侧分位点及区间长度

    { quan=c (qchisq (level/2, 2*n) , qchisq (1-level/2, 2*n) ) ; return (c (quan, quan[2]-quan[1]) ) }

    int1.mat=matrix (0, nrow=3, ncol=6) ; n=c (20, 30, 40)

    for (i in 1:3) int1.mat[i, ]=c (min1.interval (n[i], 0.05) , min1.quantile (n[i], 0.05) )

    Get_sample=function (mu=10, sigma=2, n=20)

    { set.seed (100) ; return ( rnorm (n, mean=mu, sd=sigma) ) } #使用set.seed确保数据可再现

    Get_bootstrap_int=function (x, level=0.05, Times=2000) #取得bootstrap区间估计

    { n=length (x) ; mus=numeric (Times)

    for (i in 1:Times) { nx=sample (x, n, replace=TRUE) ; mus[i]=mean (nx) ; }

    mus=sort (mus) ; left=trunc (Times*level/2) ; right=Times-left;

    return (c (mus[left], mus[right], mus[right]-mus[left]) )

    }

    min2.quantile=function (x, level=0.05) #取得常规的区间估计

    { n=length (x) ; quan=c (qt (level/2, n-1) , qt (1-level/2, n-1) ) ; mx=mean (x) ; sdx=sd (x)

    return (c (mx+quan*sdx/sqrt (n) , (quan[2]-quan[1]) *sdx/sqrt (n) ) )

    }

    int.mat=matrix (0, nrow=3, ncol=6) ; n=c (20, 30, 40)

    for (i in 1:3)

    { x=Get_sample (10, 2, n[i]) ; int.mat[i, ]=c (Get_bootstrap_int (x, 0.05, 2000) , min2.quantile (x, 0.05) ) }

    int.mat
