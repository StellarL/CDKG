    摘    要：

    调度是拟态构造Web服务器的重要机制, 现有的调度算法多数缺乏对拟态构造Web服务器异构性和Web服务质量的考虑, 不能解决调度机制带来的安全性和Web服务质量不稳定的问题。为此, 提出基于最大异构性和Web服务质量的随机种子调度算法, 通过计算拟态构造Web服务器所有阈值, 随机选出种子执行体, 根据最大异构性和Web服务质量确定调度方案。仿真结果表明, 与随机调度算法相比, 该算法具有较优的调度效果, 在安全性、Web服务质量和动态性之间达到了较好的平衡。

    关键词：

    调度算法; 异构性; 服务质量; 随机种子; 阈值;

    Executors Scheduling Algorithm for Web Server with Mimic Structure

    ZHANG Jiexin PANG Jianmin ZHANG Zheng TAI Ming ZHANG Hao NIE Guanglai

    Abstract：

    Scheduling is an important mechanism for the Web server with mimic structure.Most of the existing scheduling algorithms lack consideration about heterogeneity and Quality of Service (QoS) of the Web server with mimic structure, and do not solve the problems of security and service quality instability caused by the scheduling mechanism.Therefore, a scheduling algorithm called Random Seed algorithm based on Maximum heterogeneity and Web QoS (RSMHQ) is proposed.All the thresholds of the Web servers with mimic structure are calculated.The seed executor is randomly selected, and the scheduling scheme is determined according to the maximum heterogeneity and QoS.Simulation results show that compared with the random scheduling algorithm, the proposed algorithm has better scheduling effects and achieves an excellent balance between security, Web service quality and dynamic behaviour.

    0 概述

    当今社会已经进入“互联网+”和“万物互联”的时代, 人们的生活与网络空间的联系愈发紧密, 与此同时, 网络空间的安全问题也越来越成为当前的严峻挑战之一, 其根本原因在于网络空间中软硬件漏洞和后门无法杜绝, 防御者无法彻查出所有的漏洞和后门, 且基于漏洞和后门的攻击门槛低, 进而导致网络空间安全处于“易攻难守”的被动局面[1]。为解决上述问题, 文献[2]基于异构冗余的架构, 提出非相似余度架构 (Dissimilar Redundancy Structure, DRS) 技术, 将其广泛应用于入侵检测[3]、入侵容忍[4,5]与移动目标防御等领域[6], 为避免基于共生漏洞和后门的攻击以及扭转防御者处于网络攻防中的不利态势起到了积极作用。网络空间拟态防御技术是基于动态异构冗余 (Dynamic Heterogeneous Redundancy, DHR) 架构的新型主动防御技术, 通过在DRS架构中导入基于闭环负反馈控制的动态、随机、多样等经典防御机制, 克服了DRS架构的静态性以及运行机制确定性等缺点[7]。

    调度是一种应用广泛的技术, 研究者在不同领域对其做了大量研究[8,9,10], 但是很多调度算法只适用于确定处理机调度任务[11,12], 缺乏面向网络空间拟态防御技术的执行体调度算法。文献[12]针对实时系统的副版本调度问题, 提出一种副版本不可抢占的全局容错算法 (Fault-Tolerant Global Scheduling with Non-Preemptive Backups, FTGS-NPB) , 该算法可以有效地降低实现容错的代价。文献[13]针对异构集群系统的实时任务容错调度中缺少考虑服务质量 (Qualify of Service, QoS) 因素的问题, 提出一种具有QoS需求的调度算法。文献[14]针对异构多处理环境中的任务调度问题, 提出一种采用初始随机调度机制的基于蚁群优化的rank-ACO算法, 实验结果表明, 该算法具有较好的性能。文献[15]为异构网络体系结构的入侵检测系统 (Intrusion Detection Systems, IDS) 提供了一个调度框架, 通过综合考虑系统安全要求和可变计算资源的可用性, 合理地分配入侵检测系统的任务。文献[16]针对采用冗余异构功能等价体的系统, 采取完全随机调度策略以降低攻击者对某些漏洞和后门的探知或利用的可能性。文献[17]针对拟态防御系统的异构功能等价体调度问题, 提出了一种兼顾动态性的随机种子最小相似度 (Random Seed and Minimum Similarity, RSMS) 算法, 但是该算法没有给出量化异构功能等价体间相似度的方法, 也未考虑因调度的异构功能等价体对拟态防御系统服务质量的影响, 以及因调度机制所带来的安全性和Web服务质量不稳定的问题。

    为解决拟态构造Web服务器的执行体调度问题, 本文根据影响拟态构造Web服务器调度的2个因素, 即异构性和Web服务质量, 给出其定义与量化评估方法, 在此基础上提出基于异构性和服务质量的执行体调度算法。

    1 调度指标

    拟态构造Web服务器的执行体调度策略需要综合考虑其安全性和Web服务质量, 本文通过对其构件的复杂性和差异性量化评估得到其异构性, 并通过对其性能量化评估得到其Web服务质量。

    1.1 拟态构造Web服务器

    拟态构造Web服务器是网络空间拟态防御技术的一种典型应用, 系统架构如图1所示。本文依据拟态防御原理, 构建功能等价的、多样化的、动态化的非相似Web虚拟机池, 采用多余度表决、执行体冗余执行等技术以阻断攻击链, 基于负反馈机制的中心调度器利用动态执行体调度来增加系统的动态性和不确定性, 减小同种攻击连续发生的概率, 增大漏洞和后门的利用难度[18]。拟态构造Web服务器的调度机制使系统具有动态性, 突破了如DRS系统的静态性与确定性, 通过对异构执行体进行优化调度, 实现异构执行体的优化组合, 增加对特定执行体漏洞和后门的探知与攻击难度, 降低了相同攻击发生的概率。与此同时, 调度机制使得系统的安全性和Web服务质量存在不确定性和不稳定性等问题。因此, 调度算法的优劣直接决定了拟态构造Web服务器的安全性和可用性。

    1.2 异构性指标

    异构冗余的Web服务器提供了真实的Web服务, 是用户、攻击者所直接面对的服务实体, 其承受了大量的服务请求和攻击行为。异构性降低了共生漏洞和后门出现的概率, 显著提高了拟态构造Web服务器的安全性, 因此, 异构性是拟态构造Web服务器的调度的关键指标之一。本文采用文献[19]对异构性的定义, 将复杂性与差异性作为拟态构造Web服务器的2个特征。拟态构造Web服务器的异构性为H, 复杂性为C, 差异性为FD, 则其异构性与复杂性、差异性有如下关系:

    H=C×FD (1)

    1.3 Web服务质量指标

    拟态构造Web服务器在不改变Web服务功能属性的基础上, 使系统具有一种内生的防御机制, 但是拟态防御机制的引入对某些Web服务器性能指标造成了一定的负面影响[8]。这种负面影响可以视为拟态防御技术对Web服务质量 (QoS) 的影响, 因此, Web服务质量是拟态构造Web服务器调度的关键指标之一。

    定义8 (QoS属性集) 能够量化Web服务质量的QoS属性的集合, 记为Q={q1, q2, …, qi, …, qm}, 其中, qi是一个QoS属性, m为QoS属性数量。

    基于拟态防御原理, 结合Web服务器、防火墙等网络设备性能测试, 拟态构造Web服务器选取每秒事务处理次数 (Transactions Per Second, TPS) 、最大TCP并发连接数 (MCTCC) 、吞吐量和响应时间构成其QoS属性集[18]。有些QoS属性属于积极属性, 其值越大, QoS度量值越高, 如每秒事务处理次数;还有些QoS属性属于消极属性, 其值越大, QoS度量值越低, 如响应时间。通过测试可以得到执行体的Web服务QoS属性的测量值[22], 由于不同QoS属性的测量值可能会相差多个数量级, 而将不同的QoS属性直接进行量化会影响量化的公平性, 因此要对QoS属性进行归一化处理, 进而得到能够统一量化的无量纲的QoS属性值。

    定义9 QoS属性矩阵其中eij表示Web服务的一个QoS属性值, n为Web服务数量, m为QoS属性的数量。

    定义10 执行体的QoS属性向量LAi由QoS属性矩阵E的一行构成, 即LAi= (ei, 1, ei, 2, …, ei, m) T, 其中, m为QoS属性的数量。

    QiA=fmin (e1, i, e2, i, …, en, i) (9)

    其中, n为执行体集中执行体数量。

    定义11 执行体集的QoS属性向量LA= (Q1A, Q2A, …, QmA) T, 其中, QiA为执行体集的第i种QoS属性值, m为QoS属性的数量。

    2 随机种子调度算法

    基于负反馈机制的调度算法是拟态构造Web服务器动态性的重要体现, 如果被调度的执行体间异构性过小, 会导致拟态构造Web服务器的共生漏洞过多, 从而增加安全风险;如果被调度的执行体Web服务质量差距过大, 会导致拟态构造Web服务器的服务质量过低, 从而降低可用性;如果调度前后的拟态构造Web服务器安全性和服务质量存在较大差距, 会严重影响系统拟态构造Web服务器的安全性和可用性。

    第1节已给出拟态构造Web服务器的异构性和Web服务质量的定义与量化评估方法。以下给出综合调度指标ES计算方法:

    ES=HA+WSA (12)

    追求拟态构造Web服务器调度综合指标ES最大化, 是调度执行体的目标。但是, ES最大的执行体集可能不是最佳的调度方案。文献[24]指出, 在调度执行体时不仅要追求ES最大化, 还要避免出现局部极值所导致的非最佳调度方案的情况。

    如图2 (a) 所示, 左、右2个执行体集的综合调度指标ES明显高于中间执行体集的综合调度指标ES, 但由于左执行体集的异构性太低, 服务质量过高, 会导致系统安全性低、可用性高;右执行体集的异构性过高, 服务质量太低, 会导致系统安全性高、可用性低;而中间执行体集则兼顾了安全性和可用性。因此, 在调度执行体时, 要避免出现异构性和Web服务质量局部极值的情况。

    如图2 (b) 所示, 执行体A1、A2、A4组成的执行体集的异构性明显高于执行体A1、A2、A3组成的执行体集的异构性, 但A1、A4的差异性d14过小, 其共生漏洞数量较多。因此, 在调度执行体时, 要避免出现差异性局部极值的情况。

    如图2 (c) 所示, 执行体A1、A2、A4组成的执行体集的服务质量高于执行体A1、A2、A3组成的执行体集的服务质量, 原因是执行体A4的服务质量高于A3的服务质量, 执行体A3是导致A1、A2、A4组成的执行体集服务质量低的主要原因。因此, 调度执行体时, 要避免出现服务质量局部极值的情况。

    由上述可知, 在调度执行体时, 执行体集的异构性、执行体集的Web服务质量、执行体的Web服务质量和执行体间差异性要大于设定的阈值, 否则调度方案不成立。

    定义13 (调度向量) 拟态构造Web服务器各个执行体被调度情况所组成的向量, 记为X= (χ1, χ2, …, χN) T。其中, χi为执行体Ai的调度情况, 当χi=0时表示执行体Ai不被调度, 当χi=1时表示执行体Ai被调度, N为系统内执行体数量。

    定义14 (调度阈值集) 阈值的集合记为Ω={Hmin, WSmin, wsmin, dmin}, 其中, Hmin为执行体集的异构性阈值, WSmin为执行体集的Web服务质量阈值, wsmin为执行体的Web服务质量阈值, dmin为执行体间差异性阈值。

    本文将系统内所有执行体集异构性的平均值作为Hmin, Web服务质量的平均值作为WSmin, Web服务质量的平均值作为wsmin, 系统内所有构件的差异性的平均值作为dmin。

    定义15 (调度规则集) 拟态构造Web服务器调度执行体所采用的规则的集合, 记为R={ri|ri是一个调度规则, 且i=1, 2, …, c}, 其中, c为规则数量。

    由上述定义可以得出, 基于最大异构性和Web服务质量的随机种子调度算法的调度规则集为R={r1, r2, r3, r4}, 其中, r1表示执行体集的冗余度n, r2表示调度种子执行体As, r3表示调度的执行体和组成的执行体集满足调度阈值集Ω, r4表示综合调度指标ES最大。

    本文提出一种基于最大异构性和Web服务质量的随机种子调度算法 (Random Seed Based on Maximum Heterogeneous and Web QoS, RSMHQ) 。首先随机选出一个执行体作为种子执行体, 然后根据最大异构性和Web服务质量原则选择整体综合调度指标最大的调度方案, 算法流程如图3所示。

    算法步骤如下:

    步骤1 生成种子执行体。中心调度器根据输入参数在下线的执行体中随机选择出种子执行体As。种子执行体是RSMHQ算法最先确定的元素, 可以增加调度方案的随机性和丰富性, 提高拟态构造Web服务器的动态性[18], 降低攻击者掌握系统变换规律的可能性, 进而提高系统安全性。

    步骤2 去除与种子执行体差异性小于阈值和Web服务质量小于阈值的执行体。去除与种子执行体差异性小于阈值和Web服务质量小于阈值的执行体, 是对执行体的初步筛选, 可以降低计算量, 减少备选执行体集的数量, 筛选出的执行体的集合记为F。若筛选出的执行体数量小于n-1, 即筛选出的执行体数量与种子执行体的和小于执行体集的冗余度n, 则无法生成新的执行体集, 转到步骤1;若大于n-1, 则转到步骤3。

    步骤3 过滤备选的执行体集。步骤2中筛选出的执行体与种子执行体组成备选执行体集P, 去掉备选执行体集中不满足异构性阈值和Web服务质量阈值的执行体集, 进而筛选出符合阈值要求的执行体集。若筛选出的执行体集数量小于1, 则转到步骤1;若数量大于等于1, 则转到步骤4。

    步骤4 确定最终调度执行体集。计算步骤3中筛选出的符合阈值要求的执行体集中综合调度指标ES, 并选出综合调度指标ES最大的执行体集作为最终要调度的执行体集, 输出调度向量X。

    4 结束语

    拟态构造Web服务器是网络空间拟态防御技术的一种典型应用, 而调度机制是其中的关键技术。本文针对拟态Web服务器调度过程中安全性和Web服务质量不稳定的问题, 提出一种基于最大异构性和Web服务质量的随机种子调度算法。实验结果表明, 该算法的平均WS、H、ES较随机调度算法分别提升了15.32%、87.33%和37.45%, 其调度稳定性较随机调度算法分别提升了5.26%、71.72%和60.53%, 在安全性、Web服务质量和动态性之间达到了较好的平衡。下一步将针对实际应用环境中的安全性和Web服务质量的要求, 研究如何确定调度阈值和调度指标的权重。
