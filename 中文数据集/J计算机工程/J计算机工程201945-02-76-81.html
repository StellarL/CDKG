    摘    要：

    现有研究在重映射失效虚拟网过程中未对不同类型的虚拟网承载业务进行有效区分, 导致资源收益较低。为此, 提出一种基于业务优先级划分的虚拟网可靠性映射算法。定义虚拟节点和虚拟链路的重要性指标, 运用混合整数线性规划方法将基于业务优先级划分的虚拟网可靠性映射问题建模为最优化问题。在此基础上, 提出快速可扩展的启发式虚拟网可靠性映射算法, 以提高失效虚拟网的重映射效率。实验结果表明, 该算法在虚拟网请求接受率、服务提供商收益和利润成本比3个指标上, 表现优于FD-EVN、APG-TABU和MW-AC算法。

    关键词：

    网络虚拟化; 混合整数线性规划; 重映射; 虚拟链路; 虚拟网可靠性映射;

    Survivable Virtual Network Embedding Algorithm Based on Service Priority Partition

    YUAN Letian LI Guofeng

    Tianjin Key Laboratory of Optoelectronic Sensor and Sensing Network Technology, Nankai University

    Abstract：

    In the process of remapping invalid virtual networks, the existing research has not effectively distinguished the different types of virtual network bearer services, which leads to the reduction of resource income. Therefore, a Survivable Virtual Network Embedding (SVNE) algorithm based on service priority partition is proposed. The virtual node and virtual link importance index are defined, and the reliability mapping problem of virtual network based on service priority division is modeled as an optimization problem by using mixed integer linear programming method. On this basis, a rapid extensible heuristic Priority based Survivable Virtual Network Embedding (PSVNE) algorithm is proposed to improve the efficiency of remapping. Experimental results show that the performance of this algorithm is better than that of FD-EVN, APG-TABU and MW-AC algorithm in the three indexes of virtual network request acceptance rate, service provider profit and profit cost ratio.

    0概述

    作为解决互联网“僵化”问题的一种手段, 网络虚拟化越来越得到学术界和产业界的关注[1]。通过允许多个异构虚拟网共享底层基础设施, 网络虚拟化技术能够提高网络的灵活性、安全性和可管理性, 而在此过程中虚拟网类似于底层网络的资源分片。在网络虚拟化环境中, 传统的互联网服务提供商被分为2种角色:基础设施提供商, 负责提供和管理物理网络资源;虚拟网服务提供商, 根据业务需要在虚拟网上部署相应的网络协议和提供各种定制化服务, 并向终端用户提供网络服务。

    本文提出一种基于业务优先级划分的虚拟网可靠性映射 (Survivable Virtual Network Embedding, SVNE) 算法。根据虚拟网承载的业务类型为其分配优先级并定义虚拟节点和虚拟链路的重要性指标, 使用混合整数规划模型对基于业务优先级划分的可靠虚拟网映射进行建模, 在此基础上, 提出启发式可靠虚拟网映射算法以提高算法的运算效率。

    1 虚拟网映射相关研究

    虚拟网映射主要用于解决为虚拟网高效分配底层网络中的可用资源问题, 已经有研究从多个方面证明该问题属于NP-hard问题[2]。当前主流研究思路是通过设计启发式映射算法提高虚拟网的映射效率[3,4,5]。文献[3]提出一种基于区域资源聚集度的虚拟网映射方法, 在映射虚拟网时充分考虑到其拓扑特性。文献[4]使用马尔科夫随机游走模型对物理节点的承载能力进行评价, 并提出压力均衡的启发式映射算法。文献[5]首先将虚拟节点用K-核分解技术进行种类划分, 然后针对每类虚拟节点的特点, 用合适的算法进行映射。

    虚拟网环境中的物理资源普遍承载了多个虚拟网节点或链路, 当某项物理资源出现故障 (这在大规模网络环境中并不少见[6,7]) 时, 会导致比在传统网络环境中更加复杂的虚拟网失效问题。为应对已部署虚拟网的失效问题, 可靠虚拟网映射引起学界的广泛关注, 其主要目标是解决物理资源失效对虚拟网的影响。目前, 针对物理节点失效问题已有研究人员提出一些解决方案, 按照其机制可以划分为2类:

    1) 主备切换机制, 该机制在故障发生前为每个虚拟节点分配备份节点, 当故障发生后, 失效虚拟节点迅速切换到备份节点。文献[8,9,10]分别提出使用主备切换机制的虚拟网可靠性映射方法。文献[8]提出一种根据故障类型对虚拟网进行加强的方法, 但是该方法会引起大量的虚拟网迁移。文献[9]为虚拟网同时分配承载和备份的物理资源以实现资源的最优化利用。文献[10]提出一种两阶段虚拟网重映射算法, 在第1阶段为虚拟节点和虚拟链路添加备份节点和链路, 在第2阶段将得到的加强虚拟网映射到物理网络。主备切换机制虽然能够将失效虚拟网迅速迁移到正常工作的物理网络, 但是会降低物理资源的利用率, 成本较高。

    2) 在线迁移机制, 当故障发生后该机制使用重映射算法, 将失效的虚拟节点和链路迁移到正常工作的物理资源。文献[11,12,13,14]使用在线迁移机制对失效虚拟网进行调整。文献[11]针对物理节点失效场景, 提出一种基于迁移感知的虚拟网重映射算法, 该算法通过对其他虚拟节点和虚拟链路的迁移为失效虚拟网释放足够的物理资源。文献[12]则考虑在对物理网络升级改造后如何对虚拟网重新进行映射, 从而在提高物理资源利用率的同时降低迁移开销。文献[13,14]关注在光网络场景中的虚拟网可靠性问题, 分别提出通过预计算方式为虚拟网寻找备份物理资源的方法。在线迁移机制不需要为虚拟网预先分配备份物理资源, 而是在故障发生后通过重映射的方法将失效虚拟网迁移到正常工作的物理资源上, 从而降低资源开销, 但是由于在线迁移机制比较依赖当前的资源状况, 因此存在计算时间长和迁移成功率低的问题。

    在实际网络环境中, 不同的网络应用对于物理资源的可用程度具有不同的容忍性[15]。多媒体业务对于网络资源可用程度的容忍性较低, 一旦承载这类应用的物理资源发生故障应立即处理, 而数据备份等业务的容忍性相对较高, 则可以给予一定的恢复时间。针对网络应用的上述特点, 可靠虚拟网映射算法在为故障虚拟网重新分配资源时, 应优先处理对物理资源可用程度容忍度低的虚拟网, 但已有的研究多数都没有提出相应解决方案。

    2.2 问题描述

    给定存在一个故障物理节点 (|NfS|=1) 的物理网络GS及其承载的虚拟网络集合GV, 寻找一种资源分配方案, 将受到影响的故障虚拟节点和故障虚拟链路重映射到正常的物理节点和物理路径上, 该方案要使在所有虚拟网中恢复的故障虚拟链路的重要性之和最大, 且重映射开销最小。

    在重映射过程中需要满足如下约束条件:

    1) 同一个虚拟网的多个虚拟节点不能映射到同一个物理节点上, 但是不同虚拟网的多个虚拟节点能够被映射到同一个物理节点上。

    2) 虚拟节点和虚拟链路要映射到具有足够可用物理资源的物理节点和物理路径上。

    3) 能够正常工作的虚拟节点和虚拟链路不应该被重映射。

    2.4 问题复杂性分析

    混合整数规划问题在本质上属于NP-hard问题, 难以在多项式时间内求解。这导致在大规模网络环境中无法直接使用该模型进行求解。为解决该问题, 本文基于上述数学模型进一步提出能够对故障虚拟节点和虚拟链路进行快速重映射的启发式算法。

    5 结束语

    针对当前虚拟网恢复算法在恢复故障虚拟网时无法对其业务类型进行区分, 导致物理资源利用效率低的问题, 本文提出一种基于业务分类的虚拟网恢复算法。对虚拟节点和虚拟链路的优先级进行定义, 使用混合整数规划方法对基于业务分类的虚拟网恢复问题进行建模, 最后为解决混合整数规划方法实践复杂度过高的问题, 提出启发式虚拟网恢复算法。实验结果表明, 该算法在虚拟网请求接受率、虚拟网服务提供商收益和利润成本比指标上均能取得较好的效果。下一步将继续完善业务优先级的评价方案, 并寻找更好的启发式映射算法, 以提高资源利用率。
