    摘    要：

    为在资源受限的安全芯片中兼顾标量乘运算的安全性与效率, 设计一种抗能量分析攻击的椭圆曲线密码标量乘算法。采用带门限的非相邻形式编码方法对标量进行编码, 以提高标量乘运算的效率。在此基础上, 结合预计算和基点掩码技术, 使算法可以抵抗多种能量分析攻击。分析结果表明, 该算法不仅能够抵抗简单能量分析攻击、差分能量分析攻击、零值点能量分析攻击和修正能量分析攻击, 而且可以提升预计算效率, 减少存储空间。

    关键词：

    椭圆曲线密码; 标量乘算法; 能量分析攻击; 非相邻形式编码; 基点掩码;

    Scalar Multiplication Algorithm Resisting Power Analysis Attacks Using NAF with Threshold Window

    GU Jianguang

    College of Information Engineering, Zhengzhou University of Industrial Technology

    Abstract：

    In order to give consideration to the safety and efficiency of scalar multiplication in resource limited security chip, an Elliptic Curve Cryptography (ECC) scalar multiplication algorithm resisting power analysis attacks is presented.The Non-Adjacent Form (NAF) coding method with threshold is used to encode the scalar to improve the efficiency of scalar multiplication.On this basis, the technologies of precomputation and base point masking are used to make the algorithm can resist the power analysis attacks.Analysis results show that the proposed algorithm can not only resist the Simple Power Analysis (SPA) attack, Differential Power Analysis (DPA) attack, Zero-value Power Analysis (ZPA) attack and Refined Power Analysis (RPA) attack, but also can improve the efficiency of precomputation and reduce storage space.

    0 概述

    椭圆曲线密码 (Elliptic Curve Cryptography, ECC) 算法具有密钥长度短、处理速度快、存储空间小、带宽要求低和安全性高等优点, 因此, 其被广泛应用于安全芯片等各类资源受限的密码系统中[1,2]。ECC算法极易遭受能量分析攻击[3,4], 但如果直接在算法中施加抵抗能量分析攻击措施, 将会降低椭圆曲线密码效率。采用非相邻形式 (Non-Adjacent Form, NAF) 编码方法[5]计算ECC算法中的标量乘运算, 可以显著提高运算效率。这是因为与传统二进制编码方法相比, 采用NAF编码方法的标量乘运算虽然增加了1次倍点操作, 但同时也减少了约1/3的点加操作次数。然而, 采用NAF编码方法的标量乘运算同样无法抵抗能量分析攻击, 依然存在效率与安全性的矛盾。

    为能兼顾椭圆曲线密码标量乘运算的安全性与效率, 本文提出一种抗能量分析攻击的的椭圆曲线密码方案TWNAF。采用带门限窗口的NAF编码方法, 同时结合预计算和基点掩码技术, 提高椭圆曲线密码运算效率, 并使其能够抵抗简单能量分析 (Simple Power Analysis, SPA) 攻击、差分能量分析 (Differential Power Analysis, DPA) 攻击、零值点功耗攻击 (Zero-value Power Attack, ZPA) 与修正功耗攻击 (Refined Power Attack, RPA) , 解决椭圆曲线密码安全性与效率矛盾的问题。

    1 相关研究

    近年来, 研究者从兼顾安全性与效率方面对椭圆曲线密码标量乘算法进行了大量研究[6,7], 如采用带符号双基数系统[8]、带符号整数拆分形式[9]、带符号阶乘展开式[10]等编码方法的抗能量分析攻击方案, 都可确保椭圆曲线密码安全性与效率两方面的性能。其中, 对于采用NAF编码方法抗能量分析攻击方案及改进方案的研究也有较多成果。文献[11]为提升标量乘算法的安全性和效率, 在窗口NAF标量乘算法的基础上, 采用边信道原子块方法实现抵抗SPA, 同时利用并行计算方法对标量进行分割, 进一步提升了标量乘运算的效率。文献[12]提出RWNAF和FWNAF两种基于NAF编码方法的抗功耗攻击方案, 其中RWNAF方案通过采用改进的带窗口NAF编码方法, 同时结合随机掩码技术实现;FWNAF方案通过采用碎片窗口NAF编码方案, 同时结合随机掩码技术实现。两种方案均能有效提升标量乘运算效率并且抵抗多种功耗攻击, 而FWNAF方案比RWNAF方案更充分利用了存储资源。文献[13]针对NAF编码方法中编码系数1和-1的点加运算和点减运算消耗能量, 提出一种抗SPA的平衡能量NAF标量乘算法, 主要是通过在标量乘运算前执行一次求解-P的预处理, 将编码系数-1的点减运算变换为点加运算。该方案只增加了一次求解-P的运算, 可忽略不计, 因此, 总运算量基本保持不变, 但增强了标量乘运算抵抗SPA攻击的能力。文献[14]针对添加伪操作会降低标量乘运算效率的问题, 提出一种基于多位运算的标量乘算法, 主要是将标量的NAF编码形式分成两部分进行运算, 每一部分的循环运算都执行相同的操作, 从而避免添加伪操作并且能够抵抗SPA, 有效提升了标量乘运算效率。

    2 背景知识

    2.1 NAF标量乘算法

    采用NAF方法对标量k进行编码, 是指在标量的二进制序列中的每个位包含0、1和-1这三种形式, 且不会有相邻的非零元素出现, 即标量k的NAF编码为k= (km-1, ⋯, ki, ⋯, k1, k0) ΝAF, 其中ki∈{-1, 0, 1}, 且有km-1≠0, 具体表示形式如式下:

    k=∑i=0m-1ki2i

    标量k的NAF编码算法描述如下:

    算法1 标量k的NAF编码算法

    输入 长度为m bit的标量k

    输出k= (km-1, ⋯, ki, ⋯, k1, k0) ΝAF

    令m=0;

    如果k≥1, 则重复执行:

    当k为奇数时, 则执行:

    km=2- (kmod4) ;

    k=k-km;

    当k为偶数时, 则执行:

    km=0;

    k=k/2;

    m=m+1;

    返回 (km-1, ⋯, ki, ⋯, k1, k0) ΝAF。

    令lNAF (k) 表示标量k采用NAF编码方法的编码长度, 则有2m/3≤lNAF (k) ≤2m+1/3, 其编码长度最多比二进制形式编码长度多1, 且其非零元素平均个数为m/3, 是所有带符号编码形式中非零元素个数最少的编码方式, 而且标量k的NAF编码形式是唯一的。

    椭圆曲线密码NAF标量乘算法描述如下:

    算法2 椭圆曲线密码NAF标量乘算法

    输入 标量k, 椭圆曲线上一点P

    输出Q=kP

    采用算法1对标量k进行NAF编码得到 (km-1, …, ki, …, k1, k0) NAF;

    令Q=0;

    对于i从m-1到0, 则重复执行:

    Q=2Q;

    2.2 能量分析攻击及其防御

    简单能量分析 (SPA) 攻击通过测量密码算法的能量消耗信息来猜测在某时刻对应的运算过程及操作次数, 以此判断该时刻所对应的密钥信息。当前抵抗SPA攻击的措施主要包括添加伪操作、归一化标量乘法和统一的点加和倍点操作等。

    差分能量分析 (DPA) 攻击通过测量密码算法的多次能量消耗曲线, 然后对其进行统计分析来获取密钥, 不需要考虑密码算法的具体实现细节。当前抵抗DPA攻击的措施主要包括随机化密钥、随机化基点、随机化曲线参数和随机化投影坐标等。

    零值点能量分析 (ZPA) 攻击通过测量寄存器为0时所获得的有效能量消耗曲线, 以此来区分标量乘运算中的点加和倍点运算, 进而获取密钥信息。当前抵抗ZPA攻击的措施主要包括基点掩码和标量分割等。

    修正能量分析 (RPA) 攻击利用特殊点 (x, 0) 或 (0, y) 使随机化投影坐标的方法无法起作用, 进而可以使攻击者采用DPA攻击以获取密钥。当前抵抗RPA攻击的措施主要包括基点掩码和标量分割等。

    3 TWNAF方案设计

    传统Width-w NAF、RWNAF和FWNAF等基于NAF的标量乘算法, 虽然通过采用预计算可以大幅减少标量乘点加操作次数, 但是平均至少需要 (2w-1) /2次预计算运算, 运算量较大。通过采用门限窗口NAF编码方法可以有效减少预计算次数, 大幅提升预计算的利用率。TWNAF方案的基本设计思想是:首先采用门限窗口NAF编码方法对标量进行编码, 其中窗口宽度为w, 门限大小为T;然后通过预计算得到预计算表Y;最后引入一个随机点R实施基点掩码, 实现标量乘算法。

    首先给出标量k基于门限窗口NAF的编码算法, 如算法3所示。

    算法3 基于门限窗口NAF的编码算法

    输入 标量k, 窗口宽度w, 门限大小T

    输出TWNAF (k) = (kt-1, kt-2, …, ki, …, k1, k0)

    如果k mod 2=0, 则执行k=k+1;

    令t=0, i=0;

    如果k>1, 则重复执行:

    令ti=w;

    如果ti>1, 则重复执行:

    令k′i= (kmod2ti+2) -2w;

    如果|k′i|<Τ, 则执行ti=ti+1;

    如果|k′i|=Τ, 则执行:

    k= (k-k′i) /2ti;

    kt=k′i;

    如果|k′i|>Τ, 则执行:

    如果状态在减少, 则执行ti=ti/2;

    如果状态在增加, 则执行ti=ti-1;

    t=t+ti;

    i=i+1;

    kt=1;

    对于余下的编码位, 则补为0;

    返回TWNAF (k) = (kt-1, kt-2, ⋯, ki, ⋯, k1, k0) 。

    4 算法性能分析

    4.1 安全性分析

    算法4由于采用窗口法和预计算, 使得攻击者不能区分椭圆曲线上点的大小, 因此无法仅通过波形来获取相关密钥信息。此外, 由于计算负值的点需进行求逆运算, 将会使功耗曲线出现差异。本文在TWNAF算法中通过对负值点也进行预计算, 在标量乘主循环运算中只进行倍点运算和点加运算, 从而消除计算正值点和负值点的功耗差异, 使算法可以抵抗SPA攻击。

    在算法4中, 由于引入了随机点R, 使得在执行主循环运算时掩盖了中间运算结果和能耗之间的差异, 使得攻击者无法对所获取的能耗曲线进行统计分析, 因此TWNAF算法可以抵抗DPA。此外, 由于引入了随机点R, 攻击者无法找到特殊点 (x, 0) 或 (0, y) , 同时也无法测量寄存器为0时的功耗曲线, 攻击者不能利用特殊点或零值寄存器来获取有用的功耗曲线, 因此TWNAF算法可以抵抗ZPA和RPA攻击。

    从图2可以看出, TWNAF标量乘算法的功耗差异轨迹波形图谱没有明显波形起伏, 执行的倍点操作和点加操作在图谱上没有明显分界, 攻击者无法对大量功耗轨迹曲线进行统计分析得到相关性密钥信息, 因此, 攻击者无法利用功耗差异对所给TWNAF标量乘算法实施能量分析攻击。

    4.3 运算效率分析

    对TWNAF算法的运算效率进行分析, 具体如下:在步骤2中, 标量k的门限窗口NAF编码算法的运算量与标量乘算法的运算量相比可以忽略不计;在步骤3中, 由于求-P的运算量很小可忽略不计, 所以预计算运算量为T-1次点加运算, 即 (Τ-1) A;在步骤6中, 主循环运算需要执行t-1次倍点运算和t/ (w+2) 次点加运算, 即 (t-1) D+[t/ (w+2) ]A;在步骤5和步骤8均需执行一次点加运算;在步骤7中, 当标量k为偶数时, 则需执行一次点加运算;当标量k为奇数时, 则不需执行点加运算。TWNAF算法的总运算量为 (t-1) D+{[t/ (w+1) ]+Τ+1}A。其中, D表示倍点运算, A表示点加运算, t表示标量的二进制编码长度, w表示窗口宽度。表1给出了TWANF算法与RWNAF算法和FWNAF算法的计算量比较。

    5 结束语

    椭圆曲线密码算法被广泛应用于安全芯片等应用领域, 但是能量分析攻击是安全芯片面临的重大威胁之一, 而且由于安全芯片资源受限, 使得兼顾安全性和效率成为安全芯片应用中亟需解决的重要问题。

    本文提出的抗能量分析攻击标量乘算法TWNAF, 通过采用带门限窗口的NAF编码方法对标量进行编码, 同时结合预计算和基点掩码, 不仅能有效抵抗SPA、DPA、RPA和ZPA等多种能量分析攻击, 而且可以减小存储空间, 提高运算效率。因此, TWNAF算法适用于资源受限的安全芯片等应用系统。
