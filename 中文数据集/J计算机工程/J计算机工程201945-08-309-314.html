    摘    要：

    基于机器学习的Android平台恶意软件检测方法提取的权限信息特征维度高且类别区分能力弱, 导致检测精度低及复杂度高。为此, 提出一种基于特征占比差与加权随机森林的恶意软件检测方法。通过获取Android软件的权限信息和硬件组件信息, 分析各类特征的占比差, 并将特征属性作为分类模型的输入。在此基础上, 对随机森林中的树模型赋予不同的权值, 验证树模型对最终分类结果的影响。实验结果表明, 与神经网络方法相比, 基于特征占比差的特征构建方法所提取的特征具有较好的类别区分能力, 且改进后的随机森林能提高恶意软件检测的准确性。

    Malicious Software Detection Method Based on Feature Ratio Difference

    YAN Haisheng LI Qiang SUN Kaiwei

    School of Software Engineering, Chongqing University of Arts and Sciences School of Computer Science and Technology, Chongqing University of Posts and Telecommunications

    Abstract：

    The Android platform malicious software detection method based on machine information extracts the feature information with high feature dimension and weak class distinguishing ability, resulting in low detection accuracy and high complexity.Therefore, a malicious software detection method based on Feature Ratio Difference (FRD) and weighted Random Forest (RF) is proposed.By obtaining the permission information and hardware component information of the Android software, the ration difference of various features is analyzed, and the feature attribute is used as the input of the classification model.On this basis, different weights are assigned to the tree model in the Random Forest (RF) , and the impact of the tree model on the final classification result is verified.Experimental results show that the features extracted by the feature construction method based on the feature difference ratio have better class distinguishing ability, and the improved random forest can improve the accuracy of malware detection.

    Keyword：

    Android security; malicious software detection; feature extraction; feature ratio difference; random forest;

    0 概述

    2018年全球智能手机市场占有率调查报告指出, Android平台已占据75.6%的移动设备市场份额[1]。然而, 恶意软件会通过窃取用户隐私信息、远程控制、恶意扣费等方式攻击Android平台, 在已发现的恶意手机程序中, 97%是基于Android平台的[2]。因此, Android平台的恶意程序检测是信息安全领域的一个重要研究方向。

    近年来, 研究者将机器学习算法应用于Android恶意软件检测领域[3]。基于机器学习技术的恶意软件检测方法根据提取的特征和训练数据, 得到有效的分类模型, 以区分恶意软件和非恶意软件。文献[4]提出一种基于朴素贝叶斯的Android软件恶意行为检测策略, 通过判别软件是否滥用权限、是否存在敏感权限组合等作为分类特征属性, 实现了对恶意软件的检测。然而, 上述研究只对Android的权限信息进行检测分析, 检测范围不够全面[5]。文献[6]对软件涉及的危险 API 调用和权限信息进行组合作为输入特征, 并创建支持向量机 (Support Vector Machine, SVM) 分类器, 实现恶意软件的检测识别。文献[7]通过反编译实验样本提取权限信息及API调用序列作为特征, 采用kNN与SVM分类器实现对恶意软件的检测, 但检测精度较低。文献[8]提出手机端与服务器端相结合的恶意软件检测方法。文献[9]提出一种三层混合集成检测模型SAMADroid, 用于检测Android平台上的恶意应用。然而, 上述方法的复杂度较高。

    针对恶意软件检测方法精度低、复杂度高的问题, 本文提出一种基于特征占比差的恶意软件检测方法, 以有效识别恶意软件。提取Android软件的权限信息及硬件组件信息, 并将其作为静态特征。在此基础上, 给出基于特征属性占比差的特征筛选算法, 去掉冗余及无关特征, 保留非恶意软件和恶意软件辨识度高的特征。

    1 恶意软件特征的构建

    本文提出的恶意软件检测方法框架如图1所示, 该方法包含特征提取、特征优化及分类检测3个模块。特征提取模块主要利用Google提供的Apktool工具[10]反编译实验样本, 得到AndroidManifest.xml配置文件, 并对配置文件中的属性标签进行解析, 提取权限信息及硬件组件信息作为静态特征, 从而构造特征向量集合。特征优化模块主要解决特征冗余问题, 在分析实验样本特征占比差值的基础上, 提出一种新的特征优化算法, 以选取类别区分能力强的特征子集。分类检测模块针对随机森林算法无法区分强弱分类器的缺陷, 提出基于加权的随机森林算法。

    1.1 特征提取

    特征提取模块通过逆向处理, 提取APK文件中的权限及硬件组件信息作为特征, 利用量化特征构造特征向量, 将全部Android软件样本的特征向量合并为数据集。具体步骤描述如下:

    步骤1 运用Google提供的Apktool工具反编译收集的Android实验样本, 获取配置文件AndroidMani fest.xml。

    步骤2 编写Python程序读取AndroidManifest.xml文件中的权限及硬件组件信息, 并形成初始特征向量集合, 其对应的标签分别为<uses-permission>和<uses-feature>, Android自定义的权限信息只属于特定样本, 因此不做统计。

    步骤3 对步骤2提取的初始特征向量集合进行去重处理, 去掉重复的特征, 形成新的特征集合FS。

    步骤4 针对每个实验样本判别其是否包含FS集合中对应的元素, 如果APK中含有该属性, 则特征向量对应元素用1表示, 否则对应元素用0表示, 遍历所有实验样本组合成特征向量集合FVS, 然后在每个样本对应的特征向量末位加一个标志位, 0表示非恶意样本, 1表示恶意样本。

    1.2 特征优化

    为最大限度减少冗余特征, 提高分类检测效果, 对特征进行筛选。本文在分析恶意软件和非恶意软件样本特征占比基础上, 提出基于特征占比差的特征筛选算法。针对恶意样本和非恶意样本, 统计使用频率较高的标签项<uses-permission>和<uses-feature>的属性值信息。在<uses-permission>标签项中, 部分特征在恶意样本和非恶意样本中的出现比例 (占比) 如图2所示。在<uses-feature>的属性值信息中, 部分特征在恶意样本和非恶意样本中出现的比例 (占比) 如图3所示。对于<uses-permission>标签, 无论是恶意软件还是非恶意软件, 均使用网络相关的权限来访问网络, 如INTERNET、ACCESS_NETWORK _STATE等属性值, 但恶意软件会频繁地使用短信相关权限, 如SEND_SMS、RECEIVE_SMS、READ_ SMS等。由<uses-permission>标签的各个特征的占比分析可知, 网络相关的权限在恶意软件中出现的频率和在非恶意软件中出现的频率差异不大, 对恶意软件的检测贡献较小, 而短信相关的权限在恶意软件出现的频率明显高于在非恶意软件中出现的频率, 这部分权限对恶意软件的检测贡献较大。同样, 对<uses-feature>的属性进行占比分析, 可以看出, 恶意软件会更加频繁使用android.hardware.telephony、android.hardware.wifi等属性。

    为衡量各个特征对区分恶意软件和非恶意软件的贡献, 本文采用占比差来描述特征的区分能力和贡献的大小。占比差计算方法如式 (1) 所示。

    Δj=|bj-mj|, j=1, 2, ⋯, t          (1)

    其中, Δj为特征j在恶意和非恶意软件中占有率的差值, bj为特征j在非恶意软件中的占有率, mj为特征j在恶意软件中的占有率。

    bj=∑ibijΝ, j=1, 2, ⋯, t          (2) mj=∑imijΜ, j=1, 2, ⋯, t          (3)

    其中, t表示特征的个数, N为非恶意样本的数量, M为恶意样本的数量。bij和mij的计算公式如下:

    bij={1, 非恶意样本i包含特征j0, 非恶意样本i不包含特征j          (4) mij={1, 恶意样本i包含特征j0, 恶意样本i不包含特征j          (5)

    对于任意属性j, 其占比差Δj越大, 说明该属性用于恶意软件检测的贡献越大, 反之, 贡献越小。对所有特征, 按其占比差从大到小排序, 占比差越大说明特征的区分度越高, 特征应该被保留。特征筛选具体步骤描述如下:

    步骤1 计算每个特征属性在恶意样本和非恶意样本中的使用率bj和mj。

    步骤2 计算每个特征属性特征的属性占比差值Δj。

    步骤3 对Δj按从大到小排序, 排在前面的特征, 其特征属性越重要。

    1.3 特征子集评价

    为客观评价特征子集的优劣, 本文选用分类精度、真正率及假正率作为特征子集的衡量指标, 并定义4个基本指标量表示分类结果。其中, TP表示恶意样本且预测为恶意样本, TN表示非恶意样本且预测为非恶意样本, FP表示非恶意样本且预测为恶意样本, FN表示恶意样本且预测为非恶意样本。

    2 恶意软件分类方法

    随机森林 (Random Forest, RF) 算法是一种集成学习分类器, 利用相互独立决策树对实验样本进行训练与预测, 每个决策树根据输入测试样本集产生分类结果, 测试样本最终类别通过各个独立决策树投票决定[11,12]。RF算法的最终分类结果由独立决策树通过投票产生, 是一种集成学习算法, 较单个分类器相比具有更好的分类结果和泛化能力, 且决策树在训练过程中相互独立, 训练速度快[13], 但忽略了决策树之间的差异。

    2.1 随机森林算法

    传统RF算法具体步骤描述如下:

    步骤1 在原始训练集S中, 使用bagging技术进行有放回地抽取N个样本构成训练子集。

    步骤2 随机从M个特征属性中选取m个属性, 依据决策树生成算法选择最优属性进行节点分裂, 分裂过程中不进行剪枝处理, 生成决策树。

    步骤3 重复k次步骤1和步骤2, 共生成k棵决策树, 并构成随机森林。

    步骤4 利用步骤3生成的随机森林对测试样本进行分类, 样本分类结果由k棵决策树以投票 (少数服从多数原则) 的方式产生。

    2.2 改进的随机森林算法

    RF算法最终分类结果由决策树通过简单投票决定, 每颗决策树的权值相同, 忽略了决策树之间的强弱差异, 不能最大化随机森林分类效果。因此, 基于决策树分类器差异, 本文提出一种基于权值的随机森林 (Weighted Random Forest, WRF) 方法。

    由于在决策树生成过程中采取有放回随机抽样策略, 因此训练集中会有部分样本未被抽取, 且单个样本未被抽中的概率为ρ= (1-1/N) N。当N→∞时, ρ≈0.37, 即每个样本集中未被抽取的样本数量约37%, 未被抽取的样本称为out-of-bag (OOB) 样本[14]。设t为当前决策树, OOB样本记为Bt, t对Bt的每个样本做分类检测, Bt中的各样本真实类别已知, 与分类检测结果比较, 可得Bt样本分类正确的数量Btr。设CRt为决策树t对Bt分类的正确率, 其表达式如式 (9) 所示, CRt越大, 决策树t的分类能力越强, 反之分类能力越差。

    CRt=BtrBt          (9)

    决策树分类能力越强, 该决策树在投票过程中应该具有更高的权值, 反之权值越低。决策树的权值计算公式为:

    wt=CRt1Τ∑i=1ΤCRi          (10)

    3 实验结果与分析

    3.1 实验设置

    本文共收集1 200个恶意软件和1 200个非恶意软件样本作为实验数据集, 恶意软件来自于Drebin[15]网站, 包含19个不同家族的恶意软件样本, 非恶意软件来源于第三方应用市场, 通过网络爬虫获得。本文实验在主机Intel (R) Core (TM) i7-4771MCPU@3.50 GHz, 内存为16 GB, 系统为Win10的台式计算机上运行, 实验软件工具为Python3.6。

    4 结束语

    为有效识别Android平台恶意软件, 本文提出一种基于特征占比差与加权随机森林的Android恶意软件检测方法。选择permission信息及hardware信息作为特征属性, 分析样本特征属性使用比率差值, 给出FRD特征筛选算法。在此基础上, 对随机森林决策树无法区分强弱分类器的缺陷进行改进, 以提高分类检测的准确率。实验结果表明, 该方法的分类准确率较高。但本文只对配置文件中的权限信息和硬件特性进行检测分析, 下一步将研究API调用、Intent信息和组件使用对恶意软件的检测效果。
