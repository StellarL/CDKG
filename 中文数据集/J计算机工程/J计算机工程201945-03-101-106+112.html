    摘    要：

    在服务功能链部署过程中, 链路可靠性较低, 冗余资源开销较高。为此, 采用两阶段部署方式, 提出一种基于多路径的链路备份方案。在虚拟网络功能部署阶段, 松弛节点部署约束, 将一些虚拟网络功能部署在同一物理设施节点上, 以减少带宽资源开销。在虚拟链路部署阶段, 基于路径分割进行多路径映射, 以保证链路可靠性并降低备份资源开销。仿真结果表明, 该方案在请求接受率和备份带宽开销增益等方面具有良好性能。

    关键词：

    网络功能虚拟化; 服务功能链; 可靠性; 多路径; 备份;

    Deployment Scheme of Reliable Service Function Chain Based on Multipath

    QIU Hang YOU Wei TANG Hongbo WANG Chen NIU Ben

    National Digital Switching System Engineering and Technological R& D Center

    Abstract：

    In the deployment process of Service Function Chain (SFC) , the link reliability is low and the redundant resource overhead is high.To solve this problem, a link backup scheme based on multipath is proposed by using two-stage deployment.In the virtual network function deployment phase, the constraints of node deployment are relaxed and some virtual network functions are deployed on the same physical facility node to reduce the bandwidth resource overhead.In the virtual link deployment phase, multipath mapping based on path partitioning is implemented to ensure link reliability and reduce backup resource overhead.Simulation results show that the proposed scheme has good performance in request acceptance rate and backup bandwidth overhead gain.

    Keyword：

    Network Function Virtualization (NFV) ; Service Function Chain (SFC) ; reliability; multipath; backup;

    0 概述

    随着网络功能虚拟化 (Network Function Virtualization, NFV) 和软件定义网络 (Software Define Network, SDN) 技术的发展, 网络变得更加开放灵活。NFV可以将传统专用硬件设备虚拟化为虚拟网络功能 (Virtualized Network Function, VNF) , 并将其运行在通用设备上, 以实现网络的高效性、灵活性和扩展性, 并有效降低运营商对专用设备的依赖性[1]。网络服务请求通常由一定顺序的虚拟网络功能组成, 也称为服务功能链 (Service Function Chain, SFC, 以下简称“服务链”) 请求, 结合SDN可大幅增强网络的灵活性和扩展性, 促进基础设施资源的高效利用, 降低新业务的部署时间和成本开销。特别是基于NFV实现的网络切片技术, 在未来的5G中将成为应对多样化场景的关键技术, 从而满足差异化的性能指标需求。

    IMT-2020将增强移动宽带、海量机器通信和高可靠低时延通信作为5G的主要应用场景[2]。5G移动通信高可靠场景对可靠性的要求较高, 需要达到99.99%以上的可靠性保障。设计一种服务链映射方案, 在节点或链路失效的情况下保证服务, 称为虚拟网络生存性问题[3]。

    底层网络的单个故障可能引起多条服务链失效, 从而造成网络服务可用性和质量下降。在虚拟网络中, 文献[4,5]研究生存性问题, 但他们只保证节点的生存性, 而对虚拟网络的连接性和带宽需求关注较少, 即其成果无法直接用于解决链路的可靠性问题。链路的可靠性问题一般通过分配冗余资源 (即备份) 来完成, 这会对服务链的部署提出新的挑战。首先, 故障的特征和修复时间难以预测[6,7], 传统的专属备份方案 (即1+1备份) 不适用, 原因是无故障发生时该资源会闲置[8]。为降低资源消耗, 文献[9]提出共享备份方案, 但发生故障时其难以保证完整的带宽恢复。在保障可靠性的前提下, 如何最小化冗余资源成为亟需解决的关键性问题。文献[10]在虚拟网映射中提出路径分割策略, 用于提高服务接受率和负载平衡, 但其未考虑链路的可靠性。文献[11]提出基于路径分割的多路径策略以缓解链路故障带来的影响, 但该策略仅解决了单链路故障, 而且未进行可靠性分析。

    针对上述服务链部署中存在的可靠性问题, 本文基于路径分割进行多路径映射, 并将可靠性问题建模成一个二次规划问题, 在满足服务需求可靠性的前提下, 以资源开销最小化为目标, 提出一种两阶段的启发式算法以进行服务链部署。

    1 网络模型与问题描述

    1.1 物理网络和服务链请求

    将物理网络定义为带权重的无向图GS (NS, LS) , 其中, NS和LS分别表示物理节点集合和物理链路集合。每个物理节点u∈NS和物理链路 (u, v) ∈LS都分别关联其剩余CPU和带宽资源属性, 且分别用RuΝ和Ru, vL表示, 其他每条物理链路关联可靠性属性Au, vL=ΜΤBFΜΤBF+ΜΤΤR。其中, MTBF和MTTR分别表示承载VNFj物理节点的平均故障间隔时间和平均故障修复时间。

    将服务链请求定义为一个带权重的有向图SFC (NV, LV) , 其中, NV和LV分别表示虚拟节点集合和虚拟链路集合。本文使用CiΝ和Ci, jL分别代表虚拟节点i∈NV和虚拟链路 (i, j) ∈LV的资源需求, 并且每条虚拟链路包含可靠性指标Ai, jL。

    服务链映射定义为从SFC到GS的一个子图的映射, 该映射需要满足所有的资源约束CiΝ和Ci, jL以及链路可靠性约束Ai, jL。服务链部署可以分解为2个部分:

    1) 节点映射fN:NV■NS, 将每个虚拟节点映射到一个物理节点上。来自同一个服务链的2个虚拟节点在满足资源约束的情况下, 可以映射到同一个物理节点上, 但每一个虚拟节点的承载物理节点需要有足够的CPU资源。

    2) 链路映射fP:LV■P, 将每个虚拟链路映射到一组底层路径集合上, 包含主链路和备份链路。该集合中的底层路径承载网络通信时相应虚拟链路的数据流, 且这些底层路径连接的2个物理节点承载了被映射的虚拟链路两端的虚拟节点。每一个虚拟链路都要被映射到具有足够剩余带宽资源的底层路径上。在路径分割的情况下, 备份链路所需带宽和主链路带宽一致。为最小化物理链路故障的影响, 备份链路和主链路必须是完全不相交的, 并且构成一个满足可靠性需求的底层路径集合。

    1.2 链路故障

    由于光纤损坏、维护和错误配置等多种原因, 物理链路无法一直保持正常工作, 这些故障将会影响所有映射在该物理链路上的虚拟链路, 进而影响整个服务链的正常工作。例如, 图1 (b) 中的底层链路{ (BG) }故障将会引起虚拟链路{ (ab) , (bc) }不可用, 从而导致网络服务中断。此外, 文献[6,12]研究表明链路故障比节点故障更加频发。

    2 可靠性服务链部署方案

    服务链部署问题已经被证明是NP-hard问题[13], 本文将可靠性服务链部署分为2个阶段:

    1) 采用粒子群算法, 以最小化物理资源开销为目标进行虚拟节点部署, 为下一步虚拟链路部署做准备。

    2) 在路径分割的基础上采用多路径机制, 设计启发式算法为虚拟链路部署选择最优物理路径。

    2.1 基于粒子群算法的虚拟节点映射

    粒子群算法是一种基于群体智能的全局随机搜索算法, 与同类优化算法相比, 其具有执行速度快、效率高等优点。在该算法中, 每个粒子以一定的速度在解空间中运动, 并向自身历史最佳位置和全局历史最佳位置聚集, 实现对候选解的进化。离散粒子群优化 (Discrete Particle Swarm Optimization, DPSO) 算法对虚拟节点的部署具有较好的拟合性, 本文从全局角度考虑资源开销问题, 基于DPSO算法进行虚拟节点映射[14]。

    相较于虚拟网节点映射问题, 为提高资源利用率, 本文在映射过程中松弛虚拟节点映射条件。在资源充足的情况下, 可让多个虚拟节点部署在一个物理节点上, 因此, 同一物理节点内的虚拟网络功能之间的通信转化为内部通信, 以减少物理链路的开销并降低传输时延。DPSO算法将目标函数式 (1) 作为适应度函数f (X) , 其中, 位置向量X表示可能的映射方案。在适应度函数的计算过程中, 首先根据式 (2) 检查节点能力约束, 然后根据式 (3) ～式 (7) 采用单条最短路径算法检查当前映射方案的可行性。如果该映射可行, 则f (X) 的值表示服务链节点映射的开销;若不可行, 则f (X) 的值被设置为+∞。

    在虚拟网络功能部署完成后, 根据数据流方向按顺序进行虚拟链路映射, 本文仅需部署位于不同物理节点上虚拟网络功能之间的传输路径, 即可从总体上有效降低网络的带宽资源开销。

    2.2 基于多路径的虚拟链路映射

    2.2.1 路径分割机制

    路径分割是一种路由策略, 其允许将一个数据流分割并通过多条路径传输。多种路径分割技术 (如等价路由和多路传输控制协议) 已经被应用到IP层和TCP层。文献[10]在虚拟网络映射中引进路径分割技术, 其将一条虚拟链路映射到多条底层路径上, 并指出多路径映射策略可以有效提高请求接受率, 调整网络负载均衡。多路径映射策略可以将故障链路上受影响的流量转移到其他可选链路, 能有效降低由链路故障带来的影响。在最差情况下, 它可以收集链路故障期间其他映射链路的流量碎片, 应用一些网络编码技术进行信息恢复。在图1中, 虚拟链路{ (ab) }映射到{ (AB) , (AGB) }, 如果底层链路{ (BG) }发生故障, 虚拟链路{ (ab) }可通过底层链路{ (AB) }保留部分链路传输的信息。

    虚拟链路的可靠性由其映射在底层路径集合的可靠性决定。在多路径映射机制下, 虚拟链路的可靠性由映射底层路径集合中可用路径的数量及其可靠性计算, 为简化表达, 本文采用统一的底层路径可靠性, 即:

    多路径传输同时会带来其他网络资源开销, 如路由表的扩展和更新, 源节点的数据分割和目的节点的数据聚合开销, 甚至不同传输链路的时延也会引起一些额外的开销。过多的路径分割将会引起一些不合理的开销, 从而降低备份带来的收益。文献[11]研究表明较为合理的路径分割条数为2条～4条, 本文基于此结论进行算法设计, 将n取值为2～4。在此基础上, 结合可靠性分析, 在实际链路部署过程中以全局资源优化为目标进而确定n的取值。

    2.2.2 多链路映射算法

    在路径分割的情况下, 一对节点之间映射的多路径完全不相交且具有相同的容量。这样一方面可以大幅降低物理链路故障对同一虚拟链路的影响, 另一方面可以有效调节网络的负载均衡。在这些约束条件下, 如果虚拟链路 (i, j) 两端虚拟网络功能映射到物理节点u和v上, 全局最优化模型MP-RSFCM将约减为解决虚拟链路映射问题的局部最优化模型LM-MP。LM-MP模型描述如下:

    图3所示为不同备份算法在相同服务请求强度下的请求接受率。从图3可以看出, 随着服务请求的陆续到达, 不同备份方式的请求接受率先降低后上升, 但下降过程中不同方案的请求接受率下降幅度有所不同。在相同的底层网络资源环境和服务到达强度下, 本文备份算法MP-RSFCM的性能下降幅度最小。FBS不考虑底层资源的状态, 单一采用两倍资源开销, 在消耗较多前期资源后, 请求接受率快速下降。SBS相对FBS资源开销较少, 但为了获取较高的可靠性, 备份资源开销相对提高, 故其服务接受率也明显下降。本文MP-RSFCM算法在保证可靠性的基础上能保持较高的请求接受率, 原因是基于多路径的服务链映射方法支持主路径的路径切分, 备份方案资源开销较少, 且路径切分自身也有利于提高请求接受率。

    4 结束语

    本文针对传输可靠性要求较高的服务链部署, 在路径分割的基础上引入多路径映射思想, 提出一种两阶段的服务链部署方案。该方案包括常用的虚拟网络功能映射算法和基于多路径的虚拟链路映射算法, 虚拟链路部署阶段采用的路径分割机制可有效降低备份资源的开销并保障虚拟链路的可靠性。仿真结果表明, 该方案能够在提高请求接受率和收益成本比的同时, 有效降低服务链的冗余带宽消耗。下一步将在NFV实验环境中扩大应用与验证的规模, 并在SDN中开发支持多路径的原型系统。
