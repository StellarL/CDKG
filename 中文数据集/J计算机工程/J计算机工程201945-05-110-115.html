    摘    要：

    为降低密集网络系统层内干扰并提高系统容量, 提出一种基于分簇的用户资源分配方案。在分簇阶段, 根据小基站到簇头的最大距离方差与簇头间最小距离对初始分簇结果进行消除、合并及分裂操作, 实现网络拓扑变化情况下的自适应分簇。在资源分配阶段, 通过用户剩余资源有效利用及资源块微调, 保障用户业务速率。仿真结果表明, 与现有用户资源分配方案相比, 该方案提高了系统容量及用户公平性。

    关键词：

    密集网络; 基站分簇; 层内干扰; 资源分配; 系统容量;

    Clustering-based User Resource Allocation Scheme for Small Base Station

    XIAN Yongju ZHENG Jian XU Changbiao

    School of Communication and Information Engineering, Chongqing University of Posts and Telecommunications School of Optoelectronics Engineering, Chongqing University of Posts and Telecommunications

    Abstract：

    In order to reduce the intra-layer interference in dense network systems and increase the system capacity, a clustering-based user resource allocation scheme is proposed.In the clustering stage, the initial clustering results are eliminated, merged and split according to the maximum distance variance from Small Base Station (SBS) to cluster-heads and the minimum distance between cluster-heads so as to realize adaptive clustering under the changing network topology.In the resource allocation stage, the user service rate is guaranteed by making use of user remaining resources and the fine adjustment of resource blocks.Simulation results show that compared with the existing user resource allocation scheme, the scheme improves system capacity and user fairness.

    Keyword：

    dense network; base station clustering; intra-layer interference; resource allocation; system capacity;

    0 概述

    随着移动互联网的发展和智能终端设备的普及, 移动业务呈现快速增长趋势。从业务场景来看, 60%的移动语音业务、70%的数据业务发生在室内和热点区域[1,2]。4G移动网络在处理用户接入速率、接入质量、高速移动性支持、吞吐量提升等方面具有较大的性能优势, 但对于越来越高的用户体验速率、千亿设备连接数密度、毫秒级端到端时延, 现有技术仍无法满足[3]。在下一代通信技术的研究中, 针对上述期望业界提出了超密集网络技术, 即在传统宏蜂窝覆盖下密集部署低功率节点, 通过缩短用户与基站之间的距离、合理分配资源等, 提高系统容量及系统频谱利用率, 保证用户服务质量[4,5]。本文在上述研究基础上, 提出一种自适应分簇方案。当网络结构发生变化时, 小基站 (Small Base Station, SBS) 能根据拓扑变化进行自适应分簇, 并在分簇的基础上提出两阶段资源分配方案。

    1 研究现状

    密集网络的部署给传统网络带来了一些新的问题。由于小基站部署在宏基站 (Macro Cell Base Station, MBS) 的覆盖范围内, 当小基站与宏基站工作在相同频段时, 宏基站可能会接收到小基站用户发来的信息引起层间干扰, 同样小基站也可能会接收到宏基站用户发来的信息引起层间干扰。同时, 小基站可能会接收到来自相邻小基站的信息引起层内干扰。如何降低乃至抑制干扰是业内十分关注的问题, 因此需合理使用有限的频谱资源使系统获得更高的吞吐量。

    国内外的研究学者对密集网络进行了一系列研究。文献[6]根据不同类型的业务占比, 对宏基站和小基站分别调用比例公平调度算法和最大加权时延有限算法进行资源分配, 经过仿真验证了业务时延得到改善, 但该方案未考虑频繁切换对用户服务质量的影响。文献[7]提出基于正交频分多址 (Orthogonal Frequency Division Multiple Access, OFDMA) 的资源分配方案。该方案首先利用连续凸的几何平均近似方法进行最佳功率分配, 然后采用最佳功率值计算信道的导出成功率, 根据成功率进行资源分配。仿真结果表明, 该方案提升了分配成功率和系统容量, 但导出成功率计算依据用户服务质量 (Quality of Service, QoS) 和信干噪比 (Signal Interference Noise Ratio, SINR) , 因此方案中假设信道状态信息在理想状态下全部反馈。文献[8]提出一种基于干扰管理的宏微蜂窝资源分配方案。该方案以最大化宏基站速率为目标, 以小基站最大发射功率和宏用户设备最大可接受干扰为限制条件, 利用低复杂度的凸逼近算法进行求解, 仿真结果表明该方案减少了跨层信道干扰并提高了系统吞吐量。文献[9]提出基于OFDMA的非独占下行链路资源分配方案。该方案以小基站和宏基站系统总容量最大化为目标, 设立共享频段供宏基站和小基站使用, 仿真结果表明宏基站和小基站系统容量均有提升。但随着用户数和小基站数目的增加, 使用共享频段使得系统层间、层内干扰加剧。文献[10]提出femtocell网络中联合信道分配和功率控制的资源分配方案。该方案将可用频段分为多个部分, 然后对属于该频谱分段中的用户进行资源分配, 可在一定程度上减少层间干扰, 但降低频谱资源利用率。上述文献在一定程度上减少了系统干扰及业务时延, 提高了系统吞吐量, 但是未考虑用户公平性问题。

    目前, 传统网络采用宏基站集中控制式下的资源分配方式, 很难完全应用在密集网络中, 因为密集网络中的信令开销会随着小基站数目以二次方的趋势逐渐增长, 所以需要对小基站进行分簇。通过分簇处理, 网络拓扑结构得到降级, 系统处理时延缩短、宽带空间复用率得到提升[11,12], 因此本文提出一种基于分簇的资源分配方案。分簇方案大致可以分为2种:簇内复用和簇间复用。簇内复用方案是分簇之间分配正交资源, 簇内用户进行复用。簇间复用方案是分簇内用户分配正交资源, 簇间进行资源复用。簇内复用方案是将干扰较小的基站分在一簇, 随着小基站的增加, 分簇方案可能会采用增加新的分簇或增加簇内小基站的方式。当增加分簇时, 频谱会被划分成更小的分段, 分到每个分簇的可用资源会变少, 造成用户通信质量下降。当增加簇内的小基站时, 分簇内的用户会变多。由于簇内进行资源复用, 因此随着人数增多, 簇内干扰会加大, 最终影响用户通信质量。对于簇间复用方案, 是将干扰较大的小基站分在一簇, 使得簇间干扰最小化。簇内分配正交资源, 而簇间复用整个可用频段。随着小基站逐渐增多, 簇间复用方案可以增加新的分簇, 该方案既避免了簇内干扰, 又保证了资源的有效利用[13]。

    文献[14]中的分簇方案先将整个网络抽象成无向加权干扰图, 小基站之间的干扰值映射成权重, 然后将干扰值较小的小基站分在一簇中。当采用簇内复用时, 随着用户增多, 系统内干扰也将增加。文献[15]提出一种基于图论着色的资源分配方案, 先利用用户反馈信息构建系统干扰图并将小基站抽象成点, 再使用着色法对点进行着色, 将颜色相同的小基站分在一簇中。文献[16]提出一种基于图论聚类的资源分配方案 (HCFM) , 将小基站分簇问题转换成图论的K-CUT问题, 采用簇内资源复用, 簇间正交分配的复用方式。虽然该方案有效抑制了层间干扰, 但该分簇方法对每个分簇中的小基站个数进行了限制。文献[17]提出一种基于改进硬聚类算法的资源分配方案 (TSSG) 。该方案计算所有小基站之间的距离, 若距离大于阈值, 则对应小基站成为下一个簇头, 然后将小基站分配到距离最近簇头所在的分簇中。同时, 上述文献主要研究的是小基站分簇问题, 但没有考虑小基站灵活部署导致的网络拓扑结构动态变化问题。

    结合上述研究, 本文综合考虑系统干扰和网络拓扑动态变化, 基于优化理论和用户公平性问题, 提出簇间复用的资源分配方案。该方案包含以下2个部分:1) 对小基站进行分簇。为应对层内干扰和网络拓扑动态变化, 本文采用簇内正交分配, 簇间复用的频谱复用方式。根据分簇数目与小基站个数的关系给出分簇数目范围并按照距离远近进行初始分簇, 在此基础上利用输入参数对分簇进行消除、分裂或合并操作。该方案能够在网络拓扑动态变化时, 达到自适应分簇。2) 资源分配。针对未获得最优资源块且用户速率未得到满足的用户进行补偿分配或资源块微调, 提高资源利用率, 保障用户业务速率, 从而提升用户公平性。

    C2:∑Sn∈Ci∑k=1ΝREpkSn⋅∂k, iSn≤ΡΤ          (10)

    C3:∑Sn∈Ci∑k=1ΝRE∂k, iSn≤Ν, ∀k          (11)

    C4:∑Sn∈Ci∂k, iSn=1, ∀i          (12)

    C5:∂k, iSn∈{0, 1}, ∀Sn∈Ci, ∀Rk (13)

    其中, C1～C5为限制条件, C1表示用户业务最低速率, C2表示最大系统总功率, C3表示可用资源块数目, C4表示簇中资源块正交分配, C5表示∂k, iSn取值限制, ∂k, iSn=1表示小基站Sn服务的用户i分配到资源块k。

    将式 (8) ～式 (13) 改写成不等式约束下的拉格朗日函数并求解:

    L (∂k, iSn, λ1, λ2, λ3, μ) =   ∑i=1ΝCi∑Sn∈Ci∑k=1ΝREΔB⋅lb (1+∂k, iSnγk, iSn) +   λ1 (∑k=1ΝREΔB⋅lb (1+∂k, iSnγk, iSn) -Rmin) +   λ2 (ΡΤ-∑i=1ΝCi∑Sn∈Ci∑k=1ΝREpkSn⋅∂k, iSn) +   λ3 (Ν-∑Sn∈Ci∑k=1ΝRE∂k, iSn) +μ (∑Sn∈Ci∂k, iSn-1)           (14)

    针对含有不等式约束的优化问题, 采用KKT条件进行求解:

    ∂L (∂k, iSn, λ1, λ2, λ3, μ) ∂ (∂k, iSn) =0          (15)

    约束为:

    λ1 (∑k=1ΝREΔB⋅lb (1+∂k, iSnγk, iSn) =0          (16) λ2 (ΡΤ-∑i=1ΝCi∑Sn∈Ci∑k=1ΝREpkSn⋅∂k, iSn) =0          (17) λ3 (Ν-∑Sn∈Ci∑k=1ΝRE∂k, iSn) =0          (18) h (∂k, iSn) =∑Sn∈Ci∂k, iSn-1=0          (19) λ1≥0, λ2≥0, λ3≥0          (20)

    得到KKT条件下的最优解:

    ∂k, iSn=B⋅ (1+λ1) ln2 (λ2pkSn+λ3-u) -1γk, iSn          (21)

    本文设定功率均分, 因此pkSn可以看作定值, ∂k, iSn值随着变量γk, iSn的增大而增大或减小而减小, 即用户分配信干噪比较大的资源块的吞吐量较大。

    基本的资源调度算法有最大载干比算法、轮询算法、比例公平算法。其中比例公平算法兼顾系统容量和用户公平性, 当多个用户的最优资源块相同时, 只有一个用户可以获得最优资源块, 但其余未获得最优资源块的用户的业务速率也需要得到保障, 因此本文在比例公平算法的基础上引入加权因子w=Rave (t) -Rreq (t) , 提出基于补偿分配的用户资源分配方案, 具体步骤如下:

    步骤1 将所有用户按照信道质量高低进行排序, 用户依次选取最优资源块, 即用户所对应的γk, iSn值的最大资源块。

    步骤2 如果用户对应资源块被别的用户选取, 则该用户从剩余资源块中选取其中γk, iSn最大的资源块并对用户进行记录。

    步骤3 当多个用户的最优资源块相同时, 计算用户在资源块上产生的比特数, 将获得比特数较大的用户进行优先分配。

    步骤4 对于被记录的用户进行补偿分配, 从被记录的用户中选取w<0的用户, 按照式 (22) 对用户进行调度值计算, 用户按照调度值的高低进行排序, 从剩余资源块中依次选取最优资源块。

    y=Ri (t) Τi (t) ⋅1w=Ri (t) Τi (t) ⋅1 (Rave (t) -Rreq (t) )           (22)

    其中, Ri (t) 表示用户瞬时速率, Ti (t) 表示用户i到时刻t的平均吞吐量, Rave (t) 表示用户平均传输速率估值, Rreq (t) 表示用户传输请求速率。

    5.2 结果分析

    图3显示了系统吞吐量与小基站数目之间的对应关系, 系统容量根据香农公式进行计算。HCFM方案采用簇内复用簇间正交的复用方式, 当分簇增多时, 每个分簇的可用资源块会减小。由于簇内复用资源, 当簇内用户增加时, 系统干扰将增加, 因此该方案吞吐量较低, 增幅有限。TSSG方案采用的是簇内正交簇间复用的复用方式, 当簇数增多时, 每个分簇可以使用的资源数目不变, 频谱利用率增加。该方案按照信道质量进行贪婪分配, 系统容量高于HCFM方案。本文方案采用自适应分簇方案, 针对网络动态变化情况对分簇进行合并或分裂, 提升频谱资源空间复用率。本文方案通过剩余资源块利用及资源块微调, 提升了频谱资源利用率, 因此系统吞吐量高于TSSG方案。

    其中, |ΝU|表示集合U中的元素个数, J值越大表示公平性越高, 数值越小表示公平性越低。HCFM方案未考虑用户公平性问题, 因此公平性较低。TSSG方案优先考虑了系统容量, 按照信道质量排序分配资源块, 牺牲了部分用户的公平性以换取容量的提升。本文方案中增加了用户最低业务速率, 并对未获得最优资源块的用户进行资源补偿分配或资源块微调, 在增加系统容量的同时可保证用户业务速率, 提升用户公平性, 因此本文方案的公平性高于TSSG方案和HCFM方案。

    6 结束语

    本文提出一种基于分簇的资源分配方案, 按照距离远近完成初始分簇, 通过判断输入参数对初始分簇进行消除、分裂或合并操作, 实现自适应分簇。记录未获得最优资源块的用户并重新计算其调度值进行补偿分配及资源块微调。实验结果表明, 相比现有资源分配方案, 改进方案兼顾了系统容量和用户公平性。下一步将对用户移动性管理进行研究, 通过小基站密集部署实现用户与小基站无缝连接, 并将移动性管理与资源分配相结合, 提升系统性能。
