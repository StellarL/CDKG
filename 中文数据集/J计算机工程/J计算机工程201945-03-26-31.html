    摘    要：

    传统TF-IDF算法仅从词频与逆向文档频率的角度计算关键词与文档之间的相关性权重, 忽略了用户兴趣对权重计算的影响。为此, 以满足用户信息检索目的为研究背景, 提出一种基于日志关联的相关性权重算法。从面向用户相关性的角度出发, 通过分析用户的搜索日志建立用户兴趣模型, 并结合分布式计算的思想, 运用MapReduce编程框架实现计算任务的并行化处理。实验结果表明, 该算法在处理海量数据时, 不仅能够提高算法效率, 而且可以根据用户的历史检索记录动态地改变检索词的权重, 提升用户与系统的交互能力。

    关键词：

    分布式计算; TF-IDF算法; 日志; 兴趣模型; 信息检索;

    Design and Implementation of Correlation Weight Algorithm Based on Hadoop Platform

    GAO Jun HUANG Xiance

    College of Information Engineering, Shanghai Maritime University

    Abstract：

    The traditional TF-IDF algorithm calculates the correlation weights between keywords and documents only by using the perspective of word frequency and reverse document frequency, which ignoes the influence of user interest on weight calculation.In order to meet the purpose of user information retrieval, a correlation weight algorithm based on journal association is proposed.From the perspective of user-oriented comelation, the user interest model is built by analyzing the user's search journal, and combined with the idea of distributed computing, the MapReduce programming framework is used to realize the parallel processing of computing tasks.Experimental results show that it can not only improve the efficiency of the algorithm when dealing with massive data, but also dynamically change the weight of retrieval word according to the user's historical retrieval records, so as to enhance the interaction ability between users and the system.

    Keyword：

    distributed computing; TF-IDF algorithm; journal; interest model; information retrieval;

    0 概述

    随着网络中数据量的激增, 当今社会已经进入到大数据时代, 人们在学习和工作中都需要处理大量的数据信息。但是, 当数据量超过一定的阈值后, 在这些海量的数据中有用数据所占的比例往往很少, 大量的无用信息会严重影响人们学习和工作的效率, 这就是学术上常说的信息过载问题[1,2]。信息检索[3]技术是解决信息过载问题的常用方法之一, 该技术的核心是相关性权重的计算方式。

    Saltond在1973年提出了TF-IDF (Term Frequency-Inverse Document Frequency) 算法[4], 该算法是一种可以用于信息检索和文本挖掘的常用加权技术[5], 可以从统计学的角度来衡量一个字或词与文档集合中一份文档的相关性程度。TF-IDF算法的核心思想是关键词的重要性与其在文档中的出现次数呈正相关, 与其在文档集合中出现的频率呈反相关, 这一思想符合香农信息论。目前, TF-IDF算法已经被广泛应用于各种信息检索系统[6], 用来评估用户输入的查询语句与文档集合中文档之间的相关性权重, 具有计算简单、易于实现等特点, 但是该算法仅仅从词频和逆向文档频率的角度作为关键词权重的度量方式, 忽略了用户的兴趣对关键词权重的影响。

    本文以TF-IDF算法在信息检索领域的应用为研究背景, 提出一种基于日志关联的相关性权重计算算法, 运用MapReduce编程框架对TF-IDF算法进行改进, 并将改进算法与TF-IDF算法在信息检索系统中进行实验对比。

    1 相关工作

    近年来, 国内外学者针对TF-IDF算法的研究较多, 文献[7]在传统TF-IDF算法上加入特征词位置作为权重计算影响因子。文献[8]对关键词出现的位置及关键词的跨度进行量化, 从而决定关键词的权值大小。文献[9]在词长上对算法进行了改进。文献[10]将信息熵和信息增益加入到TF-IDF的计算公式中。文献[11,12]通过建立关键词类别库, 根据类别修正关键词的权重。

    以上文献都是基于系统相关性的研究, 即仅从关键词的频率、位置或词长等特征对算法进行改进。这种基于系统相关性的改进存在以下问题:1) 需要反复比较关键词在文档中的分布特征, 查询语句中每一个关键词权重值的计算都需要重复地扫描文档, 显然, 这种方法大大增加了系统的I/O次数, 从而降低系统的性能;2) 从系统相关性角度进行改进, 虽然在一定程度上提高了算法的有效性, 但是忽略了用户在检索信息时往往都带有主观目的的特点, 即用户想要检索的信息与其自身的兴趣领域有很大的关系。

    针对算法效率的问题, 文献[13,14]利用MapReduce编程模型实现了分布式计算关键词相关性权重, 该方法虽然在效率上对算法做出了很大改进, 但是没有对算法的有效性进行研究。文献[15]在分布式实现TF-IDF算法的基础上从关键词分布特征的角度对算法有效性进行了改进, 但依然存在基于系统相关性角度改进的缺点。

    通过对上述研究的分析, 本文根据用户日志建立用户兴趣模型, 从用户相关性的角度提出一种基于日志关联的相关性权重计算算法, 将用户行为作为相关性权重计算的影响因子, 并将该算法通过MapReduce分布式编程模型进行实现。实验结果表明, 该算法不仅在评分效果上有了改进, 而且性能也有所提高。

    2 背景知识

    2.1 MapReduce

    MapReduce是一种开源的分布式编程模型[16], 该编程模型采用“分而治之”的思想可以对海量的数据实现并行处理。MapReduce编程模型主要是通过Map函数和Reduce函数来处理计算任务, 首先对输入数据进行分片, 以<Key1, Value1>键值对的方式输入Map函数, Map函数将输入的<Key1, Value1>键值对按照用户自定义的逻辑进行处理。然后将结果映射成<Key2, Value2>键值对。Reduce函数将Map函数输出的中间结果进行分组迭加, MapReduce作业的输入和输出一般都会保存在HDFS文件系统中, MapReduce编程模型如图1所示。

    图1 MapReduce编程模型   下载原图

    在图1中, 用户只需关心自己作业的业务逻辑的实现, 任务调度与监控都由MapReduce框架负责。MapReduce任务调度的核心是Shuffle (洗牌) , 在很多情况下, Reduce需要跨节点去取被其他节点上Map输出的结果, Shuffle的任务就是负责在跨节点取得数据时尽量减少磁盘I/O对网络带宽的消耗。

    2.2 HBase

    HBase是一种基于列存储的非关系型数据库[17], 适用于需要对大量数据进行实时读写的场景。在HBase数据库中, 表是三维的存储结构, 数据单元的值由行健、列族和时间戳共同决定, 如式 (1) 所示。

    {rowkey+conlumn=<family>+<label>+version}→value (1)

    HBase的面向列存储结构本质上是<key, value>的结构, 其中, key为{rowkey+conlumn=<family>+<label>+version}, value为cell中的值。在HBase中, 通过行健和列族可以确定一个存储单元, 每个存储单元中保存同一份数据的不同版本, 版本号根据时间戳自动分配, 这个特点使得HBase适用于需要数据重写的场景, 因此, 本文选用HBase作为用户存储用户日志的数据库。表1为HBase存储的逻辑视图。

    表1 HBase存储逻辑视图    下载原表

    由表1可以看出, HBase中的列属于特定的列族, 与关系数据库不同的是, HBase在创建表时只需定义列族, 列可以在添加数据时动态增加。另外, 值为空的列不占用存储空间, 这即为HBase存储的稀疏性, 这样既可以节省存储空间又可以提高数据访问性能。

    2.3 算法分析

    词频 (Term Frequency, TF) 表示检索词在文档中出现的频率, 目的是对检索词进行归一化处理, 以防止它偏向内容长的文档。检索词与某一特定的文档之间的相关性可以表示为:

    tfi, j=ni, j∑knk, j    (2)

    其中, ni, j是检索词在文档docj中出现的次数, 而分母是文档中所有字词出现的次数之和。TF根据频率信息给不同的关键词赋予不同的权重, 从而起到区分文档的目的, 但是单纯地以关键词的词频作为评判标准, 可能会出现因关键词均匀分布在文档集合中而无法区分的情况, 因此, 需要计算逆向文档频率 (Inverse Document Frequency, IDF) 值。

    IDF是一个词语普遍重要性的度量, 其目的是给在少数的文档中出现的关键词赋予较高的权重值, 给在较多文献中出现的关键词赋予较低的权重值, 计算公式为:

    idfi=loga|Doc||j:ti∈docj|    (3)

    其中, |Doc|表示文档集合中的文档总数, |j:ti∈docj|表示包含关键词ti的文档数目。

    式 (2) 、式 (3) 相乘即可得到计算单词与文档之间相关性权重:

    tfidfi, j=tfi, j×idfi (4)

    式 (4) 在对文档进行相关性评分时单纯地以TF值和IDF值作为影响因子, 而忽略了关键词本身的语义对权重计算的影响, 这种做法难以准确地衡量用户在检索信息时的真正意图。例如, 用户想查询Hadoop教程相关的信息时, 输入检索语句“Hadoop教程”得到的结果很可能是与Hadoop没有关系的教程信息。这是因为, 关键词“Hadoop”很可能在文档中出现的次数很少, 而在“教程”出现的次数很多, “Hadoop”所对应的相关性权重值偏小, “教程”得到的相关性权重值偏大, 这明显违背了用户的检索意图。

    根据以上分析, 本文将从语义理解的层次对相关性权重算法进行改进, 通过建立用户搜索日志构建用户兴趣模型, 计算公式见式 (5) , 该方法可以智能地理解用户的搜索意图, 解决了传统方法没有深入到语义理解的层次、不具有联想功能和智能化程度比较低等问题。

    scorei=tfi, j×idfi×w (t in logas) ×coord (q, doc) (5)

    其中:

    w (t in logas) =loga (n+1) (6)

    coord (t, doc) =loga (m+1) (7)

    在式 (5) 中, scorei表示关键词ti与特定文档之间的相关性得分, w (t in logas) 是通过用户兴趣模型得到的关键词权重, 大小与用户使用次数n有关, coord (q, doc) 是协调因子, 大小与文档包含检索词的个数m有关。

    查询语句的最终得分Score可表示为:

    Score=∑tinqscorei    (8)

    其中, q表示由查询语句经过分词得到的关键词集合。

    本文所采用的实验数据是从搜狗实验室网站 (www.sogou.com/labs) 和知网 (http://www.cnki.net/) 下载, 共收集了2万余篇文档, 经过处理共产生170多万条索引记录。索引记录的格式如式 (9) 所示。

    word:filename→counts (9)

    其中, word为关键词, fileName为文档名称, counts为关键词在文档中出现的次数。

    4.2 实验结果与分析

    为了验证基于日志关联的相关性权重计算方法在信息检索系统中的评分效果, 本文分别从评分效果和时间性能2个方面对本文设计的算法与传统TF-IDF算法进行对比分析。在实验中为了避免分词方法的不同对实验结果产生影响, 2种算法都使用IKAnalyzer分词器对查询语句进行分词。

    实验1 评分效果比较。分别使用本文设计的算法和传统的TF-IDF算法作为信息检索系统的相关性评价算法。实验方案步骤如下:

    步骤1 输入查询语句“民主政治学习” , 观察系统返回查询结果的得分情况。

    步骤2 重复输入查询语句“民主”, 记录某一篇文档得分的变化情况。

    步骤3 输入查询语句“民主政治学习”, 观察系统返回的结果, 并与第一次做比较。

    图6所示为实验步骤2输入关键词“民主”, 执行100次查询任务过程中, 文档“划清中国特色社会主义民主同西方资本主义民主的界限”得分的变化情况。

    由表4可知, 第1次使用查询语句“民主政治学习”时, 文档B和文档C的得分都比文档A高, 这是因为文档B和文档C中“学习”一词出现的次数较多, 导致查询语句中的 “学习”关键词被赋予了较大的权重值, 显然这与查询目的不符。由图6可以看出, 随着用户使用关键词“民主”次数的增加, “民主”一词的权重值也在动态增加。因此, 当再一次使用查询语句“民主政治学习”时, 文档A就获得了最高的评分。

    经过以上分析可知, 基于日志关联的权重计算方法通过对用户搜索日志进行分析建立用户兴趣模型, 动态地增加关键词的权重值, 从而提高了检索结果与用户检索目的之间的相关性。

    实验2 查询性能比较。分别在单个节点环境下和由5个节点组成的集群环境下进行实验, 并记录程序运行的耗时情况。实验方案步骤如下:

    步骤1 在集群环境中, 输入1个～50个关键词, 记录不同关键词个数对应的查询时间。

    步骤2 在单节点环境中, 输入1个～50个关键词, 记录不同关键词个数对应的查询时间。

    图7为在单节点环境下和由5个节点组成的集群环境下, 使用不同数量的关键词进行检索时系统的响应时间。

    图7 单节点与集群环境性能对比   下载原图

    由图7可以发现, 在执行海量数据查询时, 集群环境中的查询性能明显优于单节点环境。随着检索词个数的增加, 集群环境下的查询时间整体变化的较为平稳;单节点环境下由于I/O交换过于频繁、CPU开销过大和JVM可用内存不足等原因, 导致算法性能严重下降。相比较而言, 集群环境由于可以并行执行查询任务, 因此具备较好的性能。

    5 结束语

    本文在TF-IDF算法的基础上, 提出一种基于日志关联的相关性权重计算算法。该算法通过用户日志建立用户兴趣模型, 从而提高了用户与信息检索系统的交互能力, 解决了传统方法没有深入到语义理解层次、智能化程度比较低等问题。另外, 本文结合分布式计算的思想, 使用MapReduce编程框架实现计算任务的并行化处理, 提高算法的效率和可扩展性。但是随着时间的推移, 用户的兴趣可能会发生变化, 需给用户最近使用过的检索词赋予较大的权重, 因此, 下一步工作重点是将关键词被使用的时间间隔引入到权重的计算方法中, 进一步提高算法性能。
