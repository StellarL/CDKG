    摘    要：

    为提高数据中心网络 (DCN) 的链路带宽利用率和吞吐量, 提出一种基于软件定义网络 (SDN) 的流概率路径选择方法。在最短路径选择方法的基础上, 利用SDN控制器对网络流进行分类处理, 同时计算每条数据流在调度过程中的带宽占用情况以减少数据链路碎片化, 并优化DCN数据流量调度, 从而实现网络负载均衡。实验结果表明, 该方法能降低网络传输时延, 提高链路利用率及负载均衡度。

    关键词：

    概率路径; 数据中心网络; 负载均衡; 传输时延; 链路使用率;

    SDN-based Flow Probability Path Selection Method for Data Center Network

    ZHANG Zhao LI Hailong DONG Siqi HU Lei MA Jingren

    Abstract：

    In order to improve the link bandwidth utilization and throughput of Data Center Network (DCN) , a Software Defined Network (SDN) based Flow Probability Path Selection (FPPS) method is proposed.On the basis of the shortest path selection method, the SDN controller is used to classify the network flow, calculate the bandwidth occupation condition of each data flow in the scheduling process.Then, it can reduce the data link fragmentation, optimize the DCN data flow scheduling, and realize the network load balancing.Experimental results show that this method can reduce network transmission delay, improve link utilization and load balance degree.

    0 概述

    在线服务和社交网络多数要求服务器具有可靠性和高效性, 传统负载均衡方法通常使用多个服务器并安装硬件负载平衡器提高服务器的高效性, 但是硬件负载平衡器价格昂贵, 同时受到供应商的控制。软件定义网络 (Software Defined Network, SDN) 技术开辟了互联网管理的新领域, 并提高了网络技术的灵活性, 例如负载平衡、入侵检测和网络拓扑获取[1]。SDN使用户能够定义自己的成本效益和灵活的应用程序来满足系统要求[2]。由于少数环节拥挤而大部分链接未被充分利用, 导致数据中心网络 (Data Center Network, DCN) 发生拥塞, 互联网基础设施经历不平衡的交通负荷。目前, 动态流量管理对于数据中心具有重要意义, 因为访问计算和存储资源的处理速度是影响用户体验质量 (Quality of Experience, QoE) 的重要因素。加载平衡可以平均分配多个流量路径, 使网络能够用更少的时间处理更多的数据流。SDN的优势在于控制器具有网络的全局视图, 因此, 可以在该过程中以更全面的方式建立转发规则, 根据状态实时调整网络流量, 最终达到负载平衡的目标[3]。

    本文采用基于SDN的数据中心网络流概率路径选择 (Flow Probability Path Selection, FPPS) 方法, 在考虑流量优先级的同时优化DCN中的链路利用率, 并结合流量特征、链路状态和应用需求进行合理的流量调度。

    1 相关研究

    基于SDN的负载均衡是目前研究的热点。现已提出许多网络流量优化模型, 通过在流量传输过程中改变流量路径来解决网络拥塞的问题, 实现负载平衡。简单的静态路径优化方法包括循环法、随机法、最少流量法、最少延迟法等。OpenQoS框架基于流量优先级以及可用的链路带宽对视频流路径进行优化[4]。OpenQoS主要侧重于实现QoS要求和维护流量优先级, 而不是负载均衡, 该方法与数据中心大规模数据流量优化调度的需求不相符。基于粒子群优化技术的能量感知SDN路由算法[5]从源主机到目的主机逐级确定交换机中最佳的传输节点, 从而得到有助于减少网络资源消耗的传输路径, 但是该方法会导致路径选择的时间过长, 同时未考虑网络带宽的碎片化。基于SDN的多路径优化方法 (LABERIO) [6]会将流量重新分配到可用的路径带宽, 但是使用链接方差的即时值来触发其路径优化算法, 导致网络开销较大。文献[7]在基于SDN的数据中心多路径负载平衡路径优化 (DLPO) 网络中提出一种多链路和单链路数据中心网络。多链路DLPO方法在整个网络中平衡链路负载, 而单链路DLPO方法执行跳到跳的重新路由以避免高度负载链接, 但是DLPO方法通常会由于重新路由而花费更多的计算时间, 使得传输时延更长。基于SDN的动态路由方法利用分层结构网络特征和实时流量统计[8], 在负载中实现多路径路由平衡, 将网络流平均分配减少延迟, 但是该方法利用网络流量均衡器统计实时流量, 大幅提高使用成本, 实用性较差。基于SDN的流量工程方法同时考虑负载平衡和功耗问题[9], 为交通负荷搜索最佳拓扑结构, 通过关闭不包含在内的元素降低功率, 但该方法需要在一定程度上调整网络拓扑, 这在现实数据中心网络通常难以实现。

    文献[10]提出一种新的带宽需求方案 (Hedera) 及可扩展的动态流量调度系统, 通过模拟退火的调度程序自适应调度多级交换结构, 达到有效利用总体网络资源的目的。但是, Hedera单方面考虑了大流的安置问题, 在一定程度上扩大了带宽利用率, 却未综合考虑当前网络环境中网络流量的复杂性, 通常会造成网络中有过多小流量安置不到位或者网络带宽碎片化的出现。

    当前数据中心网络中的数据流量构成多数为数量多但规模小的小流, 称为老鼠流。数量少但是规模较大的大流, 称为大象流。小流虽然数量上占据较大比例, 但是由于规模与大流相去甚远, 因此在网络流量中决定负载均衡与否的关键仍在于少数的大流。同时, 在完成对大流的调控后, 对于小流的后续处理是决定网络链路利用率的关键, 通常网络链路碎片化的出现以及网络链路的资源利用率低下都是由于对小流的不合理调度。鉴于此, 本文提出基于SDN的面向流的概率路径选择方法, 在对数据流量按照速率大小进行合理调度后, 完成最短路径的选择, 进而提高网络链路利用率, 实现负载均衡。

    2 基于SDN的DCN流概率路径选择

    面向流的概率路径选择方法包括大象流筛选以及路径选取2个步骤, 整体框架 (见图1) 由3个核心部分组成:

    1) 主机端。在主机处采用Mahout[11]方法发现大象流并标记, 比如修改TCP协议的Tos字段。

    2) 交换机。根据拓扑信息和链路带宽情况, 使用最大概率路径算法计算数据中心的流路径。

    3) 控制器。将计算好的路径以流表的形式下发到OpenFlow交换机。

    2.1 大象流发现与标记

    根据相关研究, 可以选择在接入层交换机处进行大象流发现并标记, 比如采用DevoFlow和Hedera, 也可以在端主机处, 比如采用Mahout, 这2种情况都是通过判断传输字节数据是否达到预先设定的阈值进行大象流标记 (DevoFlow为1 MB～100 MB, Hedera为网卡带宽的10%, Mahout为100 KB) 。文献[11]对不同方法进行了对比发现, Mahout是较有效的大象流发现方法, 所以本文采用Mahout方法发现并标记大象流。Mahout大象流标记方法是一种位于端主机的标记方法, 通过实时检测数据TCP缓存大小判断流量规模, 当超过一定阈值时, Mahout将在数据包包头中区分服务域并对其进行标记。

    2.2 数据流动态调度

    SDN控制器对流进行调度工作的优势在于可以将数据流进行灵活处理, 对于筛选出的大象流进行优先调度, 从而尽可能避免网络拥塞问题。在对大象流进行调度后, 由于并不是每条大象流都可以较好地使用分配到的带宽, 因此在后续的老鼠流调度过程中, 大象流的剩余带宽依然可供老鼠流使用, 在此基础上再进行老鼠流的调度, 从而降低链路碎片化。根据网络数据链路中各路径的实时使用情况对网络流量进行相应调度, 从而最大化每条链路的带宽使用率, 避免出现链路碎片化。相比传统网络调度方法, 该方法能够对数据流进行及时分流, 并保证最短路径的带宽利用率维持在原有的水平[12]。

    2.3 调度流程

    本文提出的大象流调度流程 (见图2) 对负载均衡和带宽碎片化进行了合理的权衡, 具体过程如下:

    1) 主机端通过Mahout对数据流是否为大象流进行判断, 然后将数据流信息传递至控制器。

    2) 控制器基于数据流的大小分别下发调度流表至交换机。

    3) 如果控制器判断是大象流, 则根据流信息和控制器维护的全局拓扑信息获得流的可选路径, 然后根据流的大小筛选出链路带宽超出其规模最少且符合最短要求的路径对该流进行路径分配。

    4) 对于数量大但占用带宽较小的老鼠流, 在完成大象流的分配后, 使用概率路径算法保证传输质量的同时将老鼠流按照等价多路径 (Equal-Cost Multipath, ECMP) 路由规则分配至大象流所在路径的剩余带宽上, 降低链路碎片化。

    2.4 路径选择算法

    本文提出的基于SDN的数据中心网络流概率路径选择方法是在原有最短路径选择方法的基础上, 通过大象流、老鼠流的分类调度优化流量调度, 同时计算每条数据流在调度过程中的带宽占用比降低数据链路碎片化。

    在检测出大象流后, 根据胖树网络的特点, 源地址和目的地址不在同一个POD中具有k2/4个等价路径, 而本文目的在于选出最佳传输路径。

    Yen算法是一种偏离路径算法[13], 其计算建立在普通最短路径的基础上, 使用最短路径算法计算得到一组最短路径后, 建立最短路径合集{P1, P2, …, Pk}并存入列表A。该方法能够满足本文从等价路径中选择最佳路径的需求, 所以FPPS首先使用Yen算法对最短路径集进行计算, 而后对所求得的最短路径集中所有路径的使用概率进行计算, 从而得到最大概率路径。

    在得出前K条最短路径{P1, P2, …, Pk}后计算Pk+1, 具体过程如下[14]:

    1) 将计算得到的每条最短路径的任意节点Pk当作潜在的下一跳目的节点 (除最终目的节点外) , 然后将该节点重新作为新源节点, 计算从该新源节点到目的地址的最短路径集合。在计算过程中, 正在使用中的最短路径中的节点不能作为下一跳目的节点, 不然会出现网络环路的死循环。同时, 当前路径节点的下一跳目的节点应与之前的下一跳目的节点出发方向保持不同, 从而避免出现重复计算的路径。

    2) 计算得出上述条件下的最短路径后, 完成正在使用的最短路径Pk+1在新源节点到最终目的地址的路径与到原源节点的路径的拼接, 从而得到路径Pk+1, 存入列表B。

    3) 在列表B中挑选最短的路径, 将其作为Pk+1更新至列表A, 重复上述步骤计算得出前K条路径[14]。

    影响Yen算法计算时间的因素主要有:

    1) 短路径计算时间。使用不同的基础最短路径算法得到最短路径集的时间不同, 例如Dijkstra算法, 计算出最短路径集的时间代价为O (m+n logan) 。

    2) 列表B路径更新至列表A的时间代价为O (logak) 。

    3) 每条路径的节点到最终目的地址最短路径的计算时间代价为O (m+n logan) 。

    4) 列表B中可用的路径数为n, 带来的时间代价为O (n) 。

    综上所述, Yen算法总的计算时间代价为O (c (n) +K (n× (m+n logan) +n+logak) ) 即O (Kn× (m+n logan) ) 。

    在完成上述计算过程后可以得到数据流可用的最短路径, 下一步即在可用的路径中选择最优化的流量调度方法, 使得数据流能够在最短的时间内, 在整体网络付出最小代价的情况下完成转发工作[14]。

    本文使用的概率路径选择方法首先根据大象流的具体信息筛选当前可用路径中所有满足要求的路径组合, 然后通过将带宽与待调度流作比较计算剩余带宽, 计算数据流量在各路径上转发时对该路径的利用率, 从而得到路径被选择的概率, 结合转发路径长度选择路径, 其中, 概率较大的路径被选择的机会较大, 虽然路径最小链路剩余带宽较大的概率较小, 但同样会被选择, 缓解了因路径最小链路剩余带宽越小而被调度分配的流越多的问题。FPPS算法流程具体如下:

    1) 根据数据流的信息确定该流的源地址与目的地址, 从而确定两者接入的交换机S1、S2。

    2) 确定数据流的源交换机与目的交换机后, 根据控制器获取的拓扑信息确定S1、S2之间的所有满足流F带宽需求的路径L{l1, l2, …, ln}, li为任意路径。

    3) 利用Yen算法计算得出2个交换机之间的最短路径合集Lmin{lm1, lm2, …, lmn}。

    4) 计算最短路径合集中流带宽W与各路径可用带宽的比Pi=W/Bandwidth (lmi) 。

    5) 计算每条路径被最终选定的概率λi, λi=Ρi /∑m=1n Ρm。

    3 实验验证与结果分析

    3.1 实验设置

    本文在轻量级仿真工具Mininet上进行实验模拟, 使用开源Floodlight控制器[15]下发流量转发规则, 利用Mininet拓扑生成器[16]模拟胖树DCN拓扑[17]。Floodlight OpenFlow控制器与OpenFlow交换机在Mininet中使用OpenFlow进行协议切换。Mininet仿真网络由虚拟主机、OVS交换机、控制器和链接组成, 网络分析工具为Wireshark和Perf[18,19]。

    本文中的SDN北向接口使用可表示状态转移的API (REST API) 。REST API能够收集操作拓扑与设备信息以及即时流量与端口统计。将Floodlight OpenFlow控制器连接到Mininet的端口6653。如图3所示, 胖树DCN由8个主机 (h1～h8) 、10个交换机 (s1～s10) 和20个链接通过运行生成在Mininet中。执行Ping all命令检查定义网络中所有主机的可达性。其余部分API收集有关主机和交换机 (如IP) 的信息地址、MAC地址、端口映射、链路比特率、传输速率、接收速率等。REST API在8080端口将信息展示为json数据格式。应用负载平衡算法在Python代码中获取路径信息进行交通管理, 计算总的链接成本和最短的主机间的所有路径。

    图3 SDN下的胖树网络拓扑   下载原图

    本文通过Perf流量工具模拟网络流量, 使用参考文献[20]的流量模型, 流的到达服从泊松分布, 设定80%的数据流为老鼠流, 20%的数据流为大象流进行实验。

    3.2 结果比较与分析

    本文设置数据流平均传输时延、链路使用率和负载均衡度[20]作为性能评价指标, 同时比较了ECMP、LABERIO、DLPO、FPPS这4种方法在上述3个指标中的性能。数据流平均传输时延指数据流从发起端通过网络链路到达目的端使用的时间。链路使用率是指数据中心网络中所有链路的使用率。负载均衡度指在流传输的时间内, 所有等价路径上承载流量的平衡度, 计算公式为 (MAX-MIN) /AVG, 其中, MAX表示路径上的最大流量, MIN表示路径上的最小流量, AVG为平均值。

    由图4可以看出, ECMP传输时延最多, 随着网络流量的增大, FPPS逐步在效率上取得领先, DLPO在网络流量较少时时延较小, 但当网络流量变大后, DLPO由于重新计算路由使得时延变大, 而LABERIO则在触发优化算法时表现较差。

    图4 4种方法的平均传输时延对比   下载原图

    由图5可以看出, LABERIO由于使用连接方差及时值触发优化算法, 因此链路使用率较低, DLPO相对较好, 而FPPS因为每条数据流都分配至最大的概率路径上而使得链路使用率较高。

    图5 4种方法的链路使用率对比   下载原图

    由图6可以看出, ECMP由于只局限于最短路径, 因此在负载均衡上表现一般, DLPO与LABERIO由于使用多路径的负载均衡方法, 负载均衡效果较好, 但是LABERIO由于存在单链路状态, 因此在实现上稍显不足。FPPS利用最大概率选择大象流路径的方法以及最优化使用各链路的思想, 使其在较快的速度下实现负载均衡。

    4 结束语

    网络流量的日益增长使得数据中心网络拥塞成为亟待解决的问题, 而传统解决方式缺乏全局观念并且效率较低。本文利用SDN数控分离技术优化控制平面上面向流的概率路径选择, 通过Yen算法计算最短路径并对数据中心中占比较大的大象流进行概率路径选择, 在缩短传输时延的同时提高链路利用率。通过实验验证, FPPS相对传统方法以及DLPO等单一负载均衡方法在降低时延、提高链路使用率和负载均衡度方面具有更好的性能。下一步将对老鼠流的资源调度展开研究, 从而实现更高效的数据流传输。
