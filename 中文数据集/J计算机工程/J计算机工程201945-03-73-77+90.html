    摘    要：

    为适应当前移动设备用户界面可用性和可复用性需求, 解决移动设备用户界面描述语言 (UIDL) 未考虑移动设备的固有属性且对于大粒度的模式构件不能较好的支持等不足, 设计一种基于可扩展标记语言的移动设备用户界面描述语言PXMUL, 并给出基于模式组件的移动用户界面的实现框架, 从界面布局、逻辑和环境3个模块定义描述界面所需的属性, 在大粒度构件基础上实现用户界面的设计与开发。实例评估结果表明, PXMUL能够实现移动设备用户界面的可行性与有效性, 同时降低学习成本, 缩短开发周期。

    关键词：

    移动设备; 用户界面描述语言; 模式构件; 可扩展标记语言; 用户界面生成; 可复用性;

    Design and Implementation of A Mobile Device User Interface Description Language

    FENG Xu HUA Qingyi FAN Pan WANG Wenjian

    School of Information Science and Technology, Northwest University

    Abstract：

    In order to adapt to the current mobile device user interface usability and reusability requirements and colve the insufficency that the mobile device User Interface Description Language (UIDL) does not consider the inherent properties of the mobile device and does not support the large-grained mode components, this paper designs a mobile device user interface description language based on eXtensible Markup Language (XML) which is called PXMUL, and gives an implementation framework of mobile user interface based on pattern component.The attributes needed to describe the interface are defined from three modules of interface layout, logic and environment, and the design and development of user interface are realized on the basis of large-grained components.Example evaluation results show that PXMUL can reduce the learning cost and shorten the development cycle while ensuring the feasibility and effectiveness of interface implementation.

    Keyword：

    mobile device; User Interface Description Language (UIDL) ; pattern component; eXtensible Markup Language (XML) ; user interface generation; reusable;

    0 概述

    完整的应用需要经历设计阶段与实现阶段, 在设计过程中引入用户界面设计模式, 能够大大缩短界面设计周期[1,2]。然而, 据调查表明, 在当前的界面实现阶段, 移动应用50%的时间成本耗费在小粒度组件工作上, 因此, 如何在保证界面可用性的同时在大粒度构件上实现移动设备的界面开发, 同时实现界面的可复用性[3,4], 引起了移动设备界面设计和开发人员的重视。

    为了实现界面设计模式辅助界面自动化或半自动化的生成, 研究人员将模式用平台编程语言进行实例化, 封装模式的布局与对话形成独立的原子构件, 并留有接口供外界调用, 目前已形成初具规模的模式构件库[5,6,7]。为实现界面的可复用性, 广泛采用的方法是使用用户界面描述语言 (UIDL) 。用户界面描述语言是一种对用户界面进行描述的通用方法, 它使用规范化的描述, 诸如XML、状态转换图、图表等方式对用户界面进行描述, 并作为界面模型的主要部分来辅助界面的开发, 实现半自动化或自动化的用户界面的生成[8,9,10]。然而, 现有的基于XML的用户界面描述语言存在以下问题:当前的界面描述语言没有针对大粒度构件描述的相关属性, 使得大粒度构件不能在界面设计阶段被复用;多数界面描述语言对移动设备的界面描述能力不足, 没有考虑移动设备的固有属性及限制性, 导致界面可用性质量不高;一般的界面描述语言对于界面间的通信控制没有明确的描述, 使得界面布局和对话控制耦合度过高, 增加了开发过程中的沟通成本。

    针对上述问题, 本文从界面实现的角度出发, 设计并实现一种模式驱动的基于XML的移动设备用户界面描述语言PXMUL。该语言基于现有描述语言的优点, 使得用户界面模式成为能够直接操作的构件应用于界面工程, 并根据设计方案从用户界面模式库中选取相应模式构件完成最终用户界面的“拼装”, 从而提高开发人员的工作效率。

    1 相关工作

    国内外对用户界面描述语言的研究都较为重视, 例如微软推出界面描述语言XAML, 并将其应用于Vista界面及Silverlight的开发中。该语言能够支持对界面各个组成部分的详细描述, 并且能够通过对自定义控件描述的支持。XUL为了支持Mozilla系列的应用程序而开发的使用者界面标记语言, 能够对基于Web的用户界面进行描述。上述的2种语言[11,12]虽被广泛应用, 但大多数此类描述语言只能生成特定语言 (例C#) 的界面。佛吉尼亚人机交互技术中心提出的UIML[13,14]实现了跨平台的界面描述机制, 使用树状结构对用户界面进行描述, 是一种扩展性较强的语言, 其可以在语言中指定界面事件对应的逻辑。但该语言只是提供了一个界面元素和布局的抽象描述, 如向最终界面转换还需要大量的工作。此外, 代表性的研究成果还有 MXML、JUDL、USIXML[15]等。

    尽管以上界面描述语言能够支持界面的描述, 但是大多数界面描述语言对移动设备的界面描述能力不足, 没有考虑移动设备的一些特殊性, 而且对于大粒度的模式构件不能很好的支持。因此, 本文在模式库的基础上提出一种基于XML的移动用户界面描述语言, 将移动用户界面的布局与交互控制的规则分别描述, 并给出该语言的界面生成框架。

    2 PXMUL的设计

    2.1 PXMUL设计目标

    PXMUL旨在帮助界面开发人员在大粒度构件上进行界面描述, 其具有模块化的描述方式, 能够支持设备平台无关性、可复用性和可扩展性等。

    1) 模块化描述。

    用户界面有多个部分组成, 除了组成界面本身的各种元素之外, 还包括界面相关的交互数据、事件处理组件等, 采用模块化的描述方式减少界面组成元素与逻辑的交叉, 增加描述语言本身的可读性, 降低了直接对描述语言进行编辑开发的界面开发人员的认知困难, 提高开发效率。

    2) 无关性。

    这里的无关性是指移动设备无关性和平台无关性。当前存在有各种不同尺寸类型的移动设备, 其中, 设备无关性是指界面描述语言能够忽略设备的差异, 平台无关性是指界面描述语言不因移动用户界面所处开发平台 (安卓平台和IOS平台) 不同而有所不同。

    3) 可复用性和可扩展性。

    可复用性不仅是界面组成部分的重复使用, 还包括对环境上下文等非界面元素的复用, 可扩展性是指能在元语言层次上对组成语言的模块进行添加和修改, 以适应不同交互设备的变化。

    2.2 PXMUL组成模块

    基于上述设计目标, 本文给出一种用户界面描述语言——PXMUL。PXMUL采用基于XML的说明式描述语言, 不依赖具体的实现语言和平台, 实现界面描述和具体实现的抽离。PXMUL包括3个模块:环境描述组件, 逻辑描述组件和界面布局描述组件, 设计框架如图1所示。

    2.2.1 环境描述组件

    环境描述组件可以隔离设备和平台等物理元素对界面组件渲染的影响, 在隐藏物理工具包的同时向用户界面描述抽象工具包。抽象工具包是一个具体模式构件和事件名称的词汇表, 封装了物理工具包中的模式构件和事件。这里的物理工具包特指具体的模式构件库的工具包。

    PXMUL提供了<Environment>元素来描述环境描述组件。Environment针对不同的环境提供了界面元素描述表示词汇和生成工具箱词汇之间的映射。Environment通过类元素 (class) 描述每个模式组件或事件。每个类都有一个唯一的名称 (Environment词表的一部分) , 映射到具体环境下模式组件或标记的映射以及与其关联的属性列表。以下示例简单演示了Environment的内容。

    2.2.2 逻辑描述组件

    从概念上讲, 软件系统由三层组成:应用程序代码, 用户界面代码以及两者之间的通信 (使用API或协议) 。逻辑组件为用户界面提供了规范的方式以便与应用程序通信, 同时隐藏有关底层协议、数据转换和方法名称等信息。为实现用户界面描述语言的可学习性, 使用程序员希望提供的方法名称和参数的词汇表作为映射。

    PXMUL提供了<logic>元素来描述逻辑描述组件。logic元素使用三组名称将用户界面中的抽象调用映射到应用程序中的物理方法。方法的实际名称及其参数是应用程序词汇表的一部分, 用户界面调用它们的方法的名称是逻辑词汇表的一部分, 接口的抽象调用是接口词汇表的一部分。logic元素包含的信息可以帮助PXMUL渲染器将应用程序词汇表中的名称映射到实际的应用程序方法。下面给出logic的内容示例。

    2.2.3 界面布局描述组件

    界面布局描述组件具有平台和语言无关性, 不仅包含组成界面的元素, 还包括一些与用户交互的组件。界面布局组件使用一组独立于设备和应用程序的抽象元素、事件和方法调用, 描述了用户和应用程序之间的对话。界面逻辑是模式构件在页面上的拼接信息, 内容表示具体某一模式构件的呈现, 代表了应用服务如何与用户界面之间能够发生的一个交互行为, 相关属性则为用户自定义界面提供了灵活的方式。

    PXMUL提供了<interface>元素来描述逻辑描述组件。interface将用户界面描述分成4个独立的部分。每个部分都描述了部件如何对用户界面作出贡献的不同方面, 具体包括界面结构 (structure) 、内容 (content) 、风格 (style) 和事件 (events) 等信息。下面给出Interface的语法结构。

    一个界面可以包含多个结构、样式、内容或事件元素, 前提是每个元素都可以按名称唯一标识。

    structure描述了用户界面的各个部分的组织结构, 是模式构件在页面上的拼接信息;structure结构是树型结构, 树的每个节点都代表一个界面呈现区域。每个节点的所有子节点集合代表对该节点所描述的呈现区域的一种划分;叶子节点代表了不能再被划分的最小呈现区域, 它作为用户界面模式的容器。

    style描述了依赖于设备的样式表信息和数据, 表示具体界面的属性;<style>部分中的每一行描述了一组具有相同类属性的界面元素。可以指定特定应用程序的各种样式属性 (如颜色、字体大小) 创建不同设备的不同样式表。

    content包含与设备无关但取决于应用程序的数据, 是呈现给用户的信息;<content>部分中的每一行都有一个名称属性, 用于标识相应的界面元素。每个内容元素的文本可以是任何有效的XML代码。

    events描述了运行时的交互行为具体表现为界面元素和后端之间传递的运行时接口事件。<events>部分可能包含多个事件描述, 每个事件都使用在接口描述中唯一的名称进行标识。

    在Pattern Set环境下, 模式构件可以是一个基础的界面Widget组件, 也可以是一个封装成特定功能的大粒度构件。应用程序的框架设计简化了构件的重用, 借助于框架的实现, 程序员可以灵活、方便地替换应用程序构件, 从而帮助程序员快速地开发程序。目前已经形成一定规模的模式构件库。本文的PXMUL即针对此用户界面软件层次结构设计及实现。

    3.2 PXMUL界面生成框架

    采用PXMUL描述的界面经过XML解析器解析, 通过如图3所示的界面生成框架和生成引擎最终实现为具体界面。生成框架在基于MVC的基础上分为3层:表示层, 控制层和数据层。主要模块包括模式构件工厂、模式管理器、数据管理者。各层的主要职责如下:

    1) 表示层:该层主要包括模式构件工厂和用户界面模式库。将模式作为构成界面的基本组件, 整个页面作为模式的容器表示为根节点, 按照提供的布局策略层组织成一颗呈现树, 即布局模型, 通过模式构件工厂创建出用户界面。

    2) 控制层:模式管理器作为控制层, 负责协调界面视图和数据模型之间的控制。其中包括用户界面事件的监听与回调, 根据事件性质去调用数据层操作数据, 数据发生变化则会影响布局模型, 与此同时, 模式管理器要控制模式构件工厂重新展示界面视图。

    3) 数据层:数据层负责从持久数据存储区检索和存储数据。数据管理者 (Data Manager) 负责过滤和转换从帮助类中返回的数据, PreferHelper、DatabaseHelper等负责Data Manager与数据库等的连接与数据交换。

    3.3 引擎生成

    生成引擎根据环境描述组件、逻辑描述组件和界面布局描述组件描述的界面进行合并解析后转化为最终用户界面。生成引擎主要包括模式构件工厂和模式管理器。模式管理器负责根据事件控制对象来管理界面上构件间的对话控制与响应, 它是引擎的核心, 主要包括API、事件处理、逻辑控制等子模块。模式构件工厂根据界面对象和布局策略组建成具体的界面布局。该工厂提供了一个适配器来建立模式控件对安卓平台和IOS平台控件的映射, 同时对事件监听接口进行封装, 使得引擎的模型和控制器不依赖于具体的界面实现。由于篇幅限制, 模式控件的设计和模式构件库的建立本文不做详细陈述。

    4 实例评估

    为了验证PXMUL的界面表达能力以及对大粒度构件的支持, 本文用PXMUL构建了校园二手交易市场应用, 该应用在功能上主要包括4个模块:个人中心模块, 二手市场模块, 讨论集会模块, 我的收藏模块。

    在实践中发现, 采用PXMUL适合大粒度构件层次的界面设计, 且使用PXMUL的生成框架也可以很好地辅助具体物理环境下界面的生成。同时得益于语言的高度抽象层次, 使得用户界面与非界面代码实现了自然分离, 减少了界面设计人员与开发人员的沟通成本。与原有的分离式开发各个移动平台原生应用方法相比, 显著缩短了开发人员得到初始版本及迭代版本的时间并大大减少了LOC (Lines of Code) 。

    为了验证PXMUL在提高开发效率、减少程序员工作量与学习难度等方面的优势, 本文组织界面开发者分别使用本语言、安卓界面描述原生语言以及IOS界面描述原生语言来对二手交易市场进行界面开发。3种界面开发方法的实验结果如表1所示。从表1可以看出, 本文语言相比于其他原生移动设备界面描述语言, 具有较低的学习成本及较高的界面复用性, 提高了界面开发人员的效率。

    5 结束语

    本文在对用户界面描述语言研究的基础上, 结合大粒度模式构件的特点, 提出一个层次化的移动用户界面描述语言PXMUL。该描述语言能够在基于大粒度模式构件的基础上实现快速地开发用户界面, 提高界面的开发效率并保证可用性, 同时支持高质量界面的自动化或半自动化生成, 并采用基于XML说明式的描述方式, 简洁清晰, 降低了学习成本。实例结果表明, 该方法能够提高开发效率且易于使用。下一步的工作是完善PXMUL的框架结构, 提高生成引擎的效率和丰富生成引擎的功能, 使之能被推广应用。
