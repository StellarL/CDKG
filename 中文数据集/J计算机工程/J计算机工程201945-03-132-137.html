    摘    要：

    为在社交网络中向用户进行更高效的好友推荐, 分析微博社交网络中存在的交互行为, 综合考虑网络结构、用户属性以及用户交互特点, 计算用户之间建立好友关系的可能性大小。在此基础上, 提出一种基于用户交互的社交网络好友推荐算法。实验结果表明, 与只考虑网络拓扑结构或用户属性的算法相比, 该算法准确率更高。

    Social Network Friend Recommendation Algorithm Based on User Interaction

    LONG Zengyan CHEN Zhigang XU Chenglin

    School of Software, Central South University Ministry of Education-China Mobile Joint Laboratory of “Mobile Health”, Central South University

    Abstract：

    In order to recommend friends to users more efficiently in social network, the interaction behavior in micro-blog social network is analyzed.Considering the network structure, user attributes and user interaction characteristics, the possibility of establishing friendship among users is calculated.On this basis, a friend recommendation algorithm for social network based on user interaction is proposed.Experimental results show that the proposed algorithm is more accurate than the algorithm which only considers the network topology or user attributes.

    0 概述

    当前, 人们的生活与学习, 尤其是社交活动, 受到互联网的较大影响。随着互联网的不断进步, 社交网络获得了大量的用户群[1], 且人们的很多生活方式都随着社交网络而逐渐发生变化[2]。社交网络通过用户交互以及其他相关活动而构建, 因此, 可以将其看成是用户现实社会关系的一种网络反映。

    社交网络用户众多, 数据庞大, 依靠其本身存在的好友推荐功能使得用户找到可能成为好友的其他用户显得十分重要。提供可用的好友推荐功能, 社交网站需要具备高效、可以发现潜在好友的算法[3]。当前, 在社交网络好友推荐领域中, 文献[4]根据共同好友来进行推荐, 该方法认为朋友的朋友最终也会成为自己的朋友, 即拥有更多共同好友的用户更加容易成为新的朋友[5]。但是, 在这种方法中, 假如2个用户本身在社交网络中的共同好友不多, 那么他们就不会被推荐为好友, 这与现实情况不符。另外, 假设存在AB和CD 2对用户节点, 其中, AB比CD拥有更多的共同好友, 但AB本身的好友数量也更大, 如果AB的共同好友在自身好友总数中占比比CD小, 则CD这对节点更容易建立好友关系, 这也与实际情况不符。文献[6]依照用户所喜欢的物品构建好友推荐模型。该模型认为2个用户所喜欢的物品相似度越大, 则越有可能成为好友。但其在推荐前需要得到用户对于某些物品的喜欢程度, 而喜欢程度是一个比较主观的评价标准, 在收集数据的过程中可能因为主观因素而产生偏差, 使得推荐效果不佳。文献[7]根据用户所处社交网络的结构信息进行好友推荐, 但其只计算网络的拓扑结构信息, 会丢失社交网络用户所具备的特有属性, 由此导致结果不准确。

    本文研究用户交互行为, 提出一种社交网络潜在好友识别算法。对微博中用户之间的交互操作进行建模并给出一种衡量算法, 突出用户交互这一显著特征。考虑网络拓扑结构, 在链路预测算法的基础上提出一种微博社交网络好友推荐算法KIFLink。

    1 相关工作

    在社交网站中, 好友推荐功能具有重要意义。一个性能较好的好友推荐方法, 既可以帮助用户更快地找到生活中的好友, 也可以使用户发现生活中不认识的人, 从而帮助其扩展人际圈。

    在通常情况下, 社交网站分析网络拓扑结构信息, 通过链路预测算法实现好友推荐[8]。这类算法首先将用户看成网络中的节点, 用户之间的好友关系看成不同节点之间的边[9], 通过计算不存在边的节点之间形成新链路的可能性来实现好友推荐。通过共同邻居方法将链路预测算法应用于好友推荐中[10]。共同邻居方法统计2个用户节点之间的共同邻居数量, 进而评估不同用户成为好友的概率。2个用户的共同邻居可以看作是其共同好友, 共同好友越多, 可以认为2个用户越可能成为新朋友。这种方法具有计算简单、易实现的特点, 时间复杂度较低, 但是, 由于只考虑了共同邻居这一个网络结构特点, 导致推荐准确度不高。文献[11]运用似然分析算法对节点之间建立新链路的可能性进行评估, 但该方法运算量较大、时间复杂度较高, 并不适用于实际大规模社交网络。文献[12]根据网络中的可达路径情况开展好友推荐, 该方法只计算单一路径, 计算复杂度较低, 但是其忽略了较长的可达路径, 从而丢失了部分信息, 使推荐精确度较低。

    文献[13]根据与用户相关的博文内容来实现好友推荐。在微博等社交网络中, 通过分析用户活动产生的发布、转发、评论等交互行为涉及的博文内容来对指定用户所感兴趣的话题进行提取, 从而计算用户感兴趣信息的关系, 由此来找到新的好友。这种方法对于不经常发布信息的用户而言效果较差, 原因是这类用户的微博内容少, 不能提取足够的兴趣信息来进行好友推荐。文献[14]采用主题模型提取主题信息然后进行好友推荐, 但其同样要求用户发布或者评论足够多的信息才能取得较好效果。文献[15]计算用户属性特征然后进行好友推荐, 但其要求用户对于自身属性特别清楚, 这一缺陷限制了该方法的通用性。

    文献[16]应用概率模型分析电子邮件数据库以实现好友推荐, 但其只适用于具备来往电子邮件的特定网站, 不能广泛应用于社交网络。

    2 KIFLink算法

    2.1 社交网络好友推荐特征分析

    社交网络好友推荐算法需要较多参数以对用户之间建立好友关系的可能性进行计算[17]。本文基于用户交互、通过链路预测方法进行好友推荐的KIFLink算法, 包含社交网络拓扑结构信息以及社交网络用户所特有的一些属性信息。在个人属性方面, 算法主要分析住址、工作、受教育程度、个人兴趣爱好等信息。一般而言, 居住在同一个地方的人有可能会对当地的一些热点话题感兴趣从而构建联系。在工作方面, 从事同一行业工作的用户也可能会产生共同话题, 从而拉近两者的关系。对于受教育程度来说, 处于一个学校的校友更加容易成为好友。在个人兴趣方面, 具有类似兴趣爱好的用户会容易产生共同话题, 从而形成好友关系。

    在微博平台中, 用户的交互活动主要包括点赞、转发和评论[18]。点赞操作对于构建好友关系的影响较弱, 但是在用户没有其他互动的情况下, 该项活动也可以作为衡量指标。转发操作可以持续多个不同的用户, 从而提供更多的候选推荐用户。评论操作则主要影响被评论人以及从评论用户转发的用户, 对于其他用户的影响较小。本文算法考虑以上情况, 以实现高效的微博网络好友推荐。

    2.2 Katz Index指数分析

    根据“朋友的朋友也可能成为自己的朋友”的理论[19], 在社交网络中, 2个节点之间的路径长度会对2个节点能否产生边带来一定影响。路径越短则形成好友关系的概率越大[20], 这也是基于最短路径的衡量方式。

    Katz Index考虑所有路径的情况, 其总结所有的路径长度以及因为路径长度而引起的影响力衰减之间的关系, 得出路径越短, 在衡量两者相似度指数时所具备的权重越大的结论。该指数的表示如式 (1) 所示。

    SxyΚatz=∑l=1∞βl|pathsxy<l>|=βAxy+β2 (A2) xy+  β3 (A3) xy+⋯    (1)

    其中, pathsxy<l>是连接节点x和节点y的长度为l的全部路径[21], β用来调整不同路径所具有的权重, 若β值较小, 式 (1) 则转化为共同邻居方法, A为网络邻接矩阵, (An) xy表示节点x和节点y之间长度为n的路径数量。

    2.3 用户属性与交互行为计算

    针对社交网络进行好友推荐, 需要考虑用户本身所具备的各项社会属性, 如住址、教育情况、个人爱好等。本文算法采用余弦相似度的方法计算用户属性相似度[22]。用2个向量计算余弦相似度, 这2个向量代表了待计算用户的属性信息。用户属性相似度计算如下:

    sim (x, y) =cos (x, y) +a (2)

    其中, y为对比节点, 也即候选节点, x为目标节点, a为调节参数, 用来应对某些极端情况下出现的相似度过低现象。

    本文在式 (2) 的基础上, 根据微博用户交互特点, 提出基于路径相似度的微博用户交互评价方法。该方法将微博用户间存在的点赞、关注、转发、评论等一系列动作看作从一个用户能够抵达另一个用户的各条路径。这4种行为的发生次数则计算为路径长度, 比如, 一个用户对另一个用户的微博内容点赞了1次, 则路径长度视为1, 其他行为发生的次数也如此计算。但是, 由于这4种行为对于一个用户与另一个用户形成好友关系所产生的影响不同, 因此本文算法对其赋予不同的权重。此外, 由于微博中的评论等行为具备方向性, 当A用户对B用户的一条微博进行评论, 但是B用户并没有对前面的评论进行回复, 以及A用户针对B用户的相关微博进行评论, 然后B用户又有针对性地进行了回复, 这2种情况将被视为2条不同的可达路径, 并且具备不同的权重。

    在微博中, 2个用户相互关注则认定成为好友。假设系统向用户M进行好友推荐, 而用户N为系统中的待计算用户, 对用户M和用户N之间的交互需要衡量的情况为:用户M针对用户N的转发情况, 用户N针对用户M的转发情况, 用户M针对用户N的点赞情况, 用户N针对用户M的点赞情况, 用户M针对用户N的评论情况, 用户N针对用户M的评论情况, 用户M针对用户N的关注情况。这7种情况被视为用户M可以到达用户N的7条不同路径, 反映交互方式的权重值设为α。对于评论这种交互方式, 可能用户N在其他用户处对用户M的内容进行了评论, 即用户N经过n个用户后对M进行了评论, 这时就记录参数β为1/n。对于点赞或者转发的交互方式, 也同样计算。因此, 定义第i条路径的权重为Wi, 其计算表达式为:

    Wi=αi·βi (3)

    设用户交互行为评分为S, 定义为:

    S=W1·Lxy+W2·Lxy2+W3·Lxy3+W4·Lxy4+W5·Lxy5+W6·Lxy6+W7·Lxy7 (4)

    式 (4) 中相关参数含义如表1所示, 用户关注的影响力评分按照关注的时间长度来衡量。

    表1 用户交互相关参数说明 导出到EXCEL

    在表1中, 只有当交互评分到达阈值时, 才认为2个用户可能成为好友, 进而明确不同交互方式的影响力。

    2.4 KIFLink算法步骤

    KIFLink好友推荐算法基于链路预测方法, 在社交网络的数据库中为用户找到有可能成为好友的用户并进行推荐。该算法考虑社交网络自身具备的拓扑结构和用户的个人属性信息, 对链路预测中基于路径的相似度评价指标Katz Index进行适当变化, 从而构建适用于微博网络的用户交互评分计算公式, 通过计算该公式来衡量用户对于待推荐好友的态度倾向。

    社交网络好友推荐系统不只面向老用户, 还要考虑新用户, 为此, KIFLink算法对新用户进行了处理。按照用户的属性相似度查找一些相似的用户进行推荐, 然后跟据用户自身添加好友的结果进行更多的好友推荐。本文算法将用户的好友视为相邻节点, 从用户现有好友出发建立待推荐节点。此外, 由于微博自身存在的交互方式, 该算法也会在点赞、转发、评论的用户中组建待推荐列表, 从而实现更广泛的好友推荐。KIFLink算法的伪代码如下。

    3 实验结果与分析

    3.1 实验设置

    本文采用网络上的新浪微博数据集进行实验, 数据集时间范围是2014年1月—2014年6月, 包括用户的部分个人属性, 如住址、教育信息、个人爱好等。数据集中共含有202 524个用户, 用户关系数为11 352 326, 用户之间的交互数据量为12 345 421。本次实验使用Matlab平台进行仿真, 对KIFLink算法、基于用户的协同过滤算法 (User-CF) [23]以及使用局部路径相似度作为评价标准的链路预测方法 (local path) [24], 在准确率[25]、召回率[26]以及F1-measure[27]指标方面进行比较与分析。

    准确率、召回率以及F1-measure是在推荐领域中衡量算法效果的3个常用标准。准确率指算法得出的推荐结果中确实是目标用户的真实好友的数量在算法推荐用户总数中的比例, 该指标数值越大, 则表示推荐列表中的结果对用户越有效。召回率指算法得出的好友数目占数据集中的好友总数的比例, 该数值越大, 表示算法效果越好。F1-measure为综合衡量指标, 其是上述两者的综合。准确率和召回率这2个指标数值都是越大越好, 但在某些情况下2个指标会不一致, 此时需要通过F1-measure标准来进行判断, F1-measure值较大说明算法的效果较好。

    准确率的计算方法如式 (5) 所示。

    Ρrecision=ΤrueΡredicted    (5)

    其中, True代表推荐结果中真正的用户好友数量, Predicted代表推荐列表总数。

    召回率的计算方法如式 (6) 所示。

    Recall=ΤrueReal    (6)

    其中, Real代表测试数据集中的好友总数。

    F1-measure的计算方法如式 (7) 所示。

    F1-measure=2Ρrecision⋅RecallΡrecision+Recall    (7)

    3.2 实验对比算法

    在推荐系统中, 当一个用户A需要个性化推荐时, 可以先根据兴趣相似度找到类似用户, 然后将用户A未知的而其余用户感兴趣的物品推荐给用户A, 这就是基于用户的协同过滤算法。

    在本文中, 利用KIFLink算法向用户推荐好友, 即计算用户的兴趣相似度, 然后将与用户A相似的用户的好友推荐给用户A, 从而实现基于用户的协同过滤好友推荐。

    基于局部路径相似度的链路预测算法, 统计从节点A到节点B的路径长度以及条数, 并通过对不同路径长度的路径赋予权值来计算节点A和节点B的相似度。局部路径相似度算法只考虑路径长度在指定范围内的路径条数, 会丢失长度较长的路径信息, 从而使得考虑的数据不够全面。该算法通过计算路径条数以及权值来推荐好友。

    4 结束语

    本文通过对链路预测算法中的路径相似性指标进行分析, 采用Katz Index指标作为基础, 分析微博社交网络中存在的点赞、转发、关注、评论行为, 提出一种基于用户交互的微博社交网络用户相似度计算算法KIFLink。该算法同时考虑用户属性特征、网络拓扑结构以及网络特有的交互方式对用户之间建立好友关系的影响, 将用户视为2个节点来判断其之间建立链路的可能性, 进而向用户推荐好友。实验结果验证了KIFLink好友推荐算法的高效性和准确性。下一步将通过该好友推荐算法发现具有共同特性的社区。
