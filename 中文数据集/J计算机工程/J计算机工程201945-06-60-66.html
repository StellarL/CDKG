    摘    要：

    为降低多属性不等值连接操作的计算代价, 提出一种基于属性优选的不等值连接操作算法MIEJoin。按照连接属性对元组进行排序, 计算各连接属性的候选集大小, 在最小候选集中根据连接谓词进行筛选得到最终的结果集。在此基础上, 为提升系统的缓存命中率, 提出一种缓存敏感的多属性不等值连接算法CMIEJoin。基于MIEJoin算法建立元组的排列顺序数组, 在内存中邻近存储连续访问的数据, 以降低缓存的缺失次数并提升算法的运行效率。在TPC-H数据集上的实验结果表明, 与BIEJoin算法和NLJoin算法相比, CMIEJoin算法具有较高的运行效率。

    关键词：

    不等值查询; 不等值连接; 最小候选集; 缓存敏感算法; 查询处理;

    Non-Equi Join Operation Algorithm for Multi-attribute

    MENG Qingqiang HE Haoqi BI Nifei ZHAO Bin JI Genlin

    NARI Group Corporation (State Grid Electric Power Research Institute) School of Computer Science and Technology, Nanjing Normal University

    Abstract：

    To reduce the computational cost of multi-attribute non-equi join operation, an non-equi join operation algorithm MIEJoin based on attribute optimization is proposed.The tuples are sorted by join attribute, the candidate set size of each join attribute is calculated, and the final result set is filtered according to the join predicate in the minimal candidate set.On this basis, a cache-sensitive multi-attribute non-equi join algorithm CMIEJoin is proposed to improve the hit rate of the system cache.Based on the MIEJoin algorithm, the array of sorted tuples is established, and the continuousty accessed data is stored in memory in proximity to reduce the number of cache missings and improve the efficiency of the algorithm.Experimental results on the TPC-H dataset show that CMIEJoin algorithm has higher efficiency than BIEJoin algorithm and NLJoin algorithm.

    Keyword：

    non-equi query; non-equi join; minimal candidate set; cache-sensitive algorithm; query processing;

    0 概述

    连接是数据库查询处理中的一项重要操作, 其按照连接谓词的不同分为等值连接和不等值连接。等值连接操作算法主要包括嵌套循环连接算法、排序归并连接算法和哈希连接算法。嵌套循环连接算法通过二重循环的方法完成等值连接操作, 文献[1]采用单指令多数据 (Single Instruction Multiple Data, SIMD) 指令改进算法, 提升了连接操作的效率。对于排序归并连接算法, 可以通过使用SIMD指令并针对非一致性内存访问 (Non-Uniform Memory Access, NUMA) 环境进行改进[1,2,3]。哈希连接算法分为无分区/有分区哈希连接和Radix哈希连接。其中, Radix哈希连接算法通过对数据进行2次分区, 有效解决了TLB缺失的问题[1,3,4]。近年来, 一些研究人员针对特定的计算机硬件, 如Intel Xeon Phi CPU[5]、GPU[6]以及CPU和GPU的联合体系[7]等, 对连接算法进行改进, 另有一些学者基于Hadoop与Spark云平台分析并改进连接算法[8,9,10]。

    不等值连接操作算法主要包括嵌套循环连接算法、基于哈希的不等值连接算法、基于索引的不等值连接算法以及基于排序的不等值连接算法。基于哈希的不等值连接算法采用特殊的哈希函数对数据表进行分区段映射以快速完成不等值连接[11]。基于索引的不等值连接操作算法通过采用线段树索引来快速过滤不满足连接条件的数据, 最终提升不等值连接操作的执行效率[12]。文献[13]提出基于排序的不等值连接操作算法IEJoin。该算法对元组按属性排序, 利用数据的有序性, 可通过索引直接定位关键位置, 减少比较次数, 同时, 采用基于Bitmap[14]索引的位向量数组提升算法效率。由于IEJoin仅能处理2个连接属性的不等值连接, 因此文献[15]提出一种基于IEJoin的多属性不等值连接操作算法。该算法从多个属性组合中选择连接代价最小的2个属性, 使用IEJoin算法进行不等值连接并得到候选集, 然后根据其他属性的连接谓词过滤候选集, 最终得到结果集。

    在复杂数据的查询中, 通常会涉及多个属性的不等值连接, 现有不等值连接算法主要用于少量属性 (小于3) 的连接操作, 无法高效进行多个属性 (大于等于3) 的连接操作, 原因是多属性不等值连接的计算代价较高。为此, 本文提出一种基于属性优选的多属性不等值连接操作算法MIEJoin。该算法对2张关系表的元组分别按照属性进行排序, 记录元组id的排列顺序并建立索引。计算每个连接属性的候选集大小, 选择其中最小的候选集, 计算该候选集与其他属性对应候选集的交集然后得到最终的结果集。在此基础上, 为减少系统的缓存缺失次数, 基于MIEJoin算法, 本文提出一种缓存敏感的多属性不等值连接操作算法CMIEJoin。

    2 传统多属性不等值连接算法

    多属性不等值连接主要包括2类:嵌套循环连接, 基于IEJoin的多属性不等值连接[11]。

    2.1 嵌套循环连接

    嵌套循环连接算法NLJoin的基本思想是通过二重循环生成R表与S表的笛卡尔积, 然后逐一判断元组是否满足不等值连接条件。虽然该方法操作简单, 但是其连接产生的候选集规模过于庞大, 导致算法的计算代价过高, 执行效率较低。因此, 嵌套循环连接算法无法高效执行数据表的不等值连接。

    2.2 基于IEJoin的多属性不等值连接

    文献[13]提出针对2个连接谓词的不等值连接算法IEJoin。该算法根据不等值条件, 将元组按照不同的属性进行排序并记录。

    文献[15]提出基于IEJoin的多属性不等值连接算法BIEJoin。该算法通过预评估的方式确定产生规模最小候选集的2个连接谓词, 然后使用IEJoin进行连接得到候选集, 在此基础上, 根据其他属性的连接谓词过滤候选集并得到最终结果集。

    3 MIEJoin算法

    BIEJoin算法产生的候选集规模依然很大, 候选集的计算代价会降低整个算法的运行效率。为此, 本文提出一种基于属性优选的多属性不等值连接算法MIEJoin, 该算法主要包括以下2个阶段:

    1) 初始化阶段, 对R表和S表中的元组按照连接属性进行排序, 使用数组记录排序后的元组id信息, 然后对元组建立哈希索引。

    2) 连接阶段, 对2张表进行不等值连接。R表中每个元组的连接过程主要分为2步:一是计算S表中每个连接属性上的候选集并统计大小;二是计算各连接属性候选集的交集得到最终结果集。在各连接属性候选集中同时出现的元组就是满足连接条件的元组, 因此, 需要计算所有候选集的交集。为最小化连接操作的计算代价并提升效率, 在上述过程中, 只处理最小候选集然后得到结果集。

    MIEJoin算法在连接过程中每次都选取所有属性候选集中最小的候选集, 然后根据索引快速判断最小候选集中的元组在其他属性上是否满足不等值连接条件。由于针对R中每个元组的计算时, S表中参与计算的候选集规模均为最小, 因此2张表连接的计算代价可降为最低。算法的时间复杂度为O (knlb n+kmlb m+knz) , 其中, z为所有最小候选集的平均大小, 其会根据数据的分布和查询条件变化, 最大值为m。排序阶段的时间复杂度为O (knlb n+kmlb m) , 建立哈希表和计算偏移量数组的时间复杂度为线性时间, 连接阶段的时间复杂度为O (knz) 。该算法的空间复杂度为O (kn+km) 。

    4 CMIEJoin算法

    MIEJoin算法在处理最小候选集的过程中可能产生大量的缓存缺失。该过程通过查询哈希表得到同一个元组在其他属性上的位置, 进而判断该元组是否存在于其他属性的候选集中。但是, 由于数据表中的元组按照各属性分别排序, 打乱了原有的排列顺序, 导致查询哈希表会产生大量的随机映射。当数据规模增大时, 哈希表的规模也会变大。由于缓存的大小有限, 因此无法将整张哈希表全部存放在缓存中。此时, 频繁查询哈希表产生的随机映射会引发大量的缓存缺失, 最终降低算法的执行效率。为解决该问题, 本文对MIEJoin算法进行改进, 提出一种缓存敏感的多属性不等值连接算法CMIEJoin。

    MIEJoin算法的缓存缺失主要来源于算法1中的第14行～第20行。当处理的数据量变大时, 此阶段将频繁查询最小候选集中的元组在哈希表HTS中的位置信息, 从而造成大量的缓存缺失。该过程有2个特点:

    1) 在处理最小候选集时, 按照候选集对应数组中元组排列顺序读取元组并查询哈希表。根据此特性, 本文将元组在哈希表中的位置信息按照产生最小候选集的属性中元组排列顺序进行连续存储, 建立特定的位置数组。在此数组上频繁地进行连续访问操作, 可以发挥程序访问的空间局部性优势, 大幅减少系统缓存的缺失次数。如图2所示, Mi为图1中的数组, A1是针对M1建立的特定位置数组, A1中的数据是M1对应位置的元组在M2和M3中的位置。例如, M1[1]=s3在M2、M3中的位置分别为3、4。

    2) 每次连接产生最小候选集的属性不确定。由于元组按照各属性单独排序会导致元组在不同属性中的排列顺序不同, 因此需要为S表中的每个连接属性单独建立一个与其元组排列顺序相同的位置数组。如图2所示, M1、M2和M3分别对应数组A1、A2和A3。

    改进后的CMIEJoin算法整体框架与MIEJoin算法相同, 主要区别在于最小候选集的处理阶段。在该阶段, CMIEJoin算法首先选择与产生最小规模候选集属性相对应的位置数组, 然后顺序扫描最小规模候选集在该数组上的范围, 依次根据位置数据判断元组是否落在其他属性的候选集中。CMIEJoin算法可大幅减少缓存的缺失次数, 提升算法的执行效率。由于可通过哈希表在线性时间内生成位置数组, 因此该算法的时间复杂度与MIEJoin算法相同, 且其位置数组的空间占用仅为哈希表HTS的两倍, 因此, 其空间复杂度与MIEJoin算法也相同, 都为O (kn+km) 。

    5.2 性能分析

    5.2.1 查询时间与数据集大小之间的关系

    当连接属性的个数固定时, 随着数据规模的增加, 各算法执行查询Q2和Q3的运行时间变化分别如图3、图4所示。可以看出, NLJoin算法的执行效率最低, 并且随着数据规模的增加, 其执行2个查询的运行时间增长趋势相同, 原因是NLJoin算法只与数据集的规模相关, 与查询条件无关。因此, 在连接属性的个数和数据集规模都相同的情况下, 连接条件的变化对NLJoin算法的运行时间没有影响。另外3个算法在数据集规模相同时, 执行查询Q3的时间是执行查询Q2时间的两倍。原因是这3个算法的计算代价与查询语句产生的结果集规模有关, 查询Q2与查询Q3产生的结果集规模相差较大, 如图5所示。在3种算法中, CMIEJoin算法耗时最少, 并且随着数据规模的增加, 算法运行时间的变化趋势始终最缓慢, 其次是MIEJoin算法。这是因为CMIEJoin算法针对MIEJoin算法在计算交集阶段产生大量缓存缺失的缺陷进行了改进, 进一步提升了执行效率。2个算法在计算交集阶段产生的缓存缺失次数统计如表2所示, 从表2可以看出, 在执行查询Q2和Q3时, CMIEJoin算法产生的缓存缺失次数明显少于MIEJoin算法。在CMIEJoin、MIEJoin、BIEJoin 3种算法中, BIEJoin算法的执行效率最低, 在执行查询Q3时运行时间与NLJoin算法相当, 原因是BIEJoin算法在执行Q3的过程中会产生大量的候选集, 对候选集的处理增加了算法的计算代价, 降低了算法的执行效率。

    5.2.2 查询时间与连接属性个数之间的关系

    在数据规模不变的情况下, 随着连接属性个数的增加, 各算法运行时间的变化趋势如图6所示。从图6可以看出, CMIEJoin算法的执行效率最高, 其次是MIEJoin算法, 且2个算法的运行时间都随着属性个数的增加先下降后小幅提升。原因是这2个算法在每个元组的连接过程中都选取最小规模候选集进行计算得到连接结果, 当大量规模最小的候选集由新添加的属性产生时, 算法的计算代价会进一步降低, 当新添加的属性对最小规模候选集的选取影响不大时, 随着属性个数的增加, 算法的运行时间会小幅提升。当连接属性较少时, BIEJoin算法的执行效率与MIEJoin算法相差不大。但是, 随着连接属性个数的增加, 2个算法的运行时间差值逐渐变大。这是因为BIEJoin算法首先通过预评估的方法确定2个连接代价最小的属性, 当属性的个数较少时, MIEJoin算法与BIEJoin算法的连接计算代价相差不大, BIEJoin算法对候选集的处理耗时较少。但随着连接属性个数的增加, MIEJoin算法每次连接都选择规模最小的候选集进行计算得到最终结果集, 与需要处理大规模候选集的BIEJoin算法相比, 大幅降低了连接的计算代价。此外, 随着连接属性个数的增加, BIEJoin算法运行时间的增长趋势与执行效率最低的NLJoin算法相同, 原因是BIEJoin算法对候选集的处理与NLJoin算法相同。从图7可以看出, 在连接属性个数增加的过程中, 前5个属性的连接条件能够过滤掉大量元组, 因此, BIEJoin和NIJoin 2个算法前期的增长趋势较缓慢。在此基础上添加的2个属性对结果集规模的影响不大, 原因是结果集规模较小, 新增加的属性产生的计算代价对算法的执行效率影响较小。因此, 随着连接属性个数的增加, 本文CMIEJoin算法执行效率最高且最稳定。

    6 结束语

    传统多属性不等值连接算法计算代价较高, 为此, 本文提出一种基于属性优选的多属性不等值连接算法MIEJoin。为提高系统的缓存命中率, 提升算法的执行效率, 基于MIEJoin算法提出一种缓存敏感的多属性不等值连接算法CMIEJoin。实验结果验证了CMIEJoin算法在不同连接条件下的可行性与高效性。下一步将研究如何降低基于多表的多属性不等值连接的计算代价。
