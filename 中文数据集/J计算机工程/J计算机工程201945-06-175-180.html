    摘    要：

    为提高入侵检测效率, 需要对数据进行特征提取以降低数据维度。结合信息增益 (IG) 和主成分分析 (PCA) , 提出一种网络入侵检测方法。通过IG提取分类能力强的属性特征, 利用PCA对其降维, 并采用Naive Bayes进行分类检测。对数据集KDDCUP99进行测试, 结果表明, 该方法的检测率为94.5%, 高于PCA-LDA、FPCA、KPCA方法。

    关键词：

    信息增益; 主成分分析; 入侵检测; 特征提取; 降维;

    Network Intrusion Detection Method Based on Information Gain and Principal Components Analysis

    WANG Xuren MA Huizhen FENG Anran XU Yi'na

    College of Information Engineering, Capital Normal University Key Laboratory of Network Assessment Technology, Institute of Information Engineering, Chinese Academy of Sciences

    Abstract：

    In order to improve the efficiency of intrusion detection, it is necessary to extract the features of data to reduce the data dimensions.This paper proposes a network intrusion detection method by combining Information Gain (IG) and Principal Components Analysis (PCA) .The method uses IG to extract the attribute features with strong classification ability, uses PCA to reduce the dimension of the feature data, and uses Naive Bayes method for classification and detection.The test results of the data set KDDCUP99 show that the detection rate of the method is 94.5%, which is much higher than those of PCA-LDA, FPCA, and KPCA methods.

    Keyword：

    Information Gain (IG) ; Principal Components Analysis (PCA) ; intrusion detection; feature extraction; dimension reduction;

    0 概述

    网络入侵检测是保障网络安全必不可少的一项技术, 其通过识别网络中的访问数据是入侵行为还是正常行为来保护网络, 本质是一个二分类问题。入侵检测主要分为误用检测和异常检测。误用检测通过与已知的攻击进行对比来识别异常或潜在的行为, 对已知的攻击具有较好的检测率, 但是检测不到新的或不常见的入侵。异常检测[1,2]根据网络中正常的数据链接或正常的访问行为建立“normal”模型, 当数据链接或访问行为不在此前建立的模型范围内, 则生成警告。机器学习[3,4]和统计方法[5,6]在网络入侵方面的应用提高了入侵检测率, 但是网络数据的冗余属性和不相关属性依然影响入侵检测的效率。提取和选择数据中主要信息或相关性高的信息对提高入侵检测的效率非常重要。

    信息增益 (Information Gain, IG) [7]作为一种特征选择算法, 被用于决策树生成过程的根节点选择。IG表示得知特征X的信息而使得类Y的信息的不确定性减少的程度。不同的特征具有不同的IG, IG大的特征具有更强的分类能力。文献[8]使用IG对KDDCUP99数据进行特征的相关性分析, 对每个属性计算IG, 并将其大小作为衡量属性对类标签的影响。

    主成分分析 (Principal Components Analysis, PCA) 将对协方差矩阵影响最大的方向上的数据值作为主成分, 用这些数据值构成新子空间表示原始数据, 从而达到降维的目的。PCA作为特征提取方法之一被广泛应用于模式识别、图像处理等领域, 同样在网络入侵检测中得到了有效的应用[9,10,11]。近期研究人员将主成分分析与线性判别分析组合 (Principal Components Analysis and Linear Discriminant Analysis, PCA-LDA) [12]、模糊主成分分析 (Fuzzy Principal Components Analysis, FPCA) [13]、核主成分分析 (Kernel Principal Components Analysis, KPCA) [14]应用于网络入侵检测, 提高了其检测率。

    本文提出一种IG与PCA相结合的特征提取方法 (简称IG_PCA方法) 。IG的特征选择可以找到对分类能力强的属性, 从而删除冗余的属性, 而PCA利用协方差矩阵来提取主成分, 在特征提取方面具有一定的优势。两者结合可以最大程度地去除冗余信息并提取主要组成信息, 充分利用数据中的有用信息, 降低数据的维度。

    1 属性相关性分析

    1.1 信息增益

    设S为具有类标签的训练样本集。假设有c个类, 第i个类包含si个训练样本, s为总的训练样本个数。训练样本集S的经验熵计算如下:

    Η (s1, s2, ⋯, sc) =-∑i=1csis lb sis          (1)

    特征f有v个不同的取值{ρ1, ρ2, …, ρv}, 根据特征f的取值将样本集S分为v个子集{S1, S2, …, Sv}, Sj是特征f取值为ρj的样本集合, 其中sij是Sj第i类的样本个数, 特征f的经验条件熵计算如下:

    E (f) =∑j=1vs1j+s2j+⋯+scjs×Η (s1j, s2j, ⋯, scj)           (2)

    则特征f的IG为:

    Gain (f) =H (s1, s2, …, sc) -E (f) (3)

    1.2 主成分分析

    PCA被广泛地应用于许多领域的数据分析和预处理, 其实际上是一种降维方式。它的主要目的是减少初始变量的维数, 同时尽可能地保留这些样本中存在的方差, 将许多相关性高的变量转化成彼此相互独立或不相关的变量。

    设一个数据集有m个对象x1, x2, …, xm, 每个对象包含n个变量。为了获得n′ (n′<<n) 个主成分, PCA过程如下:

    步骤1 原始数据标准化:

    Ζij=xij-x—jsj, i=1, 2, ⋯, m, j=1, 2, ⋯, n          (4)

    其中, x—j=∑i=1mxijm, sj2=∑i=1m (xij-x—j) 2m-1。

    步骤2 求相关系数矩阵:

    R=ΖΤΖm-1          (5)

    步骤3 求R的特征根λ1≥λ2≥…≥λn及相应的单位特征向量:

    ai=[α1iα2i⋮αni], i=1, 2, ⋯, n          (6)

    步骤4 计算主成分:

    ti=α1iZ1+α2iZ2+…+αniZn, i=1, 2, …, n′ (7)

    以n′个主成分作为新的数据向量, 取代原始数据。

    经过属性特征提取后, 可以最大程度地去除不重要或冗余的属性特征。

    1.3 IG与PCA的组合

    本文提出组合的降维方法。利用IG和PCA各自的优点最大限度地降低数据维度, 同时提高检测率。

    定义1Finit={f1, f2, …, fn}, 原始数据的属性集合Finit包含n个属性。

    定义2FIG={FIG1, FIG2, …, FIGi}, i=1, 2, …, k, 其中, FIGi表示第i个属性子集, |FΙGi|=i, FΙG (i-1) ⊆FΙGi。k为属性子集的数目, k<n。例如, FIG1={fi}, fi为IG值最大的属性, i表示Finit的第i个属性。

    定义3FPCA={FPCA1, FPCA2, …, FPCAi}, i=1, 2, …, k, k<n。其中, FPCAi表示第i个属性子集, |FPCAi|=i, FPCA (i-1) ⊆FPCAi。例如, FPCA1={t1}, t1表示最大特征值对应的主成分。

    定义4FIG_PCA={FIG1_PCA, FIG2_PCA, …, FIGi_PCA}, i=1, 2, …, k, 其中, FIGi_PCA表示由FIGi属性子集生成得集合, FIGi_PCA={FIGi_PCA1, FIGi_PCA2, …, FIGi_PCAl}, l=1, 2, …, L, 其中, FIGi_PCAl表示第l个属性子集, 且|FΙGi_ΡCAl|=l, FΙGi_ΡCA (l-1) ⊆FΙGi_ΡCAl。k、l为属性子集的数目, k<n, L≤k。例如, FIGi_PCA1={tIGi_PCA1}, tIGi_PCA1表示最大特征值对应的主成分。

    本文方法的具体步骤如下:

    1) 生成属性集FIG={FIG1, FIG2, …, FIGi}, i=1, 2, …, k。

    2) 对FIG进行字符属性数值化。

    3) 对步骤2中的每个子集进行PCA。

    4) 生成属性集FIG_PCA={FIG1_PCA, FIG2_PCA, …, FIGi_PCA}, i=1, 2, …, k。

    5) 对步骤4得到的属性子集进行评估, 选取最优子集FIGi_PCAl 。

    2 入侵检测

    2.1 数据集及评估方法

    2.1.1 数据集

    本文使用KDDCUP99数据集[15]。该数据集由1998年DARPA入侵检测数据经整理获得, 因10%的训练数据与10%的测试数据被用于1999年KDD CUP竞赛中, 成为著名的KDDCUP99数据集。KDDCUP99数据集包括训练与测试数据集, 其包含的39种攻击被分为以下4种主要类型:

    1) 拒绝服务攻击 (DOS) :攻击者设法让目标机器停止提供服务, 例如泛洪攻击等。

    2) 远程攻击 (R2L) :从远端机器上的非授权读取, 例如基于字典的口令猜测。

    3) 本地用户非法提升权限的攻击 (U2R) :非授权地获取管理员权限, 例如各种“缓冲区溢出”攻击。

    4) 网络扫描 (Probing) :监视和其他探测技术, 包括端口扫描和漏洞扫描。

    KDDCUP99数据集的每条链接由41个属性 (记为fi, i=1, 2, …, 41) 和1个类标签组成, 其特征可以分为4个主要类型:

    1) TCP连接基本特征 (f1～f9, 其中包括3个字符属性) :包含了一些连接的基本属性, 如连续时间、协议类型、传送的字节数等。

    2) TCP连接的内容特征 (f10～f22) :用于评估原始TCP数据包的有效负载, 如登录失败的次数等。

    3) 基于时间的网络流量统计特征 (f23～f31) :2 s时间单元的流量特征。

    4) 基于主机的网络流量统计特征 (f32～f41) :按照目标主机进行分类, 使用一个具有100个连接的时间窗, 统计当前连接之前100个连接与当前连接具有相同目标主机的统计信息。

    本文实验使用的训练集包含494 021个链接, 其中有97 278个正常链接, 22种攻击类型。测试集包含311 029个链接, 正常链接有60 593个, 其中14种在训练数据集中不存在的新的攻击类型。

    2.1.2 字符属性数值化

    KDDCUP99数据集有41个属性, 其中包括3个字符属性, 然而字符属性在某些分类算法 (如K-means、KNN等) 中无法使用, 在其他算法中也可能会影响分类效果。为保证数据的属性为标准格式属性, 将字符属性数值化[9]。假设一个字符属性包含m个值, 将这m个值与m个坐标一一对应, 对应的属性值记为1, 否则为0。例如, KDDCUP99数据集的“protocol type”的属性值包括tcp、udp、icmp, 如果一条链接的数据的 “protocol type”的值为tcp (udp或icmp) , 可转换为 (1, 0, 0) ( (0, 1, 0) 或 (0, 0, 1) ) 。由于“protocol type”包含3个不同的字符属性值, “service”包含70个不同的字符属性值, “flag”包含11个不同的字符属性值, 所以KDDCUP99数据集由原来的41属性变为122个属性。

    文献[16]提出了另一种字符属性数值化方法:协议类型“protocol type”可数值化为icmp-0、tcp-1、udp-2、others-3;“flag”可数值化为rej-1、rsto-2、rstr-3、s0-4、s3-5、sh-6、sf-7、others-8;由于“service”的属性值过多, 为避免数据维度过大, 该属性忽略不计。

    由于不能确定被忽略“service”属性对分类精度的影响, 因此本文采取文献[9]方法对数据进行字符属性数值化。

    本文采用检测率 (Detection Rate, DR) 、误检率 (False Positive Rate, FPR) 、精度 (Precision) 和F-Score作为评估参数衡量分类效果。一个性能好的入侵检测具有高的检测率、精度和F-Score以及低的误检率。

    3 结束语

    提高入侵检测率一直是安全类软件的目标, 而网络数据中的冗余信息严重影响检测的准确性。本文提出IG_PCA方法, 结合IG特征选择和PCA特征提取的优点对数据属性进行提取。采用Naive Bayes对KDDCUP99数据集进行分类检测, 结果表明, 本文方法将数据维度降为16时检测率比PCA-LDA、FPCA、KPCA方法高出0.3。但是该方法没有提高对各攻击类的检测率, 下一步将对此进行改进。
