    摘    要：

    针对现有调度方法多数未考虑虚拟网络功能在实例化过程中的虚拟机选择问题, 提出一种新的虚拟网络调度方法。建立基于马尔科夫决策过程的虚拟网络功能调度模型, 以最小化所有服务功能链的服务延迟时间。通过设计基于Q-learning的动态调度算法, 优化虚拟网络功能的调度顺序和虚拟机选择问题, 实现最短网络功能虚拟化调度时间。仿真结果表明, 与传统的随机虚拟机选择策略相比, 该方法能够有效降低虚拟网络功能调度时间, 特别是在大规模网络中调度时间可降低约40%。

    关键词：

    网络功能虚拟化; 服务功能链; 调度模型; 马尔科夫决策过程; Q-学习;

    Scheduling Method of Virtual Network Function Based on Q-learning

    WANG Xiaolei CHEN Yunjie WANG Chen NIU Ben

    National Digital Switching System Engineering Technological Research Center National Engineering Laboratory for Mobile Network Security

    Abstract：

    Most of the existing scheduling methods do not consider the virtual machine selection problem of the virtual network function in the instantiation process, a new virtual network scheduling method is proposed. It establishes virtual network function scheduling model based on Markov decision process, minimizes all Service Function Chain ( SFC) service delay time. By designing a dynamic scheduling algorithm based on Q-learning, and optimizing the scheduling order of virtual network functions and virtual machine selection, the Network Function Virtualization (NFV) scheduling time is the shortest. Simulation results show that compared with the traditional random virtual machine selection strategy, this method can effectively reduce the virtual network function scheduling time, especally reduce the scheduling time by about 40% in large-scale networks.

    Keyword：

    Network Function Virtualization (NFV) ; Service Function Chain (SFC) ; scheduling model; Markov decision process; Q-learning;

    0概述

    网络功能虚拟化 (Network Function Virtualization, NFV) 作为5G移动互联网的使能技术, 能够为移动运营商提供灵活的业务编排和管理能力[1]。在传统网络中, 运营商需要大量的专用硬件设备来承载相应的网络功能。因此, 实现一个新的网络功能往往伴随着部署新的专有硬件设备。硬件与功能紧耦合的部署方式不仅占用物理空间, 而且使得资本、运营和能耗成本增加。NFV将网络功能从专用硬件中解耦, 以软件的形式在商用服务器上实现各种网络功能, 即虚拟网络功能 (Virtual Network Function, VNF) 。在NFV环境下, 多个VNF以链状的形式构成服务功能链 (Service Function Chain, SFC) 完成特定的网络服务[2]。通过NFV的部署, 网络的集中管理和编排更加方便[3]。针对一个特定的网络服务, 运营商可以动态地选择一组VNF构成相应的链, 然后控制数据流通过整个服务功能链, 完成相应的网络服务。根据业务的不同, 需要实现有效的VNF部署策略, 以减少资本支出和运营支出[4], 而VNF调度是其中需要进一步研究的关键问题[5]。

    文献[6]定义了VNF调度问题, 将其建模为车间调度模型, 但是没有提出多项式时间内的解决方案。文献[7]提出NFV环境下具有较低复杂度的多资源数据包调度算法, 该算法考虑了数据包队列的特性, 但未考虑服务功能链的特性。文献[8]对服务功能链的端到端时延进行了分析, 然而其假定时延具有固定的值, 这在实际中并不适用, 不能应用在动态的网络服务中。文献[9]在FJSP模型的基础上引入数据流量在虚拟链路上的传输时延, 建立了虚拟网络功能调度问题的混合整数线性规划模型, 并设计了基于遗传算法的虚拟网络功能调度方法, 但其提出的遗传算法容易早熟, 陷入局部最优, 无法获得更好的调度方案。

    综上所述, 现有研究存在2个方面的不足, 通过预定义固定的网络参数, 将调度问题转化为静态计算问题, 但是在真实网络环境下, VNF处理时间等网络参数具有随机性 (如物理设备宕机、网络拥塞等外界因素) , 利用传统静态方法无法有效兼容系统的不确定性, 未考虑VNF实例化阶段的虚拟机选择问题。

    针对上述问题, 本文设计一种基于Q-learning的动态调度算法。该算法在求解过程中能够动态适应网络参数的随机变化, 以保证算法结果的实时性, 同时给出一种复合行为选择策略, 优化VNF调度顺序和虚拟机选择问题。

    1问题描述

    文献[5]将NFV资源调度问题分为SFC映射和VNF调度2个阶段, 本文研究的VNF调度问题不能离开SFC映射问题单独存在。为此, 本文将从2个方面分阶段进行问题描述。

    1.1 SFC映射

    SFC映射主要关注如何找到服务功能链中各个VNF在物理网络上实例化的最优节点位置, 即确定VNF和物理节点的映射关系[10], 如图1所示。图1中用户请求的服务功能链由3个VNF组成, 物理网络共有6个节点, 带有箭头的虚线表示该VNF可映射的物理节点集合。文献[11]提出严格的一一映射算法会限制VNF调度问题求解的可行域, 从而影响VNF调度算法的效果。为验证算法有效性, 本文松弛了VNF的位置约束, 采用更加灵活的映射算法获得VM候选集合, 并设计相应的虚拟机选择方法, 进一步提高调度算法性能。

    1.2 VNF调度

    在SFC映射阶段, 映射算法限制了不同VNF可映射的虚拟机范围。基于映射结果, VNF调度重点解决如何确定各个节点上SFC的执行顺序, 以及针对各个VNF如何选择候选集合中的虚拟机, 以期最小化所有服务功能链执行的总时间[12]。VNF调度问题关注的是如何确定各个节点上VNF的执行顺序, 以在满足优先级约束和位置约束的基础上, 最小化所有服务功能链执行的总时间。图2描述了一个VNF调度的示例, 并给出一种调度问题的解决方案 (位置约束并没有在图中给出) , 即如何在有限的4个VM上, 以最短的时间完成3个服务功能链请求。

    2.2基于马尔科夫决策过程的调度模型

    本文利用已定义的参数, 结合离散时间马尔可夫决策过程描述VNF调度问题。模型可表示为四元组{S, A, r, J}[13], 其中, S表示有限的VNF部署状态空间, A表示有限的行为空间, r表示收益函数, J表示折扣总收益。状态空间S、行为空间A和收益函数r是模型建立的关键元素, 其详细定义如下:

    1) 状态空间:引入可调整的状态划分参数d将连续的调度压力分割成不同的状态。表1给出一个包含6个调度状态的状态空间示例。实验通过改变状态划分参数d来改变不同调度状态下调度压力 (p) 的粒度, 细粒度的状态划分有利于提高行为选择的精确度, 但是过细的粒度会增大状态空间, 影响求解算法的收敛时间。

    表1 包含6个调度状态的状态空间示例    下载原表

    2) 行为空间:现有研究大多关注同一虚拟机中属于不同SFC的VNF调度顺序优化, 忽略了VNF实例化阶段的虚拟机选择问题。本文设计了4类“复合行为”, 在确定VNF调度顺序的同时进行虚拟机选择, 通过求解最优复合行为策略, 进一步优化VNF调度算法的的性能。

    行为1:优先选择具有最高调度优先级的 (Schedule Priority, SP) 服务功能链以及具有最快完成时间 (Earliest Finish Time, EFT) 的虚拟机实例化相应的VNF。

    行为2:优先选择最先到达的服务功能链以及具有最快完成时间的虚拟机实例化相应的VNF。

    行为3:优先选择第1个可用的虚拟机实例化相应的服务功能链上的VNF。

    行为4:保持现有状态不变。

    在SFC实例化阶段, 系统将根据当前状态选择相应的行为, 并给出当前状态下执行该行为的即时收益, 通过最大化系统的折扣总收益获得最优的调度策略。

    3) 收益函数:收益函数的设计决定了系统的优化目标, 本文模型旨在最小化全部服务功能链的平均延迟时间。综上所述, 利用EMLT定义收益函数r, 如式 (5) 所示。

    本文利用马尔可夫决策过程描述VNF调度问题, 建立了以调度压力为评价标准的状态空间, 并提出复合行为选择策略。该模型可以同时考虑同一个虚拟机上不同SFC的调度顺序和同一个SFC上不同VNF的虚拟机选择问题, 相比于传统的Job-shop模型能够进一步优化调度性能。

    3基于Q-learning的VNF调度算法

    现有研究提出基于整数规划或以遗传算法为代表的智能算法来求解VNF调度模型, 这些方法通过预定义固定的网络参数, 将调度问题转化为静态计算问题。但是在真实网络环境下, VNF处理时间等网络参数具有随机性 (如物理设备宕机、网络拥塞等外界因素) , 利用传统静态方法无法有效兼容系统的不确定性, 此时增强学习算法成为一种有效的求解手段。调度算法的伪代码如算法1所示。

    4实验结果与性能分析

    4.1实验环境和参数设置

    本文实验在Intel Core i7-4790 3.60 GHz CPU, 8 GB内存, Linux系统的PC机上运行, 底层网络拓扑由GT-ITM工具生成[15], 仿真程序在Matlab环境下实现并运行。假设每对物理转发节点之间等概率生成物理链路, 考虑到物理链路时延的随机性, 选择期望取值范围在[0, 100]ms的平稳随机过程表示时延。为了评估算法的自适应性, 令服务请求RL动态到达和离开, 且服从强度为0到1 000的泊松分布。每个服务请求由一条服务功能链表示, 服务功能链中VNF的类型和数量满足2≤l≤m≤10。在改进的Q-learning算法中, 最大学习周期Max＿t=1 500, 最大学习步长Max＿step=1 400, 初始状态下学习因子α0=0.8。

    表2给出根据不同行为进行虚拟机选择的调度结果。在不同的测试中, 固定行为选择方法的性能波动较大, 在本文测试例中, 行为2的表现明显优于其他行为, 在随机选择策略中, 算法等概率选择不同行为部署VNF, 该方法能够避免调度方法陷入较差的选择策略, 但同时其无法获得最优选择策略。本文提出的基于Q-learning的调度方法能够利用即时收益选择最优调度策略, 使调度算法的性能进一步提高, 但是基于Q-learning方法的计算复杂度较高, 相比其他方法, 算法执行需要的CPU时间更长。

    5结束语

    本文主要研究虚拟化环境下VNF调度问题, 并设计了基于Q-learning算法的调度方法进行求解。该方法引入复合行为选择策略, 同时优化同一虚拟机上不同VNF的调度顺序和VNF实例化阶段不同虚拟机的选择问题。仿真结果表明, 与传统的随机选择方法相比, 本文提出方法具有较短的调度时间。但由于算法复杂度较高, 下一步将对算法的计算复杂度进行优化, 降低其CPU时间。
