    一种领域知识图谱的本体根类型设计

    王亚强 臧根林 吴庆蓉 占春丽 谢新扬

    摘    要：

    对象根类型的分类是构建领域知识图谱的基础工作,而现有流行的公共知识库并没有按照领域数据的特点分类,如SUMO的唯一的根类型就是实体,使得领域知识表达存在一定挑战,无法完整地表达如文本、视频图片等知识和关联。领域知识图谱本体的对象根类型除了实体类型,还应增加事件类型、文本类型和多媒体类型,基于4种根类型再扩展到具体分类来表达领域的知识,采用这种方式,可以更好地描述各种典型的领域场景,在实际知识图谱的工具研发中,体现出良好的知识体系和清晰的分类思路。

    Object root types design of domain knowledge graph ontology

    WANG Ya-qiang ZANG Gen-lin WU Qing-rong ZHAN Chun-li XIE Xin-yang

    TRS Knowledge Graph Research Institute Guangzhou TRS Big Data Technology Co. Ltd.

    Abstract：

    The classification of the object root types is the basic work of constructing the domain knowledge graph, and the popular public knowledge repository is not classified according to the characteristics of the domain data. For instance, the only root type of SUMO is entity, which brings defects to the domain knowledge expression. Knowledge such as texts, videos and pictures, and their relationships cannot be fully expressed. The object root types of domain knowledge graph ontology should include not only the entity type but also the event type, the text type and the multimedia type. Based on the four root types, types are extended to express the domain knowledge. In this way, various typical domain scenarios can be well described. The proposal has a good knowledge system and clear classification ideas in the development of the actual knowledge graph tools.

    1 引言

    知识图谱KG(Knowledge Graph)旨在描述客观世界的概念、实体、事件以及它们之间的关系[1,1]。知识图谱的本质是有向图结构的知识库[2,2],是人工智能应用不可或缺的基础资源[3,3]。知识图谱分为2类:通用知识图谱和领域知识图谱[4,4]。通用的知识图谱可以认为是百科全书,比如Google 知识图谱[5]、DBpedia[6,7]、YAGO[8,9]和Freebase[10]等;领域知识图谱是基于领域数据构建的知识图谱,用于领域分析研究工作。通用知识图谱的本体(Ontology)根类型基本上就是实体(Entity),然后在实体上再分类。通用知识图谱和领域知识图谱,在基础架构上基本相同,但由于领域知识图谱面向特定的领域数据和场景,从构建和应用上来讲更加复杂。本文所研究的领域,主要是相关的情报分析、舆情分析、智库研究等特定领域。

    SUMO(Suggested Upper Merged Ontology)及其领域本体是当今存在的最大的公共本体,主要用于搜索、自然语言处理和推理等应用,包括约25 000多个词条(terms)和80 000多条公理(axiom)。本体根类型包含3层共12个类[11],如图1所示。

    领域本体模型的研究刚刚起步。目前国内学者已经做了很多探索性研究,涉及农业、医药、植物学等。暨南大学附属医院的王美琴等[12,12]提出构建医学本体库;中国农业大学的赵明等[13,13]提出了植物领域知识图谱本体非分类关系提取方法;中国科学院文献情报中心的马雨萌等[14,14]对文献情报领域的本体模型做了系统的研究,并且对不同层次的本体模型进行了分析,如顶层本体、领域上层本体和领域应用本体。

    理论上,领域知识图谱的本体可参照通用知识图谱的本体设计,但类似SUMO这种仅以实体为根类型的分类方法,在领域知识图谱工程中会遇到很多问题。一方面这种分类方法太过复杂,分析人员往往无法直接理解;另一方面在描述领域的对象如文本类型方面没有足够的针对性。领域知识图谱不是为了描述整个世界,而是要满足领域的业务需求,所以需要为领域设计专门的本体根类型。SUMO主要的根类型是实体,但除了主要的领域实体外,一些领域还关注实体相关的事件行为,如电话的通话记录、个人的订票信息、银行卡的交易记录等。很多领域需要研究新闻文章、学术论文、邮件、电子笔录等文本对象。随着计算机和互联网的发展,多媒体也成为某些领域的关注对象,比如新闻视频、短视频、监控录像等。

    Schema.org有事件(Event)、作品(CreativeWork)等概念,但第1层的8个分类从体系上是面向通用领域的,涵盖社会、人类文明、重大事件等内容。结合本文所研究的特定领域的对象类型和数据类型分析,面向特定领域进行了重新梳理和概括,本文提出在领域知识图谱中将根类型分为4种,包括实体类型、事件类型、文本类型和多媒体类型,在此基础上可构建出更适合领域分析的知识图谱。

    2 领域主要对象类型分析

    本体(Ontology)源自西方哲学,用以描述世界上的客观存在。比较流行的本体概念是美国斯坦福大学知名学者Gruber[15]提出的,即本体是关于某个实体概念体系的明确规范的说明。知识图谱中对知识数据的描述和定义,被称为知识体系(Schema)或者本体(Ontology)[3,3]。本体是重要的知识库[16,16],内容包括对象的类型Co、属性的类型Ca和关系的类型Cr,本文主要讨论对象的类型Co。

    知识图谱可用于各个行业领域的数据整合和业务分析,比如政府分析、媒体分析、电商分析、金融分析[17,17]、公安情报分析[18,18]、学术领域分析[19,19]、学科分析[20,20]、军事信息搜索[21,21]等。领域本体(Domain Ontoloty)是领域知识图谱的基础概念,面向不同领域有不同的领域本体。在领域知识图谱工程中,最先需要确定的是本体中的对象类型。各个领域都有自己关注的主要对象,领域的信息化是围绕这些对象产生和收集数据,所以在构建领域知识图谱中,领域关注的主要对象就是知识图谱中主要的对象类型。表1是我们总结的多领域的主要对象类型。

    通过梳理表1中各个领域的对象类型,可以抽象出实体类型ce和事件类型ct。实体类型包括人物、组织、电话、产品、物品、证件等,主要特点是能够与现实世界相对应,在自己的领域中能够作为主体存在;事件类型包括企业登记、通话记录、股票交易、订单、轨迹等,可以是实体的事件行为(如个人的移动轨迹),或者是实体之间的事件行为(如通话记录)。因此,很多领域知识图谱项目,把对象根类型Coroot分为实体类型和事件类型2种,如下所示。

    Coroot={ce,ct}         (1)

    对于本文研究的特定行业领域,如情报分析、舆情分析、智库研究等,文本、图片、视频等所占的空间比例非常大,表1中带下划线的对象类型就是文本类型和多媒体类型。例如,在媒体领域,每天产生大量的文章和新闻,因此文本类数据占据了很重要的地位,有在文本中进行提取分析的业务需求,所以在这类领域中,文本T应当作为一种基础对象类型C1o;在电视媒体、公安等领域,每天产生海量的新闻视频和监控视频,同样需要在视频中分析、研究、查找,所以视频V等多媒体类型,也应该作为一种对象基础类型C2o。

    3 领域的数据类型分析

    领域知识图谱的数据来源是领域数据Df。领域数据一般分为3类,第1类是结构化数据Dstructure,通常保存在关系数据库的二维表[r(⋅),c(⋅)]中;第2类是半结构化数据D′structure,典型的半结构化数据是电子邮件、百科、网页等,主要特征是以文本T为主,再加上其他结构化数据Dstructure;第3类是非结构化数据Dcommon,图片I、声音A和视频V是典型的非结构化数据,这些数据需要结构化以后,其承载的信息才能被计算机识别。结构化处理表示如下:

    Dstructure=Fstructure(Dcommon:{Ι,A,V,⋯})         (2)

    本体中的实体类型ce和事件类型ct主要来源于结构化数据Dstructure,因此有专家认为知识图谱是以结构化的方式Fstructure(·)描述世界。对于通用知识图谱来说的确如此。但是,在某些领域,文本和视频等半结构化和非结构化的内容{D′structure,Dcommon},也是领域知识的重要组成部分,是分析人员的分析对象,所以有必要将文本类型ct、多媒体类型c(I,A,V),和实体类型ce、事件类型ct结合在一起构成领域的对象根类型Coroot,实现知识图谱对领域知识的完整表达。领域数据通过知识构建,形成具有4种对象根类型的知识图谱C(G)={Ceroot,Ctroot,CΤroot,CΜroot}:

    G=g(ce)+g(ct)+g(cΤ)+g(c(Ι,A,V))=Dstructure+D′structure+Fstructure(Dcommon)         (3)

    4 领域本体的对象根类型

    实体对象和事件对象,可以由文本对象、多媒体对象通过文本分析、视频分析提取产生,所以从抽象层次上来讲,文本类型和多媒体类型在实体类型和事件类型的更高层次上。但是,实体对象和事件对象也可以从结构化数据直接构建获得,如人员对象很多情况下是直接从结构化的人员库中获得更全面的属性信息,文本中涉及的人员只是建立了和文档的关系,也可以补充一些属性,这个场景下,文本类型和实体类型是在同一层次上。同时为方便分析人员的理解,把实体类型、事件类型、文本类型和多媒体类型并列为对象的4种根类型。

    综上所述,把领域知识图谱本体的对象根类型分为:实体类型ce、事件类型ct、文本类型cT和多媒体类型cM,这4种根类型Crooto和他们之间的关系R,构成知识图谱的模型表达G。对于4种根类型的子类型,可以有继承ri和层次rl关系。从任何类型出发,通过继承关系都可以最后到达根类型Crooto。为了统一表达知识,4种根类型可以继承一个虚拟对象类型c′,从而4种类型的对象都有了统一的对象属性和对象间的关系ru。领域本体的对象根类型说明如下:

    (1)实体类型ce,对应领域中的主要对象目标,主要从结构化数据中提取。比如:个人、组织、车辆、电话等。实体类型有层次关系,比如公务员、企业人员继承个人。

    (2)事件类型ct,是指实体的事件行为,以及实体间的事件关系。比如个人的铁路订票记录、电话之间的通话记录等。

    (3)文本类型cT,对应领域中的文本类数据,包括邮件、新闻文章、报告等。可通过对正文的文本分析提取相关的实体或者事件,并建立他们之间的关系。

    (4)多媒体类型cM,对应领域中的图片、视频等多媒体数据资源,多媒体类型展示浏览的主体是图片显示或视频播放,可通过图片识别、视频分析等方法实现从多媒体类型中提取实体、事件等功能。

    4.3 文本类型的应用场景和必要性

    文本中蕴藏着大量有用的信息,知识图谱的一个很重要的应用场景就是提取文本中的知识对象和关系,提取方法有2种,一是通过智能文本分析自动提取,二是通过人机交互的方式精确提取。文本的实体提取还能追溯实体或属性的具体文本来源,对于实际分析工作具有重要的价值。

    文本提取是文本分析的基础功能,提取过程通常分为2步。首先是自动提取,提取的内容包括:人物、组织、地址、证件号码、网址、电话、车牌、邮箱等,自动提取的结果可以提醒分析人员关注到相关的实体对象,从而让分析人员在必要的时候开展进一步的提取工作,即通过人机交互的方式从文本中提取实体、属性及实体之间的关系。如图7所示的文本中,可以提取人物、组织等实体,提取人物的属性包括姓名、组织、电话、手机、邮箱等,组织的属性包括名称、地址、网站等,同时可以提取人物和组织之间的从属关系。

    通过文本分词、分类聚类、情感分析等算法,分析人员还能从文本中提取领域关键词、地区分类、文本情感态度、主要观点等分析结果,进而补充到文本的属性中。多个文本的属性可以进行聚类统计,形成对文本的分析统计结果。比如在某国际传播大数据智能分析平台项目中,利用文本智能分析和人机交互功能,可以分析提取出新闻报道中的领域关键词、国家地区、情感态度和主要观点等,构建出分析人员研究大量新闻报道的基础,在此基础上再结合搜索和可视化技术,形成对于主题、地区、媒体组织、时间段等多维度的综合量化性分析。

    文本类型是很多领域的主要研究分析对象,所以完全有必要作为根类型之一。文本也是构建领域知识图谱本体的重要来源,从文本中构建本体可以采取手工构建或者自动构建[16,16]的方法。在这个意义上来讲,文本本身也更需要成为对象的根类型,才能够追溯文本构建的本体来源。

    4.4 多媒体类型的应用场景

    多媒体是计算机时代的产物。随着智慧城市、互联网的发展,多媒体逐渐成为领域的主要数据类型。很多领域有对多媒体进行分析的需求。多媒体类型的分析方法和实体、事件、文本类型的都不同,因此也非常有必要成为一种基础类型。目前,随着人工智能技术的发展,主要研究领域为图片和视频的智能分析。通过智能分析可以进行多媒体分类、语音识别、文字识别、人物识别、标签提取等。通过人机交互的方式,可以标识重要的多媒体场景、事件、人物等,比如交警标识一个闯红灯的事件,标识一个治安盗抢案件的犯罪过程等。

    实体、事件、文本、多媒体4种对象根类型的分类方法,也很适合情报分析领域。在某部门情报分析的知识图谱工程项目中,知识对象规模达到百亿级,本体的对象类型种类超过100种。该项目的知识图谱构建采用了这种分类方式,有针对性地设计存储架构和应用方式,将结构化数据通过映射构建为实体和事件,将Email、文章、社交媒体帖子、图片、音频、视频等类型构建为文本类型和多媒体类型,并且通过计算机辅助分析和人工交互的方法,补充对象属性、生成相关的实体或事件,再结合实体对齐融合,做到持续构建完善的知识图谱系统,为领域的工作目标奠定了坚实的技术基础。

    4.5 领域知识图谱产品案例

    基于领域知识图谱本体的对象根类型模型设计理念,我们研发出了《大数据水晶球分析师平台》,该平台入选了工信部2018年度大数据优秀产品案例(优秀产品全国30个),获得了评委专家的认可,清华大学软件学院的院长王建民教授,给出的专家点评意见如下:

    《大数据水晶球分析师平台》采用动态本体理论和知识图谱技术,对海量数据资源进行整合重构,形成灵活逻辑的知识库体系,建立面向业务的应用分析体系,切实解决了各领域、各行业的大数据关联挖掘、分析展现等问题。总体来说,大数据水晶球分析师平台在技术创新、领域应用等方面具有较高水平。

    这里的动态本体理论的核心模型,就是本文研究的对象根类型模型。

    5 结束语

    领域知识图谱的构建,是为了更好地分析和研究领域问题。根据领域的数据特点和主要对象类型,本文提出将领域本体的对象根类型分为4种:实体类型、事件类型、文本类型和多媒体类型,领域知识图谱的本体对象类型全部都继承这4种根类型。我们已经将4种根类型的领域本体模型用于实际的领域知识图谱产品和工程项目中,比如在公安和情报等领域,针对不同的对象类型设计不同的存储架构和应用功能,可承载更大规模的数据和知识分析能力。在此基础上,我们会继续深入研究4种对象根类型的特征,探索更多的应用场景,从而构建更高质量的领域知识图谱和分析应用。
