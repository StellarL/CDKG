    虚拟现实技术是一门综合性技术,涉及计算机图形学、多媒体技术、人机交互和人工智能等多个领域,在教育、医疗、娱乐、军事等众多领域有非常广泛的应用。所有这些技术和应用最终都要靠计算机软件来实现,这就使得虚拟现实系统的软件变得十分庞大和复杂,涉及许多多媒体数据。传统软件开发方法和程序设计技术侧重于处理文本数据,显然不能满足开发虚拟现实软件的需要。试图以所在团队研发成功的PAR方法和PAR平台为基础,根据虚拟现实软件系统的特征,探寻虚拟现实软件系统开发的新方法,进一步扩充和完善PAR平台中已有多媒体处理技术、形式化建模技术和C#等高级语言程序自动生成系统。

    A new virtual reality software system development method

    ZHOU Zhe-hong XUE Jin-yun HUANG Jie-wen

    National-Level International S & T Cooperation Base of Networked Supporting Software,Jiangxi Normal University School of Computer Information Engineering,Jiangxi Normal University Jiangxi Provincial Key Laboratory for High Performance Computing Technology

    Abstract：

    Virtual reality technology is a comprehensive technology involving computer graphics, multimedia technology, human-computer interaction and artificial intelligence. It has a wide range of applications in education, medical, entertainment, military and many other fields. All of these technologies are ultimately implemented by computer software. This makes the software of the virtual reality system very large and complex. The programming techniques of traditional software development methods focus on processing text data, which obviously cannot meet the needs of developing virtual reality software. Based on the successful PAR method and PAR platform of the author's R&D team, according to the characteristics of the virtual reality system, this paper explores a new virtual reality software system development method, and further expands and improves the existing multimedia processing technology in the PAR platform, the formal modeling technology and the automatic generation system of C# and other high-level language programs.

    1.1 虚拟现实程序开发的重要性

    虚拟现实技术,是指将计算机、传感器技术、人工智能、多媒体等技术相结合,生成人机交互的技术。通过现有的信息技术来模拟出真实世界,使人获得身临其境般的感觉[1,1]。虚拟现实是未来行业的基础平台,由于具有强大的跨学科性,已成为科技探究途中除理论探究、科学实验之外的第3种手段。

    虚拟现实技术的特征包含沉浸感、交互性、构想性[2,2]。沉浸感是指人处在虚拟环境中,但是能够体验到和真实世界并无太大差别的感受。交互性是指人在虚拟环境中能够和其中的物体进行交互,通过人机操作界面进行反馈。构想性是指在人机交互过程中,可在任意场景漫游,观察角度、时间不受限制,借助虚拟现实环境让人产生新的想法和灵感。随着互联网、大数据技术的发展,交互环境中的体验感受到关注,存在感和模拟真实感越来越受到重视。

    虚拟现实技术的发展和普及,满足了人对于信息可视化的需求,改变了人类与计算机的交互方式,也改变了人类的生活方式,它使人能够主动地与计算机交流。

    1.2 虚拟现实领域发展现状

    我国虚拟现实技术的研究和一些发达国家相比还有很大的一段距离,随着计算机图形学、计算机系统工程等技术的高速发展,虚拟现实技术已经得到了相当的重视,引起我国各界人士的兴趣和关注,硏究与应用虚拟现实,建立虚拟环境。虚拟场景模型和分布式虚拟现实系统的开发正朝着深度和广度2个方向发展[2,2]。北京航空航天大学计算机系是国内最早进行虚拟现实研究、最有权威的研究机构之一,其虚拟实现与可视化新技术研究室集成了分布式虚拟环境,可以提供实时三维动态数据库、虚拟现实演示环境、用于飞行员训练的虚拟现实系统、虚拟现实应用系统的开发平台等,并在以下方面取得了进展:着重研究了虚拟环境中物体物理特性的表示与处理;在虚拟现实中的视觉接口方面开发出部分硬件,并提出了有关算法及实现方法。清华大学国家光盘工程研究中心研发的“布达拉宫”,采用了Quicktime技术,实现大全景虚拟现实。浙江大学国家重点实验室开发了一套桌面型虚拟建筑环境实时漫游系统。哈尔滨工业大学计算机系已经成功地合成了人的高级行为中的特定人脸图像,解决了表情的合成和唇动合成技术问题,并正在研究人说话时手势和头的动作、语音和语调的同步等[3,3]。

    1.3 虚拟现实软件实现技术的挑战

    在开发虚拟现实程序过程中,绝大部分程序都需要编写登录模块,这通常要涉及到数据的存取、修改,XML(Extensible Markup Language)、Json(Java script object notation)、SQL(Structured Query Language)等都是数据操作的常用方法。对于数据量比较大、涉及增删改查的项目来说,使用数据库是一种比较好的方式,目前比较流行的数据库有MySql、SQL Server及Oracle。但是,Unity 3D并不支持图形化的数据库操作,所以我们在采用Unity 3D进行数据库连接操作时,还是需要用到C#语言进行后台代码编写[4]。这直接导致了每编写一个连接数据库操作的程序时,都需要重新用C#编写对数据库进行连接的代码,大大增加了开发人员的时间成本。并且对于本身没有数据库SQL语言基础的开发者来说,后续对数据库中的数据操作也是极为困难的。这些重复的编码违背了开发引擎的初衷,而我们的PAR(Partition-And-Recur)平台的Apla→C#程序自动生成系统则可以完美地解决这一问题。

    不仅如此,不光是我们研究的虚拟现实程序还是普通的计算机程序,都不可避免地需要进行数据之间的交换传输,Unity 3D也没有提供对网络连接的可视化操作,开发人员还是需要一次又一次地去编写网络连接的代码。在高级网络程序设计中,有许多纷繁复杂的网络协议[5],如果每个网络协议的连接都要使用C#去编写,这样将会是一个重复且庞大的工程。

    除了上述所提到的2个普遍存在的需要用C#语言编写代码的问题之外,Unity 3D还存在着其他的降低开发效率的问题。当用Unity 3D编写虚拟现实程序时不可避免地会遇到要使用树、图等数据结构的情况。当遇到这些情况时,开发者必须调用Visual Studio用C#去编程。然而,在数据结构中,树与图用传统C#语言去编写较为复杂,并且像类似于遍历二叉树、广度或深度优先搜索算法等经典算法在C#中的实现也是极为繁琐的。这些问题都无法使用Unity 3D提供的图形化界面解决,使得虚拟现实程序的开发变得极其低效,同时也增加了开发者的时间成本。

    2 虚拟现实程序开发过程

    2.1 Unity 3D简介

    Unity 3D[4]是由Unity Technologies开发的一个让玩家轻松创建诸如三维视频游戏、建筑可视化、实时三维动画等类型互动内容的多平台的综合型游戏开发工具,是一个全面整合的专业游戏引擎。Unity类似于Director、Blender game engine、Virtools或 Torque Game Builder等软件,它们都是基于交互图型化开发环境的。其编辑器运行在Windows和Mac OS X下,可发布游戏至Windows、Mac、Wii、iPhone、WebGL(需要HTML5)、Windows phone 8和Android平台,也可以利用Unity web player插件发布网页游戏,支持Mac和Windows的网页浏览[5]。Unity 3D的跨平台性、强大的交互功能以及良好的可移植性,使其得到广泛应用与传播。

    Unity 3D的设计初衷不是用来实现虚拟现实的,但由于其创建的场景能够让开发者实时预览,实现所见即所得,因此Unity 3D成为近来热门的虚拟现实开发平台。

    在虚拟现实的交互方面,Unity 3D提供了物理碰撞器、灯光特效、网络功能、图形渲染、强大的内置包等,这些功能简化了开发者的开发过程,减少了场景渲染的时间,能够极大地提高虚拟现实开发效率。

    2.2 Unity 3D开发典型虚拟现实程序的一般过程

    通过对大量虚拟现实程序的开发过程的研究可以得出结论,开发一个典型的虚拟现实程序大致可分为3个步骤,依次是:需求分析、模型的建立与加载、C#脚本的开发与测试。下面以本文作者开发的VR(Virtual Reality)球类游戏举例说明这3个步骤:

    (1)需求分析。

    一个简单的VR球类游戏,主要玩法为:在一个盒子里有一个气球和许多小球,还有一个透明板向气球移动,当透明板触碰到气球则游戏结束。游戏通过VR设备以及摄像头捕捉用户的嘴型与动作,借此判断用户吹气球的位置与力度,使得游戏内的气球移动并撞击小球得分,从而模拟出用户用嘴去吹气球的过程。每次撞击成功都会得分并且释放一枚导弹使透明板往反方向移动,从而延长游戏时间。

    各构件功能说明。

    ①BallManager:

    检测气球是否与其他物体有碰撞,撞到小球得100分,撞到杆子得30分。

    ②CameraController:

    一共实现4个功能,分别是:初始化游戏场景、开始游戏、重新游戏以及更新气球移动后的位置。

    ③FrameManager:

    定义气球的三维状态。

    ④GameControl:

    是否全屏开始游戏。

    ⑤MissileManager:

    控制气球碰撞到小球之后发出的导弹的轨迹。

    ⑥SocketClient:

    实现后台控制程序与VR设备之间的数据传输交互。

    (2)模型的建立与加载。

    典型的虚拟现实软件系统与传统的软件系统最大的区别就在于:传统的软件系统是通过屏幕等标准输入输出设备来与用户进行交互,而虚拟现实软件系统是脱离了屏幕、键盘鼠标等传统设备的,用户通过VR设备完全沉浸于系统所生成的3D环境当中,通过VR设备侦测用户的动作来与软件进行交互。因此,3D模型的建立对于虚拟现实软件系统来说是必不可少且至关重要的一环。

    一般地,先采用如3DS MAX、MAYA等3D建模软件将模型设计好并以.fbx、.obj等文件格式存储在计算机内,之后用Unity 3D导入这些模型并给这些模型用各式各样的材质包进行上色。在导入模型的环节中,Unity 3D提供了2种加载方式:

    ①Assetbundle.Load读取和加载资源。

    由于原始的.fbx文件在Unity 3D中的不稳定以及不易于操作性,我们通常不会用Unity 3D去直接处理导入进来的.fbx文件,而是将原始的.fbx文件转换成易于处理的.prefab预制建筑文件再对其进行一些旋转、上色等操作。Assetbundle.Load方式主要的作用就是将原始的.fbx文件处理成在Unity 3D中易于操作的.prefab文件。

    ②Resources.Load动态加载资源。

    该方式就是从一个缺省打进程序包里的AssetBundle里加载资源,而一般AssetBundle文件需要用户自己创建,运行时动态加载,可以指定路径和来源。使用AssetBundle打包预设或者场景可以将与其相关的所有资源打包,这样可很好地解决资源的依赖问题,使得用户可以方便地加载GameObject且可以指定路径和来源。

    (3)C#脚本开发与测试。

    如图2和图3所示,当将所有3D模型以.prefab的形式导入进Unity 3D之后,就需要编写C#脚本对模型进行控制。所谓C#脚本就是指一系列用C#编写的用以控制3D模型的移动和功能的控制程序。这些脚本之间互相联系、互相制约,通过对收集来的用户数据控制3D模型与用户进行交互从而完成不同的功能。

    2.3 VR程序开发的关键技术

    虚拟现实程序开发的关键技术分为2大类:(1)3D建模技术;(2)C#脚本编写技术。以下分别对这2类技术进行详细说明。

    (1)3D建模技术。

    经过对大量优秀VR程序的研究发现,模型的精细与否直接影响到用户的使用体验,所以优秀的VR作品通常都配有精致的3D模型。要实现这一点则必须借助VR领域中的2个重要技术——计算机图形技术和立体显示技术。

    用计算机生成、显示、绘制图形的技术被称为计算机图形技术。计算机中的图形是以数据的形式表示的,要把图形显示出来或绘制打印,就必须把数据转换成线条。计算机图形技术不仅能快速、准确、规范地制作大量的机械图、建筑图、电路图和地理图等,成为计算机辅助设计的重要内容,而且可以制作运动图形和三维图形,使绘图做不到的事得以实现。

    立体显示是虚拟现实的一种实现方式。立体显示主要有以下几种方式:双色眼镜、主动立体显示、被动同步的立体投影设备、立体显示器、真三维立体显示、其它更高级的设备。

    (2)C#脚本编写技术。

    在用Unity 3D开发虚拟现实程序时,导入的3D模型是没有“生命”的,开发人员需要为用到的模型编写脚本使其运作起来。那么不可避免地需要用到C#语言来编写脚本(通常越大型的VR程序需要编写的C#脚本也越多且越复杂)。C#是微软公司发布的一种面向对象的、运行于.NET Framework和.NET Core(完全开源,跨平台)之上的高级程序设计语言。推荐使用与Unity 3D兼容性较好的Microsoft Visual Studio 2017编写C#脚本。

    3 虚拟现实在拓展了的PAR平台中的实现

    3.1 PAR方法与PAR平台简介

    PAR方法和PAR平台是一个综合性软件开发环境,简称PAR。PAR由自定义泛型规约和算法描述语言Radl(Recurrence-base algorithm design language)、泛型抽象程序设计语言Apla[6,6]、系列算法和程序自动生成工具、系统的算法程序设计方法和简捷的数据库应用软件开发工具构成[7,7]。PAR方法和PAR平台包含循环不变式的新定义和新的开发策略、统一的算法程序设计方法、新的算法表示方法、自定义算法设计语言和抽象程序设计语言等关键技术。它集成涵盖了泛型、生成式、模型驱动和构件组装等新型软件开发技术[8,8]。

    PAR方法由一系列的建模语言和形式化规则组成,包括新型算法建模语言Rad1、新型算法程序设计方法、新型泛型规约、抽象泛型程序建模语言Apla[6,6]、新型形式化规约变换规则[9,9]。

    PAR平台主要包含Radl→Apla程序自动转换系统、Apla→C++程序自动转换系统[6,6]、Apla→Java程序自动转换系统[10]、Apla→C#程序自动转换系统[11,11]和一系列Apla代码转换到高级程序设计语言的自动转换工具。

    3.2 对PAR平台进行拓展的思路

    在软件开发过程中,框架的选择至关重要。选择一个性能好的框架,对后续的开发是很有帮助的,但并不是每个软件框架都是完美无缺的。例如本文所用的Unity 3D开发引擎还是存在许多需要用C#编写代码的问题。对于这些问题,PAR方法与PAR平台有着先天的优势,只需要编写少量的代码就可以快速地实现一些复杂的功能,极大地提高了编程效率。

    因此,接下来将从形式化建模技术、C#高级语言程序自动生成系统和多媒体处理技术3个方面,根据虚拟现实软件系统的特征,探寻虚拟现实软件系统开发的新方法。

    3.2.1 运用PAR方法的新型形式化建模技术简化VR程序开发过程的思路

    通过对大量虚拟现实程序以及第2节介绍的VR球类游戏开发过程的研究,可以得出典型虚拟现实程序的开发过程。那么,基于一系列的建模语言和形式化规则组成的并且旨在用效率驱动的软件开发方法简化程序达到程序自动化目的的PAR方法就可以根据这个开发过程拓展到虚拟现实领域。

    简化一个典型虚拟现实程序开发过程的具体步骤为:首先PAR方法中的建模语言可以对用UML建模的典型VR程序进行简化;再用Apla语言提高编写C#脚本的效率,缩短VR程序开发周期,开发人员甚至都不需要深入了解C#语言即可编写复杂的C#脚本;最后再将用Apla语言编写的代码转化为可用的C#脚本供Unity 3D程序调用。

    3.2.2 PAR平台对数据库的拓展

    在用Unity 3D开发VR程序的过程中,绝大部分VR程序都需要用到数据库操作。然而,这些对数据库的操作用Unity 3D自身提供的图形化界面是无法实现的。这里一般会采用Visual Studio 2017编写C#脚本的方法来实现这个功能。用C#编写操作数据库的脚本程序一般会用到以下操作:

    更新数据:

    string s="server=localhost;database=u3dproject;uid=sa;pwd=qwe";

    SqlConnection conn=new SqlConnection(s);

    conn.Open();

    SqlCommand cmd=new SqlCommand();

    cmd.Connection=conn;

    cmd.CommandText="INSERT INTO UserInfo VALUES(Jom,'Boy','17','Namchang')";

    SqlDataReadersdr=cmd.ExecuteReader ();

    读取数据:

    string s="server=localhost;database=u3dproject;uid=sa;pwd=qwe";

    在Apla语言中对应的操作思路:

    在Apla语言中,由于采用了新型的泛型机制,数据库被看做是一个新的数据类型,则所有对数据库的操作都可以被看做是这个“数据库”类型的函数[12]。这大大地简化了编写代码所需的步骤,从而缩短了开发周期。不仅如此,由于将编译成机器代码的工作交给了Unity 3D内置的C#编译器去做,因而实现了算法程序的机器无关性,软件的可靠性也易于得到保证[13,13]。

    3.2.3 PAR平台对TCP通信的拓展

    在虚拟现实软件系统开发的过程中,VR设备与虚拟现实程序之间的数据通信是必不可少的一环[14,14]。这也是虚拟现实软件系统区别于传统软件系统的一个显著特征。一般地,TCP(Transmission Control Protocol)通信由于其数据传输的可靠性以及可传输大量数据的特点而被广泛应用于虚拟现实程序的开发中。然而,这和3.2.2节提到的数据库操作一样,也是用Unity 3D自身提供的图形化界面无法实现的。因此,当需要用到TCP通信时,仍然需要编写C#脚本。TCP/IP协议在C#脚本中的客户端代码如下所示[15,15]:

    using System.Collections;

    using System.Collections.Generic;

    using UnityEngine;

    using System.Net.Sockets;

    using System.Net;

    using System.Text;

    using System;

    using UnityEngine.UI;

    using System.Threading;

    public class Client_UI:MonoBehaviour {

    private Socket client;//创建客户端

    private IPAddress ip;

    private EndPoint ep;

    private byte[] data=new byte[1024];/*用来接收和发送数据*/

    private int length=0;

    private string message;

    public Text inputText;//输入栏

    public Text outputText;//显示栏

    private string allMessage;

    public Button send;

    private Thread t;

    void Start () {

    t=new Thread (SendMessageToServer);

    client=new Socket (AddressFamily.InterNetwork,SocketType.Stream,ProtocolType.Tcp);

    //绑定IP地址和端口号

    ip=IPAddress.Parse("192.168.32.60");

    ep=new IPEndPoint (ip,4567);

    //连接服务器

    client.Connect(ep);

    length= client.Receive(data);

    message=Encoding.UTF8.GetString(data,0,length);

    allMessage=message+'\n';

    outputText.text=allMessage;

    }

    public void SendMessageToServer(){

    message=inputText.text;

    inputText.text=" ";

    if (message != null && message != "") {

    allMessage+=message+'\n';

    outputText.text=allMessage;

    data=Encoding.UTF8.GetBytes(message);

    message="";

    client.Send(data);

    }

    }

    }

    在Apla语言中对应的操作思路:

    在处理TCP\IP协议时,因为Apla语言预定义组合数据类型均由PAR方法推导得到[16,16],由核心转换器生成C#可重用部件库并且集词法检查、语法检查、语义一致性检查、转换、编译、运行为一体,那么便可以将TCP\IP协议及其对应的TCP通信功能封装进一个新的“网络构件库”,再通过核心转换器生成对应的C#语言代码,从而进一步生成动态链接库。这样便完成了对TCP通信功能模块的封装。之后开发人员仅需调用该构件库即可完成TCP通信,不需要再进行复杂繁琐的C#编码,从而提高了开发效率。

    3.2.4 树在VR程序开发中的作用以及PAR平台对树的拓展

    在开发类似VR导航以及一些棋牌类的虚拟现实程序的过程中,一般都会利用到人工智能当中的α-β剪枝算法。该算法是一个搜索算法,旨在减少在其搜索树中被极大极小算法评估的节点数,常被用于寻找最短路径以及最优解的过程中筛选掉不必要的分支从而提高效率。在α-β剪枝算法中,每个节点都以树状图的形式排列。

    树是一个具有层次结构的数据的汇集。在树的应用中,二叉树的处理比多叉树要简单得多,而且任意次树都可以转换成相应的二叉树,因此这里以二叉树为例进行探究。可以构造一个btree类,用来存放二叉树的元素及操作。对于二叉树的节点,可以定义如下:

    public class bnode〈T〉 where T:IComprable{

    public T d;

    public btree〈T〉 il,ir,ip;

    }

    由于树的结构的特殊性,在推导树时引用了一个游标,它可以在树的节点上任意地移动,游标移到哪个节点上,就可以取出这个节点的值。

    下面以“判断2棵树是否相等”为例,说明利用PAR方法推导实现树的操作的过程,以下为Apla程序代码[17]:

    program samel;

    function same (T1.T2:btree(T)):boolean;

    var S1,S2:list(btree(T));

    q1,q2:(btree(T);

    begin

    q1,q2,same,S1,S2,:T1,T2,true,[],[];

    do same=true∧┐(q1=%∧S1=[]∧q2=%∧S2=[])→

    if q1≠%∧q2≠%→same:=(q1.d=q2,d);

    [] (q1≠%∧q2≠%)∨(q1≠%∧q2≠%)→same:=false;

    [] q1=%∧q2=%→same:=true;

    fi;

    if q1≠%→q1,S1:=q1.1,ql.r↑S1;

    [] q1=%∧S1≠[]→q1,S1:=S1[h],S1[h+1..t];

    fi;

    od;

    end;

    由此可见,利用PAR方法推导生成“判断2棵树是否相等”的过程十分简洁。采用同样的方法也可以推导实现其他的操作,这里不再列出。

    3.2.5 多媒体处理技术在Apla中的实现方法研究

    随着信息技术的飞速发展,计算机应用产生的数据量也越来越大,数据类型十分丰富,如文本、图形、图像、动画和声音等不同类型的数据。多媒体技术不是各种信息媒体的简单复合,它是一种把文本、图形、图像、动画和声音等形式的信息融合在一起,并通过计算机进行综合处理和控制,能支持完成一系列交互式操作的信息技术[18,19,18,19]。

    在用Unity 3D开发虚拟现实程序的过程中,也需要运用到大量的多媒体数据,包括:3D模型材质包、粒子特效和声效等。以3D模型材质包举例,Unity 3D程序先在视窗中生成原始的3D模型,之后再将开发人员选择的材质球上的材质附着到建好的模型上,最后把完整的3D模型展现给用户。然而,多媒体技术在发展,而大多数高级程序设计语言对多媒体数据的处理能力仍较为欠缺,高级程序设计语言若不能较好地处理多媒体数据,将会落后于时代的发展而逐渐被淘汰。所以,支持对多媒体数据的处理,对于高级程序设计语言来说是极为重要的。

    目前国内外众多学者基于形式化与自动化研究设计了相关建模语言,但遗憾的是大多数建模语言都不能引入多媒体数据库处理机制。而PAR方法成功引入了多媒体数据库处理机制,在Apla中,依托 Oracle 数据库,重新设计并扩展了相关核心模块,使Apla语言具有多媒体数据处理能力,并能通过转换器生成正确的可执行的C#程序。

    4 结束语

    以PAR方法和PAR平台为基础探寻虚拟现实软件系统开发的新方法是本文的主旨,也是进一步扩充和完善PAR平台功能的重要一环。在软件开发中运用PAR方法将极大地提高开发效率以及软件的可靠性。本文首先介绍虚拟现实的重要性和挑战,再通过举例分析典型虚拟现实程序的开发过程,得知传统软件开发方法的程序设计技术无法满足开发虚拟现实软件的需要;之后详细阐述了将PAR方法运用到虚拟现实程序开发中的思想以及步骤;最后具体说明了作者所在团队利用PAR方法的语言工具Apla简化虚拟现实程序开发过程的研究成果。

    目前,作者所在的研究小组在薛锦云导师的带领下已经充分理解并证明了软件形式化方法对虚拟现实程序开发的可行性与指导作用。初步完成了利用PAR方法的建模语言Apla简化虚拟现实程序开发中所遇到的复杂算法以及在建模语言Apla中多媒体数据库的应用。在今后的研究中还将以把PAR方法拓展到虚拟现实领域并加以运用为目标,完成对虚拟现实程序从需求设计到软件建模再到程序生成的自动化。
