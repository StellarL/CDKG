    负载均衡的主导资源公平分配算法

    刘梓璇 周建涛

    摘    要：

    随着使用云计算并行且可靠地处理计算问题成为一种趋势,各种云计算平台应运而生,在这些平台中,保证多种资源调度策略的公平性非常重要。主导资源公平分配算法DRF有效地实现了多种资源环境中的公平分配,但在资源分配过程中容易出现集群负载不均的情况。因此,提出在使用DRF算法分配资源过程中,通过集群中各节点的资源利用率情况对节点进行K-means聚类分析,根据聚类结果将资源分配给任务来提高集群负载均衡的能力。基于CloudSim 4.0实现了改进DRF算法的仿真实验,实验结果表明,负载均衡的DRF算法比原始的DRF算法以及基于层次分析法(AHP)改进的DRF算法更能有效地改善集群整体的负载均衡。

    A fair load-balanced dominant resource allocation algorithm

    LIU Zi-xuan ZHOU Jian-tao

    College of Computer Science,Inner Mongolia University

    Abstract：

    With the trend of using cloud computing to handle computation problems concurrently and reliably, various cloud computing platforms have emerged. In cloud computing, the fairness of multiple resource scheduling strategies is very important. Dominant resource fairness(DRF) can effectively implement the fairness in multiple resource environments, however, it can introduce uneven cluster load in the process of resource allocation. Therefore, we propose a load-balanced dominant resource fairness allocation algorithm. It uses the DRF algorithm to allocate resources, perform K-means clustering on the slaves according to the resource utilization of each slave in the cluster, and allocate the resources to the tasks according to the clustering results to improve the load balance of the cluster. Simulations of the improved DRF algorithm are implemented on CloudSim4.0. Experimental results show that the proposed load-balanced DRF algorithm can improve the load balancing of the cluster more effective than the original DRF algorithm and the improved DRF algorithm based on analytic hierarchy process(AHP).

    cloud computing; K-means clustering; fair dominant resource allocation; load balance;

    1 引言

    互联网的高速发展和大数据处理的迫切需求,使诸如Hadoop、OpenStack、Spark、Storm等云计算框架得到了快速的发展,各种云计算框架各有所长,各自着重针对某类计算问题进行处理。但是,在云计算环境中,一方面,由于实际问题的复杂性,可能需要使用几种框架协作解决实际的应用问题。另一方面,不同类型的计算任务对不同种类的资源需求是不同的,计算密集型任务通常需要更多CPU资源,I/O密集型任务则需要更多内存资源。因此,在云计算领域中通常需要将多个云计算框架部署到同一个公共的集群环境中,以达到共享集群内的数据及资源并对集群内的资源统一地进行调度和管理的目的[1,2]。在这样的背景下,软件定义数据中心的概念被提出,并由此诞生了一系列资源统一管理与调度平台,如Yarn、Mesos和Brog等[3,4,5,4]。这些平台的核心即是使用合理的资源调度算法为提交到平台中的作业分配资源,因此资源调度器成为了它们最核心的组件[6,6]。

    目前资源分配算法主要集中在对单资源分配算法的研究上,单个资源的分配是公平且高效的。而对于多资源分配的研究大多采用对单个资源进行抽象进而分配的方式[7]。在多资源分配的环境中,需要对各种资源进行衡量,以解决多资源分配的公平性问题。主导资源公平DRF(Dominant Resource Fairness)分配算法提供了在多种资源环境中的一种公平分配机制,是一种通用的多资源分配策略,因此得到了广泛的关注和应用。该算法将针对单资源分配的max-min公平分配算法扩展到多种类型资源的分配上,对主导资源所占份额应用max-min公平分配的方法来解决多资源分配的公平性问题[8]。DRF算法满足共享激励、策略规范、帕累托效率和无嫉妒性这4条特征,因此在多资源分配环境中得到了广泛应用。文献[9]提出一种称为DRFH的多资源分配机制,将主导资源公平DRF的概念从单一种类服务器扩展为多种异构服务器,根据服务器提供各种资源的能力更加公平地进行资源分配,从而提高资源利用率。在文献[10,10]中,作者通过增加机器性能评估影响因子对DRF分配算法进行改进,增加机器性能作为影响公平性的因素,从而增强多资源分配过程中的公平性,有效提高系统资源利用率。

    DRF分配算法较好地解决了多资源环境中资源分配的公平性问题,但是,在资源分配的过程中没有充分地考虑集群整体负载均衡性问题。Mesos资源统一调度平台应用DRF算法解决了CPU和内存资源的公平分配问题。但是,Mesos在资源调度过程中,没有平衡集群中节点间负载的处理,会产生集群负载不均的情况。在实际的应用中,集群中节点负载不均衡会导致集群整体性能下降,甚至导致集群中某些节点资源利用率过高,使节点出现故障的可能性增加。因此,提高集群的负载均衡能力是十分必要的。通常情况下,CPU的最优利用率为70%,内存的最优利用率为50%,当资源利用率高于该数据时,集群中slave节点的性能将会变差[11]。并且当集群中某个节点的负载在一个时间段内持续偏高时,会增加该节点服务器死机或宕机的风险。文献[12]提出一种基于人工鱼群算法的资源调整策略,在Mesos集群中使用DRF分配算法计算出资源分配方案后,用户通过使用人工鱼群算法调整slave节点和容器间的匹配来对资源分配方案进行调整,改善集群负载均衡并提高集群性能。在文献[13,13]中,作者提出使用CPU利用率、内存利用率和容器个数的平均值作为权值,通过层次分析法AHP(the Analytic Hierarchy Process)将任务分配给权值较低的节点,使集群的资源利用率相对均衡。为改善集群在多资源分配过程中的负载不均问题并使集群在资源分配过程中可以根据节点资源占用情况动态地平衡负载,本文提出一种负载均衡的DRF算法,该算法在分配过程中,根据各节点上CPU资源和内存资源的利用率,对集群中的节点进行K-means聚类,并对聚类结果进行分析,每次将任务优先分配到资源利用率较低的一组簇中的节点上,以此来达到动态地平衡集群中节点间负载的目的。

    2 DRF算法

    2.1 DRF算法介绍

    DRF是一种通用的多资源max-min公平分配策略,可应用于用户有异质需求的多资源环境中。DRF根据主导资源所占份额来实现多资源环境中的资源公平分配[14]。该算法的思想是根据框架中任务对各种资源的需求向量和系统总空闲资源向量,确定任务的主导资源并计算主导份额,通过平衡各任务主导资源所占份额进行资源分配,以实现任务分配公平性[15,15]。

    在DRF算法中,设资源种类为R1,R2,…,Rn。资源总量由向量TotalResource(TR1,…,TRn)表示,设任务为J1,J2,…,Jm,任务所需资源量由向量NeedJi(JiR1,…,JiRn)表示。已知任务Ji所需资源为NeedJi,资源总量为TotalResource,则任务Ji的主导份额为:

    DomShareJi=max(ΝeedJiΤotalResource)=max(JiR1ΤR1,JiR2ΤR2,⋯,JiRnΤRn)         (1)

    设运算操作所得结果为JiRk/TRk,则任务Ji的主导资源为资源k。

    DRF算法流程为:假设存在任务J1和J2,资源R1和R2,资源总量分别为TR1和TR2。因此:

    DomShareJ1=max(J1R1/ΤR1,J1R2/ΤR2)         (2)DomShareJ2=max(J2R1/ΤR1,J2R2/ΤR2)         (3)

    假设J1R1/TR1>J1R2/TR2且J2R1/TR1<J2R2/TR2,则J1中资源R1是主导资源,J2中资源R2是主导资源。

    设已分配给J1的资源向量是(x1,y1),已分配给J2的资源向量是(x2,y2),则J1的主导份额为x1/TR1,J2的主导份额为y2/TR2。DRF算法每次选择主导份额具有最小优势的用户,对其分配资源运行任务[7]。即若x1/TR1<y2/TR2,DRF算法选择将资源分配给任务J1;若x1/TR1>y2/TR2,DRF算法选择将资源分配给任务J2。

    2.2 DRF算法在Mesos中的应用

    Mesos提供了一种统一的资源管理与调度机制,诸如Hadoop、Spark等各种框架均可通过注册的方式连接到Mesos[16],向Mesos发送资源请求并接收集群中master节点发送的资源邀约执行各自的任务[17]。Mesos由master节点、slave节点、框架调度器和框架执行器4部分组成,采用2级框架调度机制分配资源。在第1级调度中,master节点负责管理各个注册到Mesos上的框架和slave节点,使用DRF算法将slave节点上的资源分配给各框架。slave节点管理执行在本节点上的任务,为框架的执行器分配资源。在第2级调度中,框架调度器根据分配的资源响应任务,安装在slave节点上的框架执行器则用于启动框架中的任务[18]。

    在Mesos集群中,slave节点定期将自己的空闲资源量发送给master节点。当用户提交作业时,需指定每个任务的CPU和内存需求量并由任务所在框架向master发送资源请求。master的资源分配模块决定将资源分配给某个具体框架。master节点将各节点的资源邀约发送到请求资源的框架上,框架调度器响应任务,寻找可分配的资源邀约[19,19]。master将资源分配给框架,由框架调度器执行任务。Mesos的调度流程如图1所示。

    在第1级调度中,Mesos使用DRF算法将集群中节点上的资源邀约随机分配给集群中的任务,根据该节点的资源邀约判断该节点当前空闲资源能否为任务提供所需的资源,若可以则将资源分配给该用户,否则继续判断下一个节点是否可以提供所需资源。因此,可以发现,使用DRF算法进行分配时无法保证集群的负载均衡。

    3 负载均衡的DRF算法

    本文提出一种在资源分配过程中,通过集群中各个slave上CPU和内存资源的使用率对slave进行聚类分析[20,20],根据聚类结果将资源提供给DRF主导资源公平分配算法的思想。该思想可以使得集群中资源利用率较低的节点优先被分配资源执行任务,以达到动态平衡集群负载的目的。其核心思想是,首先从集群中各slave向master发送的心跳信息中获取当前CPU和内存资源的利用率;然后根据这2种资源的利用率对集群中的节点使用K-means聚类进行分析[21,22,21,22],对比每个簇的聚类中心,得出资源利用率较小的一组节点的资源邀约,把它们推荐给需要分配的任务,并根据该组资源邀约的资源总量计算可分配的最大任务数量;最后利用DRF算法将任务分配到集群中的节点上。

    设集群中的节点为集合S,集合中有n个节点对象,将CPU和内存资源的利用率作为对象间的相似性评价指标,因此每个对象有2个属性,数据集Sn×2可用一个n×2的矩阵X表示。矩阵的每行表示集合S中的一个节点对象。用Xij表示对象i的第j个属性值,所以Xi1表示集群节点中的CPU利用率,Xi2表示集群节点中的内存利用率,则可用二元组U=(UserateCPUi,UserateMemi)表示对象i,其中,UserateCPUi表示节点的CPU利用率,UserateMemi表示节点的内存利用率。用Dij表示集合中节点对象之间的相似性,Dij值越小表示相似程度越高;反之,Dij值越大相似程度越低。

    (1)使用欧氏距离计算集合中节点对象间的相似性:

    Dij=((UserateCΡUj-UserateCΡUi)2+(UserateΜemj-UserateΜemi)2)1/2         (4)

    (2)簇内方差(Si代表第i个簇集合):

    V=∑i=1k∑xj∈Si(xj-μi)         (5)

    聚类过程中通过式(5)反复更新聚类中心,极小化簇内方差V,直至相邻2次聚类中心没有变化时聚类结束,其中xj代表第j个节点,μi代表第i个簇聚类中心。

    (3)实验显示,CPU的最优利用率为70%,内存的最优利用率为50%,当资源利用率高于该数据时,集群中slave节点的性能将会变差。设函数f(x)=(-5/7)x+b,将求出的聚类中心分别代入该函数,得到的值越小,该分组中节点的资源利用率越小,则优先将该组节点上的资源分配给任务。如图2所示。

    (4)由此,设聚类中心为Cr,坐标表示为(Cr,Mr),定义资源利用率评估值Vr=Cr/7+Mr/5。根据利用率评估值对各组进行排序,评估值越小的节点其资源利用率越低,则优先分配任务。

    在资源分配过程中,由DRF算法计算出主导资源所占份额最低的计算任务Jlow,计算出各组聚类中心的利用率评估值,根据聚类中心的利用率评估值对分组结果进行排序,将利用率评估值较小的节点上的资源优先分配给任务Jlow。通过每次都优先将集群中相对较空闲的节点上的资源分配给任务来保证集群整体的负载均衡。同时,在集群中任务量比较大时,可降低资源分配的失败率。算法1描述了该算法的主要思路,算法输入包括矩阵Utin×2,列表CList和VList。其中,矩阵Utin×2表示集群中虚拟机服务器节点的CPU和内存资源利用率,CList表示用户提交到集群中正在等待分配并执行的任务列表,VList表示集群中当前可用的虚拟机列表。首先从Utin×2中随机选取k个点作为聚类中心,重新迭代计算聚类中心直到聚类中心不再有变化时,得出划分的簇集合。通过对比聚类中心的坐标值获取利用率最低的一组簇中的虚拟机节点。然后根据获取的该组簇中的虚拟机数量计算可提供的各资源总量。判断各资源总量是否足够进行资源分配,若不足,则将利用率次小的簇中的虚拟机加入到最小簇中。最后根据总资源量计算可分配执行的最大任务数量,进行资源分配并返回已分配资源的云任务列表ACList。负载均衡的主导资源公平分配算法的具体流程描述如下:

    4 实验结果分析

    本文使用CloudSim 4.0进行仿真实验,首先,通过重写PowerVm类、Cloudlet类、CloudletScheduler类扩展相关功能函数;其次,修改DataCenterBroker类中实现调度算法的函数,实现DRF算法、基于层次分析法(AHP)改进的DRF算法和增加K-means聚类的DRF算法;最后,分别将本文提出的改进算法与通过使用DRF算法和基于层次分析法(AHP)改进的DRF算法,针对资源分配时集群中节点的平均资源利用率情况进行对比。

    在与基于层次分析法(AHP)改进DRF算法[13,13]的实验对比中,创建了7 000个待分配资源的云任务。图7表示使用2种算法进行资源分配时,集群中各个节点在执行任务过程中CPU资源的平均利用率。图8表示集群中各个节点在执行任务过程中内存资源的平均利用率。实验结果表示,使用增加K-means聚类的DRF算法可以使集群中节点间的负载更加均衡。同时,通过对算法的时间复杂度进行分析,基于AHP改进的DRF算法和增加K-means聚类的DRF算法的时间复杂度均为O(n)。因此,使用增加K-means聚类的DRF算法可以在相同时间复杂度下更好地提高集群整体的负载均衡性。

    本文实验结果表示,负载均衡的DRF算法相比于原始的DRF算法很大程度上平衡了集群中节点间的资源利用率,同时相比于使用基于AHP改进的DRF算法也能在一定程度上改善集群负载的均衡性,使集群中的节点能够更加均衡并充分应对执行任务中的负载压力,从而降低集群中个别节点因资源使用率过高而导致节点宕机的可能性,提高了集群整体的稳定性。综上所述,使用负载均衡的DRF算法比使用原始DRF算法和使用基于AHP改进的DRF算法都能更好地改善集群负载均衡。

    5 结束语

    本文对DRF算法的公平性和负载均衡性进行分析,并通过基于DRF算法进行任务调度的Mesos资源统一调度与管理平台发现DRF算法对集群的负载均衡性考虑欠缺。提出增加K-means聚类的DRF算法,该算法根据集群中各节点CPU和内存的利用率对节点进行聚类,每次将任务优先分配给聚类结果中资源利用率最少的一组,来达到平衡集群负载的目的。将增加K-means聚类的DRF算法分别与原始DRF算法和基于AHP改进的DRF进行了对比实验,结果表明增加K-means聚类的DRF算法可以根据集群中的任务量,动态地实现集群负载平衡的目的。目前的工作中只涉及到基于CPU和内存的利用率聚类,而实际环境中影响因素还有很多,比如磁盘I/O、带宽等,在之后的工作中可以将这些影响因素加入到研究范围内。
