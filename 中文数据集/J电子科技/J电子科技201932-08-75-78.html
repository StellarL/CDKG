    基于数据挖掘的集成信令存储和应用平台研究

    吴良

    中国移动通信集团广东有限公司中山分公司

    摘    要：

    针对现有信令数据存储与分析应用模块的独立式架构不利于信令数据的保存与分析的问题, 文中设计了一种基于数据挖掘的集成信令存储和应用平台架构。通过设计并行数据挖掘信令分析模块架构, 面向数据挖掘的信令存储技术和信令数据集成应用平台, 重新定义了基于数据挖掘的信令监测平台架构。利用信令数据仓库中的ETL工具, 实现信令数据存储以及应用的一体化设计。所设计的集成信令存储和应用平台通过数据挖掘可以有效分析信令数据特性, 提高信令数据存储和应用的效率, 对于大规模信令监测具有一定的意义。

    关键词：

    数据挖掘; 信令监测; 数据仓库; ETL工具;

    作者简介： 吴良 (1969-) , 男, 工程师。研究方向:通信网络与大数据技术。;

    收稿日期：2018-10-20

    Research on Integrated Signaling Storage and Application Platform Based on Data Mining

    WU Liang

    Zhongshan Branch, China Mobile Communications Group Guangdong Co., Ltd.

    Abstract：

    Aiming at the problem that the existing independent architecture of signaling data storage and analysis application module is not conducive to the preservation and analysis of signaling data, this paper designed an integrated signaling storage and application platform architecture based on data mining. By designing the signaling analysis module architecture of parallel data mining, Data Mining Oriented signaling storage technology and signaling data integration application platform, the framework of signaling monitoring platform based on data mining was redefined. The integrated design of signaling data storage and application was realized by using ETL tool in signaling data warehouse. The integrated signaling storage and application platform designed in this paper could effectively analyze the characteristics of signaling data through data mining and improved the efficiency of storage and application of signaling data, which had certain significance for large-scale signaling monitoring.

    Keyword：

    data mining; signaling monitoring; data warehouse; ETL tool;

    Received： 2018-10-20

    随着移动通信业务的发展和移动智能设备终端的增长, 传统的信令监测平台已经无法有效应对海量的信令数据采集和分析, 研究面向大规模信令数据的存储和应用技术的需求越来越迫切[1,2]。数据挖掘技术由于具有良好的大数据处理技术, 因而在处理海量信令数据时被广泛使用[3,4]。

    在信令数据分析过程中, 数据挖掘技术在LTE[5]以及互联网中[6,7,8]被用来分析用户的使用习惯及偏好。文献[9]研究了移动通信网络下的大规模信令数据的数据挖掘方案。同时, 关于信令数据挖掘的终端设备和存储方案也被广泛研究[10]。在此基础上, 文献[11]研究了基于数据挖掘技术的信令数据存储和分析一体化设计方案。

    针对大规模信令数据存储以及分析应用的需求, 本文设计了基于数据挖掘的信令存储与应用集成一体化平台。通过设计信令数据挖掘机制, 实现原始信令数据分析。在此技术上, 对信令数据的存储和应用一体化架构进行了研究, 可以有效满足实际信令数据存储和分析应用的需要。

    1 数据挖掘信令监测平台架构

    基于数据挖掘的信令监测系统架构如图1所示, 整个系统包括信令数据接口、并行数据挖掘信令分析模块、调度引擎和集成数据存储模块这4部分。

    图1 数据挖掘信令监测平台架构   下载原图

    Figure 1. Data mining signaling monitoring platform architecture

    信令数据接口主要负责原始信令数据的采集工作, 通过信令链路和相关端口, 从移动通信网中提取信令数据。通过边界路由器, 将数据汇聚到移动通信服务器和信令服务器之中。信令数据采集接口的具体结构, 如图2所示。

    图2 信令数据接口结构   下载原图

    Figure 2. Signaling data interface architecture

    并行数据挖掘信令分析模块主要分为算法服务管理、并行数据处理和并行任务处理这3个模块。调度引擎是针对并行数据挖掘信令分析模块进行统一管理, 主要包括算法管理和任务调度两个部分。负责数据挖掘算法以及其他信令分析处理算法的调用, 并对不同的任务分配相应的资源。

    集成数据存储是负责保存经过处理的信令数据结果, 通过分布式或集中式的信令存储数据库实现信令数据的存储备份, 方便对其进行查询和调用。

    2 并行数据挖掘信令分析模块架构

    如图3所示为并行数据挖掘信令分析模块架构, 整个模块包括数据库、数据仓库、数据库服务器、数据挖掘引擎、算法库以及模式评估等模块。通过这些子模块的协同处理实现基于数据挖掘的信令数据分析和处理。

    图3 并行数据挖掘信令分析模块架构   下载原图

    Figure 3. Signaling analysis module architecture for parallel data mining

    数据库和数据仓库主要存储信令采集模块的原始信令数据, 通过多种类型的数据库共享, 分布式存储各种信令数据信息。同时, 通过信令数据集成和过滤实现信令数据的预处理。数据库服务器主要作为信令数据库与上层信令处理模块的中间接口。数据库服务器根据上层处理模块的命令, 提取相应的数据库中存储的信令数据并上传。

    数据挖掘引擎是整个并行数据挖掘信令分析模块的核心部分, 包括对信令数据的预处理以及对象化和特征化分析。通过调用算法库中的数据挖掘算法以及其他相应的处理算法, 实现信令数据的分析处理。

    算法库中内置了数据挖掘算法以及其他数据处理算法, 主要包括聚类、分类、特征分析和偏差分析等算法, 可以实现对原始信令数据的分析。算法库中通过接收数据挖掘引擎发布的任务进行响应, 并根据任务需要使用相应的算法对信令数据进行处理。

    模式评估模块主要是负责与数据挖掘引擎进行交互, 从而命令其进行数据挖掘分析或者结束当前任务。数据挖掘的力度及效果, 与数据的规模、类型有关。模式评估模块通过设置阈值过滤信令数据, 并识别数据规模, 根据识别的结果调整数据挖掘模块的参数以提高信令数据挖掘分析的最终效果。

    3 面向数据挖掘的信令存储

    面向数据挖掘的信令存储仓库主要包括信令数据管理工具、ETL工具、信令源数据库以及信令建模工具等, 具体结构如图4所示。其中, ETL工具与多种类型的数据库进行交互, 是面向数据挖掘的信令存储仓库的核心模块。

    信令数据建模子模块主要是负责对信令数据进行处理, 将信令数据对象化, 方便进行存储以及数据挖掘。信令数据挖掘需要根据信令数据的特征进行分类分析, 需要信令建模工具首先对此年来源数据库和数据仓库中存数的原始信令数据进行建模和分析, 对信令数据进行初步的检验、整理和优化加工。

    信令源数据库主要负责存储原始信令数据以及信令数据模型。信令源数据库存储的源数据包括关系数据源、信令数据源以及其他相关数据源, 例如文本文件、Web文件等。信令源数据库在存储信令源数据的基础上, 利用信令数据建模工具对原始信令数据进行初步的变化, 将处理后具有数据特征的信令数据进行存储, 以方便ETL工具进行数据挖掘分析。

    ETL工具主要负责信令数据的存储管理, 包括对信令源数据的调取、组合和转化等操作。ETL工具与信令源数据库进行交互, 根据需要调取信令关系数据、信令数据以及其他通信信令数据等。并经过信令仓库进行进一步的数据处理和转化, 以进行信令数据挖掘分析。ETL工具根据一定的数据抽取和加工规则对原始信令数据进行加工, 并在信令数据加工过程中保存数据的一致性和完整性。

    数据仓库主要是负责整个信令储存模块的最终数据存储。经过预处理的信令数据在数据仓库进行缓存, 以进行数据挖掘和数据分析, 并将信令数据挖掘和分析的结果保存在数据仓库中方便前端平台对数据进行查询和调用。同时数据仓库中还保留了所采集的原始信令数据以进行备份, 进一步保证信令数据的完整性。

    数据管理工具主要负责整个信令数据存储模块的统一调度以及数据安全监测等工作。在基于信令数据挖掘分析以及存储的过程中, 数据管理工具对数据仓库进行有效监控, 确保信令存储数据安全合法。同时, 还保证缓存的信令数据不会溢出。数据管理工具不仅监控信令数据本身的安全性, 且也对上层处理模块的数据调用命令及数据操作的安全性进行评估。并实时生成动态安全报表, 以方便进行查询。

    4 信令数据集成应用

    基于数据挖掘的信令分析应用平台, 如图5所示。平台自下而上可以分为:信令共享层、信令数据分析处理层和信令应用层这3个部分。

    图5 基于数据挖掘的信令分析应用平台   下载原图

    Figure 5. Signal analysis application platform based on data mining

    应用分析模块是分析应用场景和业务需求, 根据需求对信令共享层上传的信令数据进行抽样, 利用关键词对整个信令数据进行过滤。并根据应用场景选择数据挖掘的应用模型, 对信令数据进行数据挖掘分析。

    信令共享层是将信令采集层采集的海量信令数据进行分布式共享, 使得信令分析处理层可以调用任意采集点采集的信令数据进行分析和处理。

    信令数据分析处理层对信令数据进行包括数据挖掘分析在内的一系列数据处理。根据应用分析模块确定的应用场景, 在模型库中选择合适的模型进行初始参数配置以及数据训练。信令数据分析处理层的算法库中内置了不同的数据处理算法, 根据需求在其中选择合适的算法对信令数据进行挖掘分析。并对分析的数据进行评估, 将评估结果反馈给上层应用模块。

    信令应用层是对信令数据处理结果进行应用, 根据信令数据挖掘分析的结果, 以提高移动通信业务的质量或者进行智能化应用。应用层根据应用场景将信息反馈给应用分析模块, 从而形成一整个信令数据分析与应用闭环, 提高信令数据集成存储和应用的一体化水平。

    5 结束语

    针对现有信令监测系统中信令存储和分析应用模块独立式架构不利于大规模信令数据分析与保存的问题, 本文分别从并行数据挖掘信令分析模块架构, 面向数据挖掘的信令存储技术和信令数据集成应用平台3个模块进行设计。利用数据挖掘技术和ETL工具实现了信令数据存储与分析应用一体化设计, 增强了信令监测系统的工程实用性。
