    摘    要：

    网络性能异常检测对于促进网络健康发展具有重要意义。针对目前全网性能异常检测大多通过离线检测,无法提供良好的实时在线检测性能的问题。文中采用主成分分析方法建立异常检测模型,结合历史性能数据和近期网络性能波动状况去适应性调整网络异常判断阈值,实现了异常检测的实时在线化,并在NFV网络上进行数据采集。实验结果表明,与被广泛采用的离线检测方法比较,该方法在检测的误报率上减少了5%～8%,对于网络运行商而言具有较大的使用价值。

    ZHANG Tianqi ZHANG Shunkang

    Abstract：

    Network performance anomaly detection is of great significance to promote the healthy development of the network. In order to solve the problem that the existing anomaly detection methods are mostly offline and cannot provide good real-time online detection performance, the principal component analysis method was used to establish the anomaly detection model, and the network anomaly detection threshold was adjusted adaptively by combining historical performance data with recent network performance fluctuations. The real-time online anomaly detection was realized and the data was collected on NFV network. The experimental results showed that the proposed method reduced the false alarm rate by 5% to 8% compared with the offline detection method which was widely used, indicating the greater use value of the proposed method for network operators.

    随着互联网技术的发展,人们越来越重视网络安全问题,网络异常监测也有了很大的发展,大量的研究人员致力于网络异常检测的研究。网络性能异常检测的目的是通过对网络性能的监测,以网络性能在正常情况下为基准,准确地检测网络中异常时间段。当网络异常时能够及时发出警报,提醒网络和系统管理人员采取有效的方法去维护网络的正常运行。

    国内外研究人员已经就网络异常检测进行了大量工作。运营商网络、企业网络为了确保网络的可用性和安全性,改善网络性能,需要部署网络监测系统以实时获取网络运营的性能数据。例如Cisco IPSLA[1]使用主动分组测量网络性能,其中IPSLA收集网络时延、时延抖动、丢包率、服务响应时间、服务质量等级等性能参数。测量的结果可以存储在交换机本地,通过SNMP或者CLI(命令行界面)检索;文献[2]中提出了一种轻量级的在线网络异常检测方法,该方法使用模式匹配与频繁挖掘的算法进行异常检测。但是,上述方法仅能关注单条端到端路径的性能异常,不适用于大规模网络环境,检测的精度也难以保证;文献[3]描述了一种对全网络所有链路进行有效监控的方法,利用贪婪启发式算法选择最小路径去覆盖所有链路,但是网络测量的开销会对网络性能造成不可忽略的影响;文献[4]中利用小波变换和主元分析的方法提出了一种基于多尺度主成分分析的全网络异常检测方法;文献[5]中利用流量特征的规律趋势特性,提出了一种基于势变化的全网络异常在线异常检测方法,但是上述方法值是在离线方式下使用滑动窗口进行了在线改造,不具有实时特性。

    文献[6,7]都是基于统计分析的网络异常检测技术,通过提取数据进行量化,通过训练统计相应特征的出现概率以及转移概率,从而建立马尔科夫模型以判断网络是否发生异常;文献[8,9]将数据挖掘技术运用到实时网络异常检测环境中,能够从海量的历史数据中提取知识,并快速、自动地产生异常检测模型。但该方法误报率比较高、计算复杂度比较大、训练数据多;文献[10]中提出了一种基于核递归最小二乘法的全网络在线异常检测方法,该方法具有较低的但不执行时间,能够满足实时性需求。但是该方法误报率比较高,系统可用性难以保证;文献[11]将管理信息库(Management Information Base,MIB)与信息信息熵统计方法相结合,采用固定时间窗口进行在线实时扩展,具有一定的准确性和精确性。

    本文采用边缘测量方式对全网进行性能探测,运行主成分分析算法对网络性能矩阵建立离线异常检测模型,并且结合近期网络的性能波动,进行了实时性的扩展,使得异常检测的判断基准能够具有一定的实时效应与准确性。

    1 网络性能矩阵

    早期的网络异常检测研究主要集中在互联网服务提供商中的节点或者单跳条链路级别上。这些互联网性能建模方法都是零维或者一维的,只能对因特网的局部进行性能建模分析。面对日益复杂的因特网,这些局部的建模和分析方法的局限性越来越大。为了描述网络的整体性能以及时空特性,本文一种基于矩阵的网络性能表示法[12](Matrix based Network Performance Representation,MNPR)。MNRP是一种通用的网络性能表示法,它屏蔽了网络内部结构的复杂性和异构性,刻画了网络对外呈现的性能,并且具有强大的信息表达能力。

    为了使得性能矩阵具有时域性,本文对此性能矩阵进行了改进,给定网络中测量点的数量为K,时域性网络性能矩阵将每个测量周期得到的测量值排列成一个1×P(P=K×(K-1))的向量。而将T个测量周期内获得的所有测量值排横向排列成一个T×P矩阵。第t行表示在第t个测量周期内测量值的向量,第j列表示第j个对之间的测量序列,该矩阵称X为该AS时域性能矩阵。下图1展示了一个具有5个域间出口的AS,进行了100个周期的测量,该AS的性能可以表示为一个100×20的网络时域性能矩阵。

    图1 时域性能矩阵   下载原图

    Figure 1. Time-domain performance matrix

    网络测量点数量和位置的选择也会对异常检测结果造成很大的影响。如果部署的测量点很多,会造成很大的测量开销,一定程度上会对网络的性能造成影响。而测量点位置的选择要尽可能靠近网络的出入口,扩大测量覆盖率。基于上述考虑,本文采用边缘测量方式,在网络的出入口部署测量程序。

    2 基于主成分分析的在线实时异常检测

    异常是相对于正常而言的,所以异常检测必须要建立在正常的性能模型上,然后才将测量所得的单个样本与先前建立的正常模型进行分析比较,从而确定这个测量样本是否异常。

    本文采用的主成分分析法(Principal Component analysis,PCA)是一种数据降维的方法,可以将原数据进行线性变换,通过提取最少数量的主成分去减少数据空间的维度。被提取出来的主成分必须能够最大限度的包含原始性能数据的特征,同时最小限度的损失原始数据的信息量,以期在降维的同时发现性能数据的最主要特性,发现高纬度背后更丰富的内涵,并以此作为对网络性能进行异常检测的依据。

    其检测过程主要分成两个阶段:第一个阶段是经验数据的训练过程;第二个阶段是在线的异常检测阶段。

    2.2 在线网络异常检测及在线扩展

    在建立网络正常性能的模型以后,测量得到的实时性能数据需要对其进行处理,计算其SPE,并且与离线PCA异常模型的阈值进行比较,如果超过阈值,则网络发生异常。异常检测过程如下所示:

    对于第i个测量周期的在线实时数据X(i)(X(i)为1×m的矩阵),进行中心化和标准化处理得X(i)s;然后根据式(8)求出计算当前测量周期数据的SPE

    SPE=X(i)s(I-ρkρTk)X(i)sΤ (8)

    式中,X(i)s为经过归一化处理的实时性能数据;X(i)sΤ为X(i)s的转置向量;ρk为主元向量;ρTk为主元向量的转置。

    本文考虑到网络性能可能随着时间的变化,会长期处于一种较好或者较坏的阶段,仅用根据某段正常情况下网络的历史性能数据建立出来的异常判断标准是不够的。因此,本文将网络的历史性能数据和近阶段的性能数据进行综合考虑。

    (1)处理近期数据。使用指定大小的滑动窗口的数据来发映出短期的网络性能的动态改变,这个是确定动态阈值的可靠基础。短期平方预测误差SPE(t)的均值和标准差由式(9)计算而来

    式中,M是滑动窗口的大小;SPE(t)是在实时测量阶段的第t个周期测量数据的平方预测误差;StdST(t)代表了在第t测量周期的网络性能波动情况。式(10)结合了历史训练数据的异常阈值和近期波动

    thresholdnew(t)=α×thresholdhistory+(1-α)×StdST(t) (10)

    式中,thresholdhistory表示的是在离线PCA模型中通过历史数据得出的性能阈值;α表示的是历史数据和近期网络的性能波动权重,取值范围在0～1之间。

    (2)比较实时测量的数据的SPE与thresholdnew(t),如果SPE大于thresholdnew(t),则发出警报,发生异常。

    3 基于主成分分析的在线实时异常检测

    针对上文的在线实时异常检测算法,建立了实验环境并进行试验和分析。

    3.1 试验场景与测试用例

    网络异常检测算法性能评价的数据来源常用的有两类:模拟网络数据实验方法[13,14]和网络实测数据分析[15]。模拟网络是通过仿真器人为生成一些网络参数去运作,实验细节可以完全掌握,缺点是不够真实;实测数据分析法场景真实但是比较难以获取,实验成本过大。本文采用的是通过网络功能虚拟化(Network Function Virtualization,NFV)技术虚拟化一个高逼真度的网络场景。NFV技术将网络设备的功能用软件的形式部署在标准服务器上,以较低的成本和人力即可搭建出具有和真实网络同样等价的NFV网络。

    图2 网络试验环境拓扑   下载原图

    Figure 2. Topology of network test environment

    本文通过在物理服务器上搭建出如图2所示的网络环境,NFV网络节点采用的承载容器是LXC(Linux Container),在LXC上通过部署开源软件Quagga可以将LXC配置成一个虚拟思科路由器。部署Open switch可以将LXC配置成一台标准交换机,NFV网络的设备间的连接通过网桥(Bridge)建立。在线实时异常检测算法的测试用例设计如下:

    (1)在便携机上部署控制器和分析器,控制器负责控制NFV网络性能测量过程,分析器负责处理得到的性能数据以及判断是否出现性能异常;

    (2)选择在R1～R6共6个接入路由器上部署测量探针,每个探针都会对其他探针发起RTT测量。探针依次向各测量目标发送ICMP报文,每次发送21个报文,报文长度随机变化,测量周期为30 s;每个测量周期结束,测量中间和将返回的RTT测量数据集合发送给测量分析器;

    (3)在虚拟主机上部署分布式流量发生器[16],用流量生成脚本控制流量模式(泊松流)、流量分布(通过主要路径)和流量强度(平均占带宽的30%);

    (4)在特定时刻,通过虚拟主机向网络特定部位注入异常大流,使网络这些部位产生拥塞;

    (5)测量分析器本文设计的异常检测步骤对其进行异常检测。

    4 结束语

    本文针对当前网络性能异常检测缺少动态检测方法,大多通过处理离线数据,不具备实时检测特性的问题,提出了一种基于主成分分析的实时网络性能异常方法。该方法先用主成分分析方法对网络性能历史经验性能数据分析处理,建立异常检测先验模型,然后计算实时测量时在滑动窗口下的平方预测误差的波动状况,赋予历史数据和近期波动比例权值,并动态实时更新异常判断阈值。通过在NFV网络进行的实验结果表明,本文所提出的方法不仅能够降低异常检测的误报率,而且具有实时适应性,对于网络运行商和网络管理员具有一定的使用价值。
