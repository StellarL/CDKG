    摘    要：

    针对RRT算法收敛速度慢、路径不规则的问题, 基于双向RRT算法和概率搜索策略, 提出了一种变概率策略下的双向RRT搜索优化算法。该算法根据搜索节点的不同周边状态, 采用不同的概率策略扩展目标点, 使搜索算法能够在空旷场景下向目标点快速生成, 同时避免了路径陷入局部最小值的问题。在完成初次路径搜索后, 根据路径节点位置优化不规则路线, 减少小车行驶过程中的拐弯次数和总路径长度。在仿真中进行了多场景重复试验测试, 仿真结果表明, 改进后的算法在搜索速度和路径长度上有明显改善。

    智能车; 路径规划; 双向RRT; 概率搜索; 收敛速度; 路径不规则;

    An Improved Path Planning Algorithm With Variable Probability Bidirectional RRT

    HU Huimian

    School of Optical-Electrical and Computer Engineering, University of Shanghai for Science and Technology

    Abstract：

    To deal with the problems of slow convergence speed and the irregular searching path of the RRT algorithm, a variable probability search strategy optimization algorithm based on the bidirectional RRT was proposed. This bidirectional RRT algorithm searched different surrounding states of nodes, then extended the target point with different probability strategies. Therefore, this algorithm could rapidly converge to the target in the open environment as well as avoid falling into local minimum. After the initial path search was completed, the irregular route was optimized according to the location of the path node, and the number of turning times during the car driving and the length of the total path were effectively reduced.The multi scene repeated test was carried out in the simulation, and the simulation results showed that this algorithm had a significant improvement in the search speed and path length.

    Keyword：

    intelligent vehicle; path planning; bidirectional RRT; probability search; convergence speed; irregular searching path;

    智能车 (Intelligent Vehicle) 是指一种智能轮式移动机器人, 目前广泛应用于军事、运输、交通等领域。在其技术研究中, 自主路径规划是最为重要的技术之一。路径规划是指智能车在位姿空间中找到一条初始位姿点到目标位姿点的连续无碰撞路径, 同时这条路径还要满足环境约束以及智能车本身运动特点。

    1998年, 美国爱荷华州立大学LaValle教授提出了快速搜索随机树 (Rapid-exploring Random Tree, RRT) [1]算法。该算法的核心是随机搜索方式, 能够在高维非凸空间实现路径搜索。由于其采用随机采样的规划方式, 不需要预处理, 而且搜索速度快, 在高维空间速度优势尤其明显, 近年来在机器人运动规划领域得到了广泛的应用和研究[2]。但是其本身也存在一些缺陷: (1) 采样为随机采样, 大量无效采样导致算法收敛速度慢[3]; (2) 度量函数 (最近邻算法) 在存在复杂约束的情况下可能会很难获得有效解[4]; (3) 算法的随机性会导致生成的路径不平滑, 直接使用路径时会出现大量转折与无效曲线路径。针对基本RRT 算法的上述不足, 国内外学者也对该算法进行了不断地改进, 以使其适应不同的应用环境。Kuffner和LaValle提出了双向RRT (Bidirectional RRT, Bi-RRT) 算法[5], 即从初始状态和目标状态并行生成两棵树, 从而快速实现收敛。随后他们又提出了RRT-connect[6]算法, 该算法在节点扩展时采用指向性拓展, 其指向性拓展的概率也需要在实际应用中根据情况来选取。对于RRT算法的随机性所造成的路径不平滑问题, Fraichard 和Scheuer[7]提出了用回旋曲线来做平滑处理, 但是回旋曲线求解计算量很大。Lau等人[8]采用了5次贝塞尔曲线, 但是没有考虑路径曲率的连续性和机器人的自身特点, 而且该方法计算量也较大。上述两种方式均适用于不可顶点拐弯的智能车, 但对于可顶点拐弯的智能车, 其还有进一步优化空间。总体上RRT算法的优化都是通过增加各种约束的方式来使其进一步适应不同的应用场景, 提升收敛速度, 并努力向实用化、场景化发展。

    根据上文的发展脉络, 本文主要的研究目是优化RRT算法, 为提升路径生成速率及使其适用于四轮智能车本身特点进行算法改进。本文在双向RRT算法的基础上进行优化改进, 首先通过车辆模型将车辆体积纳入路径规划中, 在状态空间图形中设定车辆体积参数, 避免车辆与障碍发生碰撞。然后, 在路径规划时根据节点周边环境状态进行目标设定的概率选取:在周边空旷时采用终点为目标点生成下一个节点;在周边存在较多遮挡时进行概率p的目标选取, 其中若概率为p则选择终点为目标点, 若概率为1-p则选择随机点为目标点。在完成路径生成后, 根据节点情况进行路线优化, 减少路径转折的出现次数并降低总的路径长度, 提升智能车运行速度以实现算法实用化。

    1 简化的车辆模型

    智能车辆由于其不同的机械结构、控制环路, 因此会存在不同的运动模型, 本文使用的智能车辆简化模型如图1所示。在状态空间中车辆状态X= (x, y, z) , 其中 (x, y) 是车辆正中心在系统坐标系下的位置;θ是车辆纵轴与坐标系之间的夹角。车辆直行时两侧车轮同向滚动, 转弯时通过两侧车轮反向滚动实现角度θ的改变。在任何瞬间, 智能车只能实现前进或转弯中的一个动作。

    2 双向RRT算法与优化

    2.1 原始双向RRT算法

    双向RRT算法的基本思想是:以起点和目标点为两个搜索树的初始点, 分别构建搜索树。每次迭代生长时, 以其中一棵树进行扩展, 尝试连接另一棵树的最近节点来扩展新节点。当两棵树上某两个节点距离小于设定值时, 即认为搜索完成。这里采用有向图G= (V, E) 来表示路径, 则一条可行的路径就是一个顶点的序列 (V1, V2, V3, …, Vn) , 其中V1=qinit, Vn=qgoal。同时 (Vi, Vi+1) ∈E, 1≤i≤n-1表示边;E为图形内可用点集合。基本的双向RRT伪代码 (算法1) 如下:

    1. V1←{qinit}, E1←∅;G1← (V1, E1) ;

    2. V2←{qgoal};E2←∅;G2← (V2, E2) ;i←0;

    3.whilei<Ndo

    4. qrand←Sample (i) ;←i+1;

    5. qnearst←Nearst (G1, qrand) ;

    6. qnew←Steer (qnearst, qrand) ;

    7.if ObstacleFree (qnearst, qnew) thn

    8.V1←V1∪{qnew};

    9. E1←E1∪{ (qnearst, qnew) };

    10. q′nearst←Nearst (G2, qnew) ;

    11. q′new←Steer (q′nearst, q′new) ;

    12. if ObstacleFree (q′nearst, q′new) then

    13. V2←V2∪{q′new};

    14. E2←E2∪{ (q′nearst, q′new) };

    15. do

    16. q″new←Steer (q′new, qnew) ;

    17. if ObstacleFree (q″new, q′new) then

    18. V2←V2∪{q″new};

    19. E2←E2∪{ (q″new, q′new) };

    20. q′new←q″new;

    21. else break;

    22. while not q′new=qnew

    23. if q′new=qnewthen return (V1, E1) ;

    24. if |V2|<|V1| then Swap (V1 , V2) ;

    上式中, 前两步是以起点qinit和终点qgoal作为两个生成树的起始点, 且检查两点均在图形内。然后每一次迭代中, 开始步骤与原始的RRT算法一样, 都是采样随机点然后进行扩展。扩展完第一棵树的新节点qnew后, 以这个新的目标点作为第二棵树扩展的方向。第二棵树扩展的方式略有不同, 首先它会扩展第一步得到q′new, 如果没有碰撞, 则继续往相同的方向扩展第二步, 直到扩展失败或者q′new=qnew表示与第一棵树相连, 则整个算法结束。

    2.2 状态空间预处理

    在实际使用中, 由于需要考虑车辆本身的体积, 因此状态空间边缘和车辆中心的 (x, y) 实际上是需要保持距离的。由此可知, 在导入状态空间图时, 需要根据车辆体积的设定进行状态空间预处理, 将状态空间边缘进行扩展保护, 防止节点过于接近状态空间边缘而导致碰撞。

    2.3 变概率目标选取策略

    根据双向RRT算法的流程可以看出其实现过程中, 扩展时采样是针对整个状态空间进行均匀随机的采样。虽然这种方式能够有效地搜索所有未知空间, 但是其在过多不必要的位置进行了大量采样并扩展出多种分支, 极大地浪费了搜索时间, 大幅降低了算法收敛速度。

    为克服这一缺陷, 本文采用了一种基于节点环境的变概率目标选取策略, 即在生成一个节点后, 采用函数detect (q) 来检测周边单个距离间隔内是否存在遮挡 (即不包含于可用状态空间的点) 。如不存在遮挡, 则选取终点为目标点进行一次扩展, 得到一个树节点;反之采用一个设定的目标偏向概率值pset (一般设定小于10%) , 当本次随机生成概率p>pset时, 则采用随机函数生成, 当<pset时, 则采用终点为目标点。这种方式既能保证算法随机性, 又能加快向目标状态收敛的速度, 同时在较为空旷的状态空间中可以取得明显的改善效果。与原算法相比, 在路径生成过程中, 增加了周边环境的约束条件, 这一约束条件能够将目标节点的选择概率进行有目的性的改变。在大部分场景下, 算法的路径都会遇到周边环境无遮挡的节点, 这种节点的快速收敛能够极大地减少实际应用时无效路径搜索次数, 减少总的搜索时间。需要指出的是, 该算法也有一定概率导致局部最小解, 因此需要保留较大随机目标点概率, 以确保在局部寻找时能够得到路径结果。优化双向RRT算法 (算法2) 如下:

    1. V1←{qinit}, E1←∅;G1← (V1, E1) ;

    2. V2←{qgoal};E2←∅;G2← (V2, E2) ;i←0;

    3.Set Pset;

    4.while i<N do

    5. if (detect (qinit) ==1) then

    6. qrand←qgoal; i←i+1;

    7. else (P<Pset)

    8. qrand←qgoal; i←i+1

    9. else

    10. qrand←Sample (i) ; i←i+1;

    11. qnearst←Nearst (G1, qrand) ;

    12. qnew←Steer (qnearst, qrand) ;

    13. Same as Bidirectional RRT

    2.4 后处理优化

    鉴于RRT算法采样的随机性, 尽管采用了目标指向的选取策略, 但是生成的路径通常都是不自然、曲折的。在实际使用中, 折点会带来智能车重复操作转向, 使其行驶不必要的路径, 并加速车辆磨损。因此, 基于实际应用提出了基于最短路径的路径后处理流程, 对生成后的路径进行去弯取直, 删除不必要的节点以缩短路径长度。Post-Process (算法3) 流程如下:

    1. Path ←obtained from Optimize Bidirectional RRT

    2. Var Q : Path

    3. Q (q0, q1, q2, …, qn) =Path;

    4. qtemp←q0;

    5. whileqtemp!=qndo

    6. for each nodeqi∈Q

    7. if checkPath (q0, qi)

    8. detect qi;

    9. else

    10. qtemp=qi;

    11. break

    后处理流程中, 首先需要导入在之前路径生成中获得的路径节点;然后以第一个节点为起点, 检查与后续节点的连接线是否与障碍空间形成交集。如没有则删除此节点, 反之以新的节点为起点继续检查后续节点连接情况。与原算法相比, 后续的处理能够让智能车的算法应用更为合理, 减少无效拐弯节点和路径, 进一步提升最终路径合理程度。

    4 结束语

    本文结合了双向RRT算法和实际智能车模型的应用场景, 首先利用状态空间预处理避免车辆在路径上发生碰撞;然后在原始双向RRT算法上实现变概率优化算法, 利用目标指向的方式提高搜索速度, 降低计算量;最后通过后处理实现路径长度和节点个数的降低。仿真结果表明, 该算法能够提升搜索速度, 同时降低路径长度和节点个数并确保了可通行性。
