    摘    要：

    目前很多电力电子装置采用非标准通讯规约,以至其无法直接与上位机通讯。文中介绍了一种嵌入式组态触摸屏与非标准通讯规约设备之间通讯的实现方法。该方法使用嵌入式组态软件提供的脚本驱动开发工具,编写非标准通讯规约的设备驱动,并将此设备驱动加载到嵌入式组态软件中,实现了两者之间的通讯。实际测试结果显示,使用该设备驱动,嵌入式组态触摸屏与非标准通讯规约设备之间通讯正常,遥测值读取准确,并且可以按照设定值定时刷新,遥调指令下发正确,从机设备可按照指令正确改变工作模式。

    A Method of Communication Between Embedded Configuration Touch Screen and Non-standard Communication Protocol Equipment

    DING Laiwei DONG Biao LI Pengcheng DU Haijiang

    College of Information and Electrical Engineering,China Agricultural University

    Abstract：

    Many power electronic devices adopt non-standard communication protocol, making them unable to communicate with the upper machine directly. This paper introduced a method to solve the communication problem between embedded configuration touch screen and non-standard communication protocol equipment. The method uses the script-driven development tool provided by the embedded configuration software to write the device driver of the non-standard communication protocol. Then the device driver was loaded into the embedded configuration software to realize the communication. The test results showed that the communication between embedded configuration touch screen and non-standard communication protocol equipment was correct by using the device driver, and the sampling values received by embedded configuration touch screen were right and refreshed correctly. At the same time, the instruction was sent to equipment correctly and the equipment could change its mode following the instruction.

    power electronic equipment; non-standard communication protocol; embedded configuration touch screen; configuration software; script-driven development tool; device driver;

    SCADA(Supervisory Control and Data Acquisition)即数据采集与监控系统,一般采用设备层-通讯层-监控管理层三层设计架构[1]。监控管理层一般使用组态软件进行HMI(人机界面)的设计和实时数据库的管理;通讯层(也称为通讯管理机)实现不同设备通讯规约的处理及与监控管理层的数据交换[2],是监控管理层和设备层的中间纽带[3];一些小型系统中使用嵌入式组态触摸屏来替代通讯层和监控管理层,直接与设备层通讯。嵌入式组态触摸屏由软件部分和硬件部分组成,用来替代传统的PC机,以达到控制系统小型化和专业化的需求[4],其软件部分包括嵌入式组态软件和嵌入式操作系统,硬件部分则包括微处理器和各种外设(包括触摸屏)[5]。

    随着电力工业的发展,已经形成了多种通讯规约,例如Modbus规约[6]、IEC101、IEC103、IEC104等[7]。有的设备厂家还会根据需要自行开发各种标准规约的变种规约,因此导致通讯规约种类繁多[8]。由于各种规约的设备驱动开发比较复杂,开发成本较高,嵌入式组态触摸屏支持的通讯规约数量有限,因此嵌入式组态触摸屏与非标准通讯规约设备之间通讯的实现是SCADA系统开发的重要工作,目前主要有以下两种方法:

    (1)直接对嵌入式组态触摸屏进行自主开发使其支持多种通讯规约,一是使用高级语言如C++,自主开发嵌入式组态触摸屏的可执行程序,在此程序中编写各种通讯规约的设备驱动,用该程序替换原有的程序以实现非标准通讯规约的开发。该方法需要具备较高的软件基础,难度较高[9];二是有些组态软件提供了与其相兼容的脚本驱动开发工具,可以使用该驱动开发工具开发各种规约的设备驱动,将开发的设备驱动加载到嵌入式组态软件中,即可实现触摸屏与设备的通讯。该方法成本低,且开发难度较小;

    (2)使用通讯管理机间接实现嵌入式组态触摸屏与非标准通讯规约设备之间的通讯[10,11]。通讯管理机自带了多种通讯规约,同时也具备规约转换功能,将其加在嵌入式组态触摸屏和非标准通讯规约设备之间,即可实现两者之间的通讯。该方法成本较高,对没有开发基础的人员比较适用。

    本文通过使用MCGS(Monitor and Control Generated System for Embeded)组态软件提供的脚本驱动开发工具,开发了非标准Modbus-RTU规约的设备驱动,并最终实现了MCGS嵌入式组态触摸屏与非标准Modbus-RTU规约设备的通讯。

    1 嵌入式组态触摸屏通讯规约开发过程

    MCGS集成了组态界面设计、标准MODBUS及常用MODBUS变种规约和实时数据库,可用于工业现场监控系统的应用[12,13,14]。通过自带的设备驱动程序,将从设备采集的数据与实时数据库中的变量关联,即可实现MCGS嵌入式组态触摸屏与设备的通讯[15],其通讯过程如图1所示。

    图1 MCGS与外部设备通讯示意图   下载原图

    Figure 1. Communication schematic between MCGS and external devices

    由图1可以看到,将组态软件自带的设备驱动以及使用脚本驱动开发工具开发的设备驱动加载到设备窗口之后,即可实现外部设备与实时数据库的连接。MCGS嵌入式组态触摸屏支持RS232串口、RS485接口、以太网口等通讯接口,其链路层参数可通过设备属性窗口进行配置。MCGS脚本驱动开发工具提供了丰富的函数库,同时支持用户自定义函数,可以实现问答式通讯规约和循环式通讯规约的设备驱动的开发。将开发的设备驱动添加到组态软件的设备窗口,即可实现嵌入式组态触摸屏与从机设备的通讯。

    脚本驱动开发工具提供了向导式窗口,对于简单的通讯规约,可不用编程,仅通过参数配置即可完成开发;比较复杂的规约则可在向导式窗口中实现基本配置后,在生成的脚本程序基础上,利用脚本语言进一步开发实现;此外也可以不用向导,直接用脚本开发。

    脚本驱动开发工具具有4种脚本:采集脚本、单通道写脚本、初始化脚本、退出脚本。采集脚本用于MCGS从下位机周期性地读取实时遥测或遥信数据,其采集周期在MCGS组态软件的设备窗口中设置;单通道写脚本用于MCGS向下位机下发遥调或遥控指令,只有当遥调或遥控量发生变化时,该脚本才会被自动调用;初始化脚本用来完成一些初始化的工作,如对全局变量和设备状态的初始化;退出脚本在驱动退出工作时,完成一些清理工作,如释放资源以及保存数据等。初始化脚本和退出脚本可根据实际需要选择编写与否。

    将脚本驱动工具开发生成的.mdr文件拷贝到组态软件drivers文件夹下,即可用设备管理窗口加载该驱动使用。加载设备驱动之前,需根据通讯规约的要求先在设备窗口中添加通用串口父设备或者通用TCP/IP父设备,将开发的设备驱动挂在父设备下,然后再对父设备以及设备驱动按照通讯规约的要求进行属性配置。

    下面以某虚拟同步机项目中非标规约开发过程为例进行分析,根据实际需要,只需开发采集脚本和单通道写脚本。

    2 非标准Modbus-RTU规约设备驱动开发

    某虚拟同步机展示平台,使用1台MGCS嵌入式组态触摸屏作为上位机,用来控制整个系统;3台光伏逆变器,采用标准Modbus-RTU规约;1台储能逆变器,使用厂家自己定义的非标准Modbus-RTU规约。

    2.1 非标准Modbus-RTU规约格式说明

    该项目所使用储能逆变器通讯规约的数据帧格式如图2所示。

    图2 非标准Modbus-RTU数据帧格式   下载原图

    Figure 2. Data frame format of non-standard Modbus-RTU

    帧头是两个固定字节0xAAAA。源地址为上位机地址,用2 Byte表示;目标地址为储能逆变器地址,用两个字节表示;控制码通知储能逆变器应该执行何种操作,01为Read指令,02为Write指令,03为Execute指令(设置储能逆变器开关机及充放电模式);功能码通知储能逆变器操作的具体内容,1 Byte;数据长度为数据区内数据的字节数,1 Byte;若为Write指令,数据区内为需要下发的数据,若为Read以及Execute指令,则数据区内为空;校验和为从帧头到数据区内所有字节的求和,2 Byte。

    2.2 非标准Modbus-RTU规约设备驱动开发流程

    设备驱动包括采集脚本和单通道写脚本,其主要开发步骤如下所示:

    (1)使用向导生成采集脚本。

    1)配置属性。

    此处主要用来配置设备地址和通讯延时。设备地址即为储能逆变器的地址,设为18;通讯延时代表读写过程的等待延时,设为200 ms,如果在该时间内完成一次读写,则返回读取的字节数;如果没有完成一次读写,则返回一个异常值;

    2)配置通道。

    设备通道是定义设备驱动和实时数据库对象之间的接口,它用于实现驱动的数据输出功能,和实时数据对象建立实时的数据交换,设备通道的功能实现流程如图3所示;

    图3 设备通道的功能实现   下载原图

    Figure 3. Function implementation of equipment channel

    3)配置通讯帧。

    通讯帧即为通讯规约的格式,包括发送帧与接收帧,按照图2中给定的规约格式配置即可;

    4)配置预览。

    查看通道配置情况。

    以上4步配置完成后,点击设置完成按钮,自动生成采集脚本程序,根据需要,可对脚本程序进行修改;

    (2)直接编程生成采集脚本。

    如上文所述,采集脚本也可以不使用向导直接编程,程序流程按文献[16]提出的有限状态机进行描述,如图4所示。

    图4 生成采集脚本流程   下载原图

    Figure 4. Process of generating collection script

    MGCS组态软件会根据设备窗口中设置的周期循环调用该采集脚本。在采集脚本中,首先根据通讯规约格式,将读取储能逆变器实时数据的指令按字节存入发送数组中,发送至储能逆变器,然后等待一定时间-200 ms,将储能逆变器发送过来的数据存入接收数组中。对接收到的数据个数进行统计,如果数量不对,则报通讯故障;如果数量正确,则将接收数组中的数据区的内容存入数据数组中。将数据数组中的数值按照规则,组成16位整型变量,将变量存入对应的设备通道,设备通道与组态软件中的实时数据对应。至此完成了采集脚本的编写;

    图5 生成单通道写脚本流程   下载原图

    Figure 5. Process of generating single channel script

    (3)生成单通道写脚本。

    对于单通道写脚本,无法使用向导生成脚本,需要直接编程,其步骤如下:

    1)右键设备窗口中的设备通道,点击添加,根据实际需要添加3个下发遥调量的写通道,分别命名为写模式,用来控制储能逆变器开关机和充放电模式;充电,用来设置蓄电池充电功率;放电,用来设置蓄电池放电功率;

    2)点击设备脚本中的单通道写脚本,按照脚本驱动开发工具定义的标准函数编写遥调指令,具体流程如图5所示。

    当MCGS组态软件中的遥调或遥控量发生改变时,MCGS将自动调用单通道写脚本。单通道写脚本首先要获取变化的遥调或遥控量所对应的设备通道号,然后根据各通道对应的遥调或遥控指令编写不同的发送帧。将发送帧下发到储能逆变器之后,储能逆变器会回复应答帧,若应答帧正确,则报写通道正常,否则,报写通道失败。

    当设备驱动开发完成之后,要将其加载到组态软件的设备窗口。MGCS嵌入式组态触摸屏与储能逆变器之间使用串口进行通讯,首先在MCGS组态软件中的设备窗口添加通用串口父设备,将开发的设备驱动挂在通用串口父设备下。然后,对通用串口父设备进行配置,采集周期设为1 000 ms,波特率按照储能逆变器通讯规约的要求设为9 600,数据位位数为8位,停止位位数为1位,数据校验方式为无校验。随后,对开发的设备驱动进行配置,采集周期设为1 000 ms,设备地址与通讯延时保持不变。完成了设备驱动的开发与加载后,通过测试虚拟同步机展示平台来验证MGCS嵌入式组态触摸屏与储能逆变器之间通讯的正确性。

    2.3 嵌入式组态触摸屏与储能逆变器通讯测试

    3 结束语

    嵌入式组态触摸屏广泛应用于各领域的监控系统中,需要与使用各种通讯规约的设备互相通讯。本文提出了一种嵌入式组态触摸屏与非标准通讯规约设备通讯的方法。该方法通过使用组态软件提供的脚本驱动开发工具开发了储能逆变器的设备驱动,实现了与嵌入式组态触摸屏的通讯,为解决嵌入式组态触摸屏的通讯问题提供了解决方案。
