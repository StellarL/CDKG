    基于Android的手机恶意代码检测与防护技术

    叶子 李若凡

    哈尔滨理工大学软件与微电子学院

    摘    要：

    随着通信技术以及移动终端的发展, Android系统由于其本身的开源性, 滋生了大量的恶意代码。为了满足Android手机用户的安全需求, 文中基于Android, 采用SVM机器学习思想, 构建了恶意代码检测模型, 并开发了一套手机恶意代码检测与防护系统, 可以对其进行快速检测和深度检测。系统经Android手机测试结果表明, 其具有较好的检测精度以及较低的恶意代码漏报率。

    手机安全; Android; 恶意代码检测; SVM; 深度检测;

    Malicious Code Detection and Protection Technology of Mobile Phone Based on Android

    YE Zi LI Ruofan

    School of Software and Microelectronics, Harbin University of Science and Technology

    Abstract：

    With the development of modern communication technology and the development of mobile terminal, Android system breeding a lot of malicious code, due to its own open source.In order to satisfy the requirement of the modern mobile secure, a professional malicious code detection model based on SVM was designed in this paper, realizing a malicious code detection and protection system. The system had been tested by Android mobile phone, which provided better detection accuracy and low false negative rate of malicious code.

    Keyword：

    mobile phone secure; Android; malicious code detection; SVM; depth detection;

    Received： 2018-11-21

    随着无线通信、移动互联网的发展与融合, 手机移动终端飞速发展, 智能手机、智能移动终端在人们日常生活、工作中成为必需品[1,2]。在多种移动设备操作系统的竞争中, Android平台凭借其优越的性能脱颖而出, 成为了最热门的手机系统之一[3,4,5,6]。但由于Android的开源、开放的特性, 基于传统PC、网络服务器传播的计算机病毒、恶意代码开始向手机、智能移动终端扩散。近年来, 手机用户重要信息泄露、手机安全等社会问题频发, 对用户的财产及人身安全造成了巨大的威胁[7]。相比于苹果公司的iOS系统, Android从底层的操作系统到上层的用户界面均是开源的, 这为Android系统内恶意代码的繁殖提供了温床, 使得恶意代码在手机内不断繁殖, 对手机内的应用程序、重要信息进行破坏[8]。在此背景下, 手机安全面临着严峻的挑战, 如何快速有效地进行恶意代码的检测与防护, 从而保证用户手机上的信息安全, 确保重要资料、个人信息不被泄露、窃取, 成为研究的热点。在手机平台上进行恶意代码的检测与防护是保证手机信息安全的重要途径[9]。本文研究了恶意代码的甄别与检测技术, 并基于Android平台建立了恶意代码检测模型, 实现了针对手机恶意代码的检测防护系统, 并对本系统的检测性能进行了全面测试[10]。

    1 系统分析与模块构建

    在对Android平台内的恶意代码检测时, 可静态的进行检测, 此时需要对该代码所占用系统权限、API调用序列、系统资源组件与代码构成的APK结构进行分解, 构建成特征向量, 输入到检测系统内, 应用相应的相似度检测算法, 与已有恶意代码库中的样本进行比对分析[11]。对于动态的恶意代码检测, 需要动态地进行恶意代码分析, 此时可重新编译Android源码为内核镜像, 利用此镜像进行模拟器加载, 实时监控系统上被检测代码的程序读写、权限调用等举动, 对代码做出的行为进行分析、模式匹配, 判断该代码是否为恶意代码, 指导系统做出防护行为[12]。经过上述分析, 结合本文在代码检测时使用的恶意代码检测模型, 采用结构化思想, 切实考虑系统在代码检测时所需的系统开销、检测程度, 在纵向上将恶意代码检测步骤分为快速检测和深度检测两步, 建立如图1所示的系统结构。

    图1 恶意代码检测模型   下载原图

    Figure 1. Malware detection model

    由于检测模型需要在恶意代码运行时快速识别, 及时杀出、止损, 因此对模型检测的速度有着较高需求。基于此需求, 本模型引入快速检测引擎, 提高检测速度。快速检测引擎可以对大量已知的恶意代码进行识别, 该引擎基于已有的恶意代码数据样本, 经过对APK的特征提取后, 在样本库中进行比对鉴别。在APK输入到系统之后, 大多数已有恶意代码经过快速检测将被鉴别。深度检测引擎通过预设的通信接口和快速检测引擎链接, 通信接口模块负责两个引擎间的数据交互。在深度检测引擎中, 通过机器学习构建的分类器对代码进行检测判别, 并对待测文件从多个维度进行特征提取和模式判别。在同一个终端设备中可以部署多个快速检测引擎与一个深度检测引擎, 由此可较好地对多组代码进行安全性检测, 同时兼顾系统的数据处理能力, 保障系统性能。在进行恶意代码检测时, 一般将深度检测引擎部署在服务器中, 而将快速检测引擎部署在手机终端上。在对恶意代码进行检测时, 存储在手机中的APK文件通过快速引擎检测, 对于无法甄别的代码通过通信接口送至服务器, 检验判别结果经服务器中的深度检测引擎检测后返回到手机终端。系统主要包括快速检测引擎、特征样本库、深度检测引擎和通信接口模块等。

    (1) 快速检测引擎。

    在该引擎中完成对APK文件的扫描, 解析代码格式, 同时在扫描过程中获取文件哈希、解析该文件的安装文件证书, 获取文件的MD5与证书签名等特征信息, MD5是一种散列函数, 可提供消息完整性保护, 在安全领域广泛使用。然后进行特征信息的匹配, 在进行匹配时, 快速检测引擎将检索文件的哈希特征库, 检索文件签名黑白名单, 判定该代码是否为恶意代码;

    (2) 特征样本库。

    主要包括文件的哈希特征库和文件的证书签名库。计算APK的哈希值是检测恶意代码最为简单、便捷的方法, 利用唯一标识文件的哈希值可有效的鉴别文件属性。文件的证书签名库由白名单库和黑名单库构成, 在经过扫描后获取文件的证书信息, 首先在白名单中检索, 若得到匹配项, 则将该APK标为可信任, 未收到地方篡改, 白名单库由可信任的第三方开发者的证书构成。对于未在白名单中出现的APK证书信息, 本模型在经过深度检测后将决定是否将其放入黑名单库;

    (3) 深度检测引擎。

    在深度搜索引擎中也有类似于快速检测引擎中的文件特征扫描提取模块, 在该模块中最为重要的是针对APK文件恶意代码的特征学习与决策模块。该特征提取模块不仅可对文件的静态特征完成提取, 还可以对APK的动态行为特征进行提取。在静态提取中, 该提取模块对文件进行反编译, 对其配置文件解析, 抽取相应特征。在动态行为提取中, 该提取模块关注APK行为的环境配置、沙盒运行、出发条件, 从而获取其日志信息。在深度检测引擎的机器学习与决策模块中, 基于支持向量机算法, 进行恶意代码判别;

    (4) 通信接口模块:

    该模块负责终端与服务器之间、快速检测引擎与深度检测引擎间的数据交互。在快速引擎到深度检测引擎的数据发送中, 包括其的检测报告与自身特征提取苦衷的特征向量信息。通信接口模块还负责监听终端到服务器之间是否有网络连接, 判断所处的网络环境, 依据消息中的控制信息来判断发送与接收。

    2 系统实现

    2.1 Android平台与SVM算法

    Android广泛应用在手机、平板电脑、电视机顶盒等移动设备中, 是Google公司开发的基于Linux的全自由开源操作系统。Android相比iOS系统具有更强的开放性, 无论是底层的操作系统还是上层的用户界面均向开发者开放[13,14]。因此, 该平台不存在阻碍移动终端创新的专利障碍。当然也正是由于Android平台的开放性, 较之iOS系统, 其有了更多的安全性问题, 使得恶意代码有机可乘。Android系统极其重视自身的安全, 建立了多种安全机制, 包括与本系统设计密切相关的沙箱机制、数字签名机制以及权限机制[15,16]。沙箱机制为不同的应用程序提供各自独立的运行空间, 避免了彼此在运行时造成序列上的混乱, 保证程序运行时的独立性。Android系统通过数字签名保证应用程序在安装时的完整性。若程序在安装时, 签名目录中的哈希值经过系统对比后与实际的有所差异, 这便会导致安装失败。Android的权限控制反映在程序的权限配置文件AndroidManifest.xml中, 该文件是程序操作与系统权限调用的映射, 保证了每次的重要权限访问均在用户允许下进行。恶意代码对系统的权限申请数量通常与安全代码有着较大区别。

    支持向量机 (SVM) 是一种流行的机器学习方法, 基于最小结构风险原则, 根据有限的样本信息下载模型, 在复杂性和学习能力中取折中。本文将恶意代码的检测抽象为分类问题, 利用支持向量机根据输入的数据产生分类结果, 具体模型如图2所示。

    图2 基于SVM的恶意代码检测模型   下载原图

    Figure 2. Malware detection model based on SVM

    在特征提取中, 本文收集了恶意样本覆盖的1 400个研究对象进行统计分析, 提取了其最长用的20种权限信息作为特征向量, 然后经过特征映射, 将其标准化处理, 作为数据输入, 再借助LIBSVM库, 采用RBF函数进行编程实现。

    3 结束语

    本文介绍了一种在Android平台下实现的恶意代码检测与防护系统, 基于支持向量机算法, 采用结构化思想, 切实考虑系统在代码检测时所需的系统开销、检测程度, 在纵向上将恶意代码检测步骤分为快速检测和深度检测两步, 实现了恶意代码的监测与防护。本模型可以有效防止恶意代码在手机上的传播, 为手机用户的重要信息、文件提供安全保证。
