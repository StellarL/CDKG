    基于云计算的数据挖掘系统设计

    蓝机满

    惠州工程职业学院

    摘    要：

    为了高效、快速地解决呈指数增长的数据处理问题, 提高数据储存、运算能力, 文中提出了基于云计算的数据挖掘系统的设计。该系统首先分析了主流云计算平台Spark的组件构成和运行机制, 深入研究其计算架构的编程原理。同时利用Spark进行了C4.5算法和K-medoids聚类算法的并行化设计, 有效提高算法的运行速度、收敛速度和结果的稳定性。测试表明, 在进行海量数据的分析处理时, 文中提出的云计算平台在分类误差内, 可有效提高整体系统的运算速度, 分类效率也大幅提高。

    关键词：

    云计算; 数据挖掘; Spark; C4.5算法; K-medoids聚类算法;

    作者简介： 蓝机满 (1983-) , 男, 讲师。研究方向:大数据数据分析技术。;

    Design of Data Mining System Based on Cloud Computing

    LAN Jiman

    Huizhou Engineering Vocational College

    Abstract：

    In order to solve exponentially increasing data processing problems and improve data storage and computing power efficiently and quickly, this paper proposed a cloud computing-based data mining system design. The system first analyzed the component composition and operation mechanism of the mainstream cloud computing platform Spark, and deeply studied the programming principle of its computing architecture. At the same time, Spark was used to parallelize the C4.5 algorithm and K-medoids clustering algorithm, which effectively improved the running speed, convergence speed and stability of the algorithm. The test showed that in the analysis and processing of massive data, the cloud computing platform proposed in this paper could effectively improve the computing speed of the whole system and improve the classification efficiency.

    Keyword：

    cloud computing; data mining; Spark; C4.5 algorithm; K-medoids clustering algorithm;

    Received： 2018-10-15

    近年来, 我国信息化网络发展进程逐步扩大, 尤其是电子商务的出现, 加速了互联网经济的生成。随之而来的是大规模、海量的用户使用数据。随着各行各业对数据的重视程度的增加, 人们开始深度挖掘数据中隐含的有用信息[1]。数据挖掘技术现在已经成功应用在生活中的各个场景[2]。然而随着数据的海量增长, 传统的数据挖掘方法在应对海量数据时, 由于其采用单机运算模式存在着速度慢、效率底下等问题。

    针对上述问题, 有公司提出了并行化云计算解决方案, 并按照业务的不同向使用公司收取一定的费用。并行化云计算解决方案具有着便捷、可靠的特点, 一经推出便得到了众多互联网公司的青睐。然而传统数据挖掘算法并不能直接应用在并行化云计算解决方案中, 需要进一步的改进[3,4,5,6]。

    在数据挖掘系统中, 聚类算法的应用使得运算速度和分类程度得到了大幅提高[7]。但值得注意的是, 传统数据挖掘系统中的聚类算法处理的数据通常有着相对规模较小的特点。这就使得当在处理海量数据时, 运用该聚类算法并不能得到良好的聚类效果。因此, 能够兼容云计算平台的聚类算法成为国内外学者研究的课题之一[8,9,10,11,12,13]。

    文中提出了基于云计算的数据挖掘系统的设计。该系统首先分析了主流云计算平台Spark的组件构成和运行机制, 深入研究其计算架构的编程原理。同时利用Spark进行了C4.5算法和K-medoids聚类算法的并行化设计, 有效提高算法的运行速度、收敛速度和结果的稳定性。

    1 基于云计算的数据挖掘系统

    文中提出的基于云计算的数据挖掘系统的设计主要由两个部分组成:云计算架构和数据挖掘算法。云计算的架构采用Spark大数据处理并行框架, 与HADOOP相比, Spark大数据处理并行框架具有效率高、适用于迭代运算等特点, 成为了主流的数据处理平台。当使用Spark大数据处理并行框架进行分布式计算程序编写时, 用户可将精力放在算法逻辑部分的实现。而对于位于大数据处理框架底层的数据的交互, 节点之间的通信和数据运算的可靠性分析等问题则可不处理。

    数据挖掘算法主要有两类算法:并行化的C4.5分类算法与改进并行化的K-medoids聚类算法。

    2 Spark大数据处理并行框架

    2.1 Spark并行架构设计思想

    Spark的原则是将数据采用分区的方式进行储存, 即将数据当作弹性分布式数据集 (Resilient Distributed Datasets, RDD) 来对待。Spark进行数据处理的过程主要为:先将待处理的数据创建出弹性分布式数据集, 再对生成的RDD施加转换和行动操作即可得到相应运算结果。

    转换是使用某些函数将旧RDD生成新RDD的过程;而行动是指使用某些函数对RDD中的数据进行运算, 并将结果返回。在运算过程中使用的RDD可以是来自RDD转换生成的, 也可以是在内存事先保存的。通常创建的RDD会被直接存放在内存中, 上一次RDD的输出结果会作为下一次RDD的输入数据直接参与运算。而对于需要经常使用的RDD, 则可以将其持久储存到内存中, 并根据使用频率的程度, 设定持久化存储的级别。甚至当储存的RDD过多, 引起内存空间不足时, 也可以将RDD储存在储存器之中, 以上方式大幅提高了迭代运算的速度。

    2.2 Spark运行架构设计

    用户在线提交应用程序, 由Spark集群接收相关程序, 并在主节点启动Master进程, 在子节点启动Worker进程;Master进程会在Worker进程上启动Driver程序进行任务的管理, Driver中的Spark Context根据任务的复杂程度, 向Master申请CPU和内存这两种资源, 同时进行Executor的初始化, 并生成有向无环图 (DAG) 传递给Task Scheduler;由后者生成Task Set, 并将Task任务提交给Executor并发执行。当所有的任务执行完毕后, 返回运算结果、关闭Spark Context和释放CPU及内存资源, 流程如图2所示。

    图2 Spark工作流程图   下载原图

    Figure 2. Spark workflow diagram

    此外, Spark另一个显著的特点便是可以实现变量的共享。共享变量, 通常指广播变量和累加器这两种。广播变量是一种只能用来读取数据的变量, 当驱动程序创建后, 该变量会被发送至计算节点处, 以改善不同程序之间通信的性能。累加器是指进行累加操作的变量, 通常用来进行计数和求和功能。当累加器这一变量被设定为全局变量时, Spark架构使得每个任务只需进行一次更新操作, 便可进行累加运算。

    3 数据挖掘算法

    通常数据挖掘按照功能的不同, 可分为分类、聚类和关联规则等功能。分类是指对大量数据, 按照某种规则的限定, 进行的划分操作;聚类是指将一个数据集合内的元素, 按照一定的属性规则, 分成不同的簇, 通过合适的相似性度量表达式, 使得每个簇内的元素具有相似的特征, 而不同簇的元素彼此有较大的区别;关联规则, 有时又称为关联分析, 是指从一定规模的数据集合中识别到某种频繁关联的项集模式的知识。

    3.1 C4.5决策树算法并行化

    C4.5算法是由ID3算法优化改进得到的决策树算法, ID3算法通过信息增益的方式来进行节点分枝属性的选择。而C4.5算法具有以下优点:

    (1) 利用信息增益率的大小进行分枝属性区分的标准, 避免了选择取值多的缺点;

    (2) C4.5算法在构造决策树的过程中, 通过修剪枝节的操作来克服过度拟合;

    (3) 可通过猜测缺失的数值, 来进行不完整数据的补充, 同时通过离散化的方式来解决连续性属性的处理。

    C4.5算法的设计思路与信息熵有关, 通过规范化处理过的信息增益来作为衡量属性分枝数据的均匀度和广度。计算公式如下

    SplitInfoA (S) =-∑j=1m|Sj|S×log2|Sj|S (1)

    GainRatio (A) =Gain (A) SplitΙnfoA (S) (2)

    式 (1) 和式 (2) 分别为信息熵与信息增益率的计算公式。由上式可知, 测试数据如果在属性A上的取值越均匀, SplitInfoA (S) 的取值就越大, 相应的信息增益率就会变得越小。由此, 可以使用信息增益率来滤除取值较多、均匀分布属性的数据。C4.5算法依据广度优先的思想进行决策树的构造, 同时逐层进行样本数据集的划分, 以便降低磁盘读写的操作。

    基于Spark实现的C4.5算法的并行化设计思路主要是, 首先调用RDD中的textFile (path, minSplits) 函数, 通过该函数将储存在HDFS系统中的训练样本数据读取出来, 直接转换成Spark RDD;转换完成之后, 调用cache (·) 函数将更新后的RDD在内存中进行缓存;再调用flatMapToPair (·) 函数将父RDD中每一个样本数据元素按照Key-value的形式转换为PairRDD。由于PairRDD中有一部分Key的元素数值相同, 利用reduceKey (·) 函数对数值相同的元素数值进行求和运算, 为下面进行信息增益率做准备。最后, 通过Spark中的Driver将上述过程中得到各类数据进行信息增益率的计算, 进而确定信息增益率最高值, 来作为最佳分枝属性的标准。

    3.2 K-medoids聚类算法

    K-medoids聚类算法是由K-means算法改进而来。K-means算法的思想是首先计算某个簇中的均值作为整个簇的中心点, 通过计算簇中其他数据距离中心点的距离, 按照规则划分不同的聚类。而K-medoids聚类算法改进了中心点的设定方法, 将簇中距离其他点距离之和最小的点设定为中心点, 这种改进防止某些离群点引起的聚类误差。具体流程如下:

    (1) 在一个有N个数据的集合里随机选择k个数值不同的数字, 作为该簇的初试中心Ci;

    (2) 分别计算该簇中步骤 (1) 选定簇中心相互之间的距离并求和, 用SUMi来表述。之后, 将这些簇中心分配到最近的簇;

    (3) 分别计算簇内某个非中心点Ctest到其他点的距离并求和, 用SUMtest来描述。若SUMtest<SUMi, 则将Ci用Ctest代替;

    (4) 重复进行步骤 (2) 和步骤 (3) , 直至中心点不再发生变化为止。

    由于K-medoids在处理海量数据时, 迭代次数过多从而导致算法进程缓慢。而Spark架构在迭代运算方面具有较快的速度, 因而文中将Spark与K-medoids算法相结合, 以实现后者的并行化。

    5 结论

    文中提出了基于云计算的数据挖掘系统的设计, 该系统采用了Spark云计算平台, 并在该平台上进行了C4.5算法和K-medoids聚类算法的并行化设计。从测试结果可知, 文中提出的系统, 可有效提高算法的运行速度、收敛速度和结果的稳定性。
